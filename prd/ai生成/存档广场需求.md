## 1. 功能概述

### 1-1 功能描述

新增"存档广场"和"云存档"两个功能模块。

**存档广场**：用户上传/下载/分享游戏存档的UGC平台，包含完美存档、通关存档、初始存档等分类。参照小黑盒存档广场，盖世差异化在于**一键导入**（不需要用户手动操作文件路径）。

**云存档**：自动备份用户本地游戏存档至云端，支持跨设备恢复。基础功能免费(3个游戏)，高级功能付费。

### 1-2 功能结构

```
存档功能
├── 存档广场（UGC存档分享）
│   ├── 存档广场首页（热门/分类/搜索）
│   ├── 存档详情页（预览+下载+一键导入）
│   ├── 上传存档
│   └── 游戏圈-存档tab（联动社区）
├── 云存档（个人存档管理）
│   ├── 云存档列表（我的页面入口）
│   ├── 自动同步设置
│   ├── 恢复存档
│   └── 付费升级
└── 游戏详情页-存档入口
```

### 1-3 本期范围

| 范围 | 包含 |
|------|------|
| 本期做 | 存档广场首页、存档详情、上传存档、一键导入(本地Wine游戏)、云存档列表、手动同步/恢复、付费体系 |
| 本期不做 | 存档编辑器、存档自动检测完整性、存档版本diff对比、非Wine游戏存档 |
| 后续迭代 | 存档编辑器(星露谷等)、MOD管理器、存档评分算法优化 |

### 1-4 名词注释

| 名词 | 说明 |
|------|------|
| 存档 | 游戏进度数据文件，位于Wine Prefix内的特定路径（如AppData/Documents/Saved Games） |
| 一键导入 | 用户下载存档后，APP自动识别目标游戏的Wine Prefix路径，将存档文件放入正确位置，无需用户手动操作 |
| 云存档 | 将本地存档文件上传至盖世云端服务器，支持跨设备下载恢复 |
| Wine Prefix | Wine模拟器为每个游戏创建的虚拟Windows文件系统 |

## 2. 界面说明

### 2-1 入口说明

| 入口位置 | 形式 | 跳转目标 |
|---------|------|---------|
| 游戏详情页 | "存档"按钮（与云存档、下载并列） | 该游戏的存档列表页 |
| 社区-游戏圈详情-存档tab | 帖子列表（分类=存档） | 帖子详情（含存档附件下载） |
| 我的页面 | "我的存档"入口 | 云存档管理页 |
| 游戏内浮窗菜单 | "存档管理"按钮 | 当前游戏的存档管理 |

### 2-2 存档广场首页

入口：游戏详情页"存档"按钮 / 社区游戏圈存档tab

#### ① 页面结构

```
[← 返回]  存档广场 · 星露谷物语  [上传]
──────────────────────────────
[🔍 搜索存档]

分类：全部 | 完美存档 | 通关存档 | 初始存档 | MOD存档
排序：最热 | 最新
──────────────────────────────
[存档卡片列表]
```

#### ② 存档卡片元素

| 元素 | 说明 |
|------|------|
| 存档名称 | 主标题，最多1行 |
| 存档描述 | 副标题，最多2行截断 |
| 存档截图 | 若有，显示缩略图(游戏内截图) |
| 分类标签 | 完美存档/通关存档/初始存档/MOD存档 |
| 上传者 | 头像+昵称 |
| 数据 | 下载量 · 收藏量 · 评分(5星制) |
| 文件大小 | 如"45MB" |
| 适用版本 | 如"Steam最新版" |

(1) 列表为单列卡片布局
(2) 下拉刷新、上拉加载更多
(3) 若该游戏无任何存档，显示空态："还没有存档，上传第一个 →"

#### ② 交互说明

(1) "下载并导入"按钮：
   - 点击后检查本地是否安装该游戏
   - 若已安装→下载存档文件→自动导入到对应Wine Prefix路径→弹toast"导入成功，启动游戏即可加载"
   - 若未安装→弹提示"请先安装该游戏"
   - 下载过程显示进度条

(2) 一键导入技术说明：
   - APP根据game_id查询该游戏的存档路径配置（由后台维护）
   - 存档路径配置示例：`drive_c/users/{user}/AppData/Roaming/StardewValley/Saves/`
   - 将下载的存档文件解压至对应路径
   - 若目标路径已有存档→弹确认"将覆盖现有存档，是否继续？[取消] [覆盖] [备份后覆盖]"

(3) 收藏：即时反馈
(4) 评价：下载并导入成功后，弹出评价引导"存档好用吗？给个评分 ⭐⭐⭐⭐⭐"

### 2-4 上传存档

入口：存档广场右上角"上传"按钮

#### ① 页面结构

```
[取消]        上传存档        [提交]
──────────────────────────────
关联游戏：[选择游戏 ▼]

── 选择存档文件 ──
[从本地选择]  ← 检测到以下可用存档：
  ☑ 存档1: Farm_01 (修改于今天 14:30, 12MB)
  ☐ 存档2: Farm_02 (修改于昨天, 8MB)

存档名称：(必填，最多30字)
存档描述：(必填，最少20字)
分类标签：[完美存档] [通关存档] [初始存档] [MOD存档]
适用版本：(必填，如"Steam v1.6.14")

截图：(选填，最多6张)
[📷 添加截图]
```

#### ② 自动检测存档

(1) 选择关联游戏后，APP自动扫描该游戏Wine Prefix内的存档文件
(2) 列出可用存档供用户勾选
(3) 若无法检测→显示"手动选择文件"入口
(4) 支持多选→打包为zip上传

#### ③ 上传限制

| 限制项 | 值 |
|--------|------|
| 单次上传文件大小 | ≤ 500MB |
| 每日上传次数 | 普通用户5次 / 创作者20次 |
| 支持格式 | 自动打包为zip |

(1) 提交后进入审核（自动审核：文件安全扫描）
(2) 审核通过→自动上架
(3) 审核失败→通知用户原因

```
│  最近同步：昨天 22:15           │

│  [同步] [恢复] [删除]          │
├──────────────────────────────┤

└──────────────────────────────┘

[+ 添加游戏到云存档]  ← 免费用户已达上限(3/3)

── 同步设置 ──

同步时机：退出游戏时 / 仅WiFi
```

#### ② 交互说明

(1) "同步"按钮：立即将本地存档上传至云端（覆盖云端版本）
(2) "恢复"按钮：将云端存档下载至本地Wine Prefix（覆盖本地版本）
   - 恢复前弹确认："将用云端存档覆盖本地存档，是否继续？"
   - 恢复过程显示进度
(3) "删除"按钮：删除该游戏的云存档，释放空间
(4) "添加游戏"：若未达上限→弹出已安装游戏列表→选择→自动首次同步
   - 若已达免费上限→弹付费引导

#### ③ 自动同步逻辑

| 条件 | 行为 |
|------|------|
| 退出游戏时 + WiFi | 自动上传存档至云端 |
| 退出游戏时 + 非WiFi | 标记"待同步"，下次WiFi时自动上传 |
| 启动游戏时 | 检查云端是否有更新版本→若有弹提示"云端有更新的存档，是否恢复？" |
| Prefix损坏/重建时 | 自动从云端恢复最新存档 |

### 2-6 云存档付费

#### ① 付费方案

| 方案 | 价格 | 游戏数 | 空间 | 历史版本 |
|------|------|--------|------|---------|
| 免费 | ¥0 | 3个游戏 | 500MB | 仅最新版本 |
| 基础版 | ¥5/月 | 10个游戏 | 2GB | 保留最近3个版本 |
| 高级版 | ¥12/月 | 不限 | 10GB | 保留最近10个版本 |

(1) 付费入口：云存档管理页"升级"按钮 / 添加游戏时达上限弹出
(2) 支付方式：支持微信/支付宝/积分兑换(800积分=基础版1个月)
(3) 到期处理：到期后不删除数据，但停止自动同步+不可新增游戏，续费后恢复

#### ② 付费引导时机

| 触发场景 | 引导文案 |
|---------|---------|
| 免费用户添加第4个游戏 | "免费版最多同步3个游戏，升级基础版支持10个游戏" |
| 空间不足 | "云存档空间已满，升级获取更多空间" |
| Prefix损坏且无云存档 | "存档已丢失。开启云存档，再也不怕丢档" |
| 游戏内退出时（首次） | "开启云存档，自动保存游戏进度 [免费开启]" |
