## 二、背景与目标

❗背景：

当前盖世用户遇到游戏启动/配置问题时，只能去QQ群发截图求助或在B站搜教程。成功跑通游戏的用户没有沉淀配置方案的地方，下一个用户还得重新踩坑。竞品233乐园已有完整游戏圈+创作者体系，底部导航给了"社区"和"创作"两个独立tab。盖世在社区维度为零。

❗目标：

通过上线"社区-游戏圈"功能，在APP内建立**以游戏为中心的用户互助生态**，实现"求助→解答→沉淀→复用"的内容闭环。核心目标是让用户的配置经验、攻略、存档在平台内流转，间接提升启动成功率和留存。

核心挑战：冷启动内容供给、UGC质量管控、配置帖结构化与机型匹配、海内外内容审核一致性。

## 三、故事介绍

### 3.1 用户与运营场景

❗用户与运营场景：

- **C端求助者（小白）**：小白下载了盖世想玩GTA5，启动失败不知道怎么配置。他进入GTA5游戏圈，搜索"骁龙8gen3"，看到一条配置帖自动带了机型和Wine参数，按帖子设置后成功启动。他在评论区回复"感谢，成功了"。

- **C端分享者（老王）**：老王花了2小时调通赛博2077的配置，想分享给其他人。他点FAB发帖，选分类"配置"，APP自动填入他的机型(小米14 Pro/骁龙8gen3/12GB)和Wine环境(9.8-staging/DXVK2.5.1)，他只需补充正文说明"分辨率720p锁30帧关阴影"，贴两张截图，发布。3小时后收到56个点赞。

- **C端围观者（路人）**：路人打开社区刷推荐流，看到"空洞骑士手柄完美映射方案"帖子，点赞收藏，点击帖子里的游戏名跳转到空洞骑士详情页，决定下载试试。

- **B端运营（李四）**：李四在后台看到GTA5游戏圈有一条高赞配置帖(234赞)，将其标记为"精选"。同时发现一条广告帖，直接删除并封禁发帖账号7天。他还创建了一个"黑神话悟空"游戏圈，配置了封面图和推荐位排序。

### 3.2 价值分析

❗价值分析：

- **间接提升启动成功率**：用户通过社区找到同机型配置方案→启动成功率从3.9%提升（目标30%用户通过社区帖子解决配置问题）
- **提升留存与粘性**：社区内容消费增加用户停留时长和回访频率→D1留存目标从20%提升至25%+
- **为商业化铺垫**：社区活跃度→积分体系→积分消费（云游戏时长/CDK）→变现闭环
- **构建竞品壁垒**：配置帖结构化+机型匹配推荐是盖世独有能力（233没有Wine环境数据）

### 3.3 核心体验路径

❗核心链路：

- **消费者核心链路**：
  进入社区 → 搜索/浏览游戏圈 → 找到目标帖子 → 阅读内容 → 评论追问/按配置操作 → 问题解决

- **创作者核心链路**：
  游戏跑通 → 点FAB发帖 → 选游戏+选分类"配置" → 设备信息自动填入 → 补充正文+截图 → 发布 → 收到点赞/评论反馈

- **运营核心链路**：
  后台查看游戏圈 → 精选高质量帖子 → 删除违规内容 → 配置推荐位排序 → 创建新游戏圈

### 3.4 产品指标预测

❗产品指标预测：

1. 社区DAU占APP DAU比例：首月预测 20%，3个月后目标 40%
2. 发帖转化率（进入发帖页→发布成功）：预测 60%
3. 帖子互动率（浏览帖子→点赞/评论/收藏）：预测 15%
4. 通过社区帖子解决配置问题的用户比例：首月预测 15%，3个月后目标 30%
5. 游戏圈关注转化率（进入游戏圈→关注）：预测 25%

### 3.5 路径规划

❗路径规划：

- **当前版本（V1.0）**：跑通社区基础闭环——游戏圈浏览+发帖(图文)+评论+搜索+关注+配置帖自动填入设备信息。后台支持帖子管理+游戏圈管理。
- **V1.1（预告）**：创作者中心+积分基础体系+发行人计划入口
- **V2.0（预告）**：存档广场+MOD分享+视频帖+积分商城

## 四、概要设计

### 4.1 模块设计

本次功能涉及的系统模块：

- 【C端社区模块】
  - 社区首页：推荐流+关注流+游戏圈列表
  - 游戏圈详情：单游戏社区空间，含分类sub-tab
  - 内容创作：发帖编辑器+配置帖设备信息自动填入
  - 内容消费：帖子详情+评论楼中楼+搜索
  - 游戏详情联动：详情页社区tab嵌入游戏圈内容

- 【B端管理中台】
  - 帖子管理：审核/精选/置顶/删除
  - 游戏圈管理：创建/排序/推荐位/关闭
  - 用户管理：禁言/封号/创作者审核
  - 内容审核：机审配置+人工审核队列

#### 4.3.3 发帖与内容创作（C端）

**发帖页结构**

| 字段 | 必填 | 说明 |
|------|------|------|
| 关联游戏 | 是 | 点击弹出游戏选择列表（搜索+热门）；未选择时"发布"按钮置灰 |
| 分类标签 | 是 | 单选：讨论(默认)/攻略/配置/求助/存档/MOD |
| 标题 | 否 | 最多50字 |
| 正文 | 是 | 最少10字，最多2000字；<10字时发布弹toast"正文至少10个字" |
| 图片 | 否 | 最多9张，单张≤10MB，支持JPG/PNG/GIF |
| 附件 | 否 | 仅分类为"存档"/"MOD"时出现；支持zip/rar/7z，≤500MB |

**配置帖自动填入**

选择分类"配置"或"求助"时，正文区域上方自动出现设备信息卡片：

| 字段 | 数据来源 | 可编辑 |
|------|---------|--------|
| 机型 | 系统读取Build.MODEL | 是 |
| 芯片 | 系统读取Build.HARDWARE | 是 |
| 内存 | 系统读取总RAM | 是 |
| Wine版本 | 读取当前APP内Wine环境版本 | 是 |
| DXVK版本 | 读取当前APP内DXVK版本 | 是 |

RS：设备信息标注"由系统自动获取"，用户可手动修改

**草稿自动保存**

- 编辑内容每30秒自动保存至本地
- APP被杀/闪退后，下次打开发帖页弹提示"检测到未发布的草稿，是否恢复？"
- 点击"恢复"→编辑器恢复之前全部内容（含图片引用）
- 点击"放弃"→清除草稿

**发布流程**

- 发布成功→跳转帖子详情页
- 发布失败→toast提示原因，编辑器内容保留
- 新用户(注册<24h)发帖→帖子状态"审核中"，仅自己可见

| 规则 | 说明 |
|------|------|
| 排序 | 默认最热；可切换最新 |

#### 4.3.5 游戏详情页-社区tab（C端）

| 帖子操作 | 置顶了帖子[{post_id}] | 点击置顶 |
| 用户操作 | 对用户[{uid}]执行禁言{days}天 | 点击禁言并确认 |
| 用户操作 | 封禁了用户[{uid}] | 点击封号并确认 |
| 游戏圈操作 | 创建了游戏圈[{game_name}] | 新建游戏圈保存 |
| 游戏圈操作 | 关闭了游戏圈[{game_name}] | 点击关闭并确认 |
| 系统 | 管理员登录后台 | 登录成功 |

### 4.4 边界条件与异常处理

#### 内容侧

| 场景 | 处理方式 |
|------|---------|
| 帖子含外链 | 白名单放行(Steam/Epic/B站/抖音)，其余替换为"[链接已屏蔽]"不可点击 |
| 配置帖Wine版本已过时 | 不下架，帖子顶部加灰色提示"该配置发布于X天前，部分参数可能已过时" |
| 存档文件含恶意代码 | 服务端自动安全扫描，未通过→拒绝上架+通知用户"文件安全检测未通过" |
| 帖子图片含二维码 | 图片OCR检测，命中→审核拦截，人工复核 |
| 大量用户反馈配置帖无效 | 帖子下方显示"X人反馈该配置无效"；负反馈>30%→从精选移除 |
| 帖子关联错误游戏 | 版主可修改关联游戏；举报选项增加"游戏关联错误" |
| 单个游戏圈帖子量<5 | 底部显示"该游戏圈还很冷清，发第一条帖子 →"；社区首页不推荐 |

#### 用户侧

| 场景 | 处理方式 |
|------|---------|
| 未登录浏览 | 允许浏览所有帖子；点击发帖/评论/点赞/关注→弹登录引导 |
| 用户改昵称/头像 | 历史帖子实时更新为最新昵称/头像 |
| 用户注销账号 | 帖子保留，作者显示"已注销用户"头像置灰；不删除内容 |
| 被封禁用户帖子 | 封禁期间帖子对他人不可见(软删除)，解封后恢复 |
| 短时间大量关注 | 单日关注上限50个游戏圈，超出toast"今日关注已达上限" |
| 批量发相同内容 | 10分钟内相似度>80%帖子超3条→第4条起进入审核队列 |
| 新注册用户发帖 | 注册<24小时发帖需审核；24小时后正常发布 |

#### 状态侧

| 场景 | 处理方式 |
|------|---------|
| 帖子审核中 | 仅自己可见，列表中标"审核中"；审核通过后公开 |
| 帖子被举报 | 暂时隐藏→审核通过恢复/不通过永久隐藏+通知作者 |
| 写帖子时APP闪退 | 本地草稿自动保存(每30秒)，下次打开提示恢复 |
| 发帖时网络断开 | "发布失败，点击重试"；内容保留不丢失 |
| 收藏的帖子被删 | 收藏列表显示"该内容已被删除"灰色文字，不可点击 |
| 主评论被删有回复 | 主评论显示"该评论已删除"占位，回复保留可见 |
| 附件下载中断 | 支持断点续传；恢复网络后继续 |

#### 权限/风控侧

| 场景 | 处理方式 |
|------|---------|
| 涉政/涉黄/暴力内容 | 发布时过机审(关键词+图片识别)，命中→拦截+"内容违规，请修改后重试" |
| 评论辱骂/人身攻击 | 机审关键词过滤；被举报3次以上→自动隐藏+人工复核 |
| 冒充官方昵称 | 注册/改名拦截含"盖世""官方""客服""管理员"关键词 |
| 批量注册发广告 | 同设备24h内注册>2个账号→第3个起需手机验证码 |
| 版主滥用权限 | 操作全留日志；给自己帖子加精需另一位版主确认 |
| 存档文件实为盗版安装包 | 文件类型白名单(zip/rar/7z内部不含exe/msi/apk)，命中→拒绝+警告 |
| 未成年保护 | 社区不展示付费引导给未实名用户；评论不展示加群/加好友引导 |
| 帖子含隐私信息 | 机审识别手机号/微信号→自动打码；举报"隐私泄露"→2小时内人工处理 |
| 举报被滥用 | 同一用户对同一帖子只能举报1次；单日上限20次；准确率<20%降低权重 |

### 4.5 非功能需求

| 需求类型 | 详细要求 |
|---------|---------|
| 性能 | 社区首页首屏加载<2秒；帖子列表滚动帧率≥50fps；图片懒加载 |
| 兼容性 | Android 9.0+；iOS 13.0+；横竖屏自适应；横屏社区改为3列瀑布流 |
| 安全 | 帖子内容过机审(关键词+图片)；存档文件服务端安全扫描；评论防XSS |
| 容错 | 接口超时5秒显示重试；图片加载失败显示占位图；点赞乐观更新+失败回退 |
| 数据一致性 | 点赞/收藏计数最终一致(允许短暂延迟)；评论发送前端乐观更新 |

## 七、非目标范围（本期不做）

| 不做的事 | 原因 |
|---------|------|
| 视频帖 | 开发成本高+审核复杂，图文够用 |
| 积分商城 | 需商品供给体系配合，先跑通内容闭环 |
| 创作者收益提现 | 需财务/税务/合规体系 |
| MOD一键安装 | 技术复杂，先做文件下载 |
| 存档一键导入 | 需维护存档路径配置库，先做文件下载 |
| 话题/热榜运营 | 冷启动帖子量不够，先靠运营手动精选 |
| 私信 | 评论区够沟通，私信带来审核和骚扰风险 |
| 发行人计划 | 需审核+财务结算，V1.1做 |
| 国际化社区 | 先跑通中文，海外后续规划 |
