# 社区功能 · 边界条件补充

---

## 1. 内容侧边界

| 场景 | 处理方式 | 是否写入PRD |
|------|---------|------------|
| 帖子正文含外链（竞品APP下载链接/钓鱼链接） | 自动识别URL，白名单内放行（Steam/Epic/B站/抖音），其余链接替换为"[链接已屏蔽]"+不可点击 | ✅ 必须写 |
| 配置帖中的Wine/DXVK版本号已过时（如Wine8.x已不维护） | 不主动下架，但在帖子顶部加灰色提示"该配置发布于X天前，部分参数可能已过时" | ✅ 写 |
| 用户上传的存档文件含病毒/恶意代码 | 上传后服务端自动扫描（ClamAV或同类），未通过则拒绝上架+通知用户"文件安全检测未通过" | ✅ 必须写 |
| 帖子图片含二维码（引流到竞品/外部群） | 图片OCR检测二维码，命中则审核拦截，人工复核 | ✅ 写 |
| 同一配置帖被大量用户反馈"按这个配置启动失败" | 帖子下方自动显示"X人反馈该配置无效"标签；累计超过30%负反馈→从"精选"中移除 | ✅ 写 |
| 帖子关联了错误的游戏（如GTA5的帖子关联到了空洞骑士） | 允许版主/运营修改帖子关联游戏；用户举报选项增加"游戏关联错误" | ✅ 写 |
| 存档文件实际适用版本与标注不符（用户下载后导入失败） | 导入失败时弹出"存档可能与当前游戏版本不兼容"提示+引导用户去评价区反馈 | ✅ 写 |
| 帖子内容完全复制另一条帖子（洗稿/搬运） | 发布时不做查重（成本高），依赖用户举报"内容搬运"→人工核实→标记原创/删除搬运 | ⚠️ 可简写 |
| 单个游戏圈帖子量极少（<5条） | 游戏圈详情页底部显示"该游戏圈还很冷清，发第一条帖子 →"引导发帖；社区首页不推荐帖子量<5的游戏圈 | ✅ 写 |

## 2. 用户侧边界

| 场景 | 处理方式 | 是否写入PRD |
|------|---------|------------|
| 未登录用户浏览社区 | 允许浏览所有帖子和评论；点击发帖/评论/点赞/关注时弹登录引导 | ✅ 必须写 |
| 用户改了昵称/头像后，历史帖子显示什么 | 全部实时更新为最新昵称/头像（不保留历史态） | ✅ 写 |
| 用户注销账号后，其帖子/评论怎么处理 | 帖子保留但作者显示"已注销用户"，头像置灰；评论同理。不删除内容（避免破坏讨论上下文） | ✅ 必须写 |
| 被封禁用户的帖子是否可见 | 封禁期间：帖子对他人不可见（软删除），解封后恢复。永久封禁：帖子永久隐藏 | ✅ 必须写 |
| 用户短时间内关注大量游戏圈（疑似机器人） | 单日关注上限50个游戏圈，超出后提示"今日关注已达上限" | ✅ 写 |
| 用户在多个游戏圈发相同内容（批量灌水） | 同一用户10分钟内发布内容相似度>80%的帖子超过3条→第4条起自动进入审核队列 | ✅ 写 |
| 新注册用户立刻发帖（可能是spam账号） | 注册后24小时内发帖需要人工审核（或通过验证码）；24小时后正常发布 | ✅ 必须写 |
| 用户设备信息自动填入配置帖，但用户用的是模拟器/非真实设备 | 设备信息标注"由系统自动获取"，不做真伪校验。用户可手动修改填入的设备信息 | ⚠️ 可简写 |

## 3. 状态侧边界

| 场景 | 处理方式 | 是否写入PRD |
|------|---------|------------|
| 帖子发布后正在审核中（新用户24h内） | 发布成功→帖子状态"审核中"→仅自己可见→审核通过后公开。发帖者看到的列表中该帖标"审核中" | ✅ 必须写 |
| 帖子被举报后进入审核 | 帖子暂时对其他用户不可见（隐藏），审核通过→恢复，审核不通过→永久隐藏+通知作者 | ✅ 写 |
| 用户正在写长帖子时APP被杀/闪退 | 本地草稿自动保存（每30秒），下次打开发帖页时提示"检测到未发布的草稿，是否恢复？" | ✅ 必须写 |
| 用户发帖时网络断开 | 发布按钮变为"发布失败，点击重试"；帖子内容保留在编辑器中不丢失 | ✅ 写 |
| 帖子已发布后，关联的游戏被下架/删除 | 帖子保留，游戏名显示但不可点击跳转；游戏圈标记"该游戏已下架" | ⚠️ 可简写 |
| 用户收藏的帖子被作者删除 | 收藏列表中该条显示"该内容已被删除"灰色文字，不可点击 | ✅ 写 |
| 评论区楼主删除了主评论，但下面有多条回复 | 主评论显示"该评论已删除"占位，回复保留可见（保持上下文） | ✅ 写 |
| 游戏圈被运营关闭/合并 | 关闭：帖子保留但不可新发帖，顶部提示"该游戏圈已关闭"。合并：帖子迁移至目标游戏圈，原圈重定向 | ⚠️ 可简写 |
| 帖子附件（存档文件）下载到一半网络断开 | 支持断点续传；已下载部分保留，恢复网络后继续 | ✅ 写 |

## 4. 异常侧边界

| 场景 | 处理方式 | 是否写入PRD |
|------|---------|------------|
| 社区服务端宕机/接口超时 | 社区tab内容区显示"加载失败，点击重试"；不影响其他tab（游戏库/排行等）正常使用 | ✅ 必须写 |
| 图片上传失败（超大/格式不支持） | 即时toast提示"图片需小于10MB，支持JPG/PNG/GIF"；已选的其他图片不受影响 | ✅ 写 |
| 存档文件上传超时（大文件+弱网） | 显示上传进度条；超时后提示"上传超时，建议在WiFi环境下重试"；已上传部分支持续传 | ✅ 写 |
| 推荐算法冷启动（新用户无行为数据） | 默认推荐：热门帖子（全站点赞Top）+ 用户已安装游戏的游戏圈帖子 | ✅ 写 |
| 搜索无结果 | 显示"未找到相关内容"+ 推荐热门游戏圈入口"试试看这些游戏圈 →" | ✅ 写 |
| 帖子详情页图片加载失败 | 图片位置显示灰色占位+重试icon，点击重新加载单张图片 | ⚠️ 可简写 |
| 用户点赞后立刻取消（快速双击） | 前端做防抖（300ms），以最后一次操作为准；接口幂等 | ⚠️ 技术文档写 |
| 评论发送成功但列表未刷新（接口延迟） | 前端乐观更新：发送成功后立刻在列表顶部插入该评论（本地态），后台异步同步 | ⚠️ 技术文档写 |
| 帖子被大量用户同时访问（热帖/爆款） | 帖子详情页做CDN缓存（5分钟TTL）；评论区实时但分页加载（每次20条） | ⚠️ 技术文档写 |

## 5. 权限/风控侧边界

| 场景 | 处理方式 | 是否写入PRD |
|------|---------|------------|
| 用户发布涉政/涉黄/暴力内容 | 发布时过机审（关键词+图片识别），命中→拦截不发布+提示"内容违规，请修改后重试"；漏网→用户举报→人工处理 | ✅ 必须写 |
| 用户在评论区辱骂/人身攻击 | 评论过机审关键词过滤；被举报3次以上→自动隐藏+人工复核 | ✅ 必须写 |
| 用户冒充官方（昵称含"盖世官方""客服"等） | 注册/改名时拦截包含"盖世""官方""客服""管理员"等关键词的昵称 | ✅ 写 |
| 同一IP/设备批量注册发广告 | 同一设备ID 24小时内注册超过2个账号→第3个起需要手机验证码 | ✅ 写 |
| 版主滥用权限（删除正常帖子/给自己帖子加精） | 版主操作全部留日志；给自己帖子加精需要另一位版主或运营确认 | ✅ 写 |
| 用户上传的存档文件实际是盗版游戏安装包 | 文件类型白名单（仅允许zip/rar/7z且内部不含exe/msi/apk）；命中→拒绝上传+警告 | ✅ 必须写 |
| 未成年用户保护 | 社区不展示付费引导（积分商城/云存档付费）给未实名认证用户；评论区不展示引导加群/加好友的内容 | ✅ 必须写（合规要求） |
| 海外用户发布中文社区不适宜的内容（如政治敏感） | 国内外共用一套审核规则；海外用户发帖同样过机审+人工 | ⚠️ 可简写 |
| 用户举报被滥用（恶意举报正常帖子） | 同一用户对同一帖子只能举报1次；单日举报上限20次；举报准确率<20%的用户降低其举报权重 | ✅ 写 |
| 帖子中包含其他用户的隐私信息（手机号/微信号） | 机审识别手机号/微信号格式→自动打码显示；用户举报"隐私泄露"→优先人工处理（2小时内） | ✅ 必须写 |

## 补充：容易漏写但影响上线的Top 5

1. **新用户24h审核期** — 不写这条，上线第一天就会被spam淹没
2. **草稿自动保存** — 不写这条，用户写了10分钟的配置帖闪退后会骂娘然后卸载
3. **存档文件安全扫描** — 不写这条，有人传恶意文件会出安全事故
4. **帖子被删后评论保留** — 不写这条，开发会把整棵评论树一起删掉，破坏讨论上下文
5. **未成年保护** — 不写这条，过不了应用商店审核/有合规风险
