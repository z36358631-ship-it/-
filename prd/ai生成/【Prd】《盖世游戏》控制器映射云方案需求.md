# 二、 背景与目标

**背景：**
目前盖世游戏云游戏平台已支持虚拟按键布局编辑功能，用户可以自定义虚拟手柄按键的位置和大小。然而，部分PC游戏（如逃离塔科夫、CS2、Valorant等）原生不支持手柄操控，用户在云游戏环境中无法直接使用虚拟手柄进行游玩。

当前用户面临的核心痛点：不支持手柄的游戏只能通过触屏模拟鼠标键盘操作，体验极差，导致这类游戏在移动端的留存和时长远低于手柄适配游戏。

**目标：**
通过"控制器映射"功能，实现**虚拟手柄按键 → 键盘/鼠标操作**的映射绑定，让用户通过虚拟按键或物理手柄来操控原本不支持手柄的游戏。同时构建映射方案的**云分享生态**（推荐方案、排行榜、分享码），降低用户的配置门槛，提升整体云游戏体验。

**核心挑战：** 映射精准度（延迟<16ms）、方案数据同步一致性、物理手柄热插拔状态管理、分享码防滥用。

# 三、 故事介绍

## 3\.1 用户与运营场景

- **C端玩家（张三）：** 张三想在手机上玩《逃离塔科夫》，进入云游戏后看到右侧操作侧边栏的"控制器映射"功能已开启，点击"控制器映射设置"进入映射广场，在"推荐"Tab中找到"逃离塔科夫-标准"方案，点击查看详情后一键应用。虚拟按键立即按该方案的映射规则工作，A键映射鼠标左键开枪，摇杆映射WASD移动。

- **C端玩家（李四）：** 李四是FPS老手，觉得社区方案不够顺手，在"自定义"Tab中点击"创建"，选择"游戏专用"类型，命名为"塔科夫-速攻"，进入编辑器逐个配置按键映射关系，保存后通过"云分享"生成分享码`GS-A8K2M5`发给队友。

- **硬件场景（王五）：** 王五游戏中途插入Xbox手柄，系统自动切换为物理手柄输入（共享同一套映射方案），虚拟按键自动隐藏。拔出手柄后，虚拟按键立即恢复显示，无需手动操作。

## 3\.2 价值分析

- **扩大可玩游戏覆盖面：** 解锁平台上所有不支持手柄的PC游戏（FPS/RTS/MMO等），预计可新增30%+可玩游戏品类。

- **降低配置门槛：** 通过推荐方案和分享码机制，新手无需逐个配置按键映射，一键应用即可上手。

- **提升社区活跃：** 映射方案的分享与排行机制形成UGC内容生态，促进玩家间互动。

## 3\.3 核心体验路径

- **C端使用核心链路：**
侧边栏开启控制器映射 → 进入映射设置页 → 浏览推荐/排行/自定义方案 → 点击方案卡片查看详情 → 应用方案 → 虚拟按键按映射规则生效。

- **C端创建核心链路：**
自定义Tab → 点击"创建" → 输入方案名+选择类型 → 进入编辑器配置映射 → 保存 → 可选云分享生成分享码。

- **C端导入核心链路：**
自定义Tab → 点击"导入" → 输入分享码 → 方案自动添加到"我的方案" → 可直接应用。

## 3\.4 产品指标预测

1. **映射功能开启率：** 进入不支持手柄游戏的用户中，预计首月 `60%` 会开启控制器映射。

2. **方案应用率：** 进入映射设置页的用户中，预计 `75%` 会应用一个方案（推荐/排行/导入）。

3. **自定义创建率：** 预计 `15%` 用户会创建自定义方案。

4. **分享码使用率：** 预计 `8%` 用户会通过分享码导入方案。

## 3\.5 路径规划

- **当前版本（V1\.0）：** 实现控制器映射核心功能（启用/禁用、方案广场3Tab、方案详情与应用、创建与编辑、分享码导入导出、设备切换逻辑）。

- **未来可能升级（预告）：** 方案评分与评论系统、按游戏自动推荐映射方案、映射宏（组合键/连招）、方案版本历史。



# 四、概要设计

## 4\.1 模块设计

本次功能涉及的系统模块归纳如下：

- **【C端游戏侧边栏模块】**

    - **控制器映射开关：** 启用/禁用映射功能，联动虚拟按键显隐。

    - **设备列表：** 显示当前活跃输入设备（虚拟手柄/物理手柄），支持点击切换。

    - **引导说明：** "?"按钮弹窗解释功能用途。

- **【映射方案广场模块】**

    - **推荐Tab：** 适合当前游戏的方案 + 全部映射方案。

    - **排行Tab：** 按使用量排序的方案列表。

    - **自定义Tab：** 我的方案 + 我的云分享，含创建/导入操作。

- **【方案管理模块】**

    - **方案详情弹窗：** 查看方案信息与操作（应用/编辑/复制/删除/云分享）。

    - **方案创建：** 命名+类型选择（游戏专用/通用）→ 进入编辑器。

    - **方案编辑：** 复用按键布局编辑器，扩展映射绑定能力。

- **【云分享模块】**

    - **分享码生成：** 导出方案为唯一分享码。

    - **分享码导入：** 输入分享码获取方案到本地。

# 五、交付设计（数据分析与埋点设计）

## 5\.1 核心业务指标

- **功能漏斗：** 进入游戏（UV） → 开启控制器映射（UV） → 进入映射设置页（UV） → 应用方案（UV） → 完成一局游戏（UV）。

## 5\.2 埋点事件表

|**事件ID**|**事件名称**|**触发时机**|**关键参数**|
|---|---|---|---|
|`mapping_toggle`|控制器映射开关|用户切换启用/禁用开关时|`uid`, `action`(enable/disable), `game_id`|
|`mapping_page_view`|映射设置页曝光|进入映射方案广场时|`uid`, `game_id`, `source`(sidebar)|
|`mapping_tab_switch`|Tab切换|切换推荐/排行/自定义Tab时|`uid`, `tab_name`(recommend/rank/custom)|
|`scheme_card_click`|方案卡片点击|点击方案卡片拉起详情弹窗时|`uid`, `scheme_id`, `scheme_type`, `tab_source`|
|`scheme_apply`|应用方案|点击详情弹窗"应用"按钮时|`uid`, `scheme_id`, `scheme_name`, `game_id`|
|`scheme_create`|创建方案|确认创建方案时|`uid`, `scheme_name`, `scope`(game/general)|
|`scheme_share_export`|导出分享码|生成分享码时|`uid`, `scheme_id`, `share_code`|
|`scheme_share_import`|导入分享码|确认导入分享码时|`uid`, `share_code`, `result`(success/fail)|
|`device_switch`|设备切换|手柄接入/拔出/手动切换时|`uid`, `device_type`(virtual/physical), `trigger`(auto/manual)|
|`scheme_search`|方案搜索|执行搜索动作时|`uid`, `keyword`, `result_count`|
|`guide_view`|引导弹窗曝光|点击?按钮展示引导弹窗时|`uid`, `game_id`|
