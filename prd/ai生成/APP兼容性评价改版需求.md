# 二、背景与目标

**背景：** 目前移动端均缺乏玩家侧的兼容性反馈能力。玩家在下载或购买游戏前，无法参考其他玩家在相同设备上的实际运行情况，导致因兼容性问题引发的退款与差评。

**目标：** 通过在游戏详情页新增兼容性评价入口，支持玩家提交带有设备信息的兼容性反馈，构建"浏览反馈 - 提交评价 - 后台治理"的完整数据闭环。

# 三、故事介绍

## 3.1 用户与运营场景

**C端玩家（小王）：**

小王在游戏详情页看到了「兼容评价」入口卡片，显示综合评分 4.3，描述文案"大多数玩家可正常游玩"。他点击进入兼容性评价详情页，浏览了其他玩家的反馈，看到设备信息「iPhone 15 Pro · iOS 17」，确认自己的设备应该没问题，随即下单购买。购买后他也点击「我要反馈」提交了自己的评价，选了 5 星并勾选了"基本可玩"标签。

**B端运营（李姐）：**

李姐在后台进入「玩家兼容性评价管理」，按「移动端」筛选后，发现某款游戏在旧版本有多条负面评价，通过版本号筛选后将旧版本评价批量隐藏；并对其中一条举报进行了「处理」操作，评价自动联动隐藏，该游戏综合评分随即重新计算更新。

## 3.2 价值分析

- **降低购买风险：** 玩家在购买决策前可参考真实设备兼容情况，减少因兼容性未知导致的退款与差评。
- **结构化数据沉淀：** 按端（Mac端/移动端）区分的评价数据，帮助研发精准定位平台兼容问题，提升排查效率。
- **内容自治闭环：** 举报机制配合后台处理能力，平台可主动干预低质内容，保障评价生态健康。

## 3.3 核心体验路径

**C端评价核心链路：**

<font color="#4e73ff">详情页入口卡片曝光 → 点击进入兼容性评价详情页 → 查看综合评分/分段占比/玩家评价列表（默认按本机配置筛选）→ 点击「我要反馈」→ 登录验证 → 平台时长门槛校验 → 填写星级+问题标签+文字/图片（可选）→ 提交成功，列表与综合评分刷新 **//2026.5.14修改**</font>

**B端管理核心链路：**

<font color="#4e73ff">进入「玩家兼容性评价管理」→ 按端/状态/游戏名/版本号筛选 → 隐藏/恢复/批量清理评价 → 综合评分联动更新；举报列表处理/驳回 → 处理后联动隐藏评价及同评价其他举报 **//2026.5.14修改**</font>

## 3.4 产品指标预测

- **入口点击率：** 首月预测详情页兼容评价卡片点击率 10%～15%（已有游戏购买意向用户关注度高）。
- **反馈提交率：** 进入评价详情页后，预计 5%～10% 的已登录用户会完成反馈提交。
- **举报处理率：** 预计举报量极低，后台运营每日处理时长 < 30 分钟。

- <font color="#4e73ff">**身份设置：** 身份标签列表，可新增/编辑/删除标签，配置权限（身份标识/置顶/静默动态），支持人工下发与撤销身份。**//2026.5.14-2修改**</font>

## 4.3 详细设计

### 4.3.1 游戏详情页入口卡片（C端）

功能 demo：兼容性评价入口卡片 demo

**入口卡片布局规则：**

- 横屏：展示完整概览卡片，含图标、综合评分数字、星级、描述文案，卡片最大宽度 50%。
- 竖屏：隐藏图标区域，描述文案单行截断，整体紧凑布局。
- 卡片右侧固定展示「兼容评价」标签和向右箭头，点击卡片整体进入兼容性评价详情页。

<font color="#4e73ff">**综合评分展示规则：** **//2026.5.14修改**</font>

| 场景 | 综合评分展示 | 描述文案 |
| --- | --- | --- |
| <font color="#4e73ff">有官方评价</font> | <font color="#4e73ff">官方评级星级</font> | <font color="#4e73ff">官方评价文案</font> |
| 无官方评价，avg ≥ 4.2 | 均分，保留 1 位小数 | 可正常游玩 |
| 无官方评价，avg ≥ 3.2 且 < 4.2 | 均分，保留 1 位小数 | 整体可玩 |
| 无官方评价，avg < 3.2 | 均分，保留 1 位小数 | 兼容问题偏多 |
| 无有效评价 | 显示 `-` | 无兼容性反馈 |

**适配规则：**

| 场景 | 展示规则 | 交互规则 |
| --- | --- | --- |
| 横屏（ls） | 完整图标+评分+描述 | 点击进入详情页，列表正常展示 |
| 竖屏（pt） | 隐藏图标，描述截断单行 | 与横屏一致 |
| 无有效评价 | 评分位置显示「-」和空状态文案 | 仍可点击进入详情页提交反馈 |

### 4.3.2 兼容性评价详情页与反馈链路（C端）

功能 demo：兼容性评价改版 Demo

**页面结构：**

1. 顶部综合评分区：评分、星级、描述文案、评价总数。
2. 中部分段占比区：1–5 星各段占比条形图。
3. <font color="#4e73ff">配置筛选 chip 栏：「同配置」（默认选中）/ 「全部」 / 「支持最多」 / 「我的」，点击切换，当前激活项高亮。**//2026.5.14修改** **//2026.5.14-2修改**</font>
4. <font color="#4e73ff">下部玩家评价列表：分页展示，含头像、昵称、评分、中性标签、文字内容、图片（缩略）、设备信息、APP版本号、发布时间、点赞数、右上角「更多（...）」菜单。**//2026.5.14修改**</font>

**评价列表展示规则：**

- 仅展示状态为「正常」的评价，已隐藏评价不展示，不计入综合评分。
- 设备信息：移动端显示「设备型号 · 系统版本」（如 `iPhone 15 Pro · iOS 17`）；字段缺失时显示「-」。
- <font color="#4e73ff">每条评价展示提交时的 APP 版本号（如 `v6.1.0`），展示在发布日期旁；字段缺失时显示「-」。**//2026.5.14修改**</font>
- 每条评价底部展示点赞数，登录用户可点赞；同一用户对同一评价限点赞一次，再次点击取消点赞。
- <font color="#4e73ff">举报/删除收进评价卡片右上角「更多（...）」菜单，不直接暴露在列表中；举报后菜单内举报选项置灰不可再次点击。**//2026.5.14修改**</font>
- 评价文字超出时截断，提供展开操作。
- <font color="#4e73ff">**官方评价置顶：** 官方特殊账号发布的评价固定置顶展示，标注「官方」标识；用户仍可正常发布自己的评价。**//2026.5.14修改**</font>
- <font color="#4e73ff">**按配置筛选（同配置 chip）：** 进入详情页时，默认选中「同配置」chip，仅展示与当前设备型号相同的评价；不足时列表为空，用户可手动切换至「全部」查看所有评价。不自动回退至全部。**//2026.5.14修改** **//2026.5.14-2修改**</font>

<font color="#4e73ff">**chip 筛选逻辑：** **//2026.5.14-2修改**</font>

| chip | 过滤规则 |
| --- | --- |
| <font color="#4e73ff">同配置（默认）</font> | <font color="#4e73ff">仅展示设备型号与当前用户设备相同的评价</font> |
| <font color="#4e73ff">全部</font> | <font color="#4e73ff">展示所有正常状态评价</font> |
| <font color="#4e73ff">支持最多</font> | <font color="#4e73ff">仅展示标签含「完美兼容」的评价</font> |
| <font color="#4e73ff">我的</font> | <font color="#4e73ff">仅展示当前登录用户提交的评价</font> |

**区服：** 国服包查看中文，海外包统一查看英语。

<font color="#4e73ff">**问题标签规范：** **//2026.5.14修改**

标签采用中性描述，不使用纯负面表述。示例标签方向：「完美兼容」「基本可玩」「有部分问题」「不可玩」。具体标签内容由后台统一配置，支持运营新增/修改。</font>

<font color="#4e73ff">**图片上传与展示规则：** **//2026.5.14修改**

- 本期仅支持图片，不支持视频上传。
- 上传时必须做图片压缩处理。
- **多图展示：** 以 1:1 比例裁切缩略图（九宫格形式）；点击任意图片查看原图完整尺寸。
- **单图展示：** 撑满卡片宽度，高度自适应，点击放大查看原图。
- 展示时自动裁切，不要求用户手动裁切。</font>

**提交反馈流程：**

1. 未登录用户点击「我要反馈」，弹出登录引导弹窗，点击「去登录」进入登录流程。
2. <font color="#4e73ff">已登录用户点击「我要反馈」，系统校验用户在平台的累计游戏时长是否达到门槛（具体时长待定，建议 ≥ 30 分钟；校验平台总时长，非单个游戏时长）。未达门槛时，Toast 提示「需要积累一定游戏时长后才能提交评价」，不打开反馈弹窗。**//2026.5.14修改**</font>
3. 达到时长门槛的已登录用户打开反馈弹窗。
4. <font color="#4e73ff">弹窗字段：星级评分（1–5 星，必填）、问题标签（多选，中性标签，可选）、评价文字（选填，最多 200 字，纯文本，不支持富文本）、图片（可选，最多 9 张，上传须压缩，不支持视频）。**//2026.5.14修改**</font>
5. <font color="#4e73ff">设备信息、APP版本号由系统静默采集，不在弹窗中展示给用户。**//2026.5.14修改**</font>
6. 点击提交后，列表刷新，入口卡片综合评分同步更新。
7. 已提交过的用户再次进入，「我要反馈」按钮置灰不可点击，展示提示：您已提交过该游戏的兼容性反馈。

<font color="#4e73ff">**举报/删除流程：** **//2026.5.14修改**

1. 登录用户点击评价卡片右上角「更多（...）」菜单，展示「举报」（所有评价可见）和「删除」（仅本人评价可见）选项。
2. 点击「举报」，弹出举报原因选择弹窗，选择原因后确认，前台对举报者屏蔽该评价（仅本人不可见），后台记录举报；提交后 Toast 提示「举报已提交，感谢反馈」；举报后菜单内「举报」选项置灰。
3. 同一用户对同一评价限举报一次。</font>
