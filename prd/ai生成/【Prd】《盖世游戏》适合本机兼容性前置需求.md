## 二、背景与目标

背景：当前新用户第一天游戏体验率不到1%，大量用户启动游戏后遭遇黑屏/闪退/兼容性问题而流失。平台已有三层兼容性数据（运营配置、用户评分、帧率上报），但未在前端呈现，用户无法提前知道哪些游戏跟自己设备兼容。

目标：在用户浏览/选择游戏阶段，前置展示经兼容性验证的游戏列表，降低用户盲目启动失败的概率，提升游戏启动成功率和首日留存。

## 三、故事介绍

### 3.1 用户场景

用户打开盖世游戏APP，在首页或游戏库浏览游戏时，看到"适合本机"入口/专题，点击进入后看到的游戏都是经过兼容性验证的。选择一款游戏进入详情页启动，启动成功率显著高于随机浏览启动。

### 3.2 价值分析

- 留存：减少因启动失败导致的新用户流失，提升首日留存
- 效率：用户无需反复试错，直接找到能玩的游戏
- 体验：避免下载大包后发现不兼容的挫败感

### 3.3 核心体验路径

- 首页推荐 → 看到"适合本机"横滑专题 → 点击游戏进入详情页 → 启动游戏
- 首页推荐 → 点击"查看更多" → 进入适合本机专题页 → 搜索/浏览 → 点击游戏
- 游戏库 → PC游戏 → 点击"适合本机"入口 → 进入子页面 → 搜索/排序/浏览 → 点击游戏

### 3.4 产品指标预测

- "适合本机"入口点击率 ≥ 15%
- 从"适合本机"列表启动的游戏成功率 ≥ 85%（对比当前全量启动成功率约56%）
- 新用户首日游戏体验率从 <1% 提升至 5%+

### 3.5 路径规划

- V1.0（当前）：运营人工配置 top 热门游戏的机型兼容数据，前端展示"适合本机"
- V2.0：接入自动化数据源（用户评分、帧率上报），扩大覆盖面

## 四、概要设计

### 4.1 模块设计

- 兼容性数据匹配模块（服务端）
- 首页"适合本机"专题模块（C端）
- 首页专题详情页（C端）
- 游戏库"适合本机"入口+子页面（C端）

### 4.2 详细设计（C端）

功能demo：`demos/首页与探索/游戏库demo-适合本机.html`

#### 4.2.1 兼容性数据匹配逻辑

客户端启动时上报当前设备信息（GPU型号、CPU型号、设备型号），服务端按以下优先级匹配：

|优先级|数据源|匹配条件|说明|
|---|---|---|---|
|P1|运营兼容性配置库|运营人工标记该机型+该游戏=兼容|最高可信度，人工验证|
|P2|用户兼容性评分|同机型评分 ≥ X 且评论数 ≥ X 人|社区反馈证实可玩|
|P3|帧率稳定性数据|同机型运行 ≥ X 分钟且人数 ≥ X|自动采集的稳定性证据|
|P4|无数据|以上均无匹配|判定为不适配，不展示|

- 匹配结果 < X 款游戏时，隐藏所有"适合本机"入口，不影响其他功能
- X 的具体阈值待定，建议 ≥ 5 款时展示

#### 4.2.2 功能模块总览

|模块名称|图示|展示&交互说明|
|---|---|---|
|首页推荐逻辑改造|参见demo首页|① "为你推荐"：推荐池优先取适配当前设备的游戏，不足时非适配补位<br>② "猜你喜欢"（全部游戏区）：适配游戏优先排序，非适配排后面<br>③ 首页不做标签标注（推荐的本身就是适配的）<br>④ 不适配游戏不在首页推荐中展示|
|搜索结果页标签|参见demo搜索结果|① 位置：搜索结果页 - 综合tab游戏部分 + 游戏tab - PC游戏<br>② 适配游戏卡片右侧显示绿色标签「本机适配」<br>③ 仅可云游（不适配本机）卡片右侧显示金色标签「可云游」<br>④ 不适配游戏卡片右侧显示灰色标签「不适配」+ 封面加半透明黑色蒙层（50%透明度）<br>⑤ 搜索结果排序：适配游戏优先，不适配排后面<br>⑥ 点击游戏卡片 → 进入游戏详情页（不改变现有链路）|
|游戏库PC游戏卡片|参见demo游戏库|① 游戏卡片封面右上角显示适配状态标签（三种状态同上）<br>② 不适配游戏封面加蒙层变暗<br>③ 来源图标（Steam/Epic/导入）保留在左上角，不变|

#### 4.2.3 适配状态标签规则

两种状态：

|状态|判定条件|标签|颜色|封面处理|
|---|---|---|---|---|
|本机适配|PC模拟器兼容（compat=perfect或good）|「本机适配」|绿色 #10b981|正常展示|
|不适配|PC模拟器不兼容|「不适配」|灰色 rgba(255,255,255,0.35)|正常展示（不加蒙层）|

**云游标识：** 不单独做"可云游"标签，支持秒玩的游戏在卡片右下角显示绿色「秒玩」按钮（复用现有组件）。

**展示位置及规则：**

|展示位置|适配游戏|不适配游戏|秒玩按钮|
|---|---|---|---|
|首页推荐（为你推荐/猜你喜欢）|不标记标签（推荐算法已过滤）|不展示（算法排除）|支持秒玩的显示|
|搜索结果页-综合tab|评分右侧显示绿色「本机适配」文字|评分右侧显示灰色「不适配」文字|—|
|搜索结果页-PC游戏tab|卡片右上角绿色「本机适配」胶囊|卡片右上角灰色「不适配」胶囊|支持秒玩的显示|
|游戏库PC游戏|卡片右上角绿色「本机适配」胶囊|卡片右上角灰色「不适配」胶囊|支持秒玩的显示|

**设计原则：**
- 不加蒙层，不降低不适配游戏的封面透明度——只用标签文字区分
- 不标"可云游"——用现有"秒玩"按钮承载云游信息，避免标签重复
- 首页不标记任何标签——推荐的本身就是适配的，标了等于满屏标签无意义

#### 4.2.4 现有推荐逻辑改动点

基于《个性化推荐需求》V1.2 的现有推荐体系，本需求新增"本机适配"作为推荐过滤/权重条件：

|改动位置|现有逻辑|新增逻辑|说明|
|---|---|---|---|
|首页"为你推荐"|ai推荐池按兴趣标签生成，取12款游戏|推荐池生成时，增加"本机适配"过滤条件：优先推送兼容当前设备的游戏|适配游戏不足12款时，用非适配游戏补齐（但适配游戏排前面）|
|首页"猜你喜欢"（全部游戏区）|ai推荐池按兴趣标签循环填充|推荐池结果按"本机适配"权重重排：适配游戏优先展示|不过滤非适配游戏，只调整排序权重|
|搜索结果页|按关键词匹配展示|搜索结果按适配优先排序 + 卡片加状态标签|不隐藏不适配游戏，标签告知用户|

**规则说明：**

- "为你推荐"：在现有 ai 推荐池 + 兴趣标签的基础上，增加一层过滤——优先从推荐结果中取本机适配的游戏；适配游戏不足时，用非适配游戏补位，保证总数 12 款不变
- "猜你喜欢"：不硬过滤（避免列表太短），而是调整排序权重——适配游戏排前面，非适配游戏排后面
- 搜索结果：不隐藏任何游戏，适配的排前面，不适配的排后面并加标签+蒙层
- 兜底逻辑不变：跳过用户仍按热度池/潜力池/口碑池循环填充，但同样优先展示适配游戏

#### 4.2.4 国内海外差异

|差异点|国内包|海外包|
|---|---|---|
|功能可用性|**国内海外均可用**|同左|
|云游戏秒玩入口|展示|**不展示**（海外无云游）|
|语言|中文|跟随系统语言|

说明："适合本机"基于 PC 模拟器内核的兼容性数据，国内海外共用同一套内核，数据源和展示逻辑一致。唯一差异是推荐列表中涉及秒玩的游戏，海外包隐藏秒玩入口。

#### 4.2.5 边界条件

|场景|处理方式|
|---|---|
|兼容游戏 < X 款|隐藏所有"适合本机"入口（首页专题模块+游戏库入口卡片）|
|设备信息获取失败|不展示"适合本机"功能，不影响其他功能|
|搜索无结果|展示"未找到相关游戏"空状态|
|用户换设备登录同一账号|按当前设备重新匹配，列表实时更新|
|游戏卡片封面加载失败|展示默认占位图|
|弱网/断网|展示"网络异常，点击重试"|
|搜索结果页|不做兼容性标注（不拦截用户主动搜索行为）|

## 五、非功能需求

|需求类型|详细要求|
|---|---|
|性能|"适合本机"列表接口响应 ≤ 500ms；设备信息上报不阻塞首页加载|
|兼容性|Android 8.0+；获取设备信息使用标准 API（Build.MODEL、Build.HARDWARE），无需额外权限|
|容错|兼容性数据服务异常时，隐藏"适合本机"入口，不影响其他功能|
|缓存|兼容游戏列表本地缓存 24 小时，减少重复请求|

### 6.1 埋点事件表

|事件ID|事件名称|触发时机|关键参数|
|---|---|---|---|
|fit_device_section_exposure|适合本机专题曝光|首页"适合本机"横滑模块曝光|`position`、`game_count`|
|fit_device_section_click|适合本机专题点击|点击"查看更多"进入专题页|`source`（home/library）|
|fit_device_entry_click|适合本机入口点击|游戏库点击"适合本机"入口卡片|`game_count`|
|fit_device_game_click|适合本机游戏点击|在适合本机列表中点击某游戏|`game_id`、`position`、`source`|
|fit_device_search|适合本机搜索|在适合本机页面搜索|`keyword`、`result_count`、`source`|
|fit_device_launch_result|适合本机启动结果|从适合本机列表启动游戏后的结果|`game_id`、`is_success`、`device_model`、`gpu`|

|参数名|类型|必填|说明|枚举/示例|
|---|---|---|---|---|

|game_id|string|是|游戏唯一标识|`steam_1245620`|
|position|int|是|游戏在列表中的位置（从1开始）|`1`、`2`、`3`|
|keyword|string|是|搜索关键词|`艾尔登`|
|result_count|int|是|搜索结果数量|`3`|

|device_model|string|是|设备型号|`Xiaomi 12S Pro`|
|gpu|string|是|GPU型号|`Adreno 730`|

## 七、运营需求

1. 首批上线需运营配置 top 热门游戏（≥50款）的机型兼容性数据
2. 配置维度：游戏ID + GPU型号/CPU型号 → 兼容/不兼容
3. 后续接入自动化数据源后，运营配置作为最高优先级覆盖项保留
4. 需按周期（建议每周）更新兼容性配置，跟随内核版本迭代
