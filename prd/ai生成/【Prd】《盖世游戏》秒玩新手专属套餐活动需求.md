## 二、背景与目标

**背景：** 秒玩（云游戏免费试用）当前付费转化率仅7-8%，93%用户看到充值页后离开。新用户缺乏首购动力，常规SKU价格缺乏新人专属感知。

**目标：** 通过新人限时特惠套餐+签到赠送机制，将新用户秒玩首购转化率从7-8%提升至12%+，同时通过6天签到拉动D1-D6回访率。

**核心挑战：** 如何在不打乱现有充值体系的前提下，给新用户一个"值得立即付费"的理由，并建立连续回访的正向循环。

## 三、故事介绍

### 3.1 用户与运营场景

**场景一（新用户首次体验秒玩）：**
小明下载盖世游戏APP，进入云游戏tab → 点击秒玩GTA5 → 体验5分钟后提示"充值金额不足" → 跳转充值中心 → 看到"新人专享限时福利"弹窗 → 选择8小时体验套餐¥6.9 → 支付成功 → 立即到账2小时继续游玩 → 后续6天每天打开APP自动获得1小时时长

**场景二（运营配置活动）：**
运营在后台配置新手套餐的SKU内容、价格、有效期、弹窗触发频率 → 活动上线后监控弹窗展示量、点击率、购买转化率 → 根据数据调整套餐定价或触发策略

### 3.2 价值分析

- 变现：降低新用户首购门槛，从¥6降至¥6.9但赠送8小时（感知价值远超实付）
- 留存：6天签到机制驱动用户连续6天回访，养成打开APP习惯
- 转化：体验完8小时后用户已形成付费习惯，续费意愿显著提升
- 数据：通过活动埋点获取新用户付费决策路径数据

### 3.3 核心体验路径

**用户侧：**
首页-云游戏tab / 充值中心 → 新手套餐弹窗展示 → 选择套餐 → 确认支付 → 时长到账 → 6天签到领取 → 时长消耗（秒玩游戏时才扣）

**运营侧：**
后台配置套餐 → 设置触发规则 → 活动上线 → 监控数据 → 调整策略

### 3.5 路径规划

- V1.0（当前）：2个新手SKU + 弹窗触发 + 悬浮球 + 6天签到 + 后台配置
- V2.0（规划）：根据数据优化定价策略、新增套餐档位、签到奖励梯度化

#### 4.3.1 边界条件-内容侧

|场景|处理方式|
|---|---|
|套餐价格配置为0|校验拦截，提示"价格不可为0"|
|原价低于现价|校验拦截，提示"原价需高于现价"|
|立即到账时长为0|校验拦截，提示"到账时长不可为0"|
|活动时间结束时间早于开始时间|校验拦截，提示"结束时间需晚于开始时间"|
|活动进行中修改套餐内容|即时生效，已购买用户不受影响，未购买用户看到新内容|
|活动进行中关闭活动|所有入口立即消失，已购买用户签到不受影响|
|删除活动|非进行中状态才可删除，二次确认后硬删除|

#### 4.3.2 边界条件-用户侧

|场景|处理方式|
|---|---|
|用户未登录时看到弹窗→点击购买|弹登录弹窗→登录成功后回到套餐弹窗继续购买|
|登录后发现账号有历史付费记录|Toast"抱歉，您不符合该套餐购买条件"→关闭弹窗→隐藏所有活动入口|
|A设备购买→B设备登录同账号|B设备同步状态为"已购买"，展示签到进度，不再展示套餐弹窗|
|支付失败/取消|返回弹窗，保持可再次购买状态|
|购买套餐A后想买套餐B|不允许，任意套餐购买成功即活动结束|
|活动有效期内套餐被后台下架|弹窗不再展示，悬浮球消失，等同活动结束|
|用户卸载重装（同账号）|以账号维度判断，重装后状态不变|
|签到期间时长全部用完|正常到账，用户下次进秒玩时有余额可用|
|新手弹窗和其他弹窗冲突|新手套餐弹窗优先级最高，其他弹窗排队等待|
|游戏暂停跳充值中心时|游戏暂停，秒玩时间不继续倒计时|
|时长有效期到期|到期时长作废，不可恢复，签到到账的时长各自独立计算30天有效期|
|网络异常时点击购买|Toast"网络异常，请稍后重试"，弹窗保留|
|活动到期后用户再进入相关页面|无任何活动入口展示，和普通用户无差异|

## 五、非功能需求

|需求类型|详细要求|

## 七、运营需求

1. 活动上线前需准备：2个SKU的定价方案、用户充值协议更新、弹窗视觉素材
2. 需配置弹窗触发频率的初始值（建议：首周1日1次）
3. 需准备活动到期前1天的Push推送文案（如："你的新人福利即将过期，还剩X小时可享受"）
4. 上线后每日监控：弹窗展示→购买转化率，如低于10%需调整定价或触发策略
5. 签到完成率如低于40%，考虑增加Push提醒"今日时长待领取"
