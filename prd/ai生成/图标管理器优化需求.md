# 二、背景与目标

**背景：** 现有的图标管理器功能较为基础，仅支持平铺式图标列表浏览，缺乏分组视图与文件夹视图的切换能力，图标选择器在横屏模式下列数固定导致图标被拉伸变形，且选择面板中的"无图标"选项在迭代过程中被意外丢失，影响了用户清除按键图标的能力。同时，导入图标包的弹窗体验存在入口不一致、封面编辑交互不直观等问题。

**目标：** 通过本次图标管理器全面优化，提升玩家自定义虚拟按键布局时的图标管理体验，支持更丰富的浏览视图切换、更规范的图标包管理流程，以及更完善的图标选择器交互，为后续图标生态扩展奠定基础。

# 三、故事介绍

在切入繁杂的功能点前，我们先通过典型用户视角，来看看本次优化功能的运转方式。

## 3.1 用户与运营场景

**C端玩家（张三）：**

张三打开虚拟按键布局编辑器，想给自己的"A"按键换一个像素风格的图标。他进入图标管理器，通过"文件夹视图"快速找到"像素风格包"，点进去后预览了几个图标，最终选中了心仪的图标并应用。当他觉得某个按键不需要任何图标时，在图标选择器中点击第一格的"无"选项，一键清除图标。此外，他还想导入自己从网上找到的一套图标资源，通过"+"按钮选择本地文件夹，系统自动将每个子文件夹识别为独立图标包，一次性完成批量导入。

**进阶玩家（李四）：**

李四是一个重度定制用户，他拥有多个自定义图标包，平时用"图标视图"整体浏览所有图标，切换到"文件夹视图"管理各个图标包，并为每个包设置了独特的封面图。他在"我的"标签下管理自己的所有资源，官方提供的图标包则在"官方"标签下单独陈列，两者互不干扰。

## 3.2 价值分析

- **提升定制效率：** 分组视图与文件夹视图双视图切换，让不同习惯的用户都能快速定位目标图标，减少无效滚动。
- **补全关键能力缺口：** 图标选择器中"无"选项的恢复，补全了清除按键图标这一高频操作，避免用户绕道删除再重建按键。
- **横屏体验一致性：** 图标选择器横屏模式修正列宽固定逻辑，杜绝图标被拉伸变形的视觉问题。
- **批量导入生态：** 支持多文件夹批量导入，自动识别包结构，加速用户自建图标生态的积累。

## 3.3 核心体验路径

**图标浏览与选择核心链路：**

进入编辑器 → 点击按键 → 打开图标选择器 → 搜索/推荐/文件夹浏览 → 选中图标（或选"无"清除） → 图标应用至按键

**图标包管理核心链路：**

进入图标管理器 → 切换官方/我的标签 → 切换图标视图/文件夹视图 → 导入图标包（单/多文件夹） → 设置封面 → 点击包进入详情 → 预览图标

## 3.4 产品指标预测

- **图标选择器打开率：** 编辑态按键点击后图标面板打开比例，预测首月 ≥ 60%。
- **"无"选项使用率：** 预测约 10%-15% 的图标选择操作会选择清除图标。
- **自定义图标包导入率：** 拥有自定义图标包的用户中，预测 30% 会在首月尝试导入自建图标包。
- **视图切换率：** 进入图标管理器的用户中，预测 40% 会主动切换一次视图模式。

## 3.5 路径规划

**当前版本（V1.0）：** 完成图标管理器双视图切换（图标视图/文件夹视图）、图标选择器"无"选项恢复、横屏多列修正、导入图标包弹窗居中改版、封面编辑功能。

**未来可能升级（预告）：** 支持图标包在线商城、官方图标包分类筛选、图标拖拽排序、图标包导出/分享。

# 四、概要设计

通过前面的背景介绍，相信大家对本次优化价值已具备体感，接下来转为研发视角的模块拆解。

## 4.1 模块设计（功能架构概览）

本次功能涉及的系统模块归纳如下：

**【图标管理器模块】**

- **主页视图层：** 官方/我的双标签、图标视图/文件夹视图切换、图标包卡片/图标格展示。
- **详情页：** 单个图标包的图标全览与大图预览弹层。
- **导入流程：** 本地文件夹导入 → 自动分包 → 完成弹窗 → 封面编辑。

**【图标选择器模块（嵌入编辑器）】**

- **推荐区：** 展示适合当前按键类型的图标，含"无"选项，支持展开收起。
- **全部图标区：** 文件夹视图入口 + 搜索过滤。
- **横屏适配：** 修正列宽，使用固定 px 列而非 1fr，避免图标拉伸。

## 4.4 非功能需求

| 需求类型 | 详细要求 |
| --- | --- |
| 性能 | 1. **图标渲染速度：** 图标管理器切换标签或视图时，列表重建响应时间不超过 300ms，无明显闪烁。2. **图片加载：** 自定义图标包的 base64 图片优先内存缓存，二次打开无需重复解码。3. **导入处理：** 单次导入 100 张以内图片时，FileReader 批处理完成时间不超过 3s，进度状态有反馈。 |
| 兼容性 | 1. **横竖屏适配：** 图标管理器与图标选择器在横屏/竖屏切换时，列数与卡片尺寸自动适应，无拉伸变形。2. **webkitdirectory 支持：** 文件夹选择功能依赖 webkitdirectory，需在不支持的环境下降级为单文件上传并提示。3. **图片格式兼容：** 支持 PNG、JPG、SVG、WEBP 等主流图片格式导入。 |
| 用户体验 | 1. **视觉一致性：** 图标管理器整体采用深色电竞风，图标包卡片圆角、背景色、悬停效果与编辑器整体设计语言一致。2. **即时反馈：** 所有点击操作（导入、封面编辑、视图切换）均在 200ms 内有视觉响应（高亮变色或 Loading 态）。3. **容错体验：** 图标加载失败时统一展示灰色占位图，不出现裂图；导入失败时以 Toast 明确告知原因。 |

# 五、交付设计（数据分析与埋点设计）

## 5.1 核心业务指标

**关键漏斗：**

进入编辑器 → 打开图标选择器 → 浏览/搜索图标 → 选中图标（含"无"） → 图标应用至按键

**辅助指标：**

- 图标管理器打开次数（UV/PV）
- 视图切换次数（图标视图 vs 文件夹视图）
- 自定义图标包导入次数
- 封面编辑次数

## 5.2 埋点事件表

| 事件ID | 事件名称 | 触发时机 | 关键参数 |
| --- | --- | --- | --- |
| icon_mgr_open | 图标管理器打开 | 用户点击进入图标管理器页面时 | uid, source (editor/settings) |
| icon_tab_switch | 图标管理器标签切换 | 点击"官方"或"我的"标签时 | tab_name (official/mine) |
| icon_view_toggle | 视图模式切换 | 点击视图切换按钮时 | from_view, to_view (icon/folder) |
| icon_pkg_open | 图标包详情打开 | 文件夹视图下点击图标包卡片时 | pkg_id, pkg_name, icon_count |
| icon_preview | 图标大图预览 | 详情页点击图标弹出预览弹层时 | pkg_id, icon_name |
| icon_picker_open | 图标选择器打开 | 编辑态点击按键后图标面板展开时 | btn_type (c/joy/r) |
| icon_picker_select | 图标选择确认 | 在图标选择器中点击任意图标格时 | icon_name, is_none (true/false), source (recommend/folder) |
| icon_picker_search | 图标搜索 | 在选择器搜索框输入关键词时 | keyword, result_count |
| icon_recommend_expand | 推荐区展开/收起 | 点击"展开全部"或"收起"时 | action (expand/collapse) |
| icon_import_trigger | 导入图标包触发 | 点击"+"导入按钮时 | source (icon_mgr/icon_picker) |
| icon_import_complete | 导入图标包完成 | 导入流程全部完成、弹窗展示时 | pkg_count, total_icon_count |
| icon_cover_edit | 封面编辑触发 | 导入完成弹窗内点击封面编辑按钮时 | pkg_id |
| icon_cover_apply | 封面编辑完成 | 用户选择图片后封面成功更新时 | pkg_id |
