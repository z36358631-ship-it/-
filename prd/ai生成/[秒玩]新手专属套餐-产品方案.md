# [秒玩] 新手专属套餐活动 — 产品方案

> 版本：V1.0 | 日期：2026-06-30
> 来源：运营活动说明 → 产品侧补全交互流程、异常处理、配置项、埋点

---

## 一、需求背景

秒玩（云游戏免费试用）当前数据：
- 15分钟免费体验 → 弹充值页 → 93%用户看到价格走了
- 付费转化仅7-8%

核心矛盾：用户体验了价值（人均游玩43分钟），但15分钟截断点太生硬，直接弹通用充值页没有新人感知的"专属优惠"。

---

## 二、活动目标

| 指标 | 当前值 | 目标 |
|------|--------|------|
| 新用户秒玩付费转化率 | 7-8% | 12%+ |
| 新用户D1留存（玩过秒玩的） | ~20% | 28%+ |
| 新用户7日回访率 | — | 50%+（签到拉动） |

---

## 三、活动对象

安装APP后7天内的新用户，同时满足：
1. 账号维度（非设备维度）无任何历史付费记录
2. 活动未结束（未购买 + 未超时）
3. 活动后台处于上线状态

---

## 四、核心用户流程

### 4.1 触发链路

```
新用户安装APP
    ↓
进入秒玩，开始免费体验
    ↓
5分钟时 → 系统提示"充值金额不足" → 游戏暂停 → 跳转充值中心
    ↓
充值中心正常展示所有SKU（底层）
    ↓
新手套餐弹窗覆盖在充值中心上层（首次触发，记录触发状态）
    ↓
同时/备选触发：
  如果用户未在充值中心触发过 → 进入首页-云游戏tab时触发弹窗
```

### 4.2 弹窗交互

**弹窗内容：**
- 标题：新人限时福利
- 倒计时：距活动结束 X天X时X分（首次弹窗开始计算7天）
- 套餐A：短时长SKU（价格、时长、描述）
- 套餐B：长时长SKU（价格、时长、描述，标注"推荐"）
- 签到说明："购买后连续7天每日自动到账X分钟"
- 关闭按钮（右上角X）

**用户操作：**
| 操作 | 结果 |
|------|------|
| 点击购买套餐A/B | → 调起支付 → 成功后到账时长 + 解锁签到 → 活动结束 |
| 点击关闭 | → 关闭弹窗 → 回到充值中心/当前页面 → 悬浮球出现 |
| 点击弹窗外区域 | → 不关闭（强引导） |

### 4.3 悬浮球

**展示条件：** 弹窗被关闭后 + 活动未结束期间
**展示位置：** 首页-推荐tab、首页-云游戏tab（页面右下角）
**内容：** "限时福利" + 倒计时
**交互：** 点击 → 打开新手套餐弹窗

### 4.4 弹窗触发频率

后台可配置，支持以下模式：
| 模式 | 说明 |
|------|------|
| 永久1次 | 只弹1次，关闭后只有悬浮球 |
| 1日1次 | 每天首次进入触发页面时弹出 |
| X日1次 | 每X天触发1次（X可配） |

触发页面：秒玩板块（每日首次进入时） + 首页-云游戏tab（如充值中心未触发过）

### 4.5 购买后签到阶段

**触发条件：** 用户成功购买任意新手套餐
**签到规则：**
- 每天打开APP → 自动到账X分钟（无需手动点击领取）
- 连续7天，断签不补发（当天未打开APP则当天奖励作废）
- 时长为累计游玩时长，只有玩秒玩时才扣

**签到进度展示位置：** 首页-推荐tab + 首页-云游戏tab
**展示内容：** 7天日历卡片（已领/今日已领/待领/已过期），剩余可领天数

**签到结束：** 7天到期后签到模块从页面消失

---

## 五、活动结束条件

满足任一条件即结束：
1. 用户购买任意一个新手套餐成功
2. 首次弹窗后超过7天仍未购买（倒计时归零）
3. 活动后台下线或套餐配置失效

活动结束后：
- 弹窗不再展示
- 悬浮球消失
- 用户无法进入活动购买页
- 签到模块不受影响（如已购买，签到按正常7天走完）

---

## 六、异常流程

| 场景 | 处理方式 |
|------|---------|
| 用户未登录时看到弹窗 → 点击购买 | 弹登录弹窗 → 登录成功后回到新手套餐弹窗继续购买流程 |
| 登录后发现该账号有历史付费记录 | Toast提示"抱歉，您不符合该套餐购买条件" → 关闭弹窗 → 隐藏所有活动入口 |
| 用户在A设备购买 → B设备登录同账号 | B设备同步活动状态为"已购买"，展示签到进度，不再展示弹窗 |
| 支付失败/取消 | 返回弹窗，保持可再次购买状态 |
| 套餐A购买后想买套餐B | 不允许，任意套餐购买成功即活动结束 |
| 活动有效期内套餐被后台下架 | 弹窗不再展示，悬浮球消失，等同活动结束 |
| 用户卸载重装（同账号） | 以账号维度判断，重装后状态不变 |
| 签到期间用户时长用完了 | 正常到账，用户下次进秒玩时有余额可用 |

## 八、状态机

```
[未触发] → 用户进入触发页面 → [已触发/活动进行中]
    ↓ 购买成功
[已购买/签到进行中] → 7天签到完成 → [签到结束]
    ↓ 超时未购买
[已过期] → 所有入口消失

状态存储维度：账号级别（非设备级别）
```

| 状态 | 弹窗 | 悬浮球 | 签到模块 |
|------|------|--------|---------|
| 未触发 | 不展示 | 不展示 | 不展示 |
| 已触发/活动中 | 按频率展示 | 展示（弹窗关闭后） | 不展示 |
| 已购买/签到中 | 不展示 | 不展示 | 展示 |
| 签到结束 | 不展示 | 不展示 | 不展示 |
| 已过期 | 不展示 | 不展示 | 不展示 |

## 九、埋点事件表

| 事件名 | 触发时机 | 关键参数 |
|--------|---------|---------|
| newbie_popup_show | 新手套餐弹窗展示 | trigger_source（充值中心/云游戏tab/秒玩）、show_count（第几次展示） |
| newbie_popup_close | 用户关闭弹窗 | duration_on_popup（弹窗停留时长） |
| newbie_popup_click_buy | 用户点击购买按钮 | sku_id、sku_price |
| newbie_purchase_success | 购买成功 | sku_id、sku_price、time_since_install（安装后第几天） |
| newbie_purchase_fail | 购买失败 | sku_id、fail_reason |
| newbie_float_show | 悬浮球展示 | page（推荐/云游戏） |
| newbie_float_click | 悬浮球点击 | remaining_days（剩余天数） |
| newbie_checkin_receive | 签到到账 | day_index（第几天）、minutes_received |
| newbie_checkin_miss | 断签（当天未打开APP） | day_index |
| newbie_activity_expire | 活动到期未购买 | total_popup_shown（总弹窗次数）、total_float_click |
| newbie_login_block | 登录后发现不符合条件 | reason（has_purchase_history） |

## 十、核心指标看板

| 指标 | 计算方式 | 预期 |
|------|---------|------|
| 弹窗展示→购买转化率 | purchase_success / popup_show | 12%+ |
| 悬浮球点击率 | float_click / float_show | 15%+ |
| 签到完成率 | 完成7天用户 / 购买用户 | 60%+ |
| 签到拉动D7留存 | 购买用户D7留存 vs 未购买用户D7留存 | +15pp |
| 平均首购时间 | 安装到购买的天数分布 | D0-D1集中 |

## 十一、补充说明

1. 活动为新用户一次性专属福利，不支持重复参与
2. 判断维度为账号级别，换设备/卸载重装不影响状态
3. 套餐与正常充值SKU互斥展示：新手弹窗覆盖在充值中心上层时，用户看到的第一层是新手套餐；关闭后才能操作底层充值中心
4. 时长为累计游玩制：只有在秒玩中才消耗，非实时倒计时
5. 签到到账时间：每日00:00重置，用户当天任意时刻打开APP即到账
