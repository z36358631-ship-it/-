# 二、 背景与目标

**背景：**
当前游戏库页面的竖版宽度（390px）不足以完整展示Steam/Epic/导入游戏三个入口卡片的主副标题，横版宽度（860px）导致游戏封面一排仅能展示4~5个，信息密度不够。同时横屏入口缺少副标题说明，用户无法快速理解各入口功能；Steam/Epic子页面在未绑定态仍显示右上角头像，交互逻辑不合理；复古游戏页搜索栏右侧分类按钮对当前内容量来说功能冗余。

**目标：**
1. 扩展游戏库容器尺寸，使横竖屏均有足够空间完整展示入口和游戏列表。
2. 横屏入口卡片增加副标题，一眼可读。
3. 修正Steam/Epic绑定态头像显隐逻辑，绑定前不展示。
4. 移除复古游戏页冗余分类按钮，简化搜索栏。

**核心挑战：** 尺寸扩展后横竖屏切换布局的流畅适配、绑定态与非绑定态的状态切换动效一致性。

# 三、 故事介绍

## 3\.1 用户与运营场景

**用户与运营场景：**

- **PC游戏用户（张三）：** 张三打开游戏库，竖屏PC游戏Tab下看到Steam/Epic/导入游戏三个入口，每个入口主标题+副标题（"查看游戏"/"手动添加"）一行展示。他点击Steam入口进入子页面，页面仅显示返回+标题，无头像干扰，居中展示"绑定Steam账号"引导。点击立即绑定后，头像出现，游戏列表展示。

- **横屏用户（李四）：** 李四横屏使用，进入游戏库后看到Steam/Epic/导入游戏入口均带副标题。进入Steam子页面绑定后，游戏封面一排展示7个，信息密度合理，快速找到目标游戏。

- **复古游戏用户（小明）：** 小明切到复古游戏Tab，搜索栏简洁无分类按钮，导入游戏入口固定宽度不撑满整行，与PC游戏导入入口视觉一致。

## 3\.2 价值分析

**价值分析**

- **提升信息密度：** 横版1200px宽度下游戏封面一排7个，浏览效率显著提升。
- **降低认知成本：** 入口副标题让用户无需试探即可理解功能。
- **交互逻辑自洽：** 未绑定不显示头像，避免用户困惑"这个头像是谁"。

## 3\.3 核心体验路径

**竖屏核心链路：**
游戏库 → PC游戏Tab（Steam/Epic/导入游戏入口）→ 点击Steam → 绑定缺省态（垂直居中）→ 立即绑定 → 头像出现+游戏列表展示 → 浏览/搜索游戏

**横屏核心链路：**
游戏库 → PC游戏/复古游戏Tab切换 → 入口卡片（带副标题）→ 进入子页面 → 游戏网格一排7个

## 3\.4 路径规划

- **当前版本（V1\.0）：** 完成尺寸调整、入口副标题、绑定态逻辑、分类按钮移除、网格列数调整。



# 四、概要设计

## 4\.1 模块设计（系统架构图概览）

本次功能涉及的系统模块归纳如下：

- **【游戏库容器模块】**

    - **竖版容器：** 宽度390→520px，高度844→836px。

    - **横版容器：** 宽度860→1200px，高度460→520px。

- **【平台入口模块】**

    - **横屏入口卡片：** Steam（副标题"查看游戏"）/ Epic（副标题"查看游戏"）/ 导入游戏（副标题"手动添加"），主副标题均一行展示。

    - **复古游戏入口：** 仅"导入游戏"一个入口，固定宽度不撑满。

- **【绑定态模块】**

    - **未绑定态：** 隐藏头像，展示绑定缺省态（垂直居中）。

    - **已绑定态：** 显示头像+游戏列表。

    - **退出登录：** 隐藏头像，回到未绑定态。

- **【复古游戏搜索模块】**

    - 移除搜索栏右侧分类按钮和排序按钮。

## 4\.4 非功能需求

|**需求类型**|**详细要求**|
