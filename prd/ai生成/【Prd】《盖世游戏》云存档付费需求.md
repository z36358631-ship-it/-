## 二、背景与目标

❗背景：

盖世用户通过Wine模拟器玩PC游戏，存档管理面临三个痛点：1）从零开始肝太耗时间；2）Wine Prefix崩溃/换手机导致存档丢失；3）想体验全收集但不想自己打。竞品已有付费存档模式（单次购买¥6/个），市场验证了付费意愿。盖世的独特优势是控制Wine运行环境，可以做到"一键导入即可玩"。

❗目标：

在APP内建立存档付费交易体系，官方提供精品存档打底，同时开放玩家上传定价。核心目标是跑通"浏览→购买→一键导入→直接玩"的商业闭环。

核心挑战：存档文件安全扫描、Wine Prefix路径自动识别与导入、支付回调一致性、虚拟商品退款策略。

## 三、故事介绍

### 3\.1 用户与运营场景

❗用户与运营场景：

- **C端买方（张三）**：张三想玩GTA5线上模式但不想肝几百小时。他在游戏详情页点击"存档"tab，看到一个"500万CR+440辆车全收集"的存档，标价¥6.00，877人已购买。他点击购买，微信支付¥6后，存档出现在"我的存档"中。他点击"一键导入"，系统自动将存档放入Wine Prefix正确路径。启动游戏后直接从满级状态开始玩。

- **C端卖方（李四）**：李四通关了艾尔登法环全Boss，他在"我的存档"中点击"上传"，选择存档文件，填写标题"全Boss击杀+全武器收集"，定价¥5.00，提交审核。审核通过后上架，一周内卖出50份，收益¥175（扣除30%平台抽成）。

- **B端运营（王五）**：王五在后台"存档管理"中为Top 20热门游戏各上传2-3个官方精品存档（通关/全收集/速通），标记为"官方精品"，设置价格¥6-15。同时审核玩家上传的存档，通过安全扫描和可用性验证后上架。

### 3\.2 价值分析

❗价值分析：

- **直接变现**：存档单价¥3-15，平台抽成30%（玩家存档）或100%收入（官方存档）
- **提升留存**：买了存档的用户有更强的游戏动力，游戏时长和回访率提升
- **供给飞轮**：玩家上传存档赚钱→更多人上传→存档覆盖面更广→更多人购买
- **差异化壁垒**：一键导入到Wine Prefix是盖世独有能力，竞品做不到

### 3\.3 核心体验路径

❗核心链路：

- **买方核心链路**：
  游戏详情页"存档"tab → 浏览存档列表 → 查看详情 → 支付购买 → 我的存档 → 一键导入 → 启动游戏

- **卖方核心链路**：
  我的页 → 我的存档 → 上传存档 → 选择游戏+文件+定价 → 提交审核 → 上架 → 收益到账

- **运营核心链路**：
  后台存档管理 → 新建官方存档 → 上传文件+配置 → 发布 → 审核玩家存档 → 通过/驳回

### 3\.4 产品指标预测

❗产品指标预测：

1. 存档浏览→购买转化率：首月预测 3-5%
2. 一键导入成功率：目标 > 95%
3. 首批覆盖：Top 20游戏，每游戏2-3个官方存档
4. 玩家上传：首月预测50+个玩家存档通过审核

### 3\.5 路径规划

❗路径规划：

- **V1.0（当前）**：跑通官方存档上架+用户购买+一键导入闭环；开放玩家上传+审核+定价
- **V1.5（预告）**：试玩功能（加载存档体验5分钟再决定买不买）
- **V2.0（预告）**：VIP用户免费下载所有存档；存档+Wine配置打包为"游戏快照"

## 四、概要设计

### 4\.1 模块设计

- 【C端】存档商城（游戏详情页"存档"tab）
- 【C端】存档详情页
- 【C端】我的存档（已购买+已上传）
- 【C端】上传存档
- 【B端】存档管理（官方上架+玩家审核）

### 4\.2 详细设计

|**模块名称**|**图示**|**展示&交互说明**|
|---|---|---|
|**存档商城（游戏详情页"存档"tab）**|（见demo）|- **入口**：游戏详情页顶部Tab栏增加"存档"tab<br>- **列表分区**：上方"官方精品"区（带金色"官方"标签）+ 下方"玩家分享"区<br>- **存档卡片**：封面图 + 标题（单行截断）+ 价格（¥X.XX）+ 已购买人数 + 作者昵称<br>- **排序**：官方存档按运营排序；玩家存档按购买人数降序<br>- **交互**：点击卡片进入存档详情页<br>- **异常处理**：该游戏无存档时显示"暂无存档，成为第一个分享者 →"（点击跳转上传页）|
|**存档详情页**|（见demo）|- **顶部**：作者头像+昵称（官方存档带金色标签）+ 右侧价格按钮<br>- **封面区**：存档截图（支持左右滑动多张预览）<br>- **信息区**：标题（加粗）+ 描述正文 + 存档内容预览截图<br>- **元数据**：适用游戏+游戏版本+存档大小+上传时间+已购买人数<br>- **购买按钮**：底部固定"¥X.XX 购买"（已购买则显示"已购买-去导入"）<br>- **不退款提示**：购买按钮上方灰色小字"存档为虚拟商品，购买后不支持退款"<br>- **交互**：点击购买→拉起支付（微信/支付宝）→支付成功→toast"购买成功"+按钮变为"已购买-去导入"<br>- **异常处理**：未登录点击购买→弹登录引导；未实名→弹实名认证引导；支付中断→订单待支付30分钟后自动取消|
|**我的存档**|（见demo）|- **入口**：我的页→"我的存档"<br>- **Tab**：推荐存档(X) / 我的存档(X)<br>- **已购买列表**：游戏名+存档标题+购买时间+"一键导入"按钮+"查看详情"<br>- **已上传列表**：标题+状态（审核中/已上架/已下架）+已售数量+收益金额+操作（编辑/下架）<br>- **一键导入交互**：点击→弹确认"导入将覆盖当前游戏存档，是否继续？"→确认→自动放入Wine Prefix正确路径→toast"导入成功，启动游戏即可体验"<br>- **异常处理**：本地无该游戏Prefix时→toast"请先安装并启动一次该游戏，再进行导入"；下载失败→"下载失败，点击重试"；已购买存档无限次重新导入|
|**上传存档（玩家）**|（见demo）|- **入口**：我的存档→"上传存档"按钮<br>- **表单字段**：<br>    - 关联游戏（必填，下拉选择）<br>    - 存档文件（必填，"从当前游戏读取存档"按钮，自动扫描Prefix中的存档文件）<br>    - 标题（必填）<br>    - 描述（必填）<br>    - 截图（选填，最多4张）<br>    - 定价（必填，¥1-20范围）<br>- **分成提示**：定价下方显示"平台抽成30%，你实得¥X.XX"<br>- **提交**：点击"提交审核"→toast"提交成功，审核通过后自动上架"→状态变为"审核中"<br>- **异常处理**：必填项为空时按钮置灰；定价超出范围自动修正；单日上传上限5个；文件安全扫描不通过→"文件安全检测未通过"|
|**后台-存档管理**|（见demo）|- **官方存档上架**：新建→选择游戏+上传文件+标题/描述/截图+定价+标记"官方精品"→发布<br>- **玩家存档审核**：待审核列表→查看详情（文件信息+上传者+内容描述）→通过/驳回<br>- **列表管理**：筛选（游戏/状态/类型）+表格（ID/封面/标题/游戏/作者/价格/状态/购买数/时间）+操作（编辑/上架/下架/删除）<br>- **异常处理**：删除时二次确认；下架不影响已购买用户|

## 五、边界条件

### 内容侧

| 场景 | 处理方式 |
|------|---------|
| 存档文件含恶意程序 | 上传后服务端安全扫描，未通过拒绝上架+通知上传者 |
| 存档与标注游戏不匹配 | 用户投诉→后台标记"不可用"→下架+退款 |
| 存档截图与实际不符 | 用户举报"描述不符"→人工核实→属实则下架+退款 |
| 官方存档因游戏更新失效 | 标记"已过期"提示，不自动下架 |
| 存档文件过大（>500MB） | 拦截上传，提示"文件过大" |

### 用户侧

| 场景 | 处理方式 |
|------|---------|
| 购买后卸载APP重装 | 购买记录绑定账号，重装登录后仍可下载导入 |
| 本地无该游戏时购买 | 允许购买，导入时检测：无Prefix则toast提示先安装游戏 |
| 自买自卖 | 禁止购买自己上传的存档，按钮显示"这是你的存档" |
| 玩家下架存档 | 已购买用户不受影响，仍可下载导入 |
| 单日上传过多 | 上限5个/天，超出toast提示 |

### 权限/风控侧

| 场景 | 处理方式 |
|------|---------|
| 虚拟商品退款 | 明确不支持退款，支付确认页展示提示文案 |
| 未成年购买 | 未实名认证禁止购买，弹实名引导 |
| 定价范围 | ¥1-20，超出不接受 |
| 卖方提现 | 累计≥¥10可提现，T+7到账 |
| 版权投诉 | 收到投诉48小时内下架对应游戏全部存档 |

## 六、埋点需求

### 6\.1 埋点事件表

|**事件ID**|**事件名称**|**触发时机**|**关键参数**|
|---|---|---|---|
|`save_tab_view`|存档tab曝光|进入游戏详情页"存档"tab|`game_id`|
|`save_card_click`|存档卡片点击|点击存档卡片进入详情|`save_id`, `game_id`, `price`, `is_official`|
|`save_purchase_click`|购买按钮点击|点击"¥X.XX购买"|`save_id`, `price`|
|`save_purchase_success`|购买成功|支付回调成功|`save_id`, `price`, `pay_method`|
|`save_import_click`|一键导入点击|点击"一键导入"|`save_id`, `game_id`|
|`save_import_success`|导入成功|存档文件成功放入Prefix|`save_id`, `game_id`|
|`save_upload_submit`|上传提交|点击"提交审核"|`game_id`, `price`|
|`save_upload_success`|上传审核通过|后台审核通过|`save_id`, `game_id`|

### 6\.2 埋点参数表

|**参数名**|**类型**|**必填**|**说明**|**枚举/示例**|
|---|---|---|---|---|
|`save_id`|string|是|存档唯一标识|SAVE_001|
|`game_id`|string|是|关联游戏标识|271590|
|`price`|float|否|存档价格|6.00|
