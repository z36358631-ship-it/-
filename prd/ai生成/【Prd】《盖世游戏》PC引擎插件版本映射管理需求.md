## 二、背景与目标

背景：PC引擎插件版本与APP版本之间存在兼容性依赖关系，不同引擎插件版本只能在特定APP版本上正常运行。当前缺乏统一管理工具，兼容关系散落在代码配置中，运营无法自助查看和维护，出现兼容问题时排查成本高。

目标：建立引擎插件版本与APP版本的映射管理后台，支持运营/技术人员可视化配置、查询和维护兼容映射规则。

核心挑战：版本号匹配逻辑复杂（支持精确值、范围、介于多种条件），且国内/海外APP版本系列独立，映射规则需分区管理。

## 三、故事介绍

### 3.1 用户与运营场景

- **技术运营（小李）**：发布引擎插件1.5.0后，需要声明它只兼容APP 6.0.0及以上版本。他打开映射管理后台，新建一条规则：引擎版本 `>= 1.5.0`，国内APP版本 `>= 6.0.0`，填写优先级2，点击确定。规则立即生效，后续系统可调用该规则做兼容性校验。

- **问题排查（小王）**：用户反馈APP 5.3.5上引擎插件1.4.2运行异常。小王在后台按引擎版本 `= 1.4.2`、APP版本 `= 5.3.5` 筛选，找到对应规则，确认状态是"禁用"，说明该组合已停止支持，反馈给用户升级APP。

### 3.2 价值分析

- **提升效率**：运营/技术可自助维护兼容映射，不再依赖代码发版修改
- **降低风险**：集中管理兼容规则，支持启用/禁用开关，出问题可快速回滚
- **可追溯**：每条规则记录操作人和更新时间，异常时可追查变更来源
- **分区管理**：国内/海外APP版本系列独立维护，互不干扰

### 3.3 核心体验路径

- **新增映射链路**：
进入映射管理 → 选择区服Tab（国内/海外） → 点击"新增映射" → 填写引擎版本条件（支持多条）+ APP版本条件（支持多条）+ 优先级 + 备注 → 确定 → 列表刷新展示新记录

- **查询映射链路**：
选择区服Tab → 设置筛选条件（引擎版本操作符+版本号、APP版本操作符+版本号、状态）→ 点击查询 → 列表展示命中规则

- **变更链路**：
列表找到目标规则 → 点击编辑修改 / 直接切换状态开关 / 点击删除（二次确认）

### 3.4 产品指标预测

1. 后台查询响应时间：<500ms
2. 版本条件匹配准确率：100%（精确匹配/范围匹配/介于匹配三种逻辑均正确）
3. 规则生效延迟：配置后<1分钟下发生效
4. 减少因兼容性配置错误引发的线上问题：目标降低60%+

### 3.5 路径规划

- **V1.0（当前）**：PC引擎插件 × APP版本映射管理，支持增删查改、多条件、国内/海外分区、启用/禁用开关
- **V2.0（规划）**：与APP发版流程打通，发版时自动检测是否存在对应映射规则，无规则则拦截提醒

#### 4.3.1 PC引擎插件&APP版本映射表

**页面结构：**
1. 顶部区服 Tab：**APP国内版** | **APP海外版**，切换时列表数据独立，筛选条件重置
2. 筛选区
3. 数据列表

**筛选区字段说明：**

| 筛选项 | 交互方式 | 说明 |
|--------|---------|------|
| 引擎插件版本 | 下拉（操作符）+ 文本输入（版本号） | 操作符：等于/大于/小于/大于等于/小于等于/介于(含)；选"介于"时显示两个输入框，分别填起始版本和截止版本 |
| APP版本 | 下拉（操作符）+ 文本输入（版本号） | 同上 |
| 状态 | 下拉+单选 | 全部/启用/禁用 |

**功能按钮：**
- **查询**：提交筛选条件，刷新列表；无结果显示"暂无数据"
- **重置**：清空所有筛选条件，恢复默认（全部），回到第一页，触发刷新
- **新建插件**：打开新建弹窗

**列表字段说明：**

| 字段 | 说明 | 示例 | 备注 |
|------|------|------|------|
| ID | 规则唯一编号，系统自动生成 | 1 | 只读 |
| APP协议 | 该规则适用的APP渠道包，多选；**国内**可选：iOS官方包、安卓官方包、安卓小米、安卓OPPO、安卓VIVO、安卓华为、安卓荣耀、安卓三星；**海外**仅有：Global | iOS官方包、安卓官方包 | 多Tag展示，新建时必填 |
| 插件文件 | 已上传的APK文件名 | engine_v1.5.0.apk | 点击可下载 |
| 插件版本号 | 用户自定义填写的版本号名称 | 1.5.0 | 用于匹配和排序 |
| 引擎插件版本条件 | 展示所有引擎版本条件，多条用换行或分隔符展示 | `= 1.4.2` / `>= 1.5.0` / `1.4.0 ~ 1.4.9 (含)` | 操作符+版本号格式化展示 |
| APP版本条件 | 展示所有APP版本条件，展示格式同上 | `= 5.3.5` / `6.0.0 ~ 6.0.9 (含)` | |
| 映射类型 | 系统根据条件组合自动推断 | 精确匹配/范围匹配/复合映射 | 只读，不需手动填写 |
| 状态 | 启用/禁用，inline开关直接切换 | 启用 | 切换后即时生效，无需额外确认 |
| 备注 | 可选，说明规则用途或背景 | 旧版兼容 | 超长截断，hover显示完整内容 |
| 操作人 | 最后一次新增或编辑的操作人账号，自动记录 | admin_li | 只读 |
| 更新时间 | 最后一次新增或编辑的时间 | 2026-06-18 10:32:00 | 只读，精确到秒 |
| 操作 | 编辑、删除 | — | 编辑打开弹窗；删除需二次确认 |

**列表排序规则：**
- 默认按插件版本号降序排列（版本号数字越大排越前）
- 支持点击"插件版本号"列切换升序/降序

**版本冲突规则：**
- 当多条规则的APP版本条件存在重合时，系统自动使用**插件版本号数字最大**的那条规则
- 示例：规则A（插件版本1.4.2，APP=5.3.5）和规则B（插件版本1.5.0，APP>=5.3.5）同时命中某用户，则使用规则B（1.5.0 > 1.4.2）

**分页：**
- 每页20条
- 显示总条数

**状态枚举与流转：**

| 状态 | 含义 | 可执行操作 |
|------|------|---------|
| 启用 | 规则生效，系统校验时会命中此规则 | 切换为禁用、编辑、删除 |
| 禁用 | 规则不生效，仅作记录保留 | 切换为启用、编辑、删除 |

#### 4.3.2 新建/编辑插件弹窗

**插件文件上传：**
- 上传APK文件，一次只能传1个
- 支持格式：.apk
- 文件大小限制：≤500MB
- 上传中显示进度条，上传完成显示文件名+文件大小
- 已有文件时，重新上传替换原文件
- 编辑时展示已上传的文件名，支持重新上传

**插件版本号：**
- 用户手动填写，格式为数字+点分隔（如 `1.5.0`）
- 必填，placeholder示例："如: 1.5.0"
- 输入校验：仅允许数字和点号，失焦时校验格式

**APP协议：**
- 多选checkbox，必填，至少选一个
- 国内Tab可选项：iOS官方包、安卓官方包、安卓小米、安卓OPPO、安卓VIVO、安卓华为、安卓荣耀、安卓三星
- 海外Tab可选项：Global

**引擎插件版本条件（支持多条）：**
- 默认一行：操作符下拉 + 版本号输入框
- 点击"+ 添加引擎版本条件"追加一行
- 每行右侧有删除按钮（单行时置灰不可删）
- 操作符选"介于(含)"时，版本号区域展示两个输入框（起始版本 + 截止版本），其他操作符展示单个输入框
- 版本号格式：数字+点分隔，如 `1.4.2`，输入框placeholder给出示例

**APP版本条件（支持多条）：**
- 结构与引擎版本条件完全一致

**其他字段：**

| 字段 | 规则 |
|------|------|
| 状态 | 单选：启用/禁用，**默认禁用**（需手动开启后生效） |
| 备注 | 可选文本域，最多200字 |

**映射类型自动推断逻辑：**
- 引擎和APP条件都是"等于"→ 精确匹配
- 有任意一个非"等于"且不含"介于"→ 范围匹配
- 包含"介于"或存在多条条件 → 复合映射

**提交校验：**
- APK文件必须上传
- 插件版本号必填，格式校验通过
- APP协议至少选一项
- 引擎版本条件至少一条，且版本号不为空
- "介于"条件的截止版本号必填
- APP版本条件同上

**操作人：** 提交时自动记录当前登录账号，不需手动填写

#### 4.3.3 边界条件-内容侧

| 场景 | 处理方式 |
|------|---------|
| 新增规则与已有规则完全重复（同操作符+同版本号） | 提示"已存在相同映射规则，请检查后重试"，不允许提交 |
| 删除启用中的规则 | 弹出二次确认："确认删除此映射规则？删除后不可恢复" |
| 版本号格式错误（非数字点分格式） | 输入框失焦时校验，提示"版本号格式不正确，请输入如 1.4.2 的格式" |
| 介于条件起始版本 > 截止版本 | 提交时提示"起始版本不能大于截止版本" |
| 无数据 | 列表显示"暂无映射规则，点击新增映射创建第一条规则" |

#### 4.3.4 边界条件-用户侧

| 场景 | 处理方式 |
|------|---------|
| 无权限用户访问 | 跳转403页面或展示"无访问权限"提示 |
| 状态开关频繁切换 | 切换请求返回前，开关不可再次操作（loading态），防止重复提交 |
| 网络超时 | 提示"网络异常，请重试"，操作不落库 |
| 并发编辑（两人同时编辑同一条规则） | 后保存的覆盖先保存的，更新时间和操作人以最后一次为准 |

## 六、埋点需求

### 6.1 埋点事件表

| 事件ID | 事件名称 | 触发时机 | 关键参数 |
|--------|---------|---------|---------|
| engine_mapping_tab_switch | 区服Tab切换 | 点击国内/海外Tab时 | region(cn/overseas) |
| engine_mapping_filter | 执行筛选查询 | 点击"查询"按钮时 | region, engine_op, app_op, status |
| engine_mapping_create | 新增映射规则 | 新增弹窗点击"确定"成功时 | region, engine_cond_count, app_cond_count, mapping_type |
| engine_mapping_edit | 编辑映射规则 | 编辑弹窗点击"确定"成功时 | rule_id, region |
| engine_mapping_delete | 删除映射规则 | 二次确认后删除成功时 | rule_id, region |
| engine_mapping_status_toggle | 状态开关切换 | inline开关切换成功时 | rule_id, new_status(enabled/disabled) |

### 6.2 埋点参数表

| 参数名 | 类型 | 必填 | 说明 | 枚举/示例 |
|--------|------|------|------|---------|
| region | string | 是 | 当前区服Tab | cn, overseas |
| engine_op | string | 否 | 引擎版本筛选操作符 | eq, gt, lt, gte, lte, between |
| app_op | string | 否 | APP版本筛选操作符 | eq, gt, lt, gte, lte, between |
| status | string | 否 | 状态筛选值 | all, enabled, disabled |
| rule_id | int | 否 | 规则ID | 1 |
| engine_cond_count | int | 否 | 引擎版本条件行数 | 1, 2, 3 |
| app_cond_count | int | 否 | APP版本条件行数 | 1, 2 |
| mapping_type | string | 否 | 映射类型 | exact, range, compound |
| new_status | string | 否 | 开关切换后的状态 | enabled, disabled |

## 七、运营需求

1. 初始化数据：上线时由技术侧导入现有兼容关系数据，确保存量规则不丢失
2. 权限分配：明确哪些角色有增删改权限，哪些角色只读
3. 操作规范：制定版本号命名规范文档，避免格式不统一导致匹配失效
