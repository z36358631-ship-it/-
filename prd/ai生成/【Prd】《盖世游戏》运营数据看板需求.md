## 二、背景与目标

背景：盖世游戏运营团队当前缺乏统一的数据监控平台，各业务数据分散在多个后台和表格中，无法快速定位问题、跟踪关键指标变化。

目标：搭建运营数据看板，覆盖用户生命周期（增长/留存/流失）、核心功能转化率、云游戏营收、社区运营等关键指标，支持国内/海外分区筛选，提供实时监控和数据导出能力。

核心挑战：76项指标的数据源统一口径；国内/海外两套登录体系和功能差异的数据隔离；离线计算任务的时效性。

## 三、故事介绍

### 3.1 用户与运营场景

- **场景一（日常巡检）**：运营每天打开工作台，查看DAU、新增、留存、充值金额等6个核心卡片数据，发现异常指标后点击卡片跳转到对应模块深入分析。
- **场景二（版本复盘）**：产品经理在版本上线后，切换到"核心功能"模块，查看新功能（如AI插帧、边玩边下）的渗透率漏斗，评估功能效果。
- **场景三（流失归因）**：用户增长负责人在"用户数据→流失分析"中，筛选近7天数据，查看卸载前/流失前行为还原（区分互动事件/停留页面），定位流失原因。
- **场景四（跨区对比）**：海外运营切换地区筛选为"海外"，对比海外版本留存和登录方式分布差异。

### 3.2 价值分析

- **效率提升**：统一数据出口，减少跨系统取数时间
- **问题发现**：告警机制自动发现异常波动
- **决策支持**：留存矩阵+流失还原为产品迭代提供依据
- **跨区管理**：国内/海外数据一站式查看

### 3.3 核心体验路径

工作台总览 → 发现异常指标 → 点击跳转对应模块 → 切换筛选条件（地区/时间/用户群） → 查看趋势/漏斗/排行 → 导出数据

### 3.4 产品指标预测

1. 看板日均访问人数：预测8-15人（运营+产品+管理层）
2. 数据导出频次：预测每天3-5次
3. 告警响应时效：从发现异常到定位原因 < 10分钟

### 3.5 路径规划

- **V1.0（当前）**：跑通7大模块核心指标展示、筛选、导出能力
- **V2.0（预告）**：自定义看板（拖拽组合指标卡片）；定时报表自动推送（邮件/飞书）；AB实验数据对比模块

## 四、概要设计

### 4.1 模块设计

1. 工作台（Dashboard）——核心指标概览+告警+日报
2. 用户数据——活跃/新增趋势 + 留存矩阵 + 流失分析（两Tab：用户数据/流失分析）
3. 新手链路——首次体验全链路漏斗
4. 游戏生态——游戏启动/运行/品类/排行
5. 云游&营收——云游戏用户+订单+收入（**仅国内**）
6. 核心功能——8大功能渗透率漏斗（表格形式）
7. 社区运营——互动/活跃/热门分区/热门话题

**侧边栏分组：**
- 总览分组：工作台
- 用户分组：用户数据、新手链路
- 业务分组：游戏生态、云游&营收、核心功能、社区运营

> 变更说明（V1.1）：
> - "转化漏斗"更名为"核心功能"，移到业务分组
> - "新手链路"移到用户分组
> - "用户分析"+"留存&流失"合并为"用户数据"（两tab：用户数据/流失分析）
> - "营收"合并到"云游&营收"
> - 删除"实时数据"模块

**全局筛选器（所有子页面顶部统一）：**
- 地区：全部地区 / 国内 / 海外（云游&营收模块无地区筛选，固定国内）
- 时间：今日 / 昨日 / 近7天 / 近30天 / 自定义日期范围
- 用户群：全部用户 / 新增用户 / 活跃用户

**全局交互规则：**
- 筛选切换后当前模块数据即时刷新
- 导出按钮点击后弹出下拉选择：CSV / Markdown
- 所有图表hover显示数值tooltip
- 所有指标卡加问号tooltip（hover显示计算方式/公式）
- 表格支持列排序（点击表头切换升序/降序）
- 分页表格每页10条，底部显示"共X条 第X/X页"+ 翻页箭头
- 空状态：无数据时表格/图表区域居中显示"暂无数据"
- 加载态：数据请求中，图表区域显示骨架屏
- 顶部面包屑导航 + 多标签页（支持同时打开多个模块）
- 饼图统一为右侧图例样式

功能demo：[运营数据看板demo](demos/后台管理/admin-运营数据看板v2.html)

### 4.3 详细设计（B端）

#### 4.3.1 工作台

概述：运营核心指标一览，快速掌握应用整体健康度。

**指标卡（6个）：**

| 指标名 | 计算方式 | 口径 |
|--------|---------|------|
| 活跃用户 | 所选时间范围内有app_launch的去重device_id数 | 设备数 |
| 新增用户 | 首次产生app_launch的device_id数 | 设备数 |
| D0游戏体验率 | 安装当天成功启动游戏用户数/当日新增用户数 | UV |
| D1留存率 | 昨日活跃用户中今日再次活跃的比例 | 设备数 |
| 人均互动时长 | 总session时长/活跃用户数 | 分钟 |
| 充值金额 | 当日pay_result(success)的amount总和 | 金额 |

**页面内容：**

- 数据趋势：业务/用户/订单/收入四个维度切换，支持日/周/月粒度。折线图展示，切换维度不影响粒度选择。
- 版本分布：饼图展示各版本占比（右侧图例带百分比和数值）。
- 需关注事项：告警列表，按优先级排序。触发条件见下方阈值表。
- 启动类型：饼图展示 PC Steam/云游戏/PC导入/复古模拟器 四种启动类型占比。
- 功能渗透率：横向条形图，展示云游戏/社区/配置方案/AI插帧/竖屏功能/云组队/边下边玩的用户渗透比例，每条标注百分比。
- Top游戏排名：支持PC游戏/云游戏 tab切换，列出前5名（游戏名+时长+用户数），按时长降序。
- 核心指标日报：表格形式，列=日期/活跃用户/新增/次日留存/启动成功率/D0体验率/人均互动时长/充值额/环比/同比。底部加"日均"行自动计算平均值。

**需关注事项触发阈值：**

| 告警类型 | 触发条件 | 优先级 |
|---------|---------|-------|
| 流失用户突增 | 日流失环比 ≥ 20% | P0 |
| 新手引导异常 | D0体验率 < 15% | P1 |
| 版本启动异常 | 启动成功率 < 80% 或环比降 > 5% | P0 |
| 社区里程碑 | 社区DAU首次突破整千关口 | P2 |
| 充值收入波动 | 日充值环比降 ≥ 30% | P1 |
| 留存率异常 | 次日留存连续3天 < 35% | P1 |
| 云游流失预警 | 云游日活环比降 ≥ 15% | P1 |

P0=实时触发+推送通知，P1=每日定时检测，P2=达标通知。指标回到正常范围24小时后自动消除。

#### 4.3.2 用户数据

概述：监控用户全生命周期数据，包含活跃/留存（用户数据tab）和流失分析（流失分析tab）。

**指标卡（6个）：**

| 指标名 | 计算方式 | 口径 |
|--------|---------|------|
| 活跃用户 | 所选时间内有app_launch的去重device_id | 设备数 |
| 新增用户 | 首次app_launch的device_id | 设备数 |
| 游客用户 | 使用游客模式登录的去重用户数 | UV |
| 匿名设备 | 无任何登录态的device_id数 | 设备数 |
| 次日留存率 | 昨日活跃中今日再次活跃的比例 | 百分比 |
| 日流失用户 | 连续15天无app_launch的device_id数 | 设备数 |

**用户数据 Tab：**

- 活跃用户趋势：折线图，展示活跃/新增/游客/匿名四条线，匿名线用虚线区分。
- 用户构成：饼图（右侧图例），展示注册活跃/游客/匿名设备三者占比和数值。
- 留存矩阵：表格形式，行=日期，列=1天后/3天后/7天后/14天后/30天后。顶部支持切换：新用户留存/活跃用户留存 × 天/周/月 × 留存率/留存数。留存值用色阶区分（>35%绿，20-35%橙，<20%红）。
- 留存率趋势：折线图展示次日/3日/7日留存率走势。
- 新老对比：折线图对比新用户次日留存 vs 老用户次日留存。

**流失分析 Tab：**

- 日流失趋势：折线图，日流失+7日累计双线。
- 卸载前行为还原：表格+横向条形，支持维度切换（互动事件/停留页面）和范围切换（最后一次/全部）。列=名称/条形/次数/次数占比/人数/人数占比，按人数占比降序。右上"导出全部"按钮。
- 流失前行为还原：同上，颜色区分（卸载=绿，流失=红）。
- 画像对比：两张折线图，分别展示流失 vs 留存用户的版本分布和渠道分布。

#### 4.3.3 新手链路

概述：监控新用户从安装到首次游戏体验的全链路转化。

**指标卡（6个）：**

| 指标名 | 计算方式 | 口径 |
|--------|---------|------|
| 引导完成率 | 完成引导流程用户数/触发引导用户数 | UV |
| 登录率(平台) | 平台账号登录成功数/APP首次启动设备数 | UV÷设备数 |
| 登录率(游客) | 游客模式进入数/APP首次启动设备数 | UV÷设备数 |
| 登录率(Steam) | Steam绑定成功数/登录用户数 | UV |
| D0游戏体验率 | 安装当天成功启动游戏用户数/当日新增用户数 | UV |
| 引导后次日留存 | 完成引导用户中次日有app_launch的比例 | 百分比 |

**国内/海外差异：** **国内"登录率(平台)"含手机号/微信/QQ/Apple；海外含邮箱/Google/Apple**

**页面内容：**

- 全链路漏斗表格：列=序号/步骤/口径/数值/总转化率/步间转化率。步骤1-3口径为设备数，步骤4+口径为UV。
- D0体验率趋势：折线图展示近N天 D0 率走势。
- 登录方式分布：饼图（右侧图例），展示各登录方式占比和数值。**国内：SIM一键/微信/QQ/游客/Apple；海外：邮箱/Google/Apple/游客**

### 4.4 边界条件

#### 4.4.1 内容侧

| 场景 | 处理方式 |
|------|---------|
| 数据延迟 | 页面顶部显示"数据更新至 HH:MM"，超过30分钟显示黄色提示 |
| 无数据 | 图表区域显示"暂无数据"占位，表格显示空行提示 |
| 数据异常（负数/超大值） | 前端做合理性校验，异常值标红并加tooltip说明 |
| 导出数据量超限 | 单次导出上限10万行，超出提示"请缩小时间范围" |
| 饼图数据全为0 | 显示空心环+中间"无数据"文字 |

#### 4.4.2 用户侧

| 场景 | 处理方式 |
|------|---------|
| 网络异常 | 图表加载失败显示"加载失败，点击重试"，可手动重试 |
| 权限不足 | 无权限页面显示"暂无权限，请联系管理员" |
| 并发操作 | 同一用户多Tab打开不冲突（只读看板） |
| 页面加载超时 | 10秒超时后显示重试按钮 |
| 筛选条件不合法 | 结束日期早于开始日期时禁用查询按钮并提示 |

## 五、非功能需求

|项目|要求|
|---|---|
|数据时效性|核心卡片指标：T+0（实时或准实时，延迟 ≤ 5分钟）；趋势图/矩阵：T+1（次日凌晨计算完毕）|
|页面加载|首屏（工作台6卡片+趋势图）加载 ≤ 2秒|
|并发|支持20人同时在线查看，不影响响应速度|
|数据保留|原始明细保留90天；聚合数据永久保留|
|权限|接入后台统一权限体系，按角色控制可见模块（如运营可见全部，客服仅可见用户数据）|
|兼容性|Chrome 90+ / Edge 90+，最小分辨率1280×720|
|导出|单次导出上限10万行，格式CSV(UTF-8 BOM)或Markdown，通过下拉菜单选择|
