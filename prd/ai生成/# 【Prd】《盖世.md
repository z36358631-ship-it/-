# 二、背景与目标

**背景：** 盖世游戏 APP 当前已支持云游戏秒玩，用户以单人体验为主。部分合作类游戏天然具备多人协作价值，如怪物猎人、茶杯头、人类一败涂地、幻兽帕鲁等。为提升云游戏使用场景和用户停留时长，需在现有玩游戏体系中补充「组队玩」功能，支持用户围绕指定云游戏创建房间、加入房间，并在同一云游戏 Session 中协作游玩。

**目标：**

- 在「玩游戏」体系下新增组队玩法入口，兼容现有云游戏秒玩链路。
- 用户可从云游戏首页、组队玩 Tab 进入组队链路。
- 覆盖游戏大厅、组队大厅、创建房间、组队房间、分享房间等核心页面与操作。
- 横竖屏布局适配，弹窗、悬浮条、房间详情等核心页面不超出容器。
- 通过快速加入、房间悬浮条、分享入口，降低组队转化路径的操作成本。

# 三、故事介绍

## 3.1 用户场景

**场景一：快速加入已有房间**

用户进入玩游戏页，在云游戏首页「组队热玩」专题发现怪物猎人当前有可加入房间，点击「快速加入」。系统校验云游戏时长后，将用户加入该游戏下合适的房间，进入组队房间页等待房主开始游戏。

**场景二：自己创建房间等队友**

用户进入「组队玩」Tab，在游戏大厅选择想玩的游戏。当前没有可加入房间，点击「创建房间」，设置房间名称和公开/私密权限后创建。创建成功后进入组队房间页，用户成为房主，可分享房间或等待组队大厅用户加入。

**场景三：房主离开房间页后快速返回**

房主创建房间后返回组队大厅或切换到其他页面，页面底部展示房间悬浮条，包含游戏图标、游戏名称、准备中状态和房间有效期。用户可通过悬浮条快速回到房间，也可退出并解散房间。

## 3.2 价值分析

- **扩展云游戏使用场景：** 从单人秒玩扩展到多人协作，提升云游戏时长消耗和用户停留。
- **提升合作类游戏转化：** 对适合组队的游戏提供快速加入和创建房间入口，降低找队友成本。
- **增强用户召回与传播：** 房间分享覆盖微信、QQ、复制链接，有利于外部邀请和回流。
- **降低房间等待流失：** 房间悬浮条让房主离开房间页后仍能感知房间状态，方便随时返回。

## 3.3 核心体验路径

**路径一：云游戏首页进入组队**

云游戏首页 → 组队热玩 → 快速加入 / 创建房间 → 进入组队房间 → 开始云游戏

**路径二：组队玩游戏大厅进入**

组队玩 Tab → 游戏大厅 → 选择游戏 → 快速加入 / 创建房间 → 进入组队房间 → 开始云游戏

**路径三：组队大厅进入**

组队玩 Tab → 组队大厅 → 搜索/浏览房间 → 加入公开房间 → 进入组队房间 → 等待房主开始

**路径四：创建后离开再返回**

创建房间 → 进入组队房间 → 返回组队大厅/其他页面 → 展示房间悬浮条 → 点击进入 → 回到组队房间

## 3.4 产品指标预测

- **组队玩入口点击率：** 玩游戏页访问用户中点击「组队玩」Tab 或「组队热玩」专题的比例。
- **快速加入点击率：** 游戏大厅中有房间游戏的「快速加入」点击次数 / 展示次数。
- **创建房间转化率：** 点击创建房间后成功进入组队房间的比例。
- **房间开始率：** 创建或加入房间后最终点击「开始游戏」并进入云游戏 Session 的比例。
- **分享触发率：** 房间页点击分享入口的比例。
- **悬浮条回房率：** 展示悬浮条后点击「进入」返回房间的比例。

## 3.5 路径规划

**当前版本（V1.0）：**

完成云游戏组队玩 C 端主链路 Demo，包含云游戏首页组队热玩、组队玩 Tab、游戏大厅、组队大厅、创建房间、组队房间页、分享弹窗、房间悬浮条、横竖屏适配等核心功能。

**未来可能升级：**

- 更多游戏类型与后台配置化管理。
- 组队历史、最近组队、再次组队。
- 更完整的房间状态管理、掉线重连、房间有效期策略。
- 更细的游戏模式筛选与匹配策略。

# 四、概要设计

## 4.1 模块设计（功能架构概览）

**【云游戏首页模块】**

- **组队热玩专题：** 展示适合组队的游戏，优先展示用户最近玩过且有房间的游戏。
- **秒玩链路保留：** 秒玩人气热游、全部秒玩等原有入口保持「秒玩」按钮，不被组队链路替代。

**【组队玩模块】**

- **游戏大厅：** 按游戏维度展示支持组队的游戏，展示房间数，支持快速加入或创建房间。
- **组队大厅：** 按房间维度展示公开房间，支持搜索房间名或游戏名，支持加入公开房间。
- **玩法说明：** 与游戏大厅/组队大厅 Tab 同行右侧，仅游戏大厅展示。

**【创建房间模块】**

- **创建入口：** 游戏大厅游戏卡片、组队大厅悬浮创建按钮、云游戏首页组队热玩卡片。
- **创建配置：** 选择游戏、设置房间名称、选择公开/私密权限。
- **创建结果：** 创建成功后进入组队房间页，当前用户成为房主。

**【组队房间模块】**

- **独立页面：** 顶部展示返回按钮和页面标题「组队房间」。
- **房间信息：** 展示游戏名称、房间 ID、成员席位、规则说明。
- **房间操作：** 分享房间、修改房间信息、解散房间、开始游戏。
- **成员席位：** 展示房主与队友 P 位，人数不足时开始游戏置灰。

**【分享与悬浮房间模块】**

- **分享弹窗：** 展示「分享到」，支持微信、QQ、复制链接。
- **房间悬浮条：** 用户创建房间后离开房间页时展示，支持进入房间或退出房间。

# 五、交付设计（数据分析与埋点设计）

## 5.1 核心业务指标

**关键漏斗：**

进入玩游戏页 → 点击组队玩/组队热玩 → 点击快速加入/创建房间/加入房间 → 进入组队房间 → 开始游戏

**辅助指标：**

- 组队热玩专题点击次数
- 游戏大厅快速加入点击次数
- 创建房间成功次数
- 组队大厅加入房间次数
- 房间分享次数
- 房间悬浮条进入次数
- 时长不足充值引导点击次数
