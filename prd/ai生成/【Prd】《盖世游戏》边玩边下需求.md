## 二、背景与目标

背景：盖世游戏用户点击"获取游戏"后，需等待大型PC游戏下载完成（通常20-40分钟）才能开始体验，等待期间用户流失率高。竞品Xbox Game Pass和TapTap（TapPlay）已实现"云游戏过渡+后台下载+切换本地"的完整链路，验证了该模式的可行性。

目标：在安装流程中嵌入"边玩边下"选项，用户点击后通过云游戏立即开玩，后台同时下载本地安装包，下载完成后同步存档并切换到本地运行，消除下载等待期。

核心挑战：云游戏与本地下载同时进行的带宽竞争；云端存档到本地的路径映射与同步；切换过程中的会话管理。

## 三、故事介绍

### 3.1 用户与运营场景

- **场景一（首次下载大型游戏）**：小王在盖世游戏发现了《生化危机2重制版》，点击"获取游戏"后弹出安装弹窗，显示游戏19GB需下载约25分钟。他看到"边玩边下"选项并标注"首次免费10分钟"，点击后云游戏立即启动，他边玩边等下载。25分钟后收到"下载完成，切换本地"提示，点击后存档自动同步，本地游戏从云游戏中断处继续。

- **场景二（下载完继续云游戏）**：小李选择了边玩边下，下载完成后弹窗提示切换，但他正在打Boss不想中断，点击"继续游玩"。弹窗关闭，左上角出现"下载完成，切换本地"按钮，他打完Boss后点击按钮完成切换。

### 3.2 价值分析

- **降低流失**：消除下载等待期，用户不会因等太久而放弃
- **云游戏收入**：过渡期按¥3/小时正常计费（首次免费10分钟除外），下载期间产生云游戏收入
- **差异化体验**：与竞品TapTap TapPlay对齐，强化"即点即玩"的产品心智

### 3.3 核心体验路径

- **用户核心链路**：
游戏详情页 → 点击"获取游戏" → 安装弹窗 → 点击"边玩边下" → 云游戏启动（自动切横屏）+ 后台下载 → 下载完成弹窗 → 点击"切换到本地" → 关闭云游戏+同步存档+启动本地游戏 → 接续进度

### 3.4 产品指标预测

1. "边玩边下"选择率：预测30-50%用户在安装弹窗中选择边玩边下（vs仅安装）
2. 存档同步成功率：目标 > 95%
3. 下载等待期流失率下降：预测降低40%+
4. 首次免费10分钟→付费转化率：预测15-25%用户在免费时长用完后继续付费游玩

### 3.5 路径规划

- **V1.0（当前）**：跑通"安装弹窗→云游戏过渡→后台下载→存档同步→切换本地"完整链路，覆盖全部云游戏库
- **V2.0（预告）**：智能带宽分配（根据网络状况动态调整云游戏码率和下载速度）；下载完成后自动切换（无需手动确认）

## 四、概要设计

### 4.1 模块设计

1. 安装弹窗（嵌入边玩边下选项，支持多场景：有云游戏/无云游戏/存档不可同步）
2. 云游戏运行态（含下载进度最小化提示）
3. 下载完成提示弹窗
4. 切换流程（关闭云游戏→同步存档→启动本地）
5. 切换成功反馈
6. 仅安装模式（无云游戏时：下载进度+小游戏推荐）
7. 下载悬浮球（多任务支持、可拖动、拖到底部×区域隐藏）
8. 下载管理页（查看所有下载任务，支持直接打开）

#### 4.2.1 边界条件-用户侧

|场景|处理方式|
|---|---|
|下载中用户主动退出云游戏|下载继续进行，用户回到游戏详情页可看到下载进度；下次启动可选择继续云游戏或等待下载完成|
|首次免费10分钟用完|自动转为付费（¥3/小时），顶部状态栏短暂显示"免费时长已用完，已转为付费模式"|
|下载中用户切到后台/锁屏|云游戏暂停，下载继续；用户回到前台时云游戏恢复|
|下载中WiFi断开切到移动数据|弹窗确认"当前使用移动数据，是否继续下载？"；云游戏侧同步提示网络变化|
|用户存储空间不足|安装弹窗中剩余空间 < 游戏大小时，"边玩边下"按钮置灰，提示"存储空间不足"|
|存档不支持同步的游戏|安装弹窗中绿色提示变为橙色"该游戏暂不支持存档同步"；下载完成弹窗中同步状态标注"⚠ 暂不支持"；切换流程跳过存档同步步骤|
|云游戏服务不可用/满载|"边玩边下"按钮置灰，提示"云游戏暂不可用"，用户只能选择"仅安装"|
|多次使用边玩边下（非首次）|不显示"首次免费10分钟"标签，卡片描述改为"云游戏按 ¥3/小时 计费"|
|该游戏无云游戏服务|安装弹窗不显示"边玩边下"卡片，只显示"取消"+"安装"按钮|
|用户在小游戏中收到下载完成|悬浮球变为绿色"安装"状态，点击进入下载管理页可直接打开|
|悬浮球被用户隐藏后|下载继续，用户可通过下载管理页入口查看进度和打开游戏|
|多个游戏同时下载|悬浮球右上角角标显示任务数量，进度显示最先开始下载的任务|

## 五、非功能需求

|**需求类型**|**详细要求**|
|---|---|
|性能|云游戏启动延迟 < 5秒；存档同步耗时 < 10秒（100MB以内）；切换全流程 < 15秒|
|兼容性|Android 11.0+；iOS 13.0+|
|安全|云游戏会话关闭后立即停止计费；存档传输使用加密通道；防止重复计费|
|容错|下载中断支持续传；存档同步失败不影响本地游戏启动；云游戏断连后自动重连（3次）|
|带宽|云游戏码率与下载带宽动态平衡，优先保证云游戏流畅度（最低保证720p 30fps），剩余带宽分配给下载|

## 七、运营需求

1. **首次免费时长配置（P0，上线前必须）**
    - 确认免费时长：10分钟
    - 确认适用范围：每个账号仅一次，不区分游戏
    - 确认到期后行为：自动转付费，无二次弹窗

2. **文案配置（P0）**
    - 安装弹窗中"边玩边下"区域的描述文案
    - 网络提示文案
    - 存档同步状态文案（支持/不支持）
    - 各异常场景的toast文案

3. **存档路径映射表（P0）**
    - 全部云游戏库中每款游戏的存档路径映射（云端路径→本地路径）
    - 标注哪些游戏支持存档同步、哪些暂不支持
    - 持续维护，新游戏上架时需同步补充

4. **带宽策略确认（P1）**
    - 云游戏最低码率保证（建议720p 30fps）
    - 下载限速策略（建议云游戏优先，下载占用剩余带宽）

5. **运营监控（P1）**
    - 关注指标：边玩边下选择率、存档同步成功率、切换成功率、免费→付费转化率
    - 告警规则：存档同步成功率 < 90% 时告警
