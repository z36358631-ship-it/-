## 二、背景与目标

背景：当前APP登录页转化率仅9.6%，新用户到达登录页后直接流失。核心问题：登录门槛过高（无跳过选项），第三方登录入口隐蔽（折叠在二级页面），Steam绑定流程复杂导致用户放弃。

目标：通过"降低登录门槛+新增游客模式+优化登录界面+简化Steam绑定"组合策略，将登录页综合转化率（含游客）提升至50%+，正式登录转化率提升至25%+。

核心挑战：游客账号数据合并（转正时如何保留游客期间数据）、SIM卡号码识别兼容性（不同运营商SDK差异）、Steam二维码登录接口稳定性。

## 三、故事介绍

### 3.1 用户与运营场景

- **新用户（小明）**：小明在应用商店下载了盖世游戏，打开后看到"我的"页。他点击"登录/注册"，弹出登录浮层，自动识别到手机号185****4686。他不想注册账号，点击"游客模式"，3秒内进入APP主功能，开始浏览和启动游戏。3天后他想绑定Steam同步游戏库，点击"绑定Steam"触发登录引导，完成手机号一键登录后自动拉起Steam绑定流程。

- **海外用户（John）**：John下载GameHub后进入"我的"页，点击"Sign Up"弹出登录浮层，输入邮箱+验证码完成注册，随后点击"Bind Steam"扫码完成绑定。

### 3.2 价值分析

- **提升转化**：游客模式消除登录门槛，将"注册才能用"变为"先用再注册"
- **提升留存**：新用户当天即可体验核心功能（PC游戏启动/秒玩），形成使用习惯后再转正
- **降低获客成本**：转化率提升=同样广告投放带来更多有效用户
- **Steam绑定率提升**：二维码前置+流程简化，预计绑定率提升30%+

### 3.3 核心体验路径

- **新用户核心链路**：
"我的"页 → 点击"登录/注册" → 登录浮层弹出 → 一键登录/游客模式 → 进入APP → 使用功能 → 触发转正引导 → 完成登录

- **Steam绑定链路**：
"我的"页 → 点击"绑定Steam账号" → Steam登录浮层弹出 → 扫码/账号密码绑定 → 绑定成功

- **游客转正链路**：
游客触发受限功能（充值/云存档/社交） → 转正引导弹窗 → 展示价值说明 → 完成登录 → 账号升级

### 3.4 产品指标预测

1. 登录页综合转化率（含游客）：9.6% → 50%+
2. 正式登录转化率：9.6% → 25%+
3. 新用户首日游戏体验率：<1% → 15%+
4. 游客7日内转正率：目标30%+
5. Steam绑定率：预计提升30%+

### 3.5 路径规划

- **V1.0（当前）**：登录页改版 + 游客模式 + Steam登录优化 + 转正引导
- **V2.0（后续）**：游客行为数据分析看板、智能转正时机推送、游客社交限制分级开放

|触发场景|弹窗标题|价值说明文案|

**弹窗结构：**
- 顶部图标（与场景对应）
- 标题 + 价值说明
- 登录方式（与登录浮层一致：国内一键登录+第三方，海外邮箱+Google/Apple）
- "暂不登录"文字按钮
    - 点击 → 关闭弹窗 + toast"操作已取消"（本次操作中断，不执行受限功能）

**转正成功后：**
- 游客账号升级为正式账号
- 游客期间的游戏记录、收藏等数据保留
- 提醒条消失，头像badge消失
- toast"登录成功，已升级为正式账号"

**账号冲突处理：**
- 如果用户登录的手机号/邮箱已有正式账号 → 弹窗提示"该账号已存在，是否切换到该账号？（游客数据将丢失）"
- 提供两个选项：确认切换 / 取消（留在游客）

#### 4.2.4 Steam登录浮层

|模块名称|展示&交互说明|
|---|---|
|**入口**|"我的"页Steam卡片的"绑定Steam账号"按钮|
|**前置判断**|未登录/游客状态 → 先弹平台登录浮层 → 登录成功后自动弹Steam浮层；已登录 → 直接弹Steam浮层|
|**浮层形式**|从底部上滑，覆盖约80%高度，深色背景（#1b2838 Steam风格），圆角16px|

**Steam浮层内容（从上到下）：**

1. Steam logo + "STEAM 登录" 标题
2. **二维码区域**（主推方式，前置展示）
    - 白底二维码图片，居中展示
    - 底部文字："通过二维码使用 Steam 手机应用登录"
3. 分割线 + "或使用账号密码登录"
4. 账户名称输入框：placeholder "用账户名称登录"
5. 密码输入框：placeholder "密码" + 右侧眼睛toggle（点击切换明文/密文）
6. "记住我" 勾选项（默认勾选，蓝色圆形✓图标）
7. "绑定" 按钮（蓝色渐变，全宽）
8. 底部提示："*登录仅验证用户的身份，不会收集和使用个人信息"

**绑定成功流程：**
- 点击"绑定"提交 → 如开启Steam Guard → 弹出验证码输入框（6位数字）
- 验证通过 → toast"Steam绑定成功" → 关闭浮层 → "我的"页Steam卡片更新为已绑定状态

**异常处理：**

|场景|处理方式|
|---|---|
|账号密码错误|输入框下方红字提示"账户名称或密码不正确"|
|Steam Guard验证超时|toast"验证超时，请重试"|
|二维码过期|二维码区域显示"已过期，点击刷新"，点击重新生成|
|网络异常|toast"网络异常，请检查网络后重试"|
|该Steam已被其他账号绑定|弹窗"该Steam账号已绑定其他盖世游戏账号，是否解绑并重新绑定？"|

#### 4.2.5 边界条件-用户侧

|场景|处理方式|
|---|---|
|SIM卡未插入/无法识别|不显示手机号，直接展示手机号输入视图|
|双卡用户|默认识别主卡号码，编辑图标进入手动输入可切换|
|验证码发送失败|toast"验证码发送失败，请稍后重试"|
|验证码输入错误|输入框下方红字"验证码错误，请重新输入"|
|登录超时|toast"登录超时，请重试"|
|弱网状态|登录按钮loading态，超时10秒后提示"网络异常"|
|频繁发送验证码|同一手机号60秒内仅可发一次，同一手机号每日上限10次|
|游客模式下卸载重装|游客账号基于设备ID，重装后可恢复（本地数据丢失）|
|游客转正时数据量过大|异步合并，toast"正在同步数据"，完成后通知|

#### 4.3.2 转正漏斗数据看板

|指标|定义|
|---|---|
|游客创建数|当日新创建的游客账号数|
|游客活跃数|当日有活动行为的游客数|
|转正触发次数|游客触发转正引导弹窗的次数|
|转正成功数|完成登录升级为正式账号的游客数|
|转正率|转正成功数 / 转正触发次数|
|平均转正天数|游客从创建到转正的平均天数|
|流失数|创建后30天内未转正且未活跃的游客数|

**看板筛选：**
- 时间范围：近7天/近30天/自定义
- 区服：国内/海外/全部
- 转正触发场景：全部/充值/Steam绑定/云存档/社交

#### 4.3.3 边界条件-内容侧

|场景|处理方式|
|---|---|
|游客账号过多占用资源|30天无活跃的游客账号标记为"待清理"，60天自动清理设备绑定关系|
|游客数据清理|仅清理关联关系，不删除游戏记录（已匿名化）|
|数据导出|支持导出CSV，字段范围：游客ID+设备ID+创建时间+状态+转正时间|

## 五、非功能需求

|需求类型|详细要求|
|---|---|
|性能|登录浮层弹出动画<350ms，一键登录响应<2s，验证码发送<3s|
|兼容性|Android 8.0+，iOS 13.0+；SIM卡识别需兼容移动/联通/电信三大运营商|
|安全|验证码有效期5分钟，错误超5次锁定30分钟；Steam密码传输全链路加密；游客账号设备ID加盐Hash存储|
|容错|SIM识别失败降级为手动输入；Steam二维码服务异常时隐藏二维码区域仅展示账号密码方式|
|合规|协议checkbox未勾选禁止登录；隐私政策弹窗首次安装必弹（国内）|
