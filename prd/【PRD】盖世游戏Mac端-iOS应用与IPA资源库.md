## 2. 背景、目标与范围

### 2.1 需求背景

盖世游戏 Mac 端当前缺少独立的 iOS App 管理能力。用户通过 PlayCover 等工具使用 iOS App 时，需要自行维护本地 App、查找 IPA、判断兼容性并处理安装更新，入口和状态分散。

本项目在盖世游戏 Mac 客户端新增“iOS 应用”一级入口，形成“选择 App 库路径 → 导入本地 IPA 或添加第三方来源 → 选择资源 → 下载与校验 → 安装 → 打开、更新或卸载”的完整闭环。

### 2.2 产品目标

- `[已确认]` 使用一个独立模块管理 iOS App，不与 PC/Mac 游戏库混合。
- `[已确认]` 用户只授权一个 App 库路径，盖世不扫描整台 Mac。
- `[已确认]` 支持本地 IPA 和用户添加的第三方 IPA 来源。
- `[已确认]` 让下载、校验、安装、更新和失败恢复状态清晰可见。
- `[已确认]` 页面采用接近 macOS“应用程序”的图标网格，避免网页商城式大卡片。
- `[建议方案]` 首次成功安装率、任务恢复率和首次打开耗时作为核心效果指标。

### 2.3 本期范围

- 盖世原客户端侧栏新增“iOS 应用”。
- “我的 App / IPA 资源”两个业务 Tab。
- 唯一 App 库路径选择、重新授权和更改路径。
- 本地一个或多个 IPA 文件导入。
- 第三方公开 HTTPS 来源校验、资源预览选择和来源管理。
- Mac 应用程序式图标网格、搜索、分类、来源和状态筛选。
- 单个与批量下载、暂停、继续、取消、校验和失败重试。
- 安装、打开、手动更新、卸载和异常恢复。
- 来源缓存、风险提示、操作日志和数据埋点。

### 2.4 本期不包含

- 多个同时生效的 App 库路径。
- 更改路径时自动迁移 App。
- 来源配置随盖世账号云同步。
- 盖世官方 IPA 资源或任意网页抓取。
- 历史版本选择、降级安装和自动更新。
- 依赖或唤起 PlayCover。
- 分辨率、设备型号、键鼠映射等高级运行参数。

## 3. 用户角色与核心场景

### 3.1 目标用户

- 已在 Mac 上使用 iOS App，希望统一管理本地 App 的用户。
- 知道 IPA 来源地址，希望从第三方来源挑选资源的进阶用户。
- 需要明确看到下载、校验、安装和失败恢复过程的用户。

### 3.2 核心场景

#### 场景 A：导入本地 IPA

用户进入“我的 App”，无论列表是否已有 App，均可点击常驻的“导入 IPA”。系统拉起 macOS 文件选择器，只允许选择一个或多个 `.ipa`。用户在确认弹窗中逐项选择或全选，最终确认后导入到当前 App 库。

#### 场景 B：添加第三方来源并选择资源

用户进入“IPA 资源”，点击“添加来源”，输入公开 HTTPS JSON 地址。系统在停止输入 500ms 后校验。校验成功后第一次点击“导入”只进入资源选择，用户自行勾选需要的资源；最终确认后才保存来源并导入所选资源。

#### 场景 C：下载、暂停和继续安装

用户点击资源的“下载”。资源依次进入排队、下载、校验和待安装状态。用户可以暂停或继续下载；校验通过后点击“安装”。取消安装确认时保留安装包，资源显示“已下载，待安装”。

#### 场景 D：更新失败保留旧版

用户对已安装 App 点击“更新”。新版独立下载和安装，旧版仍保持可打开。新版下载、校验或安装失败时，旧版不得被覆盖或删除。

## 5. 通用业务规则与状态机

### 5.1 公共业务规则

- BR-01 `[已确认]` 同一时间只存在一个活动 App 库路径。
- BR-02 `[已确认]` “导入 IPA”选择本地文件；“更改文件夹”只修改 App 库路径，两者不得复用文案或事件。
- BR-03 `[已确认]` 来源配置只保存在当前 Mac，不随账号同步。
- BR-04 `[已确认]` 来源由用户主动添加，盖世不提供或审核来源内容。
- BR-05 `[已确认]` 来源校验成功后的第一次“导入”不写入资源库，只进入资源选择。
- BR-06 `[已确认]` 更新任务与已安装旧版并存，任何更新失败不得破坏旧版。
- BR-07 `[已确认]` 缺少来源哈希的资源允许下载，但安装前必须二次确认。
- BR-08 `[已确认]` 批量全选只作用于当前筛选结果，并排除已安装、不兼容和处理中资源。
- BR-09 `[推导]` 同一资源的重复点击使用任务幂等键拦截，避免生成重复下载或安装任务。

### 5.2 三层状态模型

单个 IPA 资源由三层状态共同决定页面表现：

| 层级 | 字段 | 取值 |
|---|---|---|
| 资源属性 | compatible、hash、version、source | 兼容/不兼容、哈希存在/缺失、版本与来源 |
| 下载任务 | download_status | idle、queued、downloading、paused、validating、ready、download_failed、validation_failed |
| 安装任务 | install_status | none、installing、installed、install_failed |

“可更新”由 `resource.version > installed.version` 推导，不单独持久化。

### 5.3 状态与唯一主操作

| 显示状态 | 页面文案 | 唯一主操作 | 操作结果 |
|---|---|---|---|
| 未下载 | 未获取 | 下载 | 进入排队中 |
| 排队中 | 排队中 · 第 N 位 | 取消排队 | 回到未下载 |
| 下载中 | 下载中 N% | 暂停 | 进入已暂停 |
| 已暂停 | 已暂停 N% | 继续 | 回到下载中 |
| 校验中 | 正在校验 | 无 | 进入待安装或校验失败 |
| 已下载待安装 | 已下载，待安装 | 安装 | 进入安装确认/安装中 |
| 安装中 | 安装中 N% | 无 | 进入已安装或安装失败 |
| 已安装 | 已安装 | 打开 | 启动 App |
| 可更新 | 可更新至 X | 更新 | 下载新版，保留旧版 |
| 下载失败 | 下载失败 | 重新下载 | 新建下载尝试 |
| 校验失败 | 文件校验失败 | 删除并重下 | 删除坏包并重新下载 |
| 安装失败 | 安装失败 | 重试安装 | 使用已下载包重试 |
| 不兼容 | 展示具体原因 | 查看原因 | 打开详情，不改变状态 |
| 缺少来源哈希 | 缺少来源哈希 | 下载 | 安装前进入风险确认 |

## 7. F002 唯一 App 库路径

### 7.1 输入、处理与输出

- 输入：用户在首次空状态或“更改文件夹”中选择一个 macOS 文件夹。
- 处理：校验读写权限、剩余空间和授权凭据；保存唯一活动路径。
- 输出：展示路径，支持“在访达中显示、重新扫描、更改文件夹”。
- 异常：取消不报错；无权限不覆盖旧路径；权限失效时保留上次扫描缓存并进入只读状态。

### 7.2 关键规则

- `[已确认]` 更改路径不迁移、不复制、不删除原路径内容。
- `[已确认]` 下载、校验、安装或卸载任务进行中时禁止更改路径。
- `[已确认]` 未设置路径时点击“导入 IPA”或下载资源，先完成路径选择，再自动续接原动作。

### 7.3 验收标准

- AC-F002-01 `[客户端]` 单次只能选择一个可读写文件夹。
- AC-F002-02 `[客户端]` 取消选择后保留当前页面且不显示错误。
- AC-F002-03 `[端到端]` 新路径失败时旧路径和旧索引保持可用。
- AC-F002-04 `[客户端]` 任务进行中“更改文件夹”禁用并说明原因。

## 8. F003 本地 IPA 导入

### 8.1 客户端需求

- `[已确认]` “我的 App”工具栏常驻绿色主按钮“导入 IPA”，空列表、非空列表、筛选无结果和异常状态均显示。
- `[已确认]` 点击后拉起 macOS 文件选择器，仅允许选择一个或多个 `.ipa`。
- `[已确认]` 文件选择完成后进入玻璃拟态选择弹窗，标题为“导入 IPA”。
- `[已确认]` 弹窗右上角只有一个“全选/取消全选”；底部左侧为“已选 X / Y”，底部右侧只保留“取消/导入”。
- `[已确认]` 重复、损坏、不兼容和非 IPA 文件置灰并说明原因，不计入 Y。

### 8.2 验收标准

- AC-F003-01 `[客户端]` 已有 App 时“导入 IPA”仍保持可用。
- AC-F003-02 `[客户端]` 选择 0 项时最终“导入”置灰，选择至少 1 项后启用。
- AC-F003-03 `[客户端]` 全选只勾选可导入项，禁用项状态不改变。
- AC-F003-04 `[端到端]` 最终确认后仅导入选中 IPA，不扫描其他文件夹内容。

## 10. F005 第三方来源与资源选择

### 10.1 来源校验

- `[已确认]` 仅支持无需账号、Cookie、自定义请求头或令牌的公开 HTTPS JSON 来源。
- `[已确认]` 用户停止输入 `500ms` 后校验；URL 修改后立即清除旧成功状态。
- `[已确认]` 校验反馈区固定高度，文案变化不得造成弹窗抖动。
- `[已确认]` URL、JSON、目标资源或来源结构无效时统一提示：`链接无效，JSON无效或未找到！`
- `[已确认]` AppHub、Mirror、网络失败示例仅属于 Demo 工具，不进入正式业务弹窗。

### 10.2 两段式导入

1. 有效来源弹窗点击第一次“导入”。
2. 系统打开“选择要导入的 IPA”，此时不保存来源。
3. 资源默认不勾选，用户逐项选择或使用右上角全选。
4. 用户最终确认后，系统保存来源并导入所选资源。
5. 取消第二级弹窗时不保存来源；返回上一级时保留 URL 和校验结果。

### 10.3 来源管理

- 展示来源名称、URL、启用状态、最近成功时间和缓存状态。
- 支持刷新、停用、启用、删除和再次选择未导入资源。
- 删除来源不卸载已安装 App；其他来源仍包含的资源继续保留。

### 10.4 验收标准

- AC-F005-01 `[客户端]` 无效来源文案逐字匹配“链接无效，JSON无效或未找到！”。
- AC-F005-02 `[端到端]` 第一次“导入”后来源数和资源数均不增加。
- AC-F005-03 `[客户端]` 第二级弹窗默认选择数为 0，返回时保留校验成功状态。
- AC-F005-04 `[端到端]` 最终确认后只导入选中资源，未选资源不进入当前资源列表。

## 11. F006 IPA 资源网格与聚合

### 11.1 网格与筛选

- `[已确认]` 使用与“我的 App”一致的 Mac 图标网格。
- `[已确认]` 分类为“全部、游戏、工具、娱乐、社交、创意、其他”。
- `[已确认]` 支持名称/Bundle ID 搜索、来源筛选和状态筛选。
- `[已确认]` 状态筛选为“全部、未安装、进行中、待安装、已安装、可更新、需处理”。
- `[已确认]` 图标下固定显示名称、状态和一个紧凑主操作；版本、体积、来源、Bundle ID、哈希和兼容原因在详情浮层展示。

### 11.2 聚合规则

- 按 Bundle ID 合并同一 App。
- 使用版本号和 Build 号选择最高兼容构建。
- 同版本优先使用哈希有效且当前可访问的来源。
- 下载后核对 IPA 内部 Bundle ID 与来源元数据。
- Team ID、签名主体或关键身份变化时禁止静默覆盖。

### 11.3 验收标准

- AC-F006-01 `[客户端]` 默认状态全景可稳定展示 14 种显示状态。
- AC-F006-02 `[客户端]` 搜索、分类、来源和状态筛选可组合生效。
- AC-F006-03 `[端到端]` 相同 Bundle ID 只展示一个最高兼容版本。
- AC-F006-04 `[客户端]` 状态变化不改变图标网格单元高度，不造成页面跳动。

## 12. F007 下载任务与完整状态机

### 12.1 任务流程

```text
未下载 → 排队中 → 下载中 ⇄ 已暂停 → 校验中 → 已下载待安装
              └→ 下载失败        └→ 校验失败
```

### 12.2 任务规则

- 每个 App 使用独立任务 ID，暂停、继续、取消和重试只影响目标 App。
- 进度回调携带任务代次；任务取消或重置后，旧回调不得覆盖新状态。
- 下载失败保留失败原因，可重新下载。
- 校验失败必须删除坏包后重下。
- 无哈希资源不显示“已验证安全”。
- 切换 Tab 或筛选不终止进行中的任务。

### 12.3 验收标准

- AC-F007-01 `[客户端]` 下载中展示环形进度和整数百分比。
- AC-F007-02 `[端到端]` 暂停后进度停止，继续后从原进度恢复。
- AC-F007-03 `[端到端]` 重置或取消后旧回调不能改变当前状态。
- AC-F007-04 `[端到端]` 下载失败、校验失败拥有不同恢复动作。

## 13. F008 安装、打开、更新与卸载

### 13.1 安装

- 校验通过后进入“已下载，待安装”。
- 点击安装后展示版本、来源、兼容和风险信息。
- 缺少来源哈希时二次确认；取消后保留待安装包。
- 安装成功后进入已安装，支持打开和前往“我的 App”。
- 安装失败保留下载包，提供“重试安装”。

### 13.2 更新

- 可更新由资源版本与已安装版本比较产生。
- 更新沿用下载、校验、确认和覆盖安装流程。
- 更新过程中旧版仍可打开。
- 更新失败必须保留旧版本和用户数据。

### 13.3 验收标准

- AC-F008-01 `[客户端]` 缺少哈希时安装前出现明确风险确认。
- AC-F008-02 `[端到端]` 取消风险确认后资源保持“已下载，待安装”。
- AC-F008-03 `[端到端]` 新版失败后旧版仍能正常打开。
- AC-F008-04 `[端到端]` 安装失败重试不重新下载已通过校验的文件。

## 14. F009 批量选择与批量任务

### 14.1 客户端需求

- 正常模式不显示复选框；点击“批量选择”后显示在图标左上角。
- 页面右上角显示单一“全选/取消全选”。
- 全选作用于当前搜索、分类、来源和状态筛选结果。
- 底部玻璃操作条显示已选数量、预计大小、取消和下载。
- 退出批量模式清空选择。

### 14.2 验收标准

- AC-F009-01 `[客户端]` 不可下载资源不会被全选。
- AC-F009-02 `[客户端]` 筛选变化后自动移除不再有效的选中项。
- AC-F009-03 `[端到端]` 单项失败不阻塞其他成功项进入安装确认。
- AC-F009-04 `[客户端]` 批量模式和单项按钮不会同时触发两个任务。

## 15. F010 日志、埋点与质量监控

### 15.1 埋点

| 事件名 | 触发时机 | 关键字段 |
|---|---|---|
| ios_library_path_set | App 库路径设置成功 | path_source、scan_count、duration_ms |
| ios_local_ipa_import | 本地 IPA 最终确认 | selected_count、success_count、failed_count |
| ipa_source_validate | 来源校验完成 | result、failure_reason、latency_ms |
| ipa_source_import | 来源资源最终确认 | source_id、available_count、selected_count |
| ipa_download_action | 下载、暂停、继续、取消或重试 | resource_id、action、progress、result |
| ipa_validation_result | 文件校验完成 | resource_id、hash_present、result、failure_reason |
| ipa_install_result | 安装或更新完成 | resource_id、mode、result、failure_reason |
| ios_app_open | App 打开结果 | app_id、version、result、failure_reason |
| ios_app_uninstall | 卸载确认完成 | app_id、keep_user_data、result |

### 15.2 日志和监控

- `[建议方案]` 每个任务记录 `task_id、resource_id、generation、from_status、to_status、timestamp、error_code`。
- `[建议方案]` 监控来源校验成功率、下载成功率、校验通过率、安装成功率、任务恢复率和首次打开耗时。
- `[建议方案]` 错误日志不记录完整本地路径和来源敏感参数；上传前脱敏。

## 16. 端到端契约与数据模型

### 16.1 本地资源对象建议

```json
{
  "resource_id": "res-wechat-8052",
  "bundle_id": "com.tencent.xin",
  "name": "微信",
  "version": "8.0.52",
  "build": "8052",
  "size_bytes": 686817280,
  "source_id": "apphub",
  "download_url": "https://example.com/wechat.ipa",
  "hash": "sha256:...",
  "compatible": true,
  "download_status": "idle",
  "install_status": "installed",
  "installed_version": "8.0.50"
}
```

### 16.2 关键错误码建议

| 错误码 | 场景 | 客户端处理 |
|---|---|---|
| PATH_NO_PERMISSION | App 库不可写 | 保留原路径，引导重新授权 |
| SOURCE_INVALID | URL/JSON/来源结构无效 | 显示“链接无效，JSON无效或未找到！” |
| DOWNLOAD_FAILED | 下载失败 | 显示重新下载 |
| VALIDATION_FAILED | 包或哈希校验失败 | 显示删除并重下 |
| INCOMPATIBLE | 架构或系统不兼容 | 禁用下载/安装并可查看原因 |
| INSTALL_FAILED | 安装失败 | 保留包和旧版，显示重试安装 |
| IDENTITY_CHANGED | Bundle/Team/签名主体变化 | 阻止静默覆盖并明确警告 |

## 17. 非功能需求

- `[建议方案]` 本地搜索和筛选响应时间 P95 ≤ 100ms（1,000 个资源以内）。
- `[建议方案]` 来源校验超时为 10s；超时进入失败状态，不阻塞用户修改 URL。
- `[建议方案]` 进度刷新频率不高于 4 次/秒，避免频繁重绘。
- `[建议方案]` App 库、来源和任务状态在客户端异常退出后可恢复。
- `[已确认]` 来源和 App 库路径只保存在当前 Mac，不跟随账号同步。
- `[已确认]` UI 支持键盘焦点、按钮禁用原因和文字状态，不只依赖颜色。

## 19. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 单状态字符串覆盖组合状态 | 更新任务可能覆盖旧版安装状态 | 使用资源、下载、安装三层状态模型 |
| 第三方来源不稳定 | 刷新或下载失败 | 保留上次成功缓存，允许同版本备用来源 |
| 无哈希资源被误认为安全 | 用户安全风险 | 明确风险，不默认选择，安装前二次确认 |
| 路径权限中途失效 | 下载完成后无法安装 | 任务开始前校验，失效时引导重新授权 |
| 进度回调串任务 | 卡片状态错误 | 使用 task_id + generation 校验回调 |
