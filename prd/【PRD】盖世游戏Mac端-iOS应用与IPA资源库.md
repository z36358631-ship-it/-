## 2. 背景、问题与目标

### 2.1 背景

盖世游戏 Mac 端需要承接 PlayCover 用户的本地 iOS App 管理和 IPA 来源使用场景。用户既需要管理已安装 App，也需要添加 IPA 来源、浏览应用、下载并安装。本期以 PlayCover 3.1.0 可见能力为功能基准，在盖世原有 Mac 客户端侧栏和视觉体系中实现同等能力。

### 2.2 现存问题

- 本地 App、IPA 来源和下载状态缺少统一入口。
- 仅有网格展示时，版本、Bundle ID、大小和状态难以比较。
- App 的设置、数据清理、访达定位和卸载操作不完整。
- 来源用顶部下拉筛选不能表达多来源层级、同名来源和聚合关系。
- 将 App 归入“游戏、工具、娱乐”等分类会增加维护成本，但不帮助用户完成安装任务。
- 导入、清理和卸载属于高风险动作，需要比竞品更明确的选择与确认。

### 2.3 产品目标

- `[已确认]` 在盖世 Mac 客户端中提供独立的 iOS 应用管理模块。
- `[已确认]` 用“我的 App / IPA 资源”两个 Tab 分离本机管理与远程资源发现。
- `[已确认]` 功能与 PlayCover 3.1.0 保持一致，并保留盖世的来源校验、二级选择、哈希确认和异常恢复。
- `[已确认]` 用户可在网格与列表之间自由切换，交互语义不变。
- `[已确认]` 下载、校验、安装、更新与失败恢复状态可识别、可操作。
- `[建议指标]` 来源添加成功率、下载成功率、安装成功率、失败恢复率和首次启动成功率可度量。

### 2.4 非目标

- 不依赖或唤起外部 PlayCover 客户端。
- 不提供官方盗版 IPA、破解、签名绕过或任意网页抓取。
- 不支持多个同时生效的本地 App 库路径。
- 不自动迁移旧 App 库内容。
- 不做键盘映射布局的导入与导出。
- 不做页面级“全部、游戏、工具、娱乐、社交、创意、其他”分类。
- 不做来源账号云同步、历史版本选择、自动降级和自动更新。

## 3. 用户与核心场景

### 3.1 目标用户

- 已在 Apple Silicon Mac 上运行 iOS App，需要统一管理的用户。
- 持有合法 IPA 文件，希望单个或批量导入的用户。
- 知道公开 IPA 来源地址，希望浏览、下载和更新资源的进阶用户。
- 需要调整运行机型、分辨率、兼容和键鼠设置的用户。

### 3.2 核心场景

#### 场景 A：管理本地 App

用户进入“我的 App”，可搜索、刷新并切换网格或列表。单击选中 App 后可打开设置；右键可定位应用数据、在访达中查看、分别清理数据或卸载。

#### 场景 B：导入本地 IPA

“导入 IPA”始终显示。用户点击后拉起 macOS 文件选择器，可选择一个或多个 `.ipa`。系统解析后展示二级选择弹窗，默认不勾选，用户逐项选择或全选后确认导入。

#### 场景 C：按来源浏览 IPA

用户进入“IPA 资源”，左侧来源树展示“全部来源”和各个已添加来源。点击来源即筛选当前内容；同名来源仍保持独立。

#### 场景 D：下载和安装

用户单击选择资源、双击开始下载，或点击资源主操作。资源按排队、下载、暂停、校验、可安装、安装等状态流转。失败时可针对单项重试。

#### 场景 E：查看和调整 App 设置

用户从右键或工具栏打开设置，使用“键盘映射、图像设置、绕过、杂项、详细信息”五页设置。修改只作用于当前 App。

      ├─ 应用信息

### 4.1 页面职责

| Tab | 管理对象 | 主要动作 | 禁止混入 |
|---|---|---|---|
| 我的 App | 当前 App 库中已安装或已导入的 App | 启动、设置、定位、清理、卸载、导入 IPA | 远程来源筛选、页面分类 |
| IPA 资源 | 用户添加的远程 IPA 来源与任务 | 浏览、查看信息、下载、安装、更新、管理来源 | 本地 App 数据清理 |

## 5. 通用规则

### 5.1 唯一身份

- App 和 IPA 资源以 `bundleId` 为业务身份，以 `bundleId + version + build` 标识具体构建。
- 来源必须使用系统生成的稳定 `sourceId`，不得用域名或展示名作为唯一键。
- 同名、同域名来源允许同时存在，来源树中保持独立节点。

### 5.2 列表与网格

- 两个 Tab 均支持列表与网格。
- 单击：选中当前项目，启用依赖选中的工具栏动作。
- 点击空白：取消选择。
- 右键：打开与当前项目匹配的上下文菜单。
- IPA 资源双击：开始下载或执行当前唯一主操作。
- 切换视图后保留搜索、来源筛选、排序和任务状态。

### 5.3 刷新

- “我的 App”刷新：重新扫描唯一 App 库路径，不修改 App 数据。
- “IPA 资源”根节点刷新：刷新全部启用来源。
- “IPA 资源”来源节点刷新：只刷新当前来源。
- 刷新失败：保留上次成功缓存，提示失败原因和缓存时间。
- 刷新不得中断正在进行的下载、校验和安装任务。

### 5.4 危险操作

- 清除应用数据、清除应用偏好、清除 PlayChain 和卸载必须分别确认。
- 一次确认不得授权多个后续危险操作。
- 卸载确认可选择同时清理 PlayChain、Entitlements、应用设置、键盘映射和应用数据。
- 取消确认不得修改任何本地文件。

## 6. F001 我的 App 页面

### 6.1 工具栏

从左到右包含：

1. 搜索框：按 App 名称或 Bundle ID 搜索。
2. 常驻“导入 IPA”主按钮：空库和非空库均显示。
3. “刷新”：重新扫描本地 App 库。
4. “设置”：未选中 App 时置灰，选中后启用。
5. 网格/列表切换。

### 6.2 展示字段

| 视图 | 必须展示 |
|---|---|
| 网格 | 图标、名称、版本、关键状态角标 |
| 列表 | 图标、名称、Bundle ID、版本、最近使用时间、状态 |

### 6.3 App 右键菜单

顺序固定为：

1. 设置。
2. 显示应用数据。
3. 在访达中查看。
4. 分隔线。
5. 清除应用数据。
6. 清除应用偏好设置。
7. 清除 PlayChain 数据。
8. 分隔线。
9. 卸载应用。

`[不做]` 菜单中不展示“导入键盘映射布局”和“导出键盘映射布局”。

### 6.4 验收标准

- 已有 App 时“导入 IPA”仍显示。
- 右键菜单项目、顺序、分隔线和危险色与本文一致。
- 工具栏设置与右键设置打开同一窗口。
- 网格和列表均可选择、右键和搜索。
- 刷新完成后显示本次扫描结果；失败保留原列表。

## 7. F002 App 设置

### 7.1 设置窗口

- 窗口标题为“{App 名称} 设置”，展示 App 图标。
- 顶部为五个分段 Tab：键盘映射、图像设置、绕过、杂项、详细信息。
- 底部提供“重置设置”“重置键盘映射”和“关闭”。
- 设置仅作用于当前 App，修改后立即保存或在关闭时原子保存；技术方案二选一，但不得出现半保存状态。

### 7.2 键盘映射

- 启用键盘映射。
- 智能键盘映射。
- 鼠标滚轮映射。
- 鼠标灵敏度。

### 7.3 图像设置

- 选择 iOS 机型。
- 分辨率：自动或手动档位。
- 展示当前检测到的分辨率。
- 分辨率缩放。
- 修复窗口化显示问题及其兼容模式。
- 禁用显示器息屏。
- 允许旋转。

### 7.4 绕过

- 启用 PlayChain。
- 绕过越狱检测。
- 禁用应用内省库。
- iOS 框架注入模式。

### 7.5 杂项

- Discord Rich Presence。
- Metal HUD。
- LLDB 调试。
- 注入 PlayTools。
- root 工作目录。
- 应用程序类型元数据。

“应用程序类型”仅属于当前 App 的运行元数据，不作为页面筛选分类。

### 7.6 详细信息

- Bundle 名称。
- Bundle ID。
- 版本和 Build。
- 最低 iOS 版本。
- App 本地路径。
- 显示名称别名。

### 7.7 验收标准

- 五个 Tab 内容完整，切换不丢失未完成输入。
- 设置窗口不得出现键盘映射导入和导出入口。
- 重置设置和重置键盘映射互不替代，均需明确反馈。
- 非法数值不得保存，并在原字段就地提示。

## 8. F003 本地 IPA 导入

### 8.1 文件选择

- 点击“导入 IPA”拉起 macOS 原生文件选择器。
- 只允许选择 `.ipa`，支持单选和多选。
- 取消选择后回到原页面，不显示错误。
- 未授权 App 库路径时，先选择唯一目录，再继续原导入意图。

### 8.2 二级选择

- 文件解析完成后展示选择弹窗，默认不勾选。
- 右上角提供“全选/取消全选”，位于候选内容容器上方。
- 重复、损坏、不兼容或非 IPA 项置灰并说明原因。
- 底部展示已选数量；无选择时“导入”置灰。
- 最终确认后才创建导入任务。

### 8.3 验收标准

- 单个和批量 IPA 走同一套二级选择与确认。
- 异常文件不影响其他有效文件被选择。
- 导入后项目显示真实解析状态，不直接伪装成“已安装”。

## 9. F004 IPA 来源树与来源管理

### 9.1 来源树

左侧固定展示：

```text
IPA 资源库（全部启用来源）
├─ 来源 A
├─ 来源 B
└─ 来源 C
```

- 根节点按 Bundle ID 聚合全部启用来源并去重。
- 单个来源节点只展示该 `sourceId` 的资源。
- 来源名称相同也必须显示为不同节点。
- 当前来源失效或被删除时，自动返回根节点。
- 来源顺序决定同版本资源的聚合优先级。

### 9.2 添加来源

- 只接受公开 HTTPS JSON 地址，不接受账号、令牌或登录态。
- 输入变化后清除旧验证结果。
- 验证反馈容器固定高度，避免弹窗抖动。
- 地址、JSON 或目标资源无效时提示“链接无效，JSON 无效或未找到！”。
- 验证成功后点击“继续”进入资源二级选择，不直接保存来源。
- 最终确认后才保存来源和选中资源。

### 9.3 来源管理

每个来源支持：

- 启用/停用。
- 删除。
- 上移/下移。
- 刷新。
- 查看 URL、最近成功缓存时间和当前状态。

### 9.4 验收标准

- 根节点和来源节点刷新范围正确。
- 停用来源后根节点聚合不再使用该来源，但不删除本地已安装 App。
- 删除当前来源后回到根节点且页面无空引用。
- 刷新失败后仍可查看上次成功缓存。

## 10. F005 IPA 资源列表与网格

### 10.1 工具栏

从左到右包含：

1. 名称或 Bundle ID 搜索。
2. 添加来源。
3. 刷新当前路径。
4. 应用信息：未选择时置灰。
5. 字母排序。
6. 网格/列表切换。

### 10.2 展示字段

| 视图 | 必须展示 |
|---|---|
| 网格 | 图标、名称、版本、状态、唯一主操作、必要进度 |
| 列表 | 操作、图标、名称、Bundle ID、版本、大小、状态 |

### 10.3 右键与详情

- IPA 资源右键只有“应用信息”。
- 右键“应用信息”和顶部 `ⓘ` 打开同一详情窗口。
- 详情包括 Bundle 名称、Bundle ID、版本、iTunes 地址、IPA 地址、校验和、来源、体积、兼容性和风险。
- 图标加载失败只显示占位图，不得把资源标记为 IPA 无效。

### 10.4 验收标准

- 列表与网格的搜索、来源筛选、排序结果一致。
- 双击资源进入下载或当前唯一主操作。
- 未选择资源时应用信息按钮不可点击。
- 切换视图不清除正在进行的任务。

## 11. F006 下载、校验、安装和更新

### 11.1 状态与唯一主操作

| 状态 | 主操作 | 说明 |
|---|---|---|
| 未下载 | 下载 | 创建下载任务 |
| 排队中 | 取消排队 | 不影响其他任务 |
| 下载中 | 暂停 | 展示进度 |
| 已暂停 | 继续 | 从断点恢复 |
| 校验中 | 无 | 禁止重复操作 |
| 可安装 | 安装 | 包体已校验 |
| 安装中 | 无 | 禁止重复安装 |
| 已安装同版本 | 已安装/打开 | 不重复安装 |
| 本地版本更高 | 保留本地版 | 不自动降级 |
| 可更新 | 更新 | 独立下载新版 |
| 下载失败 | 重试 | 保留失败原因 |
| 校验失败 | 删除并重下 | 不允许安装 |
| 安装失败 | 重试安装 | 保留已校验包体 |
| 不兼容 | 查看原因 | 不允许下载或安装 |
| 缺少来源哈希 | 安装前确认 | 允许下载，不静默安装 |

### 11.2 任务规则

- 每个资源任务相互独立。
- 暂停、继续、取消和重试只影响当前资源。
- 切换 Tab、来源、搜索、排序或视图不终止任务。
- 新版下载、校验或安装失败时，旧版 App 及用户数据保持可用。
- 取消安装确认后保留已下载文件并显示“可安装”。
- 来源刷新失败不影响已创建任务继续执行。

### 11.3 聚合规则

- 根节点按 Bundle ID 聚合。
- 优先选择兼容当前 Mac 的最高版本和 Build。
- 同版本优先使用有校验和、可访问且来源排序靠前的条目。
- 实际下载前再次解析目标来源，防止缓存地址失效。
- 下载完成后校验包内 Bundle ID 与来源元数据一致。

## 12. F007 卸载与本地数据处理

### 12.1 右键清理

- “显示应用数据”和“在访达中查看”是只读导航，不弹危险确认。
- 清除应用数据只清理该 App 用户数据。
- 清除应用偏好设置只清理该 App 偏好。
- 清除 PlayChain 只清理该 App 对应 PlayChain 数据。

### 12.2 卸载

- 卸载确认展示当前 App 名称和不可逆提示。
- 用户可勾选是否同时清理 PlayChain、Entitlements、设置、键盘映射和应用数据。
- 默认勾选 PlayChain、Entitlements 和设置；键盘映射和应用数据默认不勾选。
- 卸载失败时保留 App 列表项并展示失败原因，不得形成“已卸载”假状态。

## 14. 埋点与监控

| 事件 | 关键参数 |
|---|---|
| `ios_library_tab_view` | tab、entry |
| `ios_library_refresh` | tab、sourceId、result、cacheUsed、duration |
| `ios_app_context_action` | bundleId、action、confirmResult |

## 15. 非功能要求

- 来源 URL、IPA URL 和日志中不得持久化账号 Cookie、令牌或系统隐私路径。
- 来源解析与下载必须限制响应体大小、重定向次数、超时和并发数。
- 本地路径只在当前 Mac 保存，不随账号同步。
- 列表达到 1,000 项时仍需保持可滚动与可搜索，建议使用虚拟列表。
- 状态更新不得改变卡片固定区域高度，避免界面抖动。
- 所有按钮具备 hover、disabled、focus 和键盘访问状态。
- 应用设置保存、卸载和数据清理必须具备失败回滚或明确失败态。

## 17. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 第三方来源内容失效或篡改 | 下载或安全风险 | HTTPS、缓存、校验和、Bundle ID 二次校验、安装前风险确认 |
| 同名来源误合并 | 用户无法判断资源来源 | 使用稳定 `sourceId`，显示名只用于展示 |
| 图标 CDN 失败被误判为包体失败 | 误导用户 | 图标独立降级，包体状态独立计算 |
| 设置项与运行能力不一致 | 开发返工 | 每个设置字段建立能力开关和兼容矩阵 |
| 数据清理范围不清 | 用户数据损失 | 分项操作、逐次确认、卸载清理选项显式展示 |
| 任务状态过多导致界面混乱 | 操作错误 | 每个状态仅保留一个主操作，详情承载原因 |
