# F005 用户订单、续租与售后

> 状态：评审稿 | 优先级：P0 | 业务域：用户订单
> 内容来源：最新 Demo 租号中心、订单详情、退款申请与退款详情（提交 `35c72ea5`）

## 1. 功能概述与用户价值

用户在租号中心查看完整订单生命周期，继续支付、启动、续租、取消、删除、再来一单或申请售后，并在订单详情和退款详情中看到明确进度。

## 2. 范围、前置条件与依赖

- 范围：订单列表、订单详情、服务端租期倒计时、15/5 分钟提醒、续租、重新租用、到期处理、再来一单、取消、删除、售后申请和退款进度。
- 依赖：[F003](F003-下单支付与资源锁定.md)、[F004](F004-Steam一键上号.md)、[F008](F008-订单与售后工作台.md)、[F011](F011-3天无理由退款与风控.md)。

## 3. 订单状态与操作

| 状态 | 列表/详情操作 |
|---|---|
| 待支付 | 继续支付、取消订单 |
| 租赁中 | 续租、申请售后、一键上号 |
| 已完成 | 再来一单、删除订单 |
| 已取消 | 删除订单 |
| 售后处理中/退款中 | 查看退款详情 |
| 已退款 | 退款进度、再来一单、删除订单 |

## 4. 通用业务规则

1. `[已确认]` 租号中心只保留“全部、租赁中、待支付”三个 Tab。
2. `[已确认]` 订单卡片整块可点击进入详情，不保留固定“查看详情”按钮。
3. `[已确认]` 卡片展示有效期；租赁中展示灰色剩余时长；待支付在有效期下展示支付倒计时。
4. `[已确认]` 订单状态固定在卡片右上角，金额显示在状态下方。
5. `[已确认]` 一键上号位于操作区最右侧；上号、再来一单、删除按钮不显示图标。
6. `[已确认]` “再来一单”回传上一次游戏、版本和套餐；文案不使用“再次租借”。
7. `[已确认]` 订单详情顶部首先展示订单状态和对应操作。
8. `[已确认]` 订单号在详情中展示，右侧提供复制按钮；游戏封面区域进入游戏详情。
9. `[已确认]` 订单进度横向展示：订单创建 → 订单支付 → 订单到期 → 完成。
10. `[已确认]` 不展示“履约记录”或“履约与上号”Tab。
11. `[已确认]` 退款申请使用弹窗，不做资格选择；问题类型平铺，必须填写问题描述。
12. `[已确认]` 问题类型为3天无理由、启动失败、Steam 登录失败、账号异常/频繁掉线、其他问题。
13. `[已确认]` 退款详情沿用订单详情样式，进度横向展示；申请中提示“预计 7 个工作日内处理”，最后一步为“完成”。
14. `[已确认]` 取消和删除必须使用带游戏信息的二次确认弹窗。
15. `[已确认]` 服务端 `expire_at` 是租期有效性的唯一时钟；客户端使用 `server_time` 计算剩余时长。
16. `[已确认]` 有效订单在剩余 15 分钟、5 分钟首次命中时顶部居中各提醒一次，不设置 1 分钟档位；休眠恢复同时错过两档时只补当前最近的一档。
17. `[已确认]` 续租从原 `expire_at` 顺延并延长原服务订单，不新建服务订单、不换账号、不退出当前游戏。
18. `[已确认]` 原账号后续预约部分冲突时只允许最大可续时长；完全冲突不能支付，第一期不中途自动换号。
19. `[已确认]` 原订单到期前续租未支付成功时关闭续租单；迟到回调不得延长订单或重新占用账号，只幂等原路退款。
20. `[已确认]` 已完成订单进入“重新租用”时创建新服务订单；同游戏同版本本地包可复用并直接启动，版本不同才更新或下载。
21. `[已确认]` T0 到期退出结果弹窗展示封面、游戏名与当前最低时租价 `¥X 租号`，不展示平台、操作、版本或订单状态副文案。

## 5. 客户端需求

1. 订单列表支持“搜索游戏”，筛选切换不改变订单真实状态。
2. 卡片内部按钮点击不得冒泡触发卡片跳转。
3. 支付倒计时使用服务端时间；到 0 后立即刷新为已取消。
4. 详情根据 `allowedActions` 渲染按钮，不显示不适用的退款状态。
5. 售后弹窗展示游戏封面、版本、金额、有效期、五类问题和描述输入框。
6. 描述框占位为“请输入你遇到的问题”，空描述禁止提交。
7. 提交后直接进入退款详情并展示“申请中”。
8. 退款未申请时访问退款详情，应回退到普通订单详情。
9. 复制订单号成功后给出轻提示。
10. “3天无理由”仅在支付后不超过 72 小时、订单目标游戏累计时长不超过 30 分钟且订单资格快照可用时允许选择；不可用时展示具体原因。
11. 无理由类型不可用不影响启动失败、Steam 登录失败、账号异常等履约售后类型。
12. 临期提醒不暂停游戏，支持知道了/暂不续租与立即续租；立即续租打开原订单续租页，关闭不改变订单状态。
13. 续租页展示原到期时间、选择时长、续租后到期时间、原账号最大可用时间；续租游戏和版本不可切换。
14. 到期 T0 立即刷新订单为已完成并结束目标游戏、清理租赁会话；安装包、设置和个人存档保留。

## 6. 服务端需求

1. 订单查询按登录用户隔离，支持状态、游戏关键词和分页。
2. 返回 `serverTime`、`expireAt`、`paymentExpiresAt`、15/5 提醒实际展示时间和 `allowedActions`；所有临期与到期判断统一使用 `expireAt`。
3. 续租提交在原子控制中校验订单仍有效、套餐价格、原账号绑定、`availableUntil` 和后续预约，冻结原到期时间与续租后到期时间。
4. 再来一单只返回预填参数，不复用原订单、支付或账号绑定。
5. 终态订单删除采用用户侧隐藏/软删除，不删除交易、退款和审计记录。
6. 售后单幂等创建，保存问题类型、用户原始描述、订单、账号、启动结果和诊断摘要。
7. 售后状态变化推送或供客户端轮询；退款金额与资源释放结果分别记录。
8. 服务端实时校验无理由退款的 72 小时时限、订单游戏时长和不可变资格快照，不能只依赖客户端是否禁用选项。
9. 售后单保存退款原因编码、资格判定结果、规则版本、累计游戏时长和不可用原因。
10. 续租支付成功更新原订单 `expireAt = oldExpireAt + duration` 并重置 15/5 提醒标记；原账号和现有会话不变。
11. 到期任务关闭待支付续租单，结束订单、撤销会话并释放账号；迟到支付回调按续租交易 ID 幂等进入原路退款。
12. 重新租用创建新订单并重新分配账号，返回本地包是否可复用及是否可直接启动。

## 7. 端到端契约

`[建议方案]`

```http
POST /rental/orders/{orderId}/after-sales
Idempotency-Key: as-1001
```

```json
{
  "reasonCode":"STARTUP_FAILED",
  "description":"启动后持续黑屏，重试两次仍无法进入。"
}
```

`[建议方案]` 续租提交返回原到期时间、可续上限和冻结后的新到期时间；冲突使用 `RENEWAL_DURATION_EXCEEDED` 或 `RENEWAL_RESERVATION_CONFLICT`，到期关闭后回调返回 `RENEWAL_CLOSED_REFUND_REQUIRED`。

```json
{
  "afterSalesId":"AS-1001",
  "status":"REVIEW",
  "estimatedProcessingText":"预计 7 个工作日内处理",
  "launchResult":"FAILED"
}
```

## 8. 埋点、日志与监控

- `rental_order_list_view`、`rental_order_action_click`、`rental_order_copy`、`rental_repeat_order`、`rental_after_sales_submit`、`rental_refund_detail_view`。
- `rental_expiry_reminder_show`：15/5 提醒成功展示时记录 `order_id`、`reminder_stage`、`remaining_seconds`、`scene`。
- `rental_expiry_reminder_action`：关闭或立即续租时记录 `order_id`、`reminder_stage`、`action`。
- `rental_renewal_submit`：续租交易提交时记录 `order_id`、`renewal_order_id`、`package_code`、`new_expire_at`。
- `rental_renewal_result`：续租成功、失败或关闭时记录 `order_id`、`renewal_order_id`、`result`、`reason`。
- 监控待支付超时任务、续租冲突率、售后提交失败率、描述缺失率、审核超时和退款状态滞留。

## 9. 验收标准

- `AC-F005-01 [客户端]` 订单中心只显示全部、租赁中、待支付三个 Tab。
- `AC-F005-02 [客户端]` 卡片点击进入详情，状态固定右上，金额在状态下方。
- `AC-F005-03 [客户端]` 待支付倒计时到 0 后订单变为已取消且不能继续支付。
- `AC-F005-04 [客户端]` 详情展示订单号复制、顶部状态操作和四步横向进度。
- `AC-F005-05 [端到端]` 再来一单预填原游戏、版本和套餐，但创建新订单。
- `AC-F005-06 [客户端]` 售后弹窗无资格选择，五类问题平铺，描述为空不能提交。
- `AC-F005-07 [服务端]` 售后单完整保存问题类型和用户原始描述。
- `AC-F005-08 [客户端]` 退款详情为四步横向进度，申请中显示预计 7 个工作日。
- `AC-F005-09 [端到端]` 已取消和已退款示例可在全部订单中正确展示并执行终态操作。
- `AC-F005-10 [端到端]` 无理由类型仅在 72 小时、30 分钟和订单资格三项均满足时可提交，履约售后不受高频退款限制。
- `AC-F005-11 [客户端]` 同一订单 15/5 分钟提醒最多各一次，休眠恢复只补当前最近档位。
- `AC-F005-12 [端到端]` 续租从原 `expireAt` 精确顺延，订单和账号不变；部分/完全预约冲突分别限制时长/阻止支付。
- `AC-F005-13 [服务端]` T0 关闭未支付续租单，迟到回调不延长订单且只产生一次原路退款。
- `AC-F005-14 [端到端]` 重新租用创建新订单；同版本本地包复用并直接启动，版本不一致才更新或下载。
- `AC-F005-15 [客户端]` 到期退出结果弹窗的游戏卡片与未租用、过期启动拦截样式及价格口径一致。
