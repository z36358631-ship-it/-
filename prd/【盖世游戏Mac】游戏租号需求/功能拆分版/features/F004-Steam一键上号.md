# F004 Steam 一键上号与目标游戏拦截

> 状态：评审稿 | 优先级：P0 | 业务域：履约
> 内容来源：最新 Demo 游戏详情、租赁拦截和启动结果（提交 `35c72ea5`）

## 1. 功能概述与用户价值

用户支付后无需接触明文账号凭据，由客户端完成租赁 Steam 会话。系统只允许下载和启动订单目标游戏，并将订单启动结果记录为未启动、启动但失败或启动成功过。

## 2. 范围、前置条件与依赖

- 范围：取号授权、Steam 登录、目标游戏下载/启动、非目标游戏拦截、会话清理、启动结果。
- 前置：订单有效且账号已锁定。
- 依赖：[F003](F003-下单支付与资源锁定.md)、[F007](F007-账号资源管理.md)、[F010](F010-游戏库与平台账号状态.md)、Steam。

## 3. 履约流程

1. 用户支付成功或在订单详情点击“一键上号”。
2. 服务端校验订单、用户、设备、账号和有效期，签发短期一次性授权。
3. 客户端建立租赁 Steam 会话。
4. 用户对目标游戏执行下载或启动。
5. 客户端在动作前校验 AppID 白名单；非目标游戏弹出租用询问。
6. 成功进入目标游戏后上报启动成功；失败时上报阶段和错误码。
7. 到期、退款、换号或强制结束后撤销会话并清理 token。

## 4. 通用业务规则

1. `[已确认]` 用户和运营后台均不得看到明文账号密码。
2. `[已确认]` 下载不等于启动，不改变启动结果。
3. `[已确认]` 租赁目标游戏可直接下载/启动，隐藏“租号开玩”。
4. `[已确认]` 租赁非目标游戏无论账号是否拥有，获取、下载和启动都必须拦截。
5. `[已确认]` 拦截标题为“当前游戏未租用”，询问“是否进行租用”；不强调当前已租用的游戏名。
6. `[已确认]` 拦截确认后进入当前游戏确认订单，默认时租 2 小时；原订单不变。
7. `[已确认]` 个人 Steam 账号拥有游戏时直接下载/启动并隐藏租号按钮；未拥有时“获取游戏”进入 Steam 商店，并保留“租号开玩”。
8. `[已确认]` 启动结果优先级：一旦成功过，后续失败不回退；从未成功且启动失败记为“启动但失败”。
9. `[推导]` 同一订单同一时刻只允许一个有效上号会话。
10. `[已确认]` 服务端 `expire_at` 是租赁有效性的唯一时钟；客户端本机时间和历史缓存不得放行租赁游戏。
11. `[已确认]` 当前游戏未租用及过期本地包启动拦截均展示封面、游戏名与当前最低时租价 `¥X 租号`，不展示平台或操作副文案。

**启动决策矩阵：**

| 场景 | 处理结果 |
|---|---|
| 个人 Steam 账号拥有目标游戏 | 最高优先级按个人游戏启动，不触发租号过期拦截 |
| 游戏、版本匹配且 `expire_at > server_time` | 建立或复用有效租赁会话并启动 |
| 租赁已过期且本地有安装包 | 阻断启动，提示租期已结束；弹窗展示封面、游戏名和当前最低时租价，可取消或重新租用 |
| 无网络或服务端校验失败 | 失败关闭，不放行租赁游戏，提示检查网络后重试 |

到期后的安装包、游戏设置和个人存档继续保留。重新租用成功且版本一致时复用本地包并直接启动；版本不一致时才更新或下载。

## 5. 客户端需求

1. 下载按钮显示资源大小，例如“下载 783M”；下载后变为“启动游戏”。
2. 一键上号、下载和启动期间禁止重复触发。
3. 目标校验至少覆盖主游戏 AppID、必要 DLC 和经审核运行组件。
4. 拦截弹窗支持“暂不租用”和“去租号”，关闭不执行任何下载/启动。
5. 成功启动只在确认进入目标游戏进程并通过最小存活阈值后上报。
6. 到期/退款后清理租赁 token，不影响个人 Steam 账号。
7. 日志和崩溃上报不得包含凭据、授权 token 或完整账号名。
8. 到达 `expire_at` 时立即结束目标游戏并清理租赁会话，不延后订单终止；客户端恢复或重连后再次同步服务端状态。

## 6. 服务端需求

1. 上号会话状态：`CREATED/AUTH_ISSUED/LOGGING_IN/LOGGED_IN/FAILED/EXPIRED/REVOKED`。
2. 每个订单维护目标 `gameId`、Steam `appId` 和白名单版本。
3. 启动事件至少包含订单、会话、AppID、阶段、结果、失败码、客户端时间和服务端接收时间。
4. 订单启动结果聚合：
   - 无启动事件：`NOT_STARTED`
   - 有失败且无成功：`FAILED`
   - 至少一次成功：`SUCCEEDED_ONCE`
5. 重复启动事件按 `eventId` 幂等；乱序事件不得把 `SUCCEEDED_ONCE` 回退。
6. 到期、退款、换号和强制结束撤销全部关联会话。
7. 到期执行分别返回并持久化 `exitResult`、`sessionRevokeResult`、`accountReleaseResult`；失败步骤进入重试、告警和后台人工处置。
8. 启动校验失败不得降级使用本地过期凭据；个人拥有权益在租赁订单判断之前完成。

## 7. 端到端契约

`[建议方案]`

```http
POST /rental/orders/{orderId}/launch-events
```

```json
{
  "eventId":"le-1001",
  "sessionId":"RLS-100018",
  "appId":"1245620",
  "stage":"GAME_PROCESS",
  "result":"SUCCESS",
  "errorCode":null
}
```

| 业务码 | 客户端处理 |
|---|---|
| `TARGET_GAME_MISMATCH` | 阻止动作并展示租用询问 |
| `ENTITLEMENT_EXPIRED` | 清理租赁会话，展示重新租用入口；重新租用必须创建新服务订单，不得延长已完成原订单 |
| `SESSION_CONFLICT` | 刷新会话并按策略处理旧设备 |
| `ACCOUNT_UNHEALTHY` | 停止启动，进入换号或售后 |
| `LOGIN_PROVIDER_TIMEOUT` | 使用原 sessionId 查询或重试 |
| `EXPIRED_RENTAL` | 阻断本地包启动，展示取消/重新租用 |
| `RENTAL_VALIDATION_FAILED` | 不放行，提示检查网络后重试 |

## 8. 埋点、日志与监控

- `rental_login_click`、`rental_login_result`、`rental_download_start`、`rental_launch_result`、`rental_non_target_block`。
- `rental_expiry_force_exit`：到期结束完成时记录 `order_id`、`exit_result`、`session_revoke_result`、`account_release_result`。
- `rental_expired_launch_block`：过期本地包启动被阻断时记录 `order_id`、`game_id`、`block_reason`。
- `rental_rerent_result`：重新租用履约判断完成时记录 `source_order_id`、`new_order_id`、`result`。
- 监控 Steam 登录成功率、游戏启动成功率、各阶段失败率、非目标拦截次数、白名单版本不一致和到期未清理数。

## 9. 验收标准

- `AC-F004-01 [端到端]` 非有效订单不能创建上号会话。
- `AC-F004-02 [客户端]` 租赁目标游戏直接下载/启动且不显示租号按钮。
- `AC-F004-03 [客户端]` 租赁非目标游戏在获取、下载、启动三个入口均被拦截。
- `AC-F004-04 [端到端]` 下载完成后订单仍为“未启动”。
- `AC-F004-05 [服务端]` 启动失败后显示“启动但失败”；任一次成功后显示“启动成功过”且不可回退。
- `AC-F004-06 [安全]` 客户端、服务端、后台和导出均无明文凭据。
- `AC-F004-07 [端到端]` 到期或退款后旧授权和 token 无法继续使用。
- `AC-F004-08 [端到端]` 到达 `expire_at` 时立即结束目标游戏、撤销会话并触发账号释放，三个结果字段可分别查询。
- `AC-F004-09 [客户端]` 过期本地包被阻断，校验失败不放行；个人账号拥有游戏时正常启动。
- `AC-F004-10 [端到端]` 同版本重新租用复用本地包并直接启动，且到期过程不删除安装包、设置或存档。
- `AC-F004-11 [客户端]` 当前游戏未租用及过期启动拦截卡片的游戏信息和当前最低时租价准确，且不存在 `Steam · 启动游戏/下载/获取游戏` 副文案。
