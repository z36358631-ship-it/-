# 盖世游戏 Mac 端 iOS 应用与 IPA 资源库功能点总结

- 盖世游戏 Mac 客户端侧栏在“游戏库”和“订单中心”之间新增“iOS 应用”一级入口。
- “iOS 应用”内部包含“我的 App”和“IPA 资源”两个 Tab。
- 用户首次进入时可以选择一个唯一的 App 库文件夹。
- App 库文件夹只允许选择可读写目录。
- 用户取消文件夹选择后停留原页面且不提示错误。
- 更改 App 库路径时不迁移、不复制、不删除原路径内容。
- 新路径授权失败时继续保留原 App 库路径。
- 下载、校验、安装或卸载任务进行中时禁止更改 App 库路径。
- App 库路径失效时保留上次扫描缓存并引导重新授权。
- “我的 App”工具栏常驻“导入 IPA”主按钮。
- “导入 IPA”在 App 列表为空或已有 App 时均保持显示。
- “导入 IPA”只选择本地 `.ipa` 文件，不用于选择普通文件夹。
- 本地 IPA 文件选择器支持一次选择一个或多个 IPA。
- 未设置 App 库路径时点击“导入 IPA”会先选择路径再自动继续导入。
- 本地 IPA 选择完成后进入导入确认弹窗。
- 导入确认弹窗右上角只显示一个“全选/取消全选”按钮。
- 导入确认弹窗底部左侧显示“已选 X / Y”。
- 导入确认弹窗底部右侧只显示“取消”和“导入”。
- 重复、损坏、不兼容和非 IPA 文件在导入列表中置灰并说明原因。
- “我的 App”采用 macOS 应用程序式图标网格。
- “IPA 资源”采用与“我的 App”一致的图标网格。
- 图标网格默认显示七列并可随宽度调整为六列或五列。
- App 图标加载失败时显示统一 IPA 文件图标。
- App 名称超过一行时省略并支持悬停查看完整名称。
- App 状态位使用固定高度避免状态切换导致页面抖动。
- App 版本、体积、Bundle ID、来源和兼容原因通过详情浮层查看。
- “我的 App”支持名称搜索、状态筛选和排序。
- 已安装 App 支持打开、更新、卸载和查看异常原因。
- 卸载默认只移除 App 包并保留用户数据。
- “IPA 资源”支持“全部、游戏、工具、娱乐、社交、创意、其他”分类。
- “IPA 资源”支持名称或 Bundle ID 搜索。
- “IPA 资源”支持按来源筛选。
- “IPA 资源”支持按未安装、进行中、待安装、已安装、可更新和需处理筛选。
- 用户可以添加无需账号和令牌的公开 HTTPS IPA 来源。
- 来源地址停止输入 500ms 后自动执行校验。
- 来源地址修改后立即清除旧成功状态并禁用“导入”。
- 来源校验反馈区固定高度避免弹窗抖动。
- 链接、JSON 或目标资源无效时提示“链接无效，JSON无效或未找到！”。
- 来源校验成功后的第一次“导入”只进入资源选择，不直接保存来源。
- 来源资源选择弹窗默认不勾选任何资源。
- 来源资源选择支持逐项选择和右上角全选。
- 来源资源最终确认后才保存来源并导入所选资源。
- 取消来源资源选择时不保存来源也不导入资源。
- 返回来源校验弹窗时保留已验证的 URL 和成功状态。
- AppHub、Mirror 和网络失败示例按钮只存在于标注工具区，不进入正式业务弹窗。
- 来源管理支持刷新、停用、启用和删除来源。
- 删除来源不会卸载已经安装的 App。
- 来源刷新失败时继续显示上次成功缓存。
- 多个来源按 Bundle ID 合并相同 App。
- 相同 App 默认展示版本号和 Build 号最高的兼容构建。
- 同版本资源优先使用哈希有效且当前可访问的来源。
- IPA 下载后校验包内 Bundle ID 与来源元数据一致。
- 下载状态包含未下载、排队中、下载中、已暂停、校验中和已下载待安装。
- 安装状态包含安装中、已安装和安装失败。
- 页面支持可更新、下载失败、校验失败、不兼容和缺少来源哈希状态。
- 未下载资源的主操作为“下载”。
- 排队中资源的主操作为“取消排队”。
- 下载中资源的主操作为“暂停”。
- 已暂停资源的主操作为“继续”。
- 校验中的资源不提供可点击主操作。
- 已下载待安装资源的主操作为“安装”。
- 安装中的资源不提供可点击主操作。
- 已安装资源的主操作为“打开”。
- 可更新资源的主操作为“更新”。
- 下载失败资源的主操作为“重新下载”。
- 校验失败资源的主操作为“删除并重下”。
- 安装失败资源的主操作为“重试安装”。
- 不兼容资源的主操作为“查看原因”。
- 缺少来源哈希的资源允许下载但安装前必须二次确认。
- 下载中使用环形进度和整数百分比展示进度。
- 每个 App 的暂停、继续、取消和重试只影响自己的任务。
- 切换 Tab、分类或筛选不会终止进行中的任务。
- 用户取消安装确认后保留已下载文件并显示“已下载，待安装”。
- 安装失败时保留已下载文件并支持直接重试安装。
- App 更新使用独立下载和安装任务。
- App 更新过程中旧版本仍然可以打开。
- 新版下载、校验或安装失败时保留旧版本和用户数据。
- 正常浏览状态不显示批量选择复选框。
- 进入批量选择模式后复选框显示在 App 图标左上角。
- 批量模式的“全选/取消全选”位于页面右上角。
- 批量全选只作用于当前搜索、分类、来源和状态筛选结果。
- 已安装、不兼容和处理中资源不会被批量全选。
- 批量操作条显示已选数量和预计下载大小。
- 退出批量选择模式后清空已选项目。
- 批量任务中单项失败不阻塞其他成功项进入安装确认。
- Demo 默认同屏展示十四种资源状态且不会自行变化。
- Demo 中操作某个 App 后只有该 App 的状态开始流转。
- 标注版左侧支持快速进入页面、弹窗、任务和异常状态。
- 标注版右侧支持查看交互说明和异常边界。
- 点击右侧标注可以自动复现目标状态并高亮对应元素。
- 标注编号支持显示、隐藏以及右侧面板折叠和恢复。
