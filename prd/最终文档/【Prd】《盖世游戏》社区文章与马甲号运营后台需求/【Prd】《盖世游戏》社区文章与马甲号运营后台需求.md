## 二、背景与目标

**背景：**社区上线前需要快速补充内容，现有发帖方式不支持图文混排和长文章编辑，后台也缺少可统一管理的社区运营身份。运营人员无法在保留真实员工登录态的同时，使用指定社区身份发布和追溯内容。

**目标：**在国内和海外社区后台新增“内容发布”“马甲号管理”“操作记录”，跑通“管理马甲号 → 选择发布身份 → 编辑和预览 → 提交审核 → 全链路留痕”流程。

核心挑战：真实员工与马甲号的双身份追溯、国内与海外数据隔离、多内容类型校验、多人并发编辑和批量导入失败处理。

## 三、故事介绍

### 3\.1 用户与运营场景

- <span style="background-color: #FEF794;">内容运营李明需要发布一篇新游攻略。他在“内容发布”的“文章”Tab 点击“新建文章”，在同一编辑页选择“盖世攻略君”马甲号，填写标题、富文本正文、分区和话题，可复制粘贴外部图文/视频，并通过客户端实时预览检查后提交审核 //2026.7.23修改</span>

- <span style="background-color: #FEF794;">社区管理员王璐需要建立一批海外运营账号。她进入海外社区后台的“马甲号管理”，上传 CSV 或 XLSX 模板；系统为成功记录生成 UID，跳过重复用户名并继续处理其他有效行，最终在当前弹窗展示成功与失败明细 //2026.7.23修改</span>

- 审计人员收到内容异常反馈后，在“操作记录”输入内容 ID，查看真实操作人、使用的马甲号、变更前后内容、请求 ID 和失败原因，再导出当前筛选结果。

### 3\.2 价值分析

- <span style="background-color: #FEF794;">内容效率：帖子沿用现有社区发布能力，文章独立支持长文及图文、视频混排，运营人员可在同一后台完成编辑和客户端预览 //2026.7.23修改</span>

- 身份安全：马甲号只是代操作身份，不替换真实员工登录态，减少共享账号和无法追责的风险。

- 全球运营：国内和海外后台提供相同的马甲号管理能力，并按区服隔离账号池、内容和日志。

- 风险追溯：内容和马甲号的关键变更全部记录真实员工、代操作身份和变更前后数据。

### 3\.3 核心体验路径

- <span style="background-color: #FEF794;">内容发布：内容发布列表 → 选择“帖子”或“文章”Tab → 新建对应内容 → 同页选择马甲号 → 编辑内容 → 客户端实时预览 → 确认身份和摘要 → 提交审核 //2026.7.23修改</span>

- 复制发布：内容发布列表 → 复制 → 生成副本编辑页 → 修改内容或身份 → 提交审核 → 生成新内容 ID。

- 马甲号管理：马甲号管理 → 单个创建或批量导入 → 编辑/启停 → 使用该身份发布 → 自动带入内容编辑页。

- 日志追溯：任一关键操作 → 写入操作记录 → 按真实员工/马甲号/对象 ID 筛选 → 查看详情或导出。

### 4\.2 通用业务规则

#### 4\.2\.1 真实员工与马甲号

- 真实员工登录态始终保留，马甲号只作为单次内容的发布身份，不替换真实员工会话。

- <span style="background-color: #FEF794;">进入帖子或文章编辑页后，在当前页面直接选择马甲号；不拆分为“选择身份”和“发布详情”两个步骤 //2026.7.23修改</span>

- 从“马甲号管理”点击“使用该身份发布”时，进入内容编辑页并预选对应马甲号；运营人员仍可在编辑页更换。

- 停用或已删除的马甲号不能用于新建、编辑或复制提交。若编辑期间账号被停用，提交时重新校验并拦截。

#### 4\.2\.2 国内与海外

- <span style="background-color: #FEF794;"><strong>国内和海外社区后台均显示“内容发布”“马甲号管理”“操作记录”，海外版不得隐藏或删除马甲号管理。</strong> //2026.7.23修改</span>

- 国内与海外的马甲号池、内容分区、话题、审核队列和操作日志按区服隔离，不得跨区服查询、选择或发布。

- 真实员工的国内、海外后台权限和马甲号可用范围分别配置。员工切换区服后重新获取当前区服数据。

- 海外后台的通用操作文案接入现有多语言资源；运营填写的标题和正文按原文存储，不自动翻译。

- 海外产品英文名统一使用 `GameHub`，不得使用 `GaishiGame`。

#### 4\.2\.3 权限

|角色|内容发布|马甲号管理|操作记录|
|---|---|---|---|
|内容运营|查看、新建、编辑、复制、提交；仅可使用已授权马甲号|查看已授权账号，不可新建、导入、启停或删除|查看本人及所属业务组记录|
|账号管理员|查看|查看、创建、导入、编辑、启停、删除和授权|查看马甲号管理记录|
|审计人员|只读|只读|查看详情和导出|
|超级管理员|全部|全部|全部|

- 菜单和按钮按权限隐藏或禁用，所有写操作必须由服务端再次校验，不依赖前端显隐。

- <span style="background-color: #FEF794;">“马甲号管理”配置为独立后台权限；未获该权限的员工不可进入账号池或执行创建、导入、编辑、启停、删除操作 //2026.7.23修改</span>

- <span style="background-color: #FEF794;">马甲号的官方、达人等前台身份标签不在本模块配置，仍通过既有“身份管理”按用户 UID 维护 //2026.7.23修改</span>

- <span style="background-color: #FEF794;">面向外部创作者的受限后台账号、仅查看本人内容和仅使用指定马甲号等数据权限本期不做，后续单独排期 //2026.7.23修改</span>

#### 4\.3\.1 F001 内容发布列表

##### 4\.3\.1\.1 统计与筛选

- 统计卡片显示全部内容、审核中、已发布和已驳回数量，数据与当前区服一致。

- <span style="background-color: #FEF794;">内容发布仅包含“帖子”和“文章”两个 Tab；进入页面默认选中“文章”，切换 Tab 后回到第 1 页，并显示对应的“新建帖子”或“新建文章”入口 //2026.7.23修改</span>

- 标题/ID 支持输入后模糊查询，发布身份和状态为下拉单选。多个筛选条件之间为“且”关系。

- 点击“查询”后提交条件并回到第 1 页；点击“重置”后恢复全部类型、全部身份和全部状态，并自动刷新。

##### 4\.3\.1\.2 列表字段

|字段|定义|展示规则|
|---|---|---|
|内容|内容标题和内容 ID|标题超出 2 行截断，悬停显示完整标题；ID 完整显示|
|类型|<span style="background-color: #FEF794;">帖子或文章 //2026.7.23修改</span>|<span style="background-color: #FEF794;">按内容实际详情形态显示，不根据内容中是否含图片或视频改变类型 //2026.7.23修改</span>|
|发布身份|内容使用的马甲号|显示昵称和 UID|
|分区/话题|内容所属分区和关联话题|未关联话题时显示“\-”|
|状态|当前审核或上线状态|审核中、已发布、已驳回、已下架|
|互动数据|阅读、点赞和评论数|数值为 0 时显示 0；大数缩写沿用社区现有规则|
|更新时间|最后一次内容变更时间|格式为 `YYYY-MM-DD HH:mm`|
|操作|当前记录可执行动作|预览、编辑、复制、审核详情/驳回原因、操作记录|

- 默认按更新时间倒序，时间相同时按内容 ID 倒序。

- 使用翻页器分页，默认每页 20 条，支持 20、50、100 条/页，显示总条数。返回列表时保留筛选条件、页码和滚动位置。

##### 4\.3\.1\.3 状态与操作

|当前状态|后续流转|可执行操作|
|---|---|---|
|审核中|审核通过 → 已发布；审核拒绝 → 已驳回|预览、编辑、复制、审核详情、操作记录|
|已发布|现有社区审核模块下架 → 已下架|预览、编辑、复制、审核详情、操作记录|
|已驳回|修改并提交 → 审核中|预览、编辑、复制、驳回原因、操作记录|
|已下架|修改并提交 → 审核中|预览、编辑、复制、审核详情、操作记录|

- 新建、编辑和复制提交后统一进入“审核中”。审核动作沿用现有社区审核模块，本期不新增审核入口。

- 复制后生成新内容 ID，标题追加“（副本）”，互动数据置 0，不继承原内容的审核状态。

- 编辑已发布内容的线上版本处理沿用现有社区审核策略，本期不改变审核期间的客户端可见性。

- 内容长度、格式和语义校验在失焦和提交时执行；上传数量和账号状态在每次变更后和提交时执行。

##### 4\.3\.2\.2 内容类型差异

|类型|媒体要求|处理规则|
|---|---|---|

- 图片和视频格式、单文件大小、图片尺寸和视频时长沿用社区现有上传组件规则，本期不新增一套规格。

- 多文件上传部分失败时，保留已成功文件，标记失败文件并支持单个重试。加载失败的图片显示占位图和重试入口。

##### 4\.3\.2\.3 预览、提交与离开

- <span style="background-color: #FEF794;">右侧客户端预览实时同步马甲号、标题、正文、媒体、话题和游戏卡片，并根据“帖子 / 文章”使用对应客户端详情样式；运营提交前需通过预览人工检查外部内容及媒体是否正常 //2026.7.23修改</span>

- 点击“提交审核”后先完成校验，再弹出确认框，展示马甲号昵称、UID、内容类型、标题、分区和话题。

- 确认提交后按钮进入处理中并禁止重复点击。成功后关闭编辑页、刷新列表并提示“提交成功，XX 已进入审核队列”。

- 提交失败时保留当前页面的已填内容和已上传文件，显示可识别的失败原因和重试入口。

- 有未提交修改时点击返回、取消或关闭，弹出“放弃本次编辑？”；选择放弃后不保留本次内容。本期不提供草稿保存。

##### 4\.3\.3\.3 启停、删除与使用身份发布

- 停用前弹出二次确认，说明“停用后不能用于新内容发布，历史内容不受影响”。启用成功后即可被有权员工选择。

- 已关联任意内容、审核记录或操作记录的马甲号不允许删除，提示“该马甲号已关联业务数据，请停用”。

- 无关联数据的马甲号删除前需二次确认，删除使用逻辑删除，不释放 UID 和内部标识。

- 点击“使用该身份发布”时，启用账号进入内容编辑页；停用账号按钮禁用并显示原因。

##### 4\.3\.3\.4 批量导入

- 支持 CSV 和 XLSX，单文件最多 500 行数据。文件超限时整体拦截，不执行导入。

- <span style="background-color: #FEF794;">模板不包含 UID；模板字段包含用户名、内部标识、手机号、邮箱、后台备注、隐私设置和状态，其中用户名与内部标识必填，UID 由系统在创建成功后自动生成 //2026.7.23修改</span>

- <span style="background-color: #FEF794;">导入前校验文件类型、列名、必填值、用户名/内部标识重复、字段长度和枚举值；手机号和邮箱不验证真实性、不发送验证码，仅用于与现有用户数据去重 //2026.7.23修改</span>

- <span style="background-color: #FEF794;">采用分行导入：用户名重复的行跳过，其他有效行继续创建；失败行不影响同批次成功行，完成后在当前弹窗展示总数、成功数、跳过数、失败数和逐行原因 //2026.7.23修改</span>

- <span style="background-color: #FEF794;">本期不生成失败行下载文件；运营人员根据弹窗结果修正原文件后可重新导入，系统再次按用户名、手机号和邮箱去重，不重复创建已存在账号 //2026.7.23修改</span>

#### 4\.3\.4 F003 操作记录

##### 4\.3\.4\.1 记录范围

- <span style="background-color: #FEF794;">仅记录“内容发布”和“马甲号管理”两个模块，不新增草稿箱、风险事件或其他社区页面的日志分类 //2026.7.23修改</span>

- 内容发布记录：新建并提交、编辑并提交、复制并提交、提交失败。

- 马甲号管理记录：创建、编辑、启用、停用、删除、批量导入、使用该身份发布及失败结果。

- 每条记录包含操作时间、真实操作人、马甲号 UID（未使用时为“\-”）、模块、动作、对象 ID、结果、请求 ID、终端摘要、变更前、变更后和失败原因。

##### 4\.3\.4\.2 查询、列表和详情

- 真实操作人、马甲号 UID、动作和对象 ID 支持输入查询；模块和结果为下拉单选；多个条件之间为“且”关系。

- 列表字段为操作时间、真实操作人、发布马甲号、模块/动作、对象 ID、结果、终端摘要和详情。

- 默认按操作时间倒序，时间相同时按请求 ID 倒序。默认每页 20 条，支持 20、50、100 条/页。

- 时间按当前后台设置的时区展示，格式为 `YYYY-MM-DD HH:mm:ss`；底层统一按 UTC 存储，导出文件包含时区。

- 点击行或“查看详情”打开详情，显示请求 ID、真实操作人、代操作身份、变更前、变更后和失败原因。

- 日志为只读数据，后台不提供修改和删除功能；至少在线保留 180 天，超期按现有审计归档策略处理。

##### 4\.3\.4\.3 导出

- 导出当前筛选条件下的全部记录，格式为 XLSX，导出字段与详情字段一致。

- 单次最多导出 20,000 条；超过时不截断数据，提示“记录过多，请缩小筛选范围后重试”。

- 导出成功后记录导出人、时间、筛选条件和文件摘要；导出失败时显示原因和重试入口。

#### 4\.3\.5 页面通用状态

|场景|处理方式|
|---|---|
|首次加载|列表区显示加载状态，防止重复操作|

|登录过期|拦截写操作并跳转后台登录；登录成功后返回原页，未提交表单仅在当前页会话中保留|

#### 4\.3\.6 边界条件

|场景|处理方式|
|---|---|
|多人同时编辑同一内容|打开编辑页时记录当前版本；提交时版本不一致则拦截覆盖，提示“内容已被他人更新”，支持重新加载或复制当前内容后新建|
|马甲号在编辑期间被停用|提交时拦截，提示重新选择启用马甲号，已填内容保留|
|分区、话题或游戏在编辑期间失效|提交时拦截并定位失效项，需重新选择|
|重复点击提交/保存|首次请求发出后禁用按钮；服务端根据请求 ID 保证幂等，不生成重复内容或马甲号|
|批量导入修正后重试|<span style="background-color: #FEF794;">重新逐行校验；已存在的用户名、手机号或邮箱跳过，其余有效行继续创建并生成 UID //2026.7.23修改</span>|
|区服切换|清空上一区服的筛选缓存和马甲号选择，重新校验员工权限，不复用跨区服数据|
|日志写入短暂失败|业务操作和日志记录通过可重试机制保持一致；超过重试阈值后告警，不允许出现长期无日志的成功操作|
|敏感词或媒体审查不通过|不提交审核队列，显示问题类型和定位信息，保留编辑内容|
