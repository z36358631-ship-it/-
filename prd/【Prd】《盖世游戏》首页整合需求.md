# 二、 背景与目标

**背景：**
当前盖世游戏APP底部导航为5个Tab（探索/玩游戏/游戏库/排行/我的），其中"探索页"和"游戏库"本质上解决的是同一个问题——帮用户找到想玩的游戏，但被拆成了两个独立入口。探索页内容偏薄（Banner+推荐+资讯），用户扫一眼就走；游戏库是纯工具型列表（Steam/Epic/复古分tab），缺乏发现性。两个页面之间存在认知断裂，且占掉两个Tab位，挤掉了社区——而社区恰恰是竞品233乐园留存远超盖世的核心原因。

**目标：**
将探索页（推荐流）和游戏库（分类管理）合并为统一首页，实现**"发现→筛选→启动"全链路一页完成**。同时释放一个Tab位给社区，支持横竖屏双模式自适应布局。
**核心挑战：** 横竖屏布局差异大（竖屏Feed流为主、横屏游戏库为主）、信息密度与发现效率的平衡、分类入口与推荐内容的权重分配。

# 三、 故事介绍

在切入繁杂的功能点前，我们先通过典型用户的视角，来看看新功能全貌的运转方式。

## 3\.1 用户与运营场景

**用户与运营场景：**

- **C端新用户（小明）：** 小明刚装APP，打开首页看到搜索栏下方有"最近游玩"（空态不展示），紧接着是游戏库金刚区（PC游戏/Steam/Epic/复古/导入），再往下是运营Banner和推荐流。他点击"Steam"金刚位进入Steam游戏子页面，看到搜索框和游戏网格列表，找到了想玩的游戏。

- **C端老用户（张三）：** 张三有丰富的游戏库，打开首页搜索栏下方直接展示"最近游玩"的3个游戏小卡片，他直接点击封面上的快捷启动按钮继续昨天的游戏。横屏模式下，他看到全屏背景大图+最近游戏竖版封面卡片，点击"查看详情"进入游戏详情页。

- **手柄横屏用户（李四）：** 李四用盖世手柄横屏操作，首页顶部5个Tab导航（首页/社区/玩游戏/排行榜/我的），第一屏是最近游戏大卡片墙，滚动下去是游戏库金刚区（icon+渐变卡片风格）和运营Banner，再往下是为你推荐、最新资讯、秒玩、热玩榜单等内容模块。

## 3\.2 价值分析

**价值分析**

- **提升发现效率：** 合并后用户在一个页面内完成"发现→筛选→启动"全链路，减少跨页跳转，首页→启动游戏的转化路径缩短50%。

- **释放社区入口：** 底部导航释放一个Tab位给社区，建立社交留存基线，对标233乐园的社区渗透率。

## 3\.3 核心体验路径

我们首先明确核心用户的核心体验漏斗路径：

**竖屏核心链路：**
打开APP → 搜索栏（热词轮播）→ 最近游玩（快捷启动）→ 游戏库金刚区（点击进子页面）→ Banner推荐 → 为你推荐（横滑竖封面）→ 最新资讯 → 秒玩 → 热玩榜单 → 今日推荐

**横屏核心链路：**
打开APP → 顶部Tab导航 → 最近游戏大卡片墙（查看详情/查看更多）→ 游戏库金刚区（icon卡片，点击进子页面）→ 运营Banner（一排2个）→ 为你推荐 → 最新资讯 → 秒玩 → 热玩榜单

## 3\.4 产品指标预测

**产品指标预测：**

1. **首页→启动游戏转化率：** 合并后预测提升至 `8%`（当前基线：从探索页出发需跨页操作，约4%）。

2. **首页平均浏览深度：** 合并后应 > 原探索页+游戏库的单页深度，预测首屏滚动率 `75%+`。

3. **社区Tab DAU渗透率：** 新增入口后首月预测 `15%` DAU触达社区。

## 3\.5 路径规划

基于敏捷开发（Sprint）原则，我们拆分了迭代节奏：

- **当前版本（V1\.0）：** 完成首页整合（探索+游戏库合并）、横竖屏双布局适配、底部导航重构（5Tab→首页/社区/玩游戏/排行榜/我的）、金刚区子页面交互。

- **未来可能升级（预告）：** 个性化推荐算法、社区评分体系接入、手柄焦点导航优化、游戏库智能排序。



# 四、概要设计

通过前面的背景介绍，相信大家对新功能的价值已具备体感，接下来转为研发视角的模块拆解。

## 4\.1 模块设计（系统架构图概览）

本次功能涉及的系统模块归纳如下，避免设计时大方向遗漏：

- **【首页整合模块】**

    - **竖屏布局：** 搜索栏、最近游玩（竖排标题+小卡片）、游戏库金刚区、Banner推荐、Feed流内容（为你推荐/资讯/秒玩/榜单/今日推荐）。

    - **横屏布局：** 顶部Tab导航、最近游戏Hero区（全屏背景+大卡片墙）、游戏库金刚区（icon渐变卡片）、运营Banner（一排2个）、Feed流内容。

- **【子页面模块】**

    - **游戏库子页面：** 标题（分类名）+返回按钮+搜索栏+导入按钮+游戏卡片网格（竖屏2列/横屏自适应多列）。

- **【导航重构模块】**

    - **底部导航（竖屏）：** 首页/社区/玩游戏/排行榜/我的（5Tab）。

    - **顶部导航（横屏）：** 同竖屏5Tab，icon+文字形式，居中排列。

## 4\.4 非功能需求

|**需求类型**|**详细要求**|
|---|---|
|**性能**|1\. **横竖屏切换速度：** 布局切换重新渲染响应时间不超过 `500ms`，期间无白屏或闪烁。<br>2\. **流畅度：** 首页滚动、横滑模块、子页面进出动画，帧率需稳定在 `50fps` 以上。<br>3\. **加载性能：** 所有游戏封面图采用懒加载策略，首屏接口单次响应 `<1s`。<br>4\. **浏览位置保持：** 横竖屏切换后保持当前浏览位置和分类选中态。|
|**兼容性**|1\. **操作系统：** Android 9\.0+、iOS 13\.0+、PC浏览器（Chrome/Edge/Safari最新版）。<br>2\. **屏幕适配：** 竖屏适配全面屏/刘海屏/底部手势区；横屏适配16:9主流掌机分辨率。<br>3\. **设备旋转：** 跟随系统旋转自动切换布局，2秒内反复旋转3次以上不崩溃不白屏。|
|**用户体验**|1\. **视觉规范：** 深色电竞风格，文字对比度满足WCAG AA级。<br>2\. **反馈：** 所有点击操作 `<=200ms` 视觉反馈（卡片上浮、按钮变色等）。<br>3\. **容错：** 封面图加载失败展示占位图，不出现裂图；网络断开展示离线提示+已缓存内容。<br>4\. **分类切换：** 连续快速点击不同金刚区icon不崩溃，最终展示最后一次点击对应内容。|

- **漏斗监控：** 首页曝光(UV/PV) → 金刚区点击率 → 子页面浏览深度 → 游戏详情页(PV/UV) → 启动游戏数。
