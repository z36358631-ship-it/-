# 二、 背景与目标

**背景：**
当前盖世游戏APP底部导航为5个Tab（探索/玩游戏/游戏库/排行/我的），其中"探索页"和"玩游戏"解决的是不同场景——探索页负责推荐发现，玩游戏负责管理已有游戏和云游戏。但两者被割裂成独立入口，用户在"找游戏"和"玩游戏"之间需要反复跳转，路径过长。同时底部5个Tab中缺少社区入口——而社区恰恰是竞品233乐园留存远超盖世的核心原因。

**目标：**
将探索页（推荐流）和玩游戏（游戏分类管理）合并为统一首页，通过顶部二级Tab平铺切换，实现**"发现→分类→启动"全链路一页完成**。同时释放一个Tab位给社区，支持横竖屏双模式自适应布局。

**核心挑战：** 横竖屏布局差异大、二级Tab切换的流畅性与状态保持、推荐内容与游戏分类的信息密度平衡。

# 三、 故事介绍

在切入繁杂的功能点前，我们先通过典型用户的视角，来看看新功能全貌的运转方式。

## 3\.1 用户与运营场景

**用户与运营场景：**

- **C端新用户（小明）：** 小明刚装APP，打开首页默认在"推荐"Tab，看到搜索栏、运营Banner和推荐流。他想玩云游戏，点击顶部"云游戏"Tab，直接看到云游戏列表，找到GTA5点击进入详情页。

- **C端老用户（张三）：** 张三打开首页"推荐"Tab，看到最近游玩的游戏小卡片，直接点击继续昨天的游戏。之后他切到"PC游戏"Tab浏览本地已安装的PC游戏。横屏模式下各Tab内容复用原有页面布局。

- **组队玩用户（李四）：** 李四点击"云游组队玩"Tab（紫色高亮），浏览可组队的云游戏列表，找到想和朋友一起玩的游戏。

## 3\.2 价值分析

**价值分析**

- **提升发现效率：** 合并后用户在一个页面内通过顶部Tab平铺切换完成"推荐发现→分类筛选→启动游戏"全链路，减少跨页跳转。

- **释放社区入口：** 底部导航释放一个Tab位给社区，建立社交留存基线，对标233乐园的社区渗透率。

## 3\.3 核心体验路径

我们首先明确核心用户的核心体验漏斗路径：

**竖屏核心链路：**
打开APP → 首页"推荐"Tab（搜索栏+最近游玩+Banner+为你推荐+资讯+秒玩+榜单+今日推荐）→ 切换至"PC游戏"/"云游戏"/"云游组队玩"/"复古游戏"Tab浏览对应分类 → 点击游戏进入详情页 → 启动游戏

**横屏核心链路：**
打开APP → 顶部5Tab导航 → 首页二级Tab切换（推荐/PC游戏/云游戏/云游组队玩/复古游戏）→ 各Tab复用原有页面内容

## 3\.4 产品指标预测

**产品指标预测：**

1. **首页→启动游戏转化率：** 合并后预测提升至 `8%`（当前基线：从探索页出发需跨页操作，约4%）。

2. **首页平均浏览深度：** 合并后应 > 原探索页+游戏库的单页深度，预测首屏滚动率 `75%+`。

3. **社区Tab DAU渗透率：** 新增入口后首月预测 `15%` DAU触达社区。

## 3\.5 路径规划

基于敏捷开发（Sprint）原则，我们拆分了迭代节奏：

- **当前版本（V2\.0）：** 完成首页整合（探索+玩游戏合并）、顶部平铺二级Tab（推荐/PC游戏/云游戏/云游组队玩/复古游戏）、横竖屏双布局适配、底部导航重构（5Tab→首页/社区/排行榜/游戏库/我的）。

- **未来可能升级（预告）：** 个性化推荐算法、手柄焦点导航优化、游戏库智能排序。



# 四、概要设计

通过前面的背景介绍，相信大家对新功能的价值已具备体感，接下来转为研发视角的模块拆解。

## 4\.1 模块设计（系统架构图概览）

本次功能涉及的系统模块归纳如下，避免设计时大方向遗漏：

- **【首页整合模块】**

    - **二级Tab栏：** 推荐 / PC游戏 / 云游戏 / 云游组队玩（紫色高亮）/ 复古游戏，平铺展示，点击切换内容区。

    - **推荐Tab（竖屏）：** 搜索栏、最近游玩（竖排标题+小卡片）、Banner推荐、Feed流内容（为你推荐/资讯/秒玩/榜单/今日推荐）。复用原探索页内容。

    - **PC游戏/云游戏/复古游戏Tab：** 复用原"玩游戏"页面对应分类内容，无新增改动。

    - **云游组队玩Tab：** 复用原"云游戏组队玩"页面内容，紫色高亮区分。

    - **横屏布局：** 顶部5Tab一级导航 + 二级Tab切换，各Tab内容复用原有页面。

- **【导航重构模块】**

    - **底部导航（竖屏）：** 首页/社区/排行榜/游戏库/我的（5Tab）。

    - **顶部导航（横屏）：** 同竖屏5Tab，icon+文字形式，居中排列。

## 4\.4 非功能需求

|**需求类型**|**详细要求**|
|---|---|
|**性能**|1\. **横竖屏切换速度：** 布局切换重新渲染响应时间不超过 `500ms`，期间无白屏或闪烁。<br>2\. **流畅度：** 首页滚动、二级Tab切换、横滑模块，帧率需稳定在 `50fps` 以上。<br>3\. **加载性能：** 所有游戏封面图采用懒加载策略，首屏接口单次响应 `<1s`。<br>4\. **Tab切换状态保持：** 横竖屏切换、二级Tab切换后保持当前浏览位置。|
|**兼容性**|1\. **操作系统：** Android 9\.0+、iOS 13\.0+。<br>2\. **屏幕适配：** 竖屏适配全面屏/刘海屏/底部手势区；横屏适配16:9主流掌机分辨率。<br>3\. **设备旋转：** 跟随系统旋转自动切换布局，2秒内反复旋转3次以上不崩溃不白屏。|
|**用户体验**|1\. **视觉规范：** 深色电竞风格，文字对比度满足WCAG AA级。云游组队玩Tab紫色高亮与其他Tab区分。<br>2\. **反馈：** 所有点击操作 `<=200ms` 视觉反馈（Tab切换、卡片上浮、按钮变色等）。<br>3\. **容错：** 封面图加载失败展示占位图；网络断开展示离线提示+已缓存内容。<br>4\. **Tab切换：** 连续快速切换二级Tab不崩溃，最终展示最后一次点击对应内容。|

# 五、交付设计（数据分析与埋点设计）

## 5\.1 核心业务指标

- **漏斗监控：** 首页曝光(UV/PV) → 二级Tab点击率(各Tab) → 游戏详情页(PV/UV) → 启动游戏数。
- **横竖屏使用分布：** 横屏/竖屏用户占比、平均停留时长、滚动深度对比。
- **社区渗透：** 社区Tab点击率、社区DAU/首页DAU。
