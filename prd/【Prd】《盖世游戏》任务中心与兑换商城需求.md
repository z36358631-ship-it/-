# 二、 背景与目标

**背景：**
当前盖世游戏DAU/MAU比值仅6.5%-8.7%，D7留存率3.7%，用户粘性严重不足。用户缺乏回访动力——没有签到、成就、奖励等留存抓手，打开APP只为启动游戏，游戏结束即离开。竞品233乐园已通过积分任务体系+兑换商城有效提升了用户活跃和留存。我们需要建立一套"做任务赚盖世币→兑换商城消费"的正向激励循环，让用户有理由每天回来。

**目标：**
搭建盖世游戏「任务中心」+「兑换商城」功能，实现**"每日/成长任务→赚取盖世币→兑换虚拟/实物奖品→发货履约"**的完整闭环。通过行为激励驱动用户签到、发帖、互动、启动游戏等核心行为，提升DAU和留存。

**核心挑战：** 盖世币经济体系的通胀控制（产出/消耗平衡）、实物奖品的库存与发货管理、任务作弊防刷、虚拟商品自动发放的即时性与准确性。

# 三、 故事介绍

## 3.1 用户与运营场景

**用户与运营场景：**

- **C端玩家（小明）：** 小明打开盖世游戏APP，看到个人主页右上角的盖世币icon，点击进入「我的盖世币」页面。他看到余额5000盖世币，下方列出每日任务（签到+2、启动游戏+2、浏览社区+10等）和成长任务（完善资料+20、首次发帖+10等）。他点击"去签到"完成签到，积分+2，Toast提示"每日签到，积分+2"。然后点击右上角"兑换商城"，用50盖世币兑换了一张云游戏时长30分钟卡，系统自动发放到账户。他还看到X5Lite手柄只需8000盖世币，决定继续攒币。

- **B端运营（李四）：** 李四在后台「任务配置」页面新增一条每日任务"评论/点赞社区动态帖"，设置奖励2盖世币、上限10积分/日。在「奖品配置」页面上架了一款新手柄，设置兑换价格8000盖世币、库存15件、单人限兑1次。在「兑换与发货」页面看到一条待发货订单，点击"立即发货"填入快递单号完成发货。

## 3.2 价值分析

**价值分析**

- **提升DAU与留存（行为激励）：** 签到、发帖、点赞、浏览等行为被盖世币量化，用户每天有明确的回访理由和奖励预期。

- **拉动社区活跃：** 任务引导用户进入社区浏览≥3分钟、发布动态、评论互动，直接提升社区内容量和活跃度。

- **硬件销售协同：** 手柄等实物奖品通过兑换商城展示，既是用户激励也是品牌曝光，能拉动手柄购买意愿。

- **云游戏消费引导：** 云游戏时长作为高频兑换品，培养用户云游戏使用习惯，为后续付费转化铺路。

## 3.3 核心体验路径

**玩家&运营核心链路：**

- **C端玩家核心链路：**
个人主页点击盖世币icon → 进入任务中心查看余额和任务列表 → 完成每日/成长任务赚取盖世币 → 进入兑换商城浏览商品 → 选择商品兑换（虚拟→自动发放 / 实物→填写收货地址→等待发货） → 在"我的奖励"查看兑换记录和物流。

- **B端运营核心链路：**
任务配置（新增/编辑/启停任务） → 奖品配置（新增/编辑/上下架/补库存） → 兑换与发货（查看订单→实物发货填单号→虚拟自动发放）。

## 3.4 产品指标预测

**产品指标预测：**

1. **任务完成率：** 每日任务完成率预测 `30%+`（签到类最高，社区浏览类次之）。

2. **盖世币消耗率：** 月均产出盖世币中 `60%+` 被兑换消耗（健康经济循环）。

3. **兑换商城DAU渗透率：** 预测 `10%+`（每10个活跃用户至少1人进入商城浏览）。

4. **D7留存提升：** 预测从3.7%提升至 `6%+`（签到+任务的日回访引力）。

## 3.5 路径规划

- **当前版本（V1.0）：** 跑通"每日任务+成长任务+盖世币余额+兑换商城（虚拟+实物）+发货履约+我的奖励"的完整闭环。

- **未来可能升级（预告）：** 盖世币有效期倒计时提醒、限时兑换活动、盖世币排行榜、更多任务类型（周任务/活动任务）、社区头像框/称号等个性化兑换品。



# 四、概要设计

## 4.1 模块设计（系统架构图概览）

- **【C端任务与兑换模块】**

    - **任务中心：** 盖世币余额展示、每日任务列表（签到/启动游戏/分享/浏览社区/发帖/互动）、成长任务列表（完善资料/开启推送/首次启动/首次加入社区/首次关注/首次发帖/获得精选/分享配置方案/关注公众号）。

    - **兑换商城：** 商品网格展示（云游戏时长/实物手柄等）、盖世币余额、兑换确认弹窗（数量选择+余额校验）。

    - **我的奖励：** 兑换记录列表（待填地址/已发放/已发货状态）、填写收货地址、物流单号展示。

    - **积分明细：** 获取记录/消耗记录Tab切换。

    - **规则说明：** 盖世币使用规则页面。

- **【B端管理后台】**

    - **任务配置：** 新增/编辑/启停任务、任务类型（每日/成长）、奖励分值、频率上限。

    - **奖品配置：** 新增/编辑/上下架奖品、库存管理（补库存）、商品类型（虚拟/实物）、兑换价格、单人限兑。

    - **兑换与发货：** 订单列表（筛选/搜索/导出）、订单详情（收货地址/物流信息）、实物发货（填快递公司+单号）。

## 4.5 非功能需求

|**需求类型**|**详细要求**|
|---|---|
|**性能**|1. 任务列表加载 < 1s。<br>2. 兑换操作响应 < 2s（含库存扣减+余额扣减+发放/建单）。<br>3. 余额变动实时同步至所有展示位（任务中心+商城+明细）。|
|**安全**|1. 任务完成接口需防刷（同一任务同一用户同一秒内重复调用只计1次）。<br>2. 兑换接口需做幂等校验，防止重复扣币。<br>3. 盖世币流水全量记录，支持审计追溯。<br>4. 实物商品收货地址加密存储。|
|**兼容性**|1. 兼容 Android 9.0+、iOS 13.0+。<br>2. 适配横竖屏双模式（竖屏单列任务列表，横屏双列网格）。<br>3. 适配全面屏、刘海屏。|
|**经济控制**|1. 每日任务产出上限50盖世币/人/日。<br>2. 成长任务产出上限100盖世币/账号（一次性）。<br>3. 盖世币有效期365天，到期自动扣减。<br>4. 后台可随时调整任务奖励分值和商品价格，立即生效。|

# 五、交付设计（数据分析与埋点设计）

## 5.1 核心业务指标

- **任务漏斗：** 任务中心PV/UV → 各任务完成人数/次数 → 日均盖世币产出量。

- **兑换漏斗：** 兑换商城PV/UV → 商品点击数 → 兑换确认弹窗PV → 兑换成功数/金额 → 实物发货完成率。

- **经济指标：** 盖世币总产出/总消耗/净余额、人均持有量、消耗率（消耗/产出）。

## 5.2 埋点事件表

|**事件ID**|**事件名称**|**触发时机**|**关键参数**|
|---|---|---|---|

|`redeem_confirm`|确认兑换|点击"确认兑换"按钮且成功扣币时|`uid`, `item_id`, `item_name`, `item_type`, `quantity`, `total_cost`|

|`rules_view`|规则说明曝光|进入规则说明页时|`uid`|

|`qrcode_complete`|公众号关注完成|点击"我已关注并关闭"完成任务时|`uid`|
