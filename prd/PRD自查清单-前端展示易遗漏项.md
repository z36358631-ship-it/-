# PRD自查清单\-前端展示易遗漏项

> 本文档基于过往PRD回顾 \+ 行业最佳实践，总结出在PRD撰写阶段容易遗漏、但会直接影响前端用户展示与开发排期的关键信息点。适用于C端/B端所有PRD撰写环节。

## 一、列表与数据展示类

### 1\.1 排序规则（必填）

* [ ] 默认排序方式（如：按创建时间倒序、按下载量降序、按权重升序等）

* [ ] 排序字段相同时的次级排序规则（如：下载量相同时，按上传时间倒序）

* [ ] 是否支持用户切换排序方式（如：最新/最热切换）

* [ ] 切换排序后是否重置到列表顶部

### 1\.2 分页与加载

* [ ] 分页方式：翻页器分页 or 下拉无限加载

* [ ] 每页数据条数（如：每页20条）

* [ ] 是否显示总条数/总页数

* [ ] 滚动加载时是否保留已加载内容（下拉加载需保留，翻页可不保留）

* [ ] 返回列表时是否记住之前的滚动位置和页码

### 1\.3 空状态

* [ ] 列表无数据时展示什么（文案 \+ 配图）

* [ ] 是否有引导操作按钮（如：\&\#34;去发布第一条帖子\&\#34;）

* [ ] 搜索/筛选无结果时展示什么（需区别于初始空状态）

### 1\.4 加载状态

* [ ] 首次加载展示方式：骨架屏 or 加载中动画 or 菊花图

* [ ] 加载更多时的底部提示（\&\#34;加载中…\&\#34; / \&\#34;没有更多了\&\#34;）

* [ ] 加载超时后展示（如：\&\#34;加载失败，点击重试\&\#34;）

### 1\.5 筛选与搜索

* [ ] 筛选项的默认选中值

* [ ] 多个筛选项之间是\&\#34;且\&\#34;还是\&\#34;或\&\#34;关系

* [ ] 筛选结果为空时的提示

* [ ] 搜索关键词是否高亮

* [ ] 搜索是否支持模糊匹配

* [ ] 是否有搜索建议/搜索历史/热门搜索

---

## 二、文字内容类

### 2\.1 敏感词过滤（必填）

* [ ] 明确内容可见范围：仅自己可见 or 对其他用户公开展示

* [ ] 对外展示的内容：提交时必须进行敏感词过滤，需说明过滤时机（提交时/展示时）

* [ ] 过滤触发后的用户提示（如：\&\#34;内容包含违规词汇，请修改后重新提交\&\#34;）

* [ ] 是否需要人工审核兜底（如：社区帖子人工审核）

### 2\.2 文本截断与展示

* [ ] 长文本是否截断展示（如：标题最多显示1行，超出渐出处理，不采用省略号）

* [ ] 截断后是否支持\&\#34;展开/收起\&\#34;

* [ ] 截断的字符数或行数上限

### 2\.3 富文本与特殊字符

* [ ] 是否支持富文本（加粗/换行/链接等）

* [ ] 特殊字符处理（emoji是否支持展示）

* [ ] 换行符是否保留展示

---

## 三、输入类

### 3\.1 文本输入（必填）

* [ ] 输入长度限制：最小/最大字符数（如：昵称2\-20字，评论1\-500字）

* [ ] 是否显示实时字数统计（如：\&\#34;12/500\&\#34;）

* [ ] 超出限制时的处理方式：截断 or 禁止输入 or 提示报红

* [ ] 是否允许纯空格/纯换行提交

* [ ] 是否有格式校验（如：邮箱、手机号、URL格式）

### 3\.2 下拉选择/单选/多选

* [ ] 是否有默认选中项

* [ ] 多选时的最大选择数量

* [ ] 选项为空时的展示

### 3\.3 数值输入

* [ ] 最小值/最大值

* [ ] 是否允许小数，小数位数

* [ ] 是否允许负数

* [ ] 非法输入的处理方式

### 3\.4 表单整体

* [ ] 校验时机：输入时实时校验 or 失焦校验 or 提交时统一校验

* [ ] 多个字段校验失败时是否自动定位到第一个错误

* [ ] 跨字段联动校验（如：结束日期需晚于开始日期、确认密码需一致）

* [ ] 表单填写中途离开是否提示\&\#34;放弃编辑？\&\#34;

* [ ] 草稿/编辑态是否自动保存

---

## 四、图片/文件类

### 4\.1 图片上传（必填）

* [ ] 支持的图片格式（JPG/PNG/GIF/WebP等）

* [ ] 单张图片大小限制（如：≤5MB）

* [ ] 图片数量上限（如：最多9张）

* [ ] 图片尺寸/比例要求（如：封面图推荐16:9，≥800×450px）

* [ ] 必须支持删除已上传图片

* [ ] 必须支持替换已上传图片

* [ ] 上传失败的提示与重试机制

* [ ] 多图上传时是否支持拖拽排序

### 4\.2 图片展示

* [ ] 图片加载失败时的兜底展示（默认占位图/broken图标）

* [ ] 是否支持点击放大/预览

* [ ] 多图时的排列方式（九宫格/横向滑动）

### 4\.3 图片审核

* [ ] 明确图片可见范围：仅自己可见 or 对其他用户公开展示

* [ ] 对外展示的图片：上传时必须进行内容审核（涉黄/涉政/暴力等）

* [ ] 审核中的图片是否展示（展示模糊图 or 不展示 or 显示\&\#34;审核中\&\#34;）

* [ ] 审核不通过的通知与处理

### 4\.4 文件上传/下载

* [ ] 支持的文件格式

* [ ] 文件大小限制

## 五、状态与交互类

### 5\.1 状态流转（必填）

* [ ] 所有状态枚举（如：草稿/审核中/已发布/已下架/已删除）

* [ ] 状态间的合法流转路径（哪些状态可以转到哪些状态）

* [ ] 每个状态下用户可执行的操作（如：审核中不可编辑）

* [ ] 状态变更后是否需要通知用户

### 5\.2 操作反馈

* [ ] 操作成功的反馈方式（toast提示/页面跳转/刷新列表）

* [ ] 操作失败的提示文案

* [ ] 重复操作的防护（如：按钮点击后禁用防重复提交）

* [ ] 危险操作的二次确认弹窗（如：删除 → \&\#34;确定要删除吗？删除后不可恢复\&\#34;）

### 5\.3 实时性与刷新

* [ ] 数据是否需要实时刷新（如：在线人数、弹幕）

* [ ] 手动刷新方式（下拉刷新/刷新按钮）

* [ ] 自动刷新频率（如：每30秒轮询）

### 5\.4 控件状态完整性

* [ ] 按钮/控件的不可用\(Disabled\)状态样式及原因说明

* [ ] 选中/未选中态的视觉区分

* [ ] 倒计时类控件（如验证码60秒倒计时）的规则

---

## 六、时间与数值展示类

### 6\.1 时间格式（必填）

* [ ] 绝对时间格式（如：YYYY\-MM\-DD HH:mm）

* [ ] 相对时间规则（如：刚刚/\&lt;60分钟显示\&\#34;X分钟前\&\#34;/\&lt;24小时显示\&\#34;X小时前\&\#34;/超过则显示日期）

* [ ] 时区处理方式（跟随系统时区 or 服务器时区）

### 6\.2 数值格式

* [ ] 大数字的缩写规则（如：10000\+ → \&\#34;1万\&\#34;，100000\+ → \&\#34;10万\+\&\#34;）

* [ ] 小数保留位数及四舍五入规则

* [ ] 0值时的展示（显示\&\#34;0\&\#34; or 不显示 or 显示\&\#34;\-\&\#34;）

### 6\.3 金额展示

* [ ] 货币符号与位置（如：¥12\.00）

* [ ] 小数位数（如：固定2位）

* [ ] 千分位分隔符

* [ ] 原价/现价的展示（划线价等）

---

## 七、后台管理类

### 7\.1 增删查改（必填）

* [ ] 新增：必填项/选填项标注、默认值、唯一性校验

* [ ] 删除：软删除 or 硬删除、删除后关联数据的处理、二次确认弹窗

* [ ] 查询：支持哪些筛选条件、模糊/精确搜索、导出功能

* [ ] 修改：哪些字段可修改、哪些字段创建后不可改、修改是否记录操作日志

* [ ] 复制：运营频繁操作类，支持进行复制

### 7\.2 列表页

* [ ] 表格列定义（字段名、宽度、是否可排序）

* [ ] 批量操作（批量删除/批量上下架/批量导出）

* [ ] 数据导出格式（Excel/CSV）及字段范围

### 7\.3 权限控制

* [ ] 操作角色权限矩阵（谁能看、谁能改、谁能删）

* [ ] 操作日志记录（谁在什么时间做了什么操作）

---

## 八、登录态与用户身份

### 8\.1 登录态差异（必填）

* [ ] 哪些页面/功能未登录也可使用，哪些必须登录

* [ ] 未登录用户点击需登录功能时的交互（跳转登录页 or 弹出登录弹窗）

* [ ] 登录成功后是否自动回到之前的页面/操作

* [ ] 登录过期时的提示与跳转

### 8\.2 敏感数据脱敏

* [ ] 手机号、身份证、银行卡等在展示时的脱敏规则（如：138\*\*\*\*8888）

* [ ] 敏感操作是否需要二次验证（密码/短信验证码/生物识别）

---

## 九、分享功能

### 9\.1 分享渠道与内容

* [ ] 支持哪些分享渠道（微信好友/朋友圈/QQ/微博/复制链接）

* [ ] 每个渠道的分享格式（标题/描述/缩略图/链接）

* [ ] 是否需要生成分享海报/图片

* [ ] 分享成功/失败/取消后的提示

### 9\.2 被分享者视角

* [ ] 未安装APP时打开分享链接的落地方案（H5页面/应用商店引导）

* [ ] 分享链接能否直接打开APP内对应页面（Deep Link）

* [ ] 加载超时后的提示与重试

* [ ] 是否支持离线浏览已加载内容

* [ ] 多人同时编辑同一内容的处理

* [ ] 重复提交的防护策略

* [ ] 超长文本/超长昵称的展示处理

* [ ] 特殊值：0条数据、1条数据、恰好满页的边界

### 12\.1 埋点需求

* [ ] 关键页面的PV/UV

* [ ] 核心按钮的点击事件

* [ ] 转化漏斗的关键节点

* [ ] 埋点参数列表（事件ID、参数名、参数值枚举）

## 十三、发布与灰度

### 12\.1 上线策略

* [ ] 是否需要灰度发布（按用户比例/地区/设备灰度放量）

* [ ] 是否需要A/B测试，实验分组方案

* [ ] 是否需要功能开关（远程控制功能上下线）

* [ ] 功能异常时的降级展示方案（如：推荐系统挂了→显示热门榜单/国外玩家下载了国内包→支持语言切换）

## 十四、隐私合规

### 14\.1 用户隐私

* [ ] 数据收集范围告知（收集了哪些用户信息、用于什么目的）

* [ ] 系统权限按需申请（相机/位置/通讯录等必须使用时才申请，不可启动时批量弹窗）

## 使用方式

清单纳入写PRD时的自动加载规范。每次写PRD时，依靠Ai自动对照此清单，将涉及的场景逐条落实到PRD正文中，无需人工提醒。
