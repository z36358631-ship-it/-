## 版本信息

| 时间 | 版本 | 变更人 | 变更内容 | 备注 |
|---|---|---|---|---|
| 2026.6.4 | V1.0 | 郑群超 | 初稿 | - |

## 需求概述

组队玩新增Steam联机游戏品类，复用现有组队房间框架（游戏大厅、组队大厅、世界频道、房间页），同时完善分享邀请能力——支持游戏内/房间内将邀请链接发送到微信/QQ/Steam好友/Discord。

功能仅对国区包开放，海外包分享渠道有差异（见分享章节）。

---

## 功能结构

- 玩游戏 Tab
  - 组队玩页
    - 游戏大厅（新增品类筛选：全部/云游戏/Steam）
    - 组队大厅（房间卡片带品类标签）
    - 世界频道（独立页面，品类筛选：Steam默认/云游戏）
  - 创建房间页（独立页面）
  - 组队房间页（新增分享邀请按钮、成员联机启动按钮）
  - 游戏内状态（新增分享按钮）
  - 分享落地页（H5）

## 名词注释

| 名词 | 说明 |
|------|------|
| Steam房间 | 游戏类型为Steam的组队房间，成员各自通过Steam客户端联机，不走云游戏Session |
| 联机启动 | 成员侧按钮，房主启动游戏后成员可点击，拉起本地Steam客户端加入联机 |
| 品类标签 | 游戏/房间卡片上的类型标识，蓝色=Steam，紫色=云游戏 |

---

## Steam与云游戏差异对照

| 维度 | 云游戏 | Steam |
|------|--------|-------|
| 游戏来源 | 海马云游戏列表 | 后台配置Steam联机游戏列表 |
| 启动方式 | 平台启动云Session，多人共享 | 各自启动本地Steam客户端联机 |
| 时长校验 | 需校验≥10分钟 | 不校验（无平台计费） |
| 房间有效期 | 30分钟 | 60分钟 |
| 房主按钮 | 「开始游戏」→ 拉起云Session | 「启动游戏」→ 提示各自启动 |
| 成员按钮 | 无（被动拉起） | 「联机启动」→ 拉起Steam客户端 |
| 计费 | 所有成员独立扣时长 | 无计费 |
| 游戏中加入 | 不可加入 | 可加入（未满员时） |

## 游戏大厅

### 品类筛选

①顶部新增品类筛选下拉：全部（默认）/ 云游戏 / Steam
②切换后实时过滤游戏卡片列表
③组队大厅共享该筛选状态

### 游戏卡片

①每张卡片新增品类标签（左上角）：蓝色「Steam」/ 紫色「云游戏」
②卡片内容不变：游戏封面、游戏名、当前房间数
③按钮逻辑不变：rooms > 0 展示「快速加入」+「创建房间」，rooms = 0 仅展示「创建房间」
④点击卡片整体逻辑不变

---

## 组队大厅

### 房间卡片

①每张房间卡片新增品类标签：游戏名后展示蓝色「Steam」或紫色「云游戏」
②其余展示内容不变：游戏封面、游戏名、房间名、房间ID、房主昵称、人数、状态
③品类筛选联动游戏大厅

---

## 世界频道

### 页面结构

①点击世界频道 Tab → 弹出「进入世界频道」确认弹窗（复用现有逻辑）
②确认后进入世界频道独立全屏页面（非sub-tab内嵌）
③页面顶部：返回按钮 + 标题「世界频道」+ 举报按钮

### 品类筛选

①标题下方平铺展示品类按钮：Steam（默认高亮）/ 云游戏
②切换后实时过滤聊天流中的招募卡片
③招募卡片上标注品类标签

### 其余逻辑

复用现有世界频道全部规则：聊天流框架、时间戳、快捷短语、消息频率限制、举报。

---

## 创建房间

### 页面结构

①点击「创建房间」后进入独立全屏页面（非弹窗）
②顶部：返回按钮 + 标题「创建组队房间」

### 选择游戏类型

①平铺按钮：Steam（默认高亮蓝色）/ 云游戏（紫色）
②点击切换后，下方游戏列表实时过滤为对应品类

### 选择游戏

①输入框支持搜索游戏名，实时过滤
②下方展示可滚动游戏列表（封面+游戏名+人数+模式）
③点击选中某游戏，左侧蓝色标记+勾选图标
④未选择游戏时点击创建，toast提示「请先选择一个游戏」

### 房间设置

①房间名称：文本输入，最多20字，默认「来不及解释了，快加入房间！」
②最大人数：2人 / 4人（平铺按钮切换），默认取所选游戏max值
③房间权限：公开 / 私密（下拉选择）；私密时显示密码输入框，纯数字固定4位
④Steam房间创建不校验云游戏时长；云游戏房间创建时校验时长≥10分钟

### 创建成功

①创建成功后直接跳转至组队房间页
②toast提示「公开房间创建成功，可分享邀请」或「私密房间创建成功」

---

## 组队房间页

### 房间标题

①房间名旁展示品类标签：蓝色「Steam」或紫色「云游戏」

### 房主视角

①底部按钮从左到右：「分享邀请」「解散房间」「启动游戏」
②「启动游戏」：人数≥2时可点击，＜2时置灰
③点击「启动游戏」后：
  - 房间状态变为"游戏中"
  - 消息区展示系统消息「房主已启动游戏，请各自启动Steam加入联机」
  - 成员侧「联机启动」按钮变为可点击
④「世界招募」保留（输入框旁，10秒CD）

### 成员视角

①底部按钮从左到右：「分享邀请」「退出房间」「联机启动」
②「联机启动」默认置灰，文案显示「等待房主开始」
③房主启动游戏后，按钮变亮，文案变为「联机启动」，可点击
④点击后toast「正在启动Steam游戏，加入联机...」
⑤不展示「世界招募」「解散房间」
⑥聊天中"你"不带「（房主）」标识

### 其余逻辑

复用现有房间页全部规则：成员席位、房间消息、快捷短语、麦克风/听筒开关、踢人、修改房间信息。

## 分享邀请

### 入口

①房间页底部「分享邀请」按钮（房主和成员均可见）
②游戏内右上角控制按钮行新增分享按钮（房主可见）
③房间页顶部「...」菜单内「分享房间」保留

### 分享弹窗

①点击后弹出分享弹窗，展示房间信息预览（游戏名、房间名、人数、房间ID、品类）
②下方展示分享渠道选项：

**国内版渠道：**
- 微信好友：调起微信分享组件
- QQ：调起QQ分享
- Steam好友：拉起Steam好友页面
- 复制链接：写入剪贴板，toast「链接已复制」

**海外版渠道：**
- Steam好友：拉起Steam好友页面
- Discord：打开频道选择器
- 复制链接：写入剪贴板

### Discord频道选择器

①点击Discord后打开频道选择器弹窗
②展示用户已加入的服务器列表和文字频道
③顶部搜索框支持按服务器名/频道名过滤
④选中频道后点击「发送到频道」，发送富链接消息
⑤发送成功后关闭选择器，消息区展示系统提示「邀请已发送到Discord #频道名」

### 分享卡片内容

- 微信/QQ分享卡片：游戏封面缩略图 + 标题「[游戏名] 组队邀请」+ 描述（房间名+人数）+ 加入链接
- Discord富链接：Open Graph标签（og:title/og:description/og:image）

### 分享成功反馈

①分享成功后房间消息区展示系统提示（如「邀请已发送给微信好友」）
②分享按钮无CD限制（与世界招募独立）

## 分享落地页

### 页面内容

①游戏封面（16:9）
②游戏名
③房间信息卡片：房间名、房间状态（准备中/游戏中）、房主头像+昵称、当前人数/最大人数
④品类标签（Steam联机 / 云游戏）
⑤「加入房间」按钮
⑥底部提示「已安装盖世游戏？将自动打开App」

### 交互逻辑

①已安装App → 点击「加入房间」→ 唤起App并跳转至目标房间页
②未安装App → 展示下载引导（应用商店按钮），安装后首次打开自动跳转目标房间
③唤起App 2秒超时 → 展示「打开失败？点击手动打开」
④未登录用户点击加入 → 跳转登录页 → 登录成功后自动加入目标房间
⑤通过分享链接加入私密房间无需输入密码（链接自带邀请凭证，有效期24小时）

### 异常状态

| 落地页查询结果 | 展示 |
|-------------|------|
| 房间正常（准备中） | 正常展示，可加入 |
| 房间游戏中（Steam） | 正常展示，可加入（Steam允许中途加入） |
| 房间已满 | 展示「房间已满」+ 「查看其他房间」按钮 |
| 房间已解散/过期 | 展示「房间已结束」+ 「查看其他房间」按钮 |
| 链接无效/被篡改 | 展示「链接无效或已过期」 |

## 房间状态机

复用现有4种状态：创建中/游戏中/已过期/已解散。

**Steam房间差异：**

| 维度 | 云游戏 | Steam |
|------|--------|-------|
| 有效期 | 30分钟 | 60分钟 |
| 游戏中是否可加入 | 不可加入 | 可加入（未满员时） |
| 房主退出 | 房间解散 | 房主转移给最早加入的成员 |
| 全员退出 | 自动解散 | 自动解散 |

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 设备未安装微信/QQ | toast「未检测到应用」，不崩溃 |
| Steam好友页拉起失败 | toast「未检测到Steam客户端」|
| Discord发送失败 | toast「发送失败，请检查网络后重试」，保留频道选择状态 |
| 用户已在其他房间 | toast「您已在房间中，无法加入其他房间」 |
| Steam房间满员 | toast「该房间已满员，请选择其他房间」 |
| 创建房间未选游戏 | toast「请先选择一个游戏」 |
| 网络异常 | 列表加载失败展示「加载失败，点击重试」；操作失败展示toast「操作失败，请检查网络」 |

## 自查清单落实

| 清单项 | 落实情况 |
|--------|---------|
| 1.1 排序规则 | 游戏大厅按后台配置权重排序，组队大厅按房间创建时间倒序 |
| 1.2 分页 | 下拉无限加载，每页20条，保留已加载内容 |
| 1.3 空状态 | 游戏大厅无数据「暂无支持组队的游戏」；组队大厅无房间「暂无公开房间，快去创建一个吧」+创建房间按钮；搜索无结果「暂无相关组队内容」 |
| 1.4 加载状态 | 首次骨架屏，底部「没有更多了」，超时「加载失败，点击重试」 |
| 1.5 筛选 | 品类筛选默认「全部」，筛选间为且关系，结果为空展示对应空状态 |
| 2.1 敏感词 | 房间名创建时走敏感词过滤，提交时校验，提示「内容包含违规词汇，请修改」 |
| 3.1 文本输入 | 房间名2-20字，显示实时字数，超出禁止输入；密码固定4位纯数字 |
| 5.1 状态流转 | 见房间状态机章节 |
| 5.2 操作反馈 | 创建成功toast+跳转；分享成功消息区系统提示；按钮点击后禁用防重复 |
| 6.1 时间格式 | 房间有效期倒计时MM:SS；消息时间戳YYYY/M/D HH:mm |
| 8.1 登录态 | 分享落地页可浏览，加入房间需登录，登录后自动回跳 |
| 9.1 分享 | 国内：微信/QQ/Steam好友/复制链接；海外：Steam好友/Discord/复制链接；分享成功/失败/取消均有toast提示 |
| 9.2 被分享者 | 未安装App展示下载引导；Deep Link唤起App跳转房间 |
| 10.1 网络异常 | 弱网toast提示，加载超时重试，断网保留已加载内容 |
