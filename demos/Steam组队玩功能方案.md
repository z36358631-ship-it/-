# 二、背景与目标

**背景：** 云游戏组队玩已上线，用户可围绕云游戏创建房间、组队联机。当前 Steam 也支持联机，且玩家找队友的需求散落在微信群/Discord频道等外部平台。为扩大组队玩的覆盖面，需在现有框架中新增 Steam 联机游戏品类，同时完善分享邀请能力，打通微信/QQ/Steam好友/Discord 外部分享链路。

**目标：**

- 将组队玩从云游戏扩展到 Steam 联机游戏，提升组队玩整体活跃
- 通过外部分享降低组队匹配门槛，提升房间填充率和邀请回流

# 三、故事介绍

## 3.1 用户场景

**场景一：Steam 玩家想找队友联机**

用户进入组队玩页，在游戏大厅切换品类筛选到「Steam」，看到恐鬼症当前有可加入房间，点击「快速加入」。系统将用户加入房间后进入房间页，等待房主启动游戏，房主启动后用户点击「联机启动」拉起本地 Steam 客户端加入联机。

**场景二：房主创建 Steam 房间并分享到微信群**

用户选择「双人成行」创建 Steam 房间，进入房间页后点击「分享邀请」，选择微信好友发送邀请卡片。微信群里的朋友点击链接打开落地页，点击「加入房间」唤起盖世游戏 App 直接进入房间。

**场景三：游戏内补位分享**

联机过程中有人退出，房主点击游戏内分享按钮，选择 Steam 好友发送邀请。Steam 好友收到后通过链接加入房间，点击「联机启动」加入进行中的游戏。

**场景四：海外用户通过 Discord 邀请**

海外用户创建房间后点击「分享邀请」，选择 Discord，在频道选择器中选择目标频道发送富链接消息。频道内其他玩家看到后点击链接加入房间。

## 3.2 价值分析

- **扩展组队覆盖面：** 从云游戏延伸到 Steam 联机，覆盖更多游戏品类和用户群体。
- **打通外部社交链路：** 分享到微信/QQ/Steam/Discord，将邀请投递到玩家真实社交关系链，降低组队匹配门槛。
- **提升回流与拉新：** 分享落地页引导未安装用户下载，已安装用户一键进入房间，缩短转化路径。
- **降低 Steam 玩家找队友成本：** 不用在多个平台发帖找人，一个入口解决。

## 3.3 核心体验路径

**路径一：Steam 游戏快速加入**

组队玩 → 游戏大厅（筛选 Steam）→ 快速加入 → 进入房间 → 等待房主启动 → 联机启动

**路径二：创建 Steam 房间并分享**

创建房间（选 Steam 游戏）→ 进入房间 → 分享邀请（微信/QQ/Steam/Discord）→ 队友通过链接加入 → 房主启动游戏 → 各自联机

**路径三：被邀请者通过链接加入**

收到分享链接 → 打开落地页 → 点击加入 → 唤起 App 进入房间 → 联机启动

**路径四：世界频道找 Steam 队友**

世界频道 Tab → 确认进入 → 切换 Steam 筛选 → 看到招募卡片 → 立即加入

## 3.4 产品指标预测

- **Steam 房间创建量：** 每日 Steam 品类房间创建数。
- **分享触发率：** 房间页/游戏内点击「分享邀请」的用户占比。
- **分享落地页转化率：** 落地页访问 → 点击加入房间 → 成功进入房间的漏斗。
- **外部回流新增用户：** 通过分享链接首次下载并注册的用户数。
- **Steam 房间填充率：** 创建后成功≥2人并启动游戏的房间占比。

## 3.5 路径规划

**当前版本（V1.0）：**

完成 Steam 组队玩 C 端主链路 + 分享邀请（微信/QQ/Steam好友/Discord）+ 分享落地页。

**未来可能升级：**

- 微信小程序落地页（提升微信内转化）
- Discord Bot 自动推送房间状态变更
- Steam 好友列表同步（自动展示在线好友）

# 四、概要设计

## 4.1 模块设计（功能架构概览）

**【游戏大厅模块 - 新增】**

- **品类筛选：** 游戏大厅/组队大厅顶部新增品类下拉：全部/云游戏/Steam，切换后列表实时过滤。
- **品类标签：** 游戏卡片/房间卡片新增品类标签（蓝色=Steam，紫色=云游戏）。

**【创建房间模块 - 改造】**

- **独立全屏页面：** 创建房间改为独立页面（非弹窗），支持游戏搜索和品类切换。
- **游戏类型选择：** 平铺按钮 Steam（默认）/ 云游戏，切换后过滤游戏列表。
- **时长校验差异：** Steam 房间不校验云游戏时长。

**【世界频道模块 - 改造】**

- **独立全屏页面：** 确认进入后跳转独立页面，顶部品类筛选 Steam（默认）/ 云游戏。
- **招募卡片标签：** 卡片标注品类。

**【组队房间模块 - 新增】**

- **分享邀请按钮：** 底部新增，房主+成员均可见。
- **启动游戏/联机启动：** 房主按钮「启动游戏」，成员按钮「联机启动」（置灰→房主启动后可点）。
- **Steam 房间有效期 60 分钟。**

**【分享邀请模块 - 新增】**

- **分享弹窗：** 国内渠道（微信/QQ/Steam好友/复制链接）、海外渠道（Steam好友/Discord/复制链接）。
- **Discord 频道选择器：** 展示服务器+频道列表，支持搜索，选中后发送。
- **游戏内分享：** 右上角控制按钮新增分享入口（房主可见）。

**【分享落地页模块 - 新增】**

- **H5 落地页：** 展示游戏封面、房间信息、品类标签、加入按钮。
- **Deep Link 唤起 App；** 未安装引导下载。

**【房间状态机 - 差异】**

|维度|云游戏|Steam|
|---|---|---|
|有效期|30分钟|60分钟|
|游戏中可加入|不可|可加入（未满员时）|
|房主退出|房间解散|房主身份转移|
|时长校验|≥10分钟|不校验|

### 4.2.2 房间状态机（Steam差异）

复用现有5种状态：创建中/启动中/游戏中/已过期/已解散。

**Steam房间差异：**

|维度|云游戏|Steam|
|---|---|---|
|有效期|30分钟|60分钟|
|游戏中可加入|不可加入|可加入（未满员时）|
|房主退出|房间解散|房主身份转移给最早加入的成员|
|全员退出|自动解散|自动解散|
|启动中状态|云游戏排队/启动Session|无（直接→游戏中）|

### 4.2.3 异常处理

|场景|处理方式|
|---|---|
|设备未安装微信/QQ|toast「未检测到应用」|
|Steam好友页拉起失败|toast「未检测到Steam客户端」|
|Discord发送失败|toast「发送失败，请检查网络后重试」，保留频道选择状态|
|用户已在其他房间|toast「您已在房间中，无法加入其他房间」|
|Steam房间满员|toast「该房间已满员，请选择其他房间」|
|创建房间未选游戏|toast「请先选择一个游戏」|
|房间名含敏感词|toast「内容包含违规词汇，请修改后重新提交」|
|网络异常|列表「加载失败，点击重试」；操作toast「操作失败，请检查网络」|
|落地页唤起App失败|2秒超时→「打开失败？点击手动打开」+应用商店|
