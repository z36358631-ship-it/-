| 时间 | 版本 | 变更人 | 变更内容 | 备注 |
|---|---|---|---|---|
| 2026.6.4 | V1.0 | 郑群超 | 初稿 | - |

## 需求概述

组队玩新增 Steam 联机游戏品类，复用现有组队房间框架（游戏大厅、组队大厅、世界频道、房间页），同时完善分享邀请能力——支持房间内/游戏内将邀请链接发送到微信/QQ/Steam好友/Discord。功能仅对国区包开放，海外包分享渠道有差异。

---

## 核心入口与页面结构

### 入口位置

用户进入「玩游戏」Tab → 点击「组队玩」→ 进入组队玩页面，整体框架不变。

### 整体页面层级

- 玩游戏 Tab
  - 组队玩页
    - 游戏大厅（新增品类筛选）
    - 组队大厅（房间卡片带品类标签）
    - 世界频道（独立全屏页面，品类筛选）
  - 创建房间页（独立全屏页面）
  - 组队房间页（新增分享邀请 + 成员联机启动）
  - 游戏内状态（新增分享按钮）
  - 分享落地页（H5）

### Steam 与云游戏差异对照

| 维度 | 云游戏 | Steam |
|------|--------|-------|
| 游戏来源 | 海马云游戏列表 | 后台配置 Steam 联机游戏列表 |
| 启动方式 | 平台启动云 Session，多人共享 | 各自启动本地 Steam 客户端联机 |
| 时长校验 | 需校验≥10分钟 | 不校验（无平台计费） |
| 房间有效期 | 30分钟 | 60分钟 |
| 房主按钮 | 「开始游戏」→拉起云 Session | 「启动游戏」→提示各自启动 |
| 成员按钮 | 无（系统被动拉起） | 「联机启动」→拉起 Steam 客户端 |
| 计费 | 所有成员独立扣时长 | 无计费 |
| 游戏中加入 | 不可加入 | 可加入（未满员时） |
| 房主退出 | 房间解散 | 房主身份转移给最早加入的成员 |

## 游戏大厅

### 品类筛选

①游戏大厅顶部操作栏新增品类筛选下拉：全部（默认）/ 云游戏 / Steam
②切换后实时过滤游戏卡片列表，仅展示对应品类游戏
③组队大厅共享该筛选状态，切换后联动过滤房间列表

### 卡片列表

每张游戏卡片包含：
- 游戏封面图
- 游戏名
- 品类标签（蓝色「Steam」/ 紫色「云游戏」，展示在封面左上角）
- 当前房间数量

### 卡片按钮逻辑

与云游戏一致：rooms > 0 展示「快速加入」+「创建房间」并排，rooms = 0 仅展示「创建房间」。

### 卡片点击逻辑

与云游戏一致：rooms > 0 执行快速加入，rooms = 0 打开创建房间页。Steam 游戏快速加入不校验时长。

## 组队大厅

### 卡片列表

每张房间卡片展示内容新增品类标签：
- 游戏封面/图标
- 游戏名 + 品类标签（蓝色「Steam」/ 紫色「云游戏」）
- 房间名
- 房间 ID
- 房主昵称
- 当前人数 / 最大人数
- 房间状态（准备中 / 游戏中）

### 卡片点击逻辑

点击房间卡片整体直接执行加入房间逻辑。Steam 房间加入不校验时长；加入前校验是否已在房间中。

### 品类筛选

与游戏大厅共享筛选状态，切换后联动过滤房间列表。

## 世界频道

### 页面结构

①点击世界频道 Tab → 弹出「进入世界频道」确认弹窗（复用现有逻辑）
②点击「寻找队友」后，进入世界频道独立全屏页面
③点击「我再逛逛」后，关闭弹窗，切回上一个 Tab

RS：世界频道改为独立全屏页面，不再作为 sub-tab 内嵌内容

### 独立页面布局

①顶部：返回按钮（←）+ 标题「世界频道」+ 举报按钮
②标题下方：品类筛选平铺按钮
③下方：聊天流区域 + 快捷短语 + 输入框

### 品类筛选

①平铺展示两个按钮：Steam（默认高亮）/ 云游戏
②切换后实时过滤聊天流中的招募卡片，仅展示对应品类
③招募卡片上标注品类标签（蓝色「Steam」/ 紫色「云游戏」）

### 其余逻辑

聊天流框架、时间戳格式、快捷短语、消息频率限制（10秒/条）、举报按钮——全部复用现有规则，无变化。

## 创建房间

### 页面结构

①点击任何「创建房间」入口后，进入创建房间独立全屏页面
②顶部：返回按钮（←）+ 标题「创建组队房间」

RS：创建房间改为独立全屏页面，不再使用弹窗，因为支持组队的游戏可能较多需要搜索

### 选择游戏类型

①平铺展示两个按钮：Steam（默认高亮蓝色）/ 云游戏（紫色）
②点击切换后，下方游戏列表实时过滤为对应品类

### 选择游戏

①搜索输入框：placeholder「搜索游戏名...」，输入时实时过滤列表
②下方可滚动游戏列表，每行展示：游戏封面缩略图 + 游戏名 + 最大人数 + 联机模式
③点击选中游戏，左侧高亮标记 + 勾选图标表示已选
④未选择游戏时点击创建按钮，toast「请先选择一个游戏」

### 房间设置

①房间名称：文本输入，最多20字，默认「来不及解释了，快加入房间！」，显示实时字数统计
②最大人数：平铺按钮 2人 / 4人，默认取所选游戏 max 值
③房间权限：下拉选择「公开房间」/「私密房间」；选择私密后显示密码输入框
④密码规则：纯数字，固定4位

### 时长校验

- Steam 房间：不校验云游戏时长，直接创建
- 云游戏房间：校验时长≥10分钟，不足时弹出充值引导弹窗

### 创建成功

创建成功后直接跳转至组队房间页。toast「公开房间创建成功，可分享邀请」或「私密房间创建成功」。

## 组队房间页

### 房间标题

房间名旁新增品类标签：蓝色「Steam」或紫色「云游戏」。

### 房主视角

①底部按钮从左到右：「分享邀请」「解散房间」「启动游戏」
②「启动游戏」：人数≥2时可点击，＜2时置灰不可点
③房主点击「启动游戏」后：
- 房间状态变为"游戏中"
- 消息区展示系统消息「房主已启动游戏，请各自启动 Steam 加入联机」
- 成员侧「联机启动」按钮由置灰变为可点击
④「世界招募」按钮保留（输入框旁），10秒 CD

RS：Steam 房间「启动游戏」不拉起云 Session，仅变更状态并通知成员

### 成员视角

①底部按钮从左到右：「分享邀请」「退出房间」「联机启动」
②「联机启动」初始状态置灰，按钮文案「等待房主开始」
③房主点击「启动游戏」后，「联机启动」变为可点击状态，文案变为「联机启动」
④成员点击「联机启动」后，toast「正在启动 Steam 游戏，加入联机...」
⑤成员不展示「世界招募」「解散房间」按钮
⑥聊天中自己的昵称显示「你」，不带「（房主）」标识

### 其余逻辑

成员席位、房间消息聊天流、快捷短语、麦克风/听筒开关、踢人、修改房间信息——全部复用现有规则，无变化。

## 分享邀请

### 入口

①房间页底部「分享邀请」按钮——房主和成员均可见
②游戏内右上角控制按钮行新增分享按钮（<i class="fas fa-share-from-square"></i>）——房主可见
③房间页顶部「...」菜单内「分享房间」保留

### 分享弹窗

①点击分享入口后弹出分享弹窗
②弹窗上方展示房间信息预览：游戏名 + 房间名 + 人数 + 房间 ID + 品类
③下方展示分享渠道选项（区分国内/海外）

### 国内版渠道

| 渠道 | 点击行为 |
|------|---------|
| 微信好友 | 调起微信分享组件，发送分享卡片 |
| QQ | 调起 QQ 分享组件，发送分享卡片 |
| Steam好友 | 拉起 Steam 客户端好友页面 |
| 复制链接 | 写入剪贴板，toast「链接已复制」 |

### 海外版渠道

| 渠道 | 点击行为 |
|------|---------|
| Steam好友 | 拉起 Steam 客户端好友页面 |
| Discord | 打开 Discord 频道选择器 |
| 复制链接 | 写入剪贴板，toast「链接已复制」 |

### Discord 频道选择器

①点击 Discord 后弹出频道选择器弹窗
②展示用户已加入的服务器列表和文字频道
③顶部搜索框支持按服务器名/频道名实时过滤
④点击选中某频道，高亮标记
⑤点击「发送到频道」，将房间富链接消息发送到目标频道
⑥发送成功后关闭选择器，房间消息区展示系统提示「邀请已发送到 Discord #频道名」

### 分享卡片内容

- 微信/QQ 分享卡片：游戏封面缩略图 + 标题「[游戏名] 组队邀请」+ 描述（房间名 + 人数）+ 加入链接
- Discord 富链接：Open Graph 标签（og:title / og:description / og:image）

### 分享成功反馈

①分享成功后房间消息区展示系统提示（如「邀请已发送给微信好友」「邀请已发送给 Steam 好友」）
②分享按钮无 CD 限制，与世界招募独立

## 分享落地页

### 页面内容

①游戏封面图（16:9 圆角）
②游戏名
③房间信息卡片：房间名、房间状态标签（准备中/游戏中）、房主头像+昵称、当前人数/最大人数
④品类标签（蓝色「Steam 联机」/ 紫色「云游戏」）
⑤「加入房间」按钮
⑥底部提示文字「已安装盖世游戏？将自动打开 App」

### 交互逻辑

①已安装 App：点击「加入房间」→ Deep Link 唤起 App 跳转至目标房间页
②未安装 App：展示下载引导（应用商店按钮），安装后首次打开自动跳转目标房间
③唤起 App 2秒超时：展示「打开失败？点击手动打开」按钮
④未登录用户点击加入：跳转登录页 → 登录成功后自动加入目标房间
⑤通过分享链接加入私密房间无需输入密码（链接自带邀请凭证，有效期24小时）

### 落地页异常状态

| 查询结果 | 展示 |
|---------|------|
| 房间正常（准备中） | 正常展示，「加入房间」可点 |
| 房间游戏中（Steam） | 正常展示，「加入房间」可点（Steam 允许中途加入未满员房间） |
| 房间游戏中（云游戏） | 展示状态「游戏中」，按钮置灰，toast「游戏已开始，无法加入」 |
| 房间已满 | 展示「房间已满」+「查看其他房间」按钮 |
| 房间已解散/过期 | 展示「房间已结束」+「查看其他房间」按钮 |
| 链接无效/被篡改 | 展示「链接无效或已过期」 |

## 搜索适配

①全局搜索结果页「云游组队玩」Tab 内，游戏卡片和房间卡片均展示品类标签
②搜索可同时命中云游戏和 Steam 游戏，混合展示带标签区分
③搜索匹配规则不变：游戏名 / 房主昵称 / 房间 ID

## 房间状态机

复用现有4种状态：创建中 / 游戏中 / 已过期 / 已解散。

### Steam 房间差异

| 维度 | 云游戏 | Steam |
|------|--------|-------|
| 有效期 | 30分钟 | 60分钟 |
| 游戏中是否可加入 | 不可加入 | 可加入（未满员时） |
| 房主退出 | 房间解散 | 房主身份转移给最早加入的成员 |
| 全员退出 | 自动解散 | 自动解散 |

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 设备未安装微信/QQ | toast「未检测到应用」，不崩溃 |
| Steam 好友页拉起失败 | toast「未检测到 Steam 客户端」 |
| Discord 发送失败 | toast「发送失败，请检查网络后重试」，保留频道选择状态 |
| 用户已在其他房间 | toast「您已在房间中，无法加入其他房间」 |
| Steam 房间满员 | toast「该房间已满员，请选择其他房间」 |
| 创建房间未选游戏 | toast「请先选择一个游戏」 |
| 房间名含敏感词 | toast「内容包含违规词汇，请修改后重新提交」 |
| 网络异常 | 列表加载失败「加载失败，点击重试」；操作失败 toast「操作失败，请检查网络」 |
| 落地页唤起 App 失败 | 2秒超时展示「打开失败？点击手动打开」+ 应用商店链接 |
