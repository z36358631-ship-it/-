# 二、背景与目标

**背景：** 目前 Mac 客户端与移动端均缺乏玩家侧的兼容性反馈能力。玩家在下载或购买游戏前，无法参考其他玩家在相同设备上的实际运行情况，导致因兼容性问题引发的退款与差评。与此同时，运营和研发团队也缺少结构化的兼容性数据来源，无法有效区分 Mac 端与移动端的兼容问题，难以针对性地进行优化排查。

**目标：** 通过在游戏详情页新增兼容性评价入口，支持玩家提交带有设备信息的兼容性反馈，构建"浏览反馈 - 提交评价 - 后台治理"的完整数据闭环。核心挑战：横竖屏两套布局自适应、设备信息静默采集、举报与隐藏的状态联动、后台按端分类治理。

# 三、故事介绍

在切入繁杂的功能点前，先通过两个典型角色的视角，来看看新功能全貌的运转方式。

## 3.1 用户与运营场景

**C端玩家（小王）：**

小王在游戏详情页看到了「兼容评价」入口卡片，显示综合评分 4.3，描述文案"大多数玩家可正常游玩"。他点击进入兼容性评价详情页，浏览了其他玩家的反馈，看到设备信息「iPhone 15 Pro · iOS 17」，确认自己的设备应该没问题，随即下单购买。购买后他也点击「我要反馈」提交了自己的评价，选了 5 星并勾选了"无明显问题"标签。

**B端运营（李姐）：**

李姐在后台进入「玩家兼容性评价管理」，按「移动端」筛选后，发现某款游戏有多条「崩溃闪退」标签的评价，将几条内容不实的评价隐藏处理，并对其中一条举报进行了「处理」操作，评价自动联动隐藏，该游戏综合评分随即重新计算更新。

## 3.2 价值分析

- **降低购买风险：** 玩家在购买决策前可参考真实设备兼容情况，减少因兼容性未知导致的退款与差评。
- **结构化数据沉淀：** 按端（Mac端/移动端）区分的评价数据，帮助研发精准定位平台兼容问题，提升排查效率。
- **内容自治闭环：** 举报机制配合后台处理能力，平台可主动干预低质内容，保障评价生态健康。

## 3.3 核心体验路径

**C端评价核心链路：**

详情页入口卡片曝光 → 点击进入兼容性评价详情页 → 查看综合评分/分段占比/玩家评价列表 → 点击「我要反馈」→ 登录验证 → 填写星级+问题标签+文字（可选）→ 提交成功，列表与综合评分刷新

**B端管理核心链路：**

进入「玩家兼容性评价管理」→ 按端/状态/游戏名筛选 → 隐藏/恢复评价 → 综合评分联动更新；举报列表处理/驳回 → 处理后联动隐藏评价及同评价其他举报

## 3.4 产品指标预测

- **入口点击率：** 首月预测详情页兼容评价卡片点击率 10%～15%（已有游戏购买意向用户关注度高）。
- **反馈提交率：** 进入评价详情页后，预计 5%～10% 的已登录用户会完成反馈提交。
- **举报处理率：** 预计举报量极低，后台运营每日处理时长 < 30 分钟。

## 3.5 路径规划

**当前版本（V1.0）：** 跑通移动端与 Mac 端的评价入口、详情页展示、反馈提交、举报机制，以及后台评价管理与举报处理全流程。

**未来可能升级（预告）：** 支持按设备型号聚合展示兼容情况、开发者官方回复入口。

# 四、概要设计

## 4.1 模块设计（系统架构图概览）

本次功能涉及的系统模块归纳如下：

**【C端展示与交互模块】**

- **入口卡片：** 横竖屏两套布局，含综合评分、星级、描述文案、「兼容评价」标签+箭头。
- **评价详情页：** 综合评分区、分段占比条形图、玩家评价列表（含头像、昵称、评分、标签、文字、设备信息、点赞数、「举报」文字按钮）。
- **反馈弹窗：** 星级评分（必填）、问题标签（多选）、评价文字（选填，≤200字）、设备信息静默采集。
- **举报链路：** 举报原因选择弹窗、前台本地屏蔽、后台记录。

**【B端管理模块】**

- **评价管理：** 列表展示、按端/状态/游戏名筛选、隐藏/恢复操作。
- **举报处理：** 举报列表、处理（联动隐藏评价）/驳回操作、误处理修正。

## 4.2 功能清单结构

| 序号 | 模块 | 功能简述 | 优先级 |

| 2 | 兼容性评价页 | 展示综合评分、评分分布、玩家评价列表 | P0 |
| 3 | 反馈弹窗 | 登录用户提交评分+问题标签+设备信息 | P0 |
| 4 | 评价点赞 | 登录用户可对评价点赞/取消点赞，每条评价限一次 | P1 |
| 5 | 举报功能 | 对评价发起举报，前台屏蔽后提交后台处理 | P1 |
| 6 | 后台评价管理 | 列表展示、按端筛选、隐藏/恢复操作 | P0 |
| 7 | 后台举报处理 | 举报列表、处理/驳回、联动隐藏评价 | P1 |

## 4.3 详细设计

### 4.3.1 游戏详情页入口卡片（C端）

功能 demo：兼容性评价入口卡片 demo

**入口卡片布局规则：**

- 横屏（`.shell.ls`）：展示完整概览卡片，含图标、综合评分数字、星级、描述文案，卡片最大宽度 50%。
- 竖屏（`.shell.pt`）：隐藏图标区域，评分字体 15px，星星 9×9px，描述文案单行截断，整体紧凑布局。
- 卡片右侧固定展示「兼容评价」标签和向右箭头，点击卡片整体进入兼容性评价详情页。

**综合评分展示规则：**

| 场景 | 综合评分展示 | 描述文案 |
| --- | --- | --- |
| 有评价，avg ≥ 4.2 | 均分，保留 1 位小数 | 大多数玩家可正常游玩 |
| 有评价，avg ≥ 3.2 且 < 4.2 | 均分，保留 1 位小数 | 整体可玩 |
| 有评价，avg < 3.2 | 均分，保留 1 位小数 | 兼容问题偏多 |
| 无有效评价 | 显示 `-` | 暂无玩家兼容性反馈 |

**适配规则：**

| 场景 | 展示规则 | 交互规则 | 说明 |
| --- | --- | --- | --- |
| 横屏（ls） | 完整图标+评分+描述，卡片最大宽度 50% | 点击进入详情页，列表正常展示 | 默认完整展示 |
| 竖屏（pt） | 隐藏图标，评分 15px，星星 9×9px，描述截断单行 | 与横屏一致 | 竖屏空间受限，精简展示 |
| 无有效评价 | 评分位置显示「-」和空状态文案 | 仍可点击进入详情页提交反馈 | — |

### 4.3.2 兼容性评价详情页与反馈链路（C端）

功能 demo：兼容性评价详情页 demo

**页面结构：**

1. 顶部综合评分区：大字号评分、星级、描述文案、评价总数。
2. 中部分段占比区：1–5 星各段占比条形图。
3. 下部玩家评价列表：分页展示，含头像、昵称、评分、标签、文字内容、设备信息、发布时间、点赞数、「举报」文字按钮。

**评价列表展示规则：**

- 仅展示状态为「正常」的评价，已隐藏评价不展示，不计入综合评分。
- 设备信息：Mac 端显示「CPU 型号 · 内存GB」（如 `M2 · 16GB`）；移动端显示「设备型号 · 系统版本」（如 `iPhone 15 Pro · iOS 17`）；字段缺失时显示「-」。
- 每条评价底部展示点赞数，登录用户可点赞；同一用户对同一评价限点赞一次，再次点击取消点赞。
- 举报入口为文字按钮「举报」，不使用旗子图标；举报后该按钮变为灰色不可点击状态。
- 评价文字超出时截断，提供展开操作。

**提交反馈流程：**

1. 未登录用户点击「我要反馈」，弹出登录引导弹窗，点击「去登录」进入登录流程。
2. 已登录用户点击「我要反馈」，打开反馈弹窗。
3. 弹窗字段：星级评分（1–5 星，必填）、问题标签（多选，可选）、评价文字（选填，最多 200 字）。
4. 设备信息由系统静默采集，不在弹窗中展示给用户。
5. 点击提交后，列表刷新，入口卡片综合评分同步更新。
6. 已提交过的用户再次进入，「我要反馈」按钮置灰不可点击，展示提示：您已提交过该游戏的兼容性反馈。

**举报评价流程：**

1. 登录用户在评价列表项点击「举报」文字按钮，弹出举报原因选择弹窗。
2. 选择举报原因后确认，前台对举报者屏蔽该评价（仅本人不可见），后台记录举报；「举报」按钮变为灰色不可点击。
3. 同一用户对同一评价限举报一次。

**异常状态处理：**

| 场景 | 展示方式 | 文案 |
| --- | --- | --- |
| 未登录点击「我要反馈」 | 弹出登录引导弹窗 | 请先登录后再提交反馈 |
| 已提交过反馈 | 「我要反馈」按钮置灰 | 您已提交过该游戏的兼容性反馈 |
| 评价列表为空 | 空状态图+文案 | 暂无玩家兼容性反馈 |
| 网络异常/接口失败 | Toast 提示 | 加载失败，请稍后重试 |
| 提交反馈失败 | Toast 提示 | 提交失败，请检查网络后重试 |
| 已举报过该评价 | Toast 提示 | 您已举报过该评价 |
| 设备信息采集失败 | 对应字段显示「-」 | — |

### 4.3.3 后台评价管理系统（B端）

功能 demo：兼容性评价后台 demo

**页面归属：** 后台「游戏管理」模块下新增「玩家兼容性评价管理」页面，含两个子视图：评价管理 / 举报处理。

| 模块 | 功能说明 | 字段与操作规范 |
| --- | --- | --- |
| 评价管理 | 展示所有玩家兼容性评价，支持按端/状态/游戏名筛选，支持隐藏/恢复操作 | **列表字段：** 游戏名称；用户信息（昵称+UID）；端（Mac端蓝色标签/移动端绿色标签，无字段显示「-」）；设备信息（Mac端：CPU·内存GB；移动端：设备型号·系统版本；缺失显示「-」）；评分（1–5星星级展示）；问题标签（多标签并排）；评价内容（超长截断可展开）；状态（正常/已隐藏）；发布时间（精确到分钟）；操作（隐藏/恢复）。**操作规则：** 隐藏：将评价状态改为「已隐藏」，前台不再展示，同步重新计算该游戏综合评分（不计入已隐藏评价）。恢复：将评价状态改回「正常」，前台重新展示，重新计入综合评分。 |
| 举报处理 | 展示所有玩家举报记录，支持处理/驳回操作，处理结果联动隐藏被举报评价 | **列表字段：** 举报时间（精确到分钟）；举报人信息（昵称+UID）；被举报评价（游戏名+评价内容截断）；举报原因（举报人选择的原因标签）；状态（待处理/已处理/已驳回）；操作（处理/驳回）。**操作规则：** 处理：将被举报评价隐藏，同时将该评价下其他未处理举报同步标记为「已处理」，结果统一记为「隐藏评价」。驳回：驳回该举报，评价保持原状态，举报标记为「已驳回」。误处理修正：管理员可对「已处理」举报重新操作，修正处理结果。 |

**筛选条件：**

- 端：全部 / Mac端 / 移动端
- 状态（评价管理）：全部 / 正常 / 已隐藏
- 状态（举报处理）：全部 / 待处理 / 已处理 / 已驳回
- 游戏名称：文本搜索

**「端」标签视觉规范：**

| 端类型 | 背景色 | 字色 |
| --- | --- | --- |
| Mac端 | #f0f5ff | #2f54eb |
| 移动端 | #f6ffed | #389e0d |

**兼容性说明：** 历史评价无 `clientType` 字段时，后台「端」列显示「-」，不报错，不要求补采。

## 4.4 非功能需求

| 需求类型 | 详细要求 |
| --- | --- |
| 性能 | 评价列表接口响应时间 ≤ 2 秒（正常网络环境）。 |
| 防重 | 评价提交具备重复提交防护，同一用户对同一游戏仅可提交一次；重复请求拒绝并提示。举报提交具备防重机制，同一用户对同一评价仅可举报一次。 |
| 数据采集 | 设备信息静默采集，不展示给用户，不需用户授权确认。Mac 端采集 CPU 型号+内存；移动端采集设备型号+系统版本。 |
| 后台分页 | 后台列表分页，每页默认展示 20 条。 |
| 审计追溯 | 评价隐藏/恢复操作需记录操作人和操作时间，供审计追溯。 |
| 兼容性 | 历史评价无 `clientType` 字段时不报错；旧版兼容性相关入口上线后需确认是否清除（待确认）。 |

# 五、交付设计（数据分析与埋点设计）

## 5.1 核心业务指标

**漏斗监控：**

详情页兼容评价卡片曝光（UV）→ 点击入口进入评价详情页（UV/PV）→ 点击「我要反馈」（UV）→ 提交成功（UV）→ 举报提交数

## 5.2 埋点事件表

| 事件ID | 事件名称 | 触发时机 | 关键参数 |
| --- | --- | --- | --- |
| compat_entry_click | 兼容性评价入口点击 | 用户点击游戏详情页兼容性评价入口卡片 | game_id, client_type, screen_orientation |
| compat_page_view | 兼容性评价页访问 | 用户进入兼容性评价详情页 | game_id, uid（登录时必填）, client_type |
| compat_feedback_submit | 兼容性反馈提交 | 用户点击提交完成评价 | game_id, uid, score, tags, client_type, has_text |
| compat_like_click | 评价点赞/取消点赞 | 用户点击评价列表项点赞按钮 | game_id, uid, target_feedback_id, action（like/unlike） |
| compat_report_submit | 评价举报提交 | 用户提交举报 | game_id, uid, target_feedback_id, reason |

## 5.3 埋点参数表

| 参数名 (Parameter) | 类型 | 是否必填 | 参数说明 | 枚举值 / 示例 |
| --- | --- | --- | --- | --- |
| game_id | string | 是 | 被评价的游戏唯一标识 ID | 1245620 |
| uid | string | 是（登录态） | 当前登录用户的唯一标识 ID | u882910 |
| client_type | string | 是 | 客户端类型 | mac, mobile |
| screen_orientation | string | 是 | 当前屏幕方向 | portrait（竖屏）, landscape（横屏） |
| score | int | 是 | 用户提交的评分 | 1, 2, 3, 4, 5 |
| tags | array | 否 | 用户选择的问题标签列表 | ["画面异常", "崩溃闪退"] |
| has_text | bool | 是 | 是否填写了评价文字内容 | true, false |
| target_feedback_id | string | 是 | 被点赞/举报评价的唯一标识 ID | feedback_00123 |
| action | string | 是 | 点赞操作类型 | like（点赞）, unlike（取消点赞） |
| reason | string | 是 | 举报人选择的举报原因标签 | "虚假评价", "广告垃圾" |
