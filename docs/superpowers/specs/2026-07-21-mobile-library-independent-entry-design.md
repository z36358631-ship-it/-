# 手游库独立入口调整设计

## 结论

盖世游戏 Mac 端不再把 iOS 应用管理并入“游戏库”。侧边栏在“游戏库”下方新增独立入口“手游库”，进入后展示“已安装 / IPA 资源”两个 Tab。现有 App 管理、IPA 来源、下载、安装、右键设置与 ACE 风险提醒全部保留。

## 信息架构

- 侧边栏顺序：探索、云游戏、数据、找游戏、游戏库、手游库、订单中心。
- 点击“游戏库”：只进入原 PC 游戏库，不显示“PC游戏库 / IOS应用库”一级 Tab。
- 点击“手游库”：侧边栏只高亮“手游库”，页面标题显示“手游库”。
- 手游库二级 Tab：已安装、IPA 资源；默认进入“已安装”，客户端可恢复上次选择。
- “手游库”是入口名称，首期仍承接当前已规划的 iOS App 与 IPA 管理能力，不增加应用类别筛选。

## 页面与交互

### 已安装

- 常驻“导入 IPA”“刷新”“设置”、网格/列表和搜索。
- 空库显示“导入 IPA”；搜索无结果只提示调整关键词。
- 保留 App 右键菜单、五页应用设置、卸载与数据清理流程。

### IPA 资源

- 保留来源树、添加来源、刷新、应用信息、排序、网格/列表和搜索。
- 初始无来源或无资源显示“添加来源”；搜索无结果不显示添加按钮。
- 保留下载、暂停、校验、安装、更新、失败与不兼容等状态。

### ACE 风险

- 本地导入解析后、远程资源安装前检测 `tersafe.framework`。
- 命中后提示可能存在账号风控和封号风险，并说明盖世游戏不承担因第三方 IPA 导致的账号封禁损失。
- 关闭或取消返回原流程；确认按 IPA SHA-256 留存，新包体重新提示。

## 路由与状态

- 移除页面内 `library=pc|ios` 平台切换状态；手游库只维护 `apps|resources` Tab 状态。
- 旧的合并式 iOS 入口或深链统一迁移到“手游库”，并保留原业务 Tab。
- 切换游戏库与手游库不取消正在进行的解析、下载、校验、安装任务。
- 搜索、来源、排序、视图和选中项按手游库页面规则保存，不与 PC 游戏库串状态。

## 文档与截图

- PRD 版本记录追加独立“手游库”入口调整，不改写历史版本记录。
- 所有当前方案、核心路径、功能表、埋点和飞书功能点改为独立入口口径。
- 重新生成真实交互截图，截图必须同时展示侧边栏“手游库”选中态、页面标题和当前 Tab。
- PRD 图片使用固定 Git 提交的 jsDelivr PNG 链接，并逐一检查 HTTP 200 和 `image/png`。

## 验证

- 自动检查侧边栏顺序、选中态、页面标题和二级 Tab。
- 自动检查页面不再出现“PC游戏库 / IOS应用库”一级切换。
- 复跑导入、来源、状态、右键菜单、空状态和 ACE 完整流程。
- 检查 JavaScript 页面错误、关键截图可读性和线上 Pages 文案。

## 三视角评审结论

- 用户视角：独立入口更容易发现；“IPA 资源”空状态需要保留明确的添加来源动作。
- 交互视角：必须只高亮“手游库”，截图完整展示入口、标题和当前 Tab。
- 开发视角：必须移除合并路由残留，并保证跨入口切换不影响后台下载与安装任务。
