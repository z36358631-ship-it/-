# Mac 租号确认订单价格布局调整设计

## 1. 背景与目标

当前确认订单页存在三个信息层级问题：游戏原价位于左侧游戏卡片中，与右侧订单金额割裂；“首单5折”标签文字对比度不足；标准租价显示在实付价左侧，不符合本次确认的阅读顺序。

本次调整只优化确认订单页的价格信息结构，不改变首单资格、折扣算法、套餐价格、支付流程或订单金额字段。

## 2. 已确认布局

### 2.1 左侧游戏信息

- 游戏卡片只展示封面和游戏名称。
- 移除游戏卡片内的游戏原价。

### 2.2 右侧价格区域

价格区域自上而下分为两行，标题列和金额列固定对齐：

1. 游戏原价：显示 `游戏原价 ¥198`。
   - 位于“订单金额”上方。
   - “游戏原价”标题使用白色文字。
   - 金额使用普通灰色文字，不使用删除线。
   - 原价金额与下方实付金额的右边缘垂直对齐。
2. 订单金额：标题显示 `订单金额`，首单场景在标题右侧紧跟 `首单5折` 标签。
   - “订单金额”标题使用白色文字。
   - 标签使用不透明绿色背景、白色文字和清晰字重。
   - 删除标题下方“2小时 · 已按当前最优单价计算”等计价说明副文案。
   - 右侧价格顺序固定为：红色放大的实付价 → 灰色标准租价。
   - 标准租价只显示金额 `¥3.4`，不展示“标准租价”文案；金额使用删除线并位于实付价右侧。

## 3. 订单状态规则

四种确认订单状态共用同一套价格区域结构，通过统一状态控制优惠元素显隐：

| 状态 | 游戏原价 | 首单5折标签 | 标准租价删除线 | 应付金额 |
|---|---|---|---|---|
| 首次新租 | 展示 | 展示 | 展示，位于实付价右侧 | 5折实付价，红色放大 |
| 非首单新租 | 展示 | 隐藏 | 隐藏 | 标准租价，红色放大 |
| 续租 | 展示 | 隐藏 | 隐藏 | 续租应付金额，红色放大 |
| 重新租用 | 展示 | 隐藏 | 隐藏 | 重新租用应付金额，红色放大 |

游戏原价只用于展示游戏本体的市场参考价，不参与租赁折扣计算。首单5折仍以标准租价为基准，订单 `standardAmount` 和 `amount` 的现有含义保持不变。

## 4. 视觉规格

- `游戏原价` 和 `订单金额`：标题均使用白色文字。
- `首单5折`：不透明绿色底、白色文字，字号不小于 10px，字重不低于 700，保证在当前缩放比例下可读。
- 实付价：保持红色和当前大字号层级。
- 标准租价：灰色、小于实付价，只显示金额并使用删除线。
- 游戏原价金额：普通灰色、不划线，字号低于订单实付价，与实付价右边缘对齐。
- 实付价与标准租价在同一基线排列，标准租价位于实付价右侧。

## 5. 交互与数据

- 套餐、时长或版本变化时，标准租价、实付价和单位价格继续实时刷新。
- 首单资格变化时，只切换标签和标准租价删除线的展示，不改变游戏原价节点。
- 支付确认弹层和最终订单金额继续使用实际应付金额，不新增价格计算分支。

## 6. 验收标准

- AC1：左侧游戏卡片中不存在游戏原价。
- AC2：`游戏原价` 和 `订单金额` 标题均为白色；`¥198` 使用普通灰色且无删除线，位于实付价正上方并按右边缘对齐。
- AC3：首单场景中，`首单5折` 紧跟在“订单金额”文字旁，使用绿色底白色文字且清晰可读。
- AC4：首单实付价位于价格组左侧并红色放大；标准租价在其右侧只显示划线金额，不显示“标准租价”文案。
- AC5：非首单、续租和重新租用均不展示首单标签及标准租价删除线。
- AC6：四种状态的应付金额、支付确认金额和订单 `amount` 保持一致。
- AC7：确认订单截图重新生成后，无文字重叠、裁切或低对比度问题。
- AC8：价格区不展示“2小时 · 已按当前最优单价计算”或同类计价说明副文案。

## 7. 范围边界

本次不修改折扣比例、首单资格来源、游戏原价数据来源、套餐定价、支付逻辑、订单状态机、其他页面价格展示或后台运营逻辑。
