## 1. 需求概述

### 1.1 背景

macOS 26 取消旧版全屏 Launchpad 后，用户仍可通过 Spotlight、访达或程序坞启动应用，但失去了全屏大图标集中浏览和按既有习惯启动应用的入口。盖世游戏 Mac 客户端当前也缺少一个面向本地游戏优先、同时覆盖本机应用的集中启动入口。

本功能定位为“盖世全屏启动台”：用户已进入盖世客户端后，可在 macOS 原生全屏空间中浏览、搜索并启动本地游戏或其他 Mac 应用。它不是系统级 Launchpad 替代品，也不是盖世商城、云游戏或租号内容页。

### 1.2 用户问题

1. 升级 macOS 26 后，习惯使用旧 Launchpad 的用户需要在多个入口之间寻找应用。
2. 本地游戏与普通应用混在系统应用列表中，缺少“游戏优先”的集中视图。
3. 盖世客户端已有本地游戏信息，但没有形成“进入盖世—浏览本机游戏—启动游戏—回到启动台”的连续闭环。

### 1.3 产品目标

| 目标 | 成功判断 |
|---|---|
| 提供可发现的集中启动入口 | macOS 26 用户可从盖世左侧一级导航进入启动台 |
| 降低寻找本地游戏的成本 | 默认显示可直接启动的本地游戏，并支持搜索与键盘导航 |
| 保持连续启动体验 | 游戏全屏启动，结束或返回后重新显示启动台 |
| 保证本地能力可用性 | 无服务端依赖；部分目录扫描失败时仍可浏览缓存和其他目录 |

### 1.4 产品验证指标

| 指标 | 口径 | 首期观察目标 |
|---|---|---|
| 启动台渗透率 | 使用启动台的 macOS 26 日活用户数 / macOS 26 日活用户数 | 上线后建立基线，不预设虚假增长值 |
| 7 日重复使用率 | 首次使用后 7 日内再次进入启动台的用户占比 | 上线后建立基线 |
| 游戏启动成功率 | 启动成功次数 / 发起启动次数 | ≥ 98%（排除应用损坏和系统明确拒绝） |
| 启动台到游戏耗时 | 进入启动台至目标应用确认置前 | 统计 P50、P95 |
| 搜索无结果率 | 无结果搜索次数 / 搜索次数 | 用于校验扫描覆盖和命名匹配质量 |
| 人工分类纠正率 | 添加到游戏或从游戏移除的应用数 / 被展示应用数 | 用于校验自动识别质量 |

### 1.5 目标用户与场景

**核心用户：** 已升级 macOS 26、安装盖世游戏 Mac 客户端、希望集中浏览本机游戏和应用的用户。

**场景 A｜正常启动游戏：** 用户在盖世左侧点击“启动台”，系统进入全屏空间并默认显示游戏列表；用户点击一款本地游戏，当前卡片显示启动中；系统确认游戏启动后将游戏全屏置前；用户结束游戏后回到启动台原会话。

**场景 B｜寻找未被识别的游戏：** 用户在“游戏”中没有找到目标应用，切换到“全部应用”，搜索应用名称，通过右键“添加到游戏”完成纠正；以后进入“游戏”即可看到该应用。

**场景 C｜启动失败：** 用户点击已损坏或受 Gatekeeper 限制的应用，启动台保留在当前页面，卡片恢复可操作，页面展示失败原因和重试入口；用户可重试、启动其他应用或退出启动台。

## 2. 范围

### 2.1 本期范围

- 盖世客户端左侧一级导航“启动台”；
- macOS 原生全屏空间；
- “游戏”和“全部应用”两个分类；
- 标准应用目录扫描、去重、图标读取和本地缓存；
- 游戏识别及用户人工纠正；
- 当前分类实时搜索、纵向滚动和键盘导航；
- 启动未运行应用、激活已运行应用；
- 启动成功、启动失败、首次扫描、空状态和部分失败降级；
- 游戏结束或返回后回到启动台；
- 本地行为埋点上报到盖世现有数据体系。

### 2.2 本期不做

- macOS 25 及以下版本入口；
- 系统级快捷键、菜单栏入口、系统触控板手势或独立 Launcher 应用；
- 拖拽排序、横向分页、文件夹、隐藏、重命名和卸载；
- 云游戏、租号游戏、PlayCover 游戏、未安装游戏、推荐内容和广告；
- 外接磁盘和用户自定义扫描目录；
- 账号同步、云端应用库、服务端业务接口和运营后台。

## 4. 公共业务规则

| 编号 | 规则 |
|---|---|
| BR-01 | 仅 macOS 26 及以上版本显示启动台入口，低版本完全隐藏。 |
| BR-02 | 启动台只展示本机可直接启动的 `.app`，不混入商城、云游戏、租号、网页链接、未安装内容或广告。 |
| BR-03 | 应用唯一标识优先使用 Bundle ID；缺失时使用规范化绝对路径。 |
| BR-04 | 游戏分类优先级为：用户人工分类 ＞ 盖世本地游戏库匹配 ＞ 应用分类或已知游戏标识 ＞ 仅进入全部应用。 |
| BR-05 | 同一时间只允许一个启动请求；请求结束前其他应用不得再次发起启动。 |
| BR-06 | 启动成功不销毁启动台；启动台保留在后层，游戏结束或返回后重新显示启动台。 |
| BR-07 | 搜索有内容时，首次按 Esc 只清空搜索；搜索为空时按 Esc 才退出启动台。 |
| BR-08 | 人工分类不得被重新扫描覆盖；应用卸载后清理对应缓存和人工分类。 |
| BR-09 | 本期全部业务数据保存在本机，不新增运营后台和业务服务端。 |

## 5. 核心流程与状态机

```text
盖世客户端
→ 点击启动台
→ 进入原生全屏空间
→ 加载缓存 / 首次扫描
→ 游戏（默认）或全部应用
→ 浏览、搜索、人工纠正分类
→ 单击应用
   ├─ 已运行：激活并全屏置前
   ├─ 未运行：显示启动中 → 启动成功 → 全屏置前
   └─ 失败：留在启动台 → 恢复卡片 → 展示原因与重试
→ 游戏结束或返回
→ 回到启动台
→ 启动台内按 Esc
→ 返回盖世客户端
```

| 当前状态 | 触发 | 下一状态 |
|---|---|---|
| 盖世客户端 | 点击“启动台” | 全屏切换中 |
| 全屏切换中 | 原生全屏成功 | 扫描中或正常列表 |
| 全屏切换中 | 原生全屏失败 | 盖世客户端并提示失败 |
| 正常列表 | 点击应用 | 启动中 |
| 启动中 | 启动或激活成功 | 目标应用全屏运行 |
| 启动中 | 失败或超时 | 启动失败并留在列表 |
| 目标应用全屏运行 | 游戏结束、关闭或主动返回 | 启动台正常列表 |
| 正常列表 | 搜索为空时按 Esc | 盖世客户端 |

## 6. 功能需求

### F001 版本判断与启动台入口

**输入：** 盖世客户端启动或主导航渲染，系统提供当前 macOS 主版本号。

**处理：**

1. 判断系统版本是否为 macOS 26 或以上；
2. 满足条件时，在现有左侧一级导航中展示“启动台”；
3. 不满足条件时不渲染入口，不展示置灰态或升级提示。

**输出：** 入口与探索、找游戏、游戏库等一级导航同层级，使用九宫格/启动台语义图标。

**异常：** 无法读取系统版本时按不满足条件处理，并记录本地错误日志。

**端侧职责：** 仅 Mac 客户端；服务端和运营后台不涉及。

### F002 原生全屏启动台容器

**输入：** 用户点击 F001 入口。

**处理：**

1. 在盖世客户端当前所在显示器创建或激活原生全屏窗口与独立 Space；
2. 首次进入默认选中“游戏”，搜索词为空，滚动位置为顶部；
3. 同一次全屏会话内分别保存“游戏”和“全部应用”的滚动位置；
4. 退出并重新进入后重置为“游戏”顶部。

**输出：** 全屏启动台使用盖世 Mac 深色视觉；游戏页四列横版封面，全部应用页六列大图标。

**异常：** 原生全屏创建失败时留在盖世客户端，提示“暂时无法打开启动台，请稍后重试”，不得出现空白全屏窗口。

**开发基准：** Demo 内层基准画布为 `1076×734`；真实客户端必须响应当前显示器分辨率和安全区，不固定为 Demo 尺寸。

### F003 本地应用扫描、去重与缓存

**输入：** 首次进入启动台、应用安装/删除通知或后续增量刷新。

**扫描目录：**

- `/Applications`；
- `~/Applications`；
- `/System/Applications`；
- `/System/Applications/Utilities`。

**处理：**

1. 读取可启动 `.app` 的名称、Bundle ID、规范化路径和图标；
2. 过滤 `.app` 包内部的 Helper、Updater、Renderer、Crash Handler 等辅助进程；
3. Bundle ID 相同时保留标准目录中可启动的优先版本；
4. Bundle ID 缺失时以规范化绝对路径作为临时主键；
5. 同名但 Bundle ID 不同的应用保留为独立条目；
6. 有缓存时先显示缓存并在后台增量扫描，无缓存时展示首次扫描状态；
7. 扫描结果写入本地缓存并更新时间戳。

**输出：** 去重后的可启动应用集合。

**异常：** 单个目录不可访问时跳过该目录并继续；图标读取失败时使用系统通用图标；扫描中断时继续使用上次缓存。

### F004 游戏与全部应用内容组织

**输入：** F003 应用集合、盖世本地游戏库匹配结果、应用分类信息和人工分类记录。

**处理：** 按 BR-04 计算游戏分类；无法确定的应用只进入“全部应用”。

**输出：**

- “游戏”默认页：只展示可直接启动的本地游戏，使用四列 `16:9` 横版封面；封面下方展示游戏名、分类/平台和可用评分；
- “全部应用”：展示 F003 的全部应用，使用六列大图标和应用名称；
- 两个分类均纵向连续滚动，不做横向分页。

**异常：** 游戏为空时展示“还没有可启动的游戏”和“前往全部应用添加游戏”；封面读取失败时使用带游戏名的通用占位封面。

### F005 浏览、搜索与键盘导航

**输入：** 鼠标、触控板、键盘和搜索框输入。

**处理与输出：**

| 操作 | 系统行为 |
|---|---|
| 鼠标或触控板纵向滚动 | 浏览当前分类内容 |
| 点击“游戏/全部应用” | 即时切换分类，并恢复该分类在当前会话的滚动位置 |
| 输入搜索词 | 不区分大小写实时过滤当前分类，不需按回车 |
| Command + F | 聚焦搜索框 |
| 方向键 | 按当前网格列数移动卡片焦点 |
| Enter | 启动当前焦点应用 |
| Tab | 在分类、筛选、搜索和卡片区域之间移动焦点 |
| Esc（有搜索词） | 清空搜索并保留当前分类 |
| Esc（无搜索词） | 退出启动台并返回盖世客户端 |

**异常：** “游戏”搜索无结果时提示可切换到全部应用；不自动替用户切换分类。

### F006 游戏分类人工纠正

**输入：** 用户在“全部应用”右键或 `Control + 单击`应用。

**处理：**

1. 当前不是游戏时只显示“添加到游戏”；
2. 当前是游戏时只显示“从游戏中移除”；
3. 点击后立即更新当前视图并写入本地人工分类；
4. 从游戏移除只影响“游戏”，不得从“全部应用”删除；
5. 重新扫描遵循 BR-08，不覆盖人工分类。

**输出：** 分类结果立即生效。

**异常：** 应用在操作过程中被卸载时关闭菜单、移除条目并提示“应用已不存在”。右键菜单不提供卸载、隐藏、重命名、排序或文件夹功能。

### F007 启动、激活与返回启动台

**输入：** 单击应用卡片或在焦点卡片上按 Enter。

**处理：**

1. 按 BR-05 锁定本次启动请求，当前卡片显示启动中；
2. 应用已运行时激活已有进程和窗口，不重复拉起；
3. 应用未运行时调用系统能力启动；
4. 系统确认目标应用已置前后，将目标游戏全屏显示；
5. 启动台不关闭，保留在后层；
6. 目标游戏结束、关闭或用户主动返回后，恢复启动台“游戏”页；
7. 点击“盖世游戏”自身图标时退出启动台并回到原客户端窗口。

**输出：** 形成“启动台 → 全屏游戏 → 返回启动台”的闭环。

**异常：** 启动失败、系统拒绝或超时后释放启动锁，恢复卡片，留在启动台并展示原因及重试入口。

**关键约束：** 返回路径不得跳到盖世客户端首页；只有退出启动台的操作才返回客户端。

### F008 状态、异常与降级

| 状态/异常 | 展示 | 用户可执行操作 |
|---|---|---|
| 首次扫描 | 骨架屏＋“正在查找应用” | 等待或 Esc 退出 |
| 后台增量更新 | 保留现有列表，不阻塞浏览 | 正常浏览和启动 |
| 游戏为空 | 空状态说明＋前往全部应用 | 切换全部应用 |
| 搜索无结果 | 当前分类无结果说明 | 修改搜索或切换分类 |
| 启动中 | 当前卡片进度态，阻止重复请求 | 等待结果 |
| 启动失败 | 页内原因＋重试 | 重试、启动其他应用或退出 |
| 应用已删除 | 从列表移除 | 选择其他应用 |
| 部分目录失败 | 继续展示其他目录和旧缓存 | 正常使用 |
| 全屏进入失败 | 留在盖世客户端并提示失败 | 稍后重试 |

### F009 本地数据与行为埋点

**本地数据：** 每个应用至少保存名称、Bundle ID、规范化路径、图标缓存标识、是否可启动、自动分类结果与来源、人工分类结果、最近扫描时间和最后发现时间。

**服务端/运营后台：** 不新增业务服务端和运营后台；行为数据复用盖世现有埋点上传链路。

**埋点事件：**

| 事件 | 触发时机 | 关键字段 | 分析目的 |
|---|---|---|---|
| `launcher_entry_click` | 点击启动台入口 | `os_version`, `source_nav` | 入口渗透率 |
| `launcher_ready` | 启动台可操作 | `has_cache`, `ready_duration_ms`, `app_count`, `game_count` | 进入性能与内容覆盖 |
| `launcher_category_switch` | 切换分类 | `from_category`, `to_category` | 分类使用分布 |
| `launcher_search` | 搜索结果刷新 | `category`, `keyword_length`, `result_count` | 搜索使用率与无结果率 |
| `launcher_game_override` | 人工纠正分类 | `bundle_id_hash`, `action`, `previous_source` | 自动识别质量 |
| `launcher_app_launch` | 发起启动 | `bundle_id_hash`, `category`, `is_running` | 启动漏斗 |
| `launcher_app_launch_result` | 启动成功或失败 | `bundle_id_hash`, `result`, `error_code`, `duration_ms` | 成功率与失败原因 |
| `launcher_return` | 游戏结束后回启动台 | `bundle_id_hash`, `run_duration_sec` | 闭环完成率 |
| `launcher_exit` | 退出启动台 | `exit_method`, `session_duration_sec` | 退出路径与使用时长 |

**隐私要求：** 不上报应用绝对路径和原始 Bundle ID；使用稳定哈希值用于聚合分析。

## 7. 视觉与交互规格

| 区域 | 规格 |
|---|---|
| 启动台背景 | 盖世 Mac 深灰色，低饱和描边和克制高亮 |
| 左侧启动台导航 | Demo 基准宽 `82px`，悬浮圆角容器 |
| 游戏网格 | 四列，封面比例 `16:9`，列间距 `16px`，圆角 `12px` |
| 全部应用网格 | 六列大图标，名称最多两行，悬停显示完整名称 |
| 搜索框 | 工具栏右侧，输入即时过滤 |
| 全屏游戏 | 占满当前显示器；Demo 中占满中间演示区，不显示窗口外边距 |
| 返回 | 游戏结束/主动返回后显示启动台；启动台按 Esc 才返回盖世客户端 |

视觉以交互 Demo 为开发基准；真实实现需适配当前显示器分辨率、缩放比例、安全区和多显示器环境。

## 8. 非功能需求

| 类别 | 要求 |
|---|---|
| 性能 | 有缓存进入可操作 P95 ≤ 500ms；首次扫描首批结果 P95 ≤ 1.5s；搜索刷新 P95 ≤ 100ms |
| 流畅度 | 200 个应用下滚动和键盘焦点移动无明显阻塞；目标 60fps，最低不连续低于 45fps |
| 稳定性 | 单目录扫描失败不得导致启动台不可用；连续 100 次进入/退出无残留全屏窗口 |
| 兼容性 | macOS 26 及以上；Intel/Apple Silicon 的支持范围与盖世客户端主版本一致 |
| 可访问性 | 所有卡片可键盘聚焦；焦点环清晰；文本对比度不低于 WCAG AA 建议值 |
| 隐私 | 应用列表和人工分类仅本地保存；埋点不上传绝对路径和原始 Bundle ID |
| 日志 | 记录扫描目录失败、去重冲突、启动结果、全屏进入失败；日志不得包含用户私有路径明文 |

## 9. 原生技术预研

正式排期前必须完成技术 Spike，并输出可运行验证结果：

1. 标准目录扫描及系统应用读取权限；
2. Bundle ID 缺失、重复和多副本处理；
3. 图标读取、横版封面来源和缓存性能；
4. 盖世本地游戏库与本地应用的匹配方式；
5. 通过 `NSWorkspace` 冷启动应用；
6. 激活已运行应用而不是重复拉起；
7. 启动失败、系统拒绝和超时结果获取；
8. 原生全屏 Space、目标应用置前与返回启动台的时序；
9. 多显示器下使用盖世窗口所在显示器；
10. 安装、删除和路径变化后的增量更新。

技术预研未通过的能力不得仅凭 HTML Demo 判定为可开发完成。

## 10. 验收标准

### F001

- `AC-F001-01` `[客户端]` macOS 26 及以上左侧显示“启动台”，低版本不显示。
- `AC-F001-02` `[客户端]` 点击入口不在普通内容区打开页面，而是进入全屏流程。

### F002

- `AC-F002-01` `[客户端]` 从盖世当前所在显示器进入原生全屏空间。
- `AC-F002-02` `[客户端]` 首次进入默认显示“游戏”顶部，搜索词为空。
- `AC-F002-03` `[客户端]` 同一会话切换分类后可恢复各自滚动位置。
- `AC-F002-04` `[客户端]` 全屏进入失败时留在客户端且无空白残留窗口。

### F003

- `AC-F003-01` `[客户端]` 扫描四个指定目录且不展示 Helper 等辅助进程。
- `AC-F003-02` `[客户端]` Bundle ID 相同的应用不重复展示；缺失 Bundle ID 时仍可展示和启动。
- `AC-F003-03` `[客户端]` 单个目录无权限时其他目录和旧缓存仍可使用。
- `AC-F003-04` `[客户端]` 有缓存进入可操作 P95 ≤ 500ms，首次扫描首批结果 P95 ≤ 1.5s。

### F004

- `AC-F004-01` `[客户端]` 游戏页仅展示本机可直接启动的游戏，不出现商城、云游戏、租号和广告。
- `AC-F004-02` `[客户端]` 游戏页为四列 `16:9` 封面；全部应用为六列大图标。
- `AC-F004-03` `[客户端]` 游戏为空时可一键进入全部应用添加游戏。

### F005

- `AC-F005-01` `[客户端]` 搜索只过滤当前分类，P95 刷新时间 ≤ 100ms。
- `AC-F005-02` `[客户端]` 方向键、Enter、Tab 和 Command + F 行为符合功能表。
- `AC-F005-03` `[客户端]` 有搜索词时 Esc 只清空搜索，无搜索词时 Esc 才退出启动台。

### F006

- `AC-F006-01` `[客户端]` 全部应用中右键或 Control + 单击可添加到游戏或从游戏移除。
- `AC-F006-02` `[客户端]` 人工分类立即生效，重新扫描和重启客户端后仍保留。
- `AC-F006-03` `[客户端]` 从游戏移除不影响全部应用中的条目。

### F007

- `AC-F007-01` `[客户端]` 已运行应用被激活，未运行应用被启动，同一时间只有一个启动请求。
- `AC-F007-02` `[客户端]` 启动成功后目标游戏全屏置前，启动台保留在后层。
- `AC-F007-03` `[端到端]` 结束、关闭或主动返回游戏后显示启动台，不跳到盖世首页。
- `AC-F007-04` `[客户端]` 启动失败后卡片恢复、启动台保留，并提供原因和重试。

### F008

- `AC-F008-01` `[客户端]` 首次扫描、游戏为空、搜索无结果、启动中和启动失败均有明确状态。
- `AC-F008-02` `[客户端]` 部分扫描失败和图标读取失败时启动台仍可使用。
- `AC-F008-03` `[客户端]` 应用卸载后条目被移除，不保留不可点击的失效卡片。

### F009

- `AC-F009-01` `[客户端]` 本地缓存和人工分类不依赖网络即可读取。
- `AC-F009-02` `[数据平台]` 表中九类事件均能按定义触发，必填字段完整。
- `AC-F009-03` `[端到端]` 埋点不包含应用绝对路径和原始 Bundle ID。

## 11. 发布与回滚

1. 首期仅随支持 macOS 26 的盖世 Mac 版本发布，不设置服务端运营开关；如客户端已有本地实验框架，可先向内部和灰度用户开放。
2. 上线前完成原生技术预研、功能验收、100 次连续进入/退出稳定性测试和多显示器测试。
3. 若出现全屏窗口残留、应用重复拉起或大面积扫描失败，版本回滚或通过客户端版本策略隐藏入口；本地缓存格式需向后兼容或可安全重建。
