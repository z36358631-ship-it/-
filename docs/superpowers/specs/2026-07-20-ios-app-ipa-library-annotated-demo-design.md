# 盖世游戏 Mac 端 iOS 应用与 IPA 资源库标注版 Demo 设计

> 日期：2026-07-20
> 状态：设计已确认，进入实现
> 原始 Demo：`Mac端demo/mac端租号功能/盖世游戏Mac端-iOS应用与IPA资源库demo.html`
> 目标文件：`Mac端demo/mac端租号功能/盖世游戏Mac端-iOS应用与IPA资源库demo-标注版.html`

## 1. 目标

在完整保留现有 iOS 应用与 IPA 资源库业务交互的基础上，生成一个供产品、设计、开发和验收人员使用的单文件交互标注版。

标注版需要让评审者同时完成三件事：

1. 按真实路径操作 Demo。
2. 快速跳转到关键页面和中间状态。
3. 查看交互规则、全局约束和异常边界，并定位到对应界面元素。

## 2. 方案选择

采用“新建独立标注版单文件”方案：

- 原 Demo 不修改、不覆盖。
- 标注版包含三栏框架、完整业务 Demo、标注数据和联动逻辑。
- 不使用 iframe，不依赖运行时加载原 Demo。
- 不拆分 CSS、JavaScript 或标注数据文件。

未采用方案：

- 直接修改原 Demo：会增加业务回归风险，且无法保留无标注版本。
- iframe 引用原 Demo：不满足单文件交付要求，页面状态与标注联动也更复杂。

### 3.1 页面外层

- 页面背景：`#12141e`

- 左侧和右侧独立滚动，中间只负责展示 Demo 舞台。

### 3.2 中间高保真舞台

- 业务舞台逻辑尺寸固定为 `1440 × 900px`，作为一张完整 macOS 桌面画布，而不是直接铺满一个普通网页应用。
- 舞台顶部绘制 `28px` macOS 系统状态栏；左侧显示 Apple 标识与当前应用名称，右侧显示网络、电量和时间等状态信息。
- 桌面背景使用用户参考图中的紫红至深蓝渐变，并叠加低对比度柔光，禁止使用纯色背景代替。
- 盖世客户端窗口在桌面中水平居中，逻辑尺寸为 `1240 × 780px`，顶部距舞台 `72px`；窗口使用 `16px` 圆角、深色半透明材质、细描边和大范围柔和阴影。
- 客户端顶部保留 `44px` 窗口标题栏，左侧使用 macOS 红、黄、绿三色窗口控制按钮，颜色、直径和间距保持系统视觉比例。
- 客户端标题栏、业务侧栏与内容区属于同一窗口，圆角和裁切保持连续，不能呈现为三个独立卡片。
- 客户端内容画布保持桌面端固定排版，不因标注框架可用宽度变化而重排。
- 使用 `ResizeObserver` 监听中间可用尺寸。
- 缩放比例为宽度比例与高度比例中的较小值，并设置合理下限。
- `1440 × 900px` 桌面舞台始终居中并整体等比缩放；空间不足时允许中间区域横向滚动，不压缩业务卡片，也不单独缩小客户端侧栏。
- 右侧面板折叠或展开后立即重新计算缩放。
- 业务弹窗、Toast、批量操作栏全部限制在舞台内部。

### 3.3 高保真复用原则

- 客户端外壳、标题栏、业务侧栏、按钮、输入框、卡片、弹窗和状态反馈均按原盖世 Mac Demo 的 CSS 参数复用，不重新设计一套通用后台风格。
- 原有组件参数能直接复用时采用原值；只有本次新增的“iOS 应用”页面和状态组件允许补充样式。
- 新增组件优先复用原客户端的颜色、圆角、边框、字号、间距和交互状态，确保程序可直接按标注开发，无需 UI 再次翻译。
- 图标统一使用与原 Demo 相同的线性图标体系，描边粗细、画布尺寸和激活状态一致；禁止以文字字母或 Emoji 代替正式图标。
- 视觉验收采用同尺寸截图叠加比对：原有客户端外壳与业务侧栏不得出现可感知的结构偏移，新增内容只出现在明确新增的位置。

- 进入“iOS 应用”后，侧栏保持固定，右侧内容区显示本次新增的模块标题、“我的 App / IPA 资源”二级 Tab 与完整业务流程。

- 首次选择目录、来源校验、安装确认等弹层必须呈现在客户端窗口内部，遮罩只覆盖客户端内容，不遮挡 macOS 桌面、外层标注导航或右侧标注面板。
- “我的 App”工具栏常驻盖世绿色主按钮“导入 IPA”，无论当前 App 库为空、已有 App、存在可更新项或搜索无结果都不能隐藏。
- “导入 IPA”与“更改文件夹”是两个独立操作：“导入 IPA”选择本地 `.ipa` 文件；“更改文件夹”只修改唯一 App 库的存储路径，文案、图标和事件不得混用。
- 工具栏右侧操作顺序固定为“导入 IPA / 重新扫描 / 在访达中显示 / 更改文件夹”，主次层级以“导入 IPA”为最高。

### 5.4 导入 IPA 选择确认弹窗（复用原 Steam 游戏库母版）

导入弹窗不使用当前 Demo 的普通不透明深色表单样式，必须以用户提供的原“Steam 游戏库”导入弹窗为视觉和布局母版：

- 客户端内容区先覆盖深色半透明遮罩并进行背景模糊，弹窗本体使用大圆角玻璃材质、低对比度描边和柔和阴影。
- 弹窗头部左侧依次展示返回箭头、标题“导入 IPA”和当前 App 库路径；返回箭头、标题、路径沿用原 Steam 导入弹窗的字号、间距和弱化层级。
- 右上角只放一个“全选 / 取消全选”状态按钮：存在未选中的可导入项时显示“全选”，所有可导入项均已选中时显示“取消全选”。
- “全选 / 取消全选”不得放在底部操作区，也不得同时出现两个按钮；已导入、文件损坏等禁用项不计入可选总数。
- 列表行沿用原弹窗横向布局：选择框、App 图标、名称与版本/体积、右侧状态。可导入项允许勾选，已导入和损坏项置灰且不能勾选。
- 底部左侧只展示选择统计，格式为“已选 X / Y”；`Y` 表示当前可导入项数量。
- 底部右侧仅保留“取消”和“导入”两个操作。“导入”在 `X=0` 时置灰，`X>0` 时使用盖世主操作绿色。
- 导入说明放在标题区与列表之间，但视觉层级弱于列表，不使用当前 Demo 中占满整行的黄色提示卡抢占注意力。

### 5.5 添加 IPA 来源弹窗

- 弹窗外观继续复用 5.4 的玻璃材质、圆角、描边、关闭按钮和底部操作区，不单独使用另一套普通表单风格。
- 业务弹窗内部只包含标题与说明、来源地址输入框、固定高度校验反馈区，以及底部“取消 / 导入”按钮。
- 用户停止输入 `500ms` 后执行校验；校验期间显示“正在验证来源…”，反馈区高度保持固定，禁止因文案变化造成弹窗抖动。
- 地址不是有效来源、返回内容不是合法 JSON、目标资源不存在或无法识别来源结构时，统一展示准确文案：`链接无效，JSON无效或未找到！`
- 校验失败时反馈区使用红色错误状态，“导入”保持置灰；校验成功时展示来源名称和可识别资源数，并启用“导入”。
- 校验成功后的“导入”仅代表进入下一步资源选择，不直接保存来源，也不直接把全部资源写入 IPA 资源库。
- “使用 AppHub 示例”“使用 Mirror 示例”“演示网络失败”等按钮属于 Demo 状态复现工具，不是正式产品能力，必须全部移出业务弹窗容器。
- 上述示例入口放入外层标注框架左侧的“快捷演示状态”区域；点击后自动打开来源弹窗并填充对应状态，正式客户端弹窗中不显示任何示例按钮。

### 5.6 来源资源选择弹窗

用户在有效来源弹窗中点击“导入”后，打开第二级“选择要导入的 IPA”弹窗，由用户自行决定导入哪些资源：

- 弹窗继续复用 5.4 的原 Steam 游戏库导入母版，不能直接跳回资源列表或显示导入成功。
- 头部左侧展示返回箭头、标题“选择要导入的 IPA”和来源名称/地址；点击返回回到“添加 IPA 来源”，保留已验证的 URL 与成功状态。
- 右上角显示单一“全选 / 取消全选”按钮，位置和状态逻辑与“导入 IPA”选择确认弹窗一致。
- 初次进入时默认不勾选任何资源，由用户主动逐项选择或点击“全选”，避免无意识导入整个来源。
- 资源行展示选择框、App 图标、名称、版本、体积、兼容状态和来源信息；不兼容、元数据缺失或重复资源置灰并说明原因，不计入可选总数。
- 底部左侧显示“已选 X / Y”；底部右侧只显示“取消”和“导入”。`X=0` 时“导入”置灰，`X>0` 时启用。
- 只有用户在本弹窗点击最终“导入”后，才保存来源配置并将所选资源加入 IPA 资源库；未选资源不进入当前资源列表。
- 取消第二级弹窗时不保存来源、不导入资源；返回上一级时保留校验结果，用户可修改 URL 后重新验证。
- 已保存来源后，用户仍可从“来源管理”再次进入资源选择，补充导入此前未选择的资源。

### 5.7 本地 IPA 导入

- 用户点击“我的 App”中的常驻“导入 IPA”后，正式客户端拉起 macOS 原生文件选择器，筛选并只允许选择 `.ipa` 文件。
- 文件选择器支持一次选择一个或多个 `.ipa`；取消选择后保持当前页面和列表不变，不显示错误。
- 若尚未设置 App 库路径，点击“导入 IPA”后先进入唯一 App 库路径选择；路径设置成功后自动恢复未完成动作并拉起 IPA 文件选择器，用户不需要再次点击按钮。
- 文件选择完成后进入导入确认弹窗，沿用 5.4 的玻璃弹窗与选择布局，展示每个 IPA 的名称、版本、体积、Bundle ID 与可导入状态。
- 可导入项由用户主动勾选或全选；重复 App、不兼容文件、损坏文件和非 IPA 文件置灰并说明原因。
- 最终确认后，将所选 IPA 安装到当前 App 库路径并出现在“我的 App”；不扫描或导入用户另选的普通文件夹内容。
- App 库中已经存在其他 App 时流程完全相同，“导入 IPA”按钮不得因列表非空而消失或变为“导入文件夹”。

### 5.8 Mac 应用程序式图标网格

“我的 App”和“IPA 资源”不再使用网页商城式大卡片，统一采用用户提供的 macOS“应用程序”窗口作为布局母版：

- 内容区正常状态不绘制卡片底色和卡片边框，只展示应用图标、单行名称、固定高度状态位和紧凑操作。
- 目标宽度下默认显示 7 列；单元格宽度 `120px`，图标 `80 × 80px`，水平/纵向间距为 `24px`。内容区变窄时自适应为 6 列、5 列，不缩小图标到 `72px` 以下。
- 图标使用真实 App 图标或来源返回的图标；加载失败时使用统一 IPA 文件图标，不得用彩色方块加单字代替。
- 应用名称位于图标下方居中显示，最多一行，超出省略；完整名称通过悬停提示显示。
- 状态位固定占一行，字号 `11px`，避免不同状态导致网格跳动。
- 操作使用紧凑文字按钮或图标按钮，不再使用占满单元格宽度的绿色按钮；每个状态只突出一个主操作。
- 版本、构建号、体积、Bundle ID、来源、哈希和兼容原因通过点击图标打开轻量信息浮层查看，不常驻堆叠在网格中。
- 一级业务 Tab 继续保留“我的 App / IPA 资源”；IPA 资源内部增加“全部、游戏、工具、娱乐、社交、创意、其他”分类导航，来源和状态使用紧凑筛选控件。
- “我的 App”和“IPA 资源”共用同一套网格、选中、悬停、焦点、加载失败和无障碍规则，避免两个 Tab 视觉割裂。

### 5.9 IPA 资源三层状态模型

不得使用单个状态字符串覆盖资源全部生命周期，单个资源由以下三层共同决定界面：

1. `resource`：版本、来源、兼容性、哈希风险等稳定属性。
2. `downloadTask`：未下载、排队中、下载中、已暂停、校验中、待安装、下载失败、校验失败。
3. `installTask`：未安装、安装中、已安装、安装失败。

“可更新”由资源版本与本地已安装版本比较得出，不单独写入持久状态。旧版已安装与新版下载/安装任务允许并存；更新失败时旧版仍可打开。

### 5.10 状态全景与操作规则

标注版 Demo 默认使用静态状态夹具，让 14 种状态以七列两行的 macOS 图标网格同屏稳定展示，不在页面加载时自动启动计时器。网格顶部用“可获取、进行中、已完成、需处理”图例说明状态语义，不拆成四个纵向分组：

| 显示状态 | 视觉规则 | 唯一主操作 | 后续状态 |
|---|---|---|---|
| 未下载 | 正常图标，灰色“未获取” | 下载 | 排队中 |
| 排队中 | 时钟角标与队列序号 | 取消排队 | 未下载 |
| 下载中 | 图标外围环形进度和百分比 | 暂停 | 已暂停 |
| 已暂停 | 保留下载进度，黄色暂停标识 | 继续 | 下载中 |
| 校验中 | 环形不确定进度 | 无，等待 | 待安装或校验失败 |
| 已下载待安装 | 绿色完成角标 | 安装 | 安装中 |
| 安装中 | 安装进度环 | 无，等待 | 已安装或安装失败 |
| 已安装 | 绿色状态点 | 打开 | 保持已安装 |
| 可更新 | 蓝色更新角标与目标版本 | 更新 | 排队中，同时保留旧版可打开 |
| 下载失败 | 红色感叹号 | 重新下载 | 排队中 |
| 校验失败 | 红色盾牌 | 删除并重下 | 排队中 |
| 安装失败 | 红色错误角标 | 重试安装 | 安装中 |
| 不兼容 | 图标降饱和 | 查看原因 | 不变 |
| 缺少来源哈希 | 橙色风险盾牌 | 下载；安装前二次确认 | 正常下载链路 |

静态夹具只负责初始演示。用户操作某个图标后，该任务才启动独立计时器；计时回调必须通过 `taskId + generationToken` 校验，重置 Demo 时清理所有计时器并递增全局代次，避免进度串到其他 App。

### 5.11 批量选择

- 正常浏览状态不展示复选框；用户点击“批量选择”后才进入选择模式。
- 复选框位于图标左上角；“全选 / 取消全选”位于页面右上角，只作用于当前搜索、分类、来源和状态筛选结果。
- 已安装、不兼容、正在处理和其他不可下载项自动排除，不计入可选总数。
- 页面底部显示玻璃悬浮操作条，包含已选数量、预计大小、“取消”和“下载”。
- 退出批量模式时清空选择；筛选条件变化时保留仍在结果集中的可选项并移除失效项。

## 7. 标注清单

### 7.1 全局规则

| 编号 | 标题 | 说明 |
|---|---|---|
| G | iOS 应用独立模块 | 不并入原 PC/Mac 游戏库，内部包含“我的 App”和“IPA 资源” |
| G | 客户端侧栏兼容新增 | 原侧栏 1:1 保留，只在“游戏库”与“订单中心”之间新增“iOS 应用”入口 |
| G | Mac 应用程序式网格 | “我的 App”和“IPA 资源”采用真实图标网格，不使用网页商城式大卡片 |
| G | 三层状态模型 | 资源属性、下载任务、安装任务独立；更新失败不得破坏已安装旧版 |
| G | 唯一 App 库路径 | 只能存在一个当前活动路径，导入与后续安装共用该目录 |
| G | 第三方来源责任 | 来源由用户主动添加，仅保存在本机，盖世不提供或审核资源内容 |

### 7.2 交互说明

| 编号 | 目标状态 | 标注内容 |
|---|---|---|
| 1 | 首次进入 | 选择 App 库文件夹，取消选择不产生错误 |
| 2 | 文件夹选择 | 只允许选择一个可读写文件夹，空目录同样有效 |
| 3 | 目录扫描确认 | 首次设置已有 App 目录时确认识别结果，异常项置灰并说明原因，不移动原文件 |
| 4 | 本地 IPA 导入 | “我的 App”常驻“导入 IPA”；选择一个或多个本地 `.ipa` 后进入选择确认，已有 App 时仍可继续导入 |
| 5 | 我的 App | 搜索、筛选、排序、启动、更新和卸载 |
| 6 | 添加来源 | URL 输入后延迟 500ms 校验，反馈区高度固定；无效时显示“链接无效，JSON无效或未找到！”；首次点击“导入”只进入资源选择，不直接写入资源库；Demo 示例入口位于弹窗外 |
| 7 | 选择来源资源 | 默认不勾选；用户逐项选择或使用右上角“全选 / 取消全选”，最终确认后才保存来源并导入所选资源 |
| 8 | 来源与聚合 | 多来源按 Bundle ID 合并，只展示最新兼容构建 |
| 9 | 批量下载 | 逐项展示下载、校验和失败状态，单项失败不阻塞其他项 |
| 10 | 安装确认 | 下载完成后强确认，无哈希资源不默认勾选 |
| 11 | 安装结果 | 停留结果页，支持失败重试、打开和主动前往我的 App |

### 7.3 异常与边界

| 编号 | 异常场景 | 产品处理 |
|---|---|---|
| E1 | 选择空目录 | 仍保存为有效 App 库，并引导浏览 IPA 资源 |
| E2 | 目录无权限或权限失效 | 不覆盖原路径，保留上次扫描缓存并引导重新授权 |
| E3 | 来源 URL、JSON 或目标资源无效 | 统一显示“链接无效，JSON无效或未找到！”，“导入”按钮保持置灰 |
| E4 | 来源刷新失败 | 保留上次缓存，不清空资源列表 |
| E5 | IPA 缺少来源哈希 | 标记风险，不默认勾选，安装前再次确认 |
| E6 | 下载或校验部分失败 | 失败项置灰，成功项仍可进入安装确认 |
| E7 | 任务进行中更改路径 | 禁止更改，并说明需等待下载或安装完成 |
| E8 | 更新或安装失败 | 保留原 App 版本，避免旧版本被破坏 |

## 8. 业务与标注事件隔离

### 8.1 业务事件

- 业务事件只处理舞台内的 `data-action`、`data-input` 和 `data-change`。
- 业务选择器查询范围限制在 `#demoStage`。
- 原有状态、下载计时器、来源验证计时器和 Smoke Test 保持独立。

### 8.2 标注事件

- 标注系统使用 `data-anno-action`，不复用业务 `data-action`。
- 标注点击、标号开关、面板折叠由外层控制器处理。
- 标号不直接写入业务渲染 HTML，统一放在 `#annotationOverlay`。
- 每次业务 `renderApp()` 完成后重新计算标号位置。

### 8.3 弹层和固定元素

- 原业务 `position:fixed` 元素改为相对舞台定位。
- 标注版的遮罩只覆盖业务舞台，不遮挡左右标注框架。
- Toast 和任务提示显示在中间舞台顶部，不覆盖右侧标注面板。

## 9. 标注联动数据

每条标注至少包含：

```js
{
  id: '5',
  type: 'interaction',
  page: 'resources',
  state: 'source-validation',
  target: 'source-validation-box',
  title: '来源 URL 即时校验',
  body: '停止输入 500ms 后校验，修改 URL 后立即清除旧成功状态。'
}
```

标注控制器按以下顺序执行：

```text
applyDemoState(annotation.state)
→ renderApp()
→ requestAnimationFrame
→ positionAnnotationBadge(annotation.target)
→ flashAnnotationTarget(annotation.target)
```

## 10. 响应式与可读性

- 目标评审分辨率为 `1440 × 900px` 及以上。
- 中间 `1440 × 900px` macOS 桌面舞台整体等比缩放，不重新排版业务内容；客户端业务侧栏始终保持逻辑 `72px` 宽。
- 右侧面板默认展开，但用户可随时折叠以放大 Demo。
- 标注面板正文不小于 12px，标题不小于 13px。
- 低于 1200px 时优先保持左右面板可操作，中间区域允许横向滚动。
