# macOS 26 盖世全屏启动台设计

> **历史版本提示：** 本文件为 V1.0“游戏优先”方案，已被 [`2026-07-22-macos-26-gamehub-launchpad-parity-design.md`](./2026-07-22-macos-26-gamehub-launchpad-parity-design.md) 替代。

> 日期：2026-07-20  
> 状态：设计已逐段确认，待书面评审  
> 适用端：盖世游戏 Mac 客户端，macOS 26 及以上  
> 参考 Demo：`Mac端demo/mac端租号功能/Mac端租号功能-标注版.html`

## 1. 背景与产品定位

macOS 26 取消了旧版全屏 Launchpad，系统仍提供 Spotlight“应用”视图等应用启动方式。用户真正失去的是大图标全屏浏览、集中查找和长期形成的启动习惯，而不是完全无法启动应用。

本功能不做系统级 Launchpad 替代品。入口仅位于盖世游戏 Mac 客户端内，定位为：

> 盖世全屏启动台：让已经进入盖世客户端的用户，在一个 macOS 原生全屏空间中集中查找和启动本地游戏及其他 Mac 应用。

“参考应用宝 Mac 版”只表示参考其游戏化内容组织和 Mac 客户端体验。公开资料没有证明应用宝 Mac 版存在一个可直接复刻的独立全屏 Launcher，也没有明确其真实入口，因此不把应用宝未验证的交互写成本需求事实。

## 2. 用户与核心问题

### 2.1 目标用户

- 已升级至 macOS 26；
- 已安装并使用盖世游戏 Mac 客户端；
- 希望以全屏、大图标方式浏览本地游戏和应用；
- 不希望通过文件夹或 Spotlight 逐个查找应用。

### 2.2 核心问题

用户在盖世客户端内缺少一个集中、直观且具有系统熟悉感的本地应用启动入口，尤其难以优先浏览已经安装并可直接运行的游戏。

### 2.3 核心闭环

```text
点击盖世左侧“启动台”
→ 进入 macOS 原生全屏空间
→ 在“游戏 / 全部应用”中浏览或搜索
→ 点击应用
→ 启动未运行应用或激活已有窗口
→ 目标游戏在当前显示器全屏运行，启动台保留在后层
→ 结束或返回目标游戏后重新显示启动台
```

## 3. 目标与非目标

### 3.1 本期目标

1. 在盖世客户端内提供稳定、可发现的“启动台”入口；
2. 使用 macOS 原生全屏空间承载系统感大图标网格；
3. 默认突出当前设备可直接启动的本地游戏；
4. 覆盖标准应用目录中的本地可启动应用；
5. 支持分类、搜索、纵向滚动、键盘操作和应用启动；
6. 允许用户轻量纠正游戏识别结果；
7. 通过本地缓存保证二次进入速度。

### 3.2 本期不做

- 系统级快捷键、菜单栏、触控板系统手势或独立 Launcher 应用；
- 拖拽排序、横向分页、文件夹、隐藏、重命名和卸载；
- 云游戏、租号游戏、PlayCover 游戏、未安装游戏和推荐内容；
- 商城、活动、广告或运营推荐位；
- 账号同步、云端应用库、运营后台；
- macOS 25 及以下版本入口。

## 4. 已确认的产品决策

| 决策项 | 结论 |
|---|---|
| 应用范围 | 全部本地应用，默认突出游戏 |
| 入口 | 盖世客户端左侧主导航“启动台” |
| 全屏形式 | macOS 原生全屏窗口和独立桌面空间 |
| 页面风格 | 高保真复用盖世 Mac 端深色视觉，保留 Launchpad 的集中启动信息架构 |
| 网格浏览 | 游戏使用四列横版封面，全部应用使用大图标网格，均纵向连续滚动 |
| 游戏页内容 | 只展示当前设备可直接启动的本地游戏 |
| 游戏识别纠错 | 右键或 Control + 单击添加到游戏/从游戏中移除 |
| 搜索范围 | 只搜索当前分类 |
| 退出方式 | 无搜索内容时按 Esc 退出启动台；结束目标游戏后回到启动台 |
| 数据与后台 | 全部本地处理，无服务端和运营后台 |

## 5. 信息架构

```text
盖世游戏 Mac 客户端
└─ 左侧主导航：启动台
   └─ macOS 原生全屏空间
      ├─ 游戏（默认）
      │  └─ 当前设备可直接启动的本地游戏
      └─ 全部应用
         └─ 标准应用目录中的本地可启动应用
```

“游戏”不是盖世商城或推荐页。未安装、仅云端可玩、仅租号可玩或仅在盖世目录中存在的游戏不得进入启动台。

## 6. 界面设计

### 6.1 盖世客户端入口

- 在现有左侧主导航增加固定入口“启动台”；
- 与探索、找游戏、游戏库等一级入口同层级；
- 图标使用九宫格/启动台语义；
- macOS 26 及以上显示，低版本不显示；
- 点击后不在客户端内容区打开普通页面，而是进入原生全屏空间。

### 6.2 全屏启动台

- 使用用户提供的盖世 Mac 端截图作为视觉基准：深灰底色、左侧悬浮圆角导航、大标题、低饱和描边和克制的高亮色；
- 内层开发基准画布为 `1076×734`，左侧启动台导航占 `82px`，内容区左右留白分别为 `31px / 36px`；
- 标题下方显示“游戏 / 全部应用”分段切换，默认选中“游戏”，搜索框位于工具栏右侧；
- “游戏”使用四列 `16:9` 横版封面，列间距 `16px`、封面圆角 `12px`，名称、分类和评分位于封面下方；
- “全部应用”使用六列大图标网格，应用名称最多两行，悬停时显示完整名称；
- 列表使用纵向连续滚动；
- 底部使用低干扰文字提示“Esc 退出启动台”；
- 不显示盖世商城导航、价格、活动角标、推荐标签和广告。

### 6.3 图标状态

| 状态 | 展示 |
|---|---|
| 默认 | 应用图标＋名称 |
| 悬停 | 图标轻微放大，名称完整提示 |
| 键盘焦点 | 系统色焦点环 |
| 启动中 | 图标显示小型进度动画，阻止重复点击 |
| 读取失败 | 系统通用应用图标 |
| 已失效 | 不继续展示，扫描后移除 |

## 7. 核心交互

### 7.1 进入启动台

1. 用户点击左侧“启动台”；
2. 系统在盖世客户端当前所在显示器进入 macOS 原生全屏空间；
3. 有缓存时先展示缓存并在后台增量扫描；
4. 无缓存时展示图标骨架屏和“正在查找应用”；
5. 默认进入“游戏”，滚动位置为顶部，搜索词为空。

同一次全屏会话内，“游戏”和“全部应用”分别保留滚动位置。退出并重新进入后重置为“游戏”顶部。

### 7.2 浏览与搜索

- 鼠标或触控板纵向滚动浏览；
- 方向键在网格中移动焦点，Enter 启动当前应用；
- Tab 在分类切换、搜索框和应用网格之间移动；
- Command + F 聚焦搜索框；
- 输入关键词后实时过滤当前分类，不需要按回车；
- 搜索匹配应用名称；匹配不区分大小写；
- “游戏”无结果时提示“可切换到全部应用继续搜索”；
- 搜索有内容时按 Esc 清空搜索并保留当前分类；
- 搜索为空时按 Esc 退出启动台。

### 7.3 启动或激活应用

```text
单击应用
├─ 应用已运行 → 激活已有窗口 → 目标应用全屏置前
├─ 应用未运行 → 显示启动中 → 启动成功 → 目标应用全屏置前
└─ 启动失败/超时 → 保留启动台 → 恢复图标 → 显示原因和重试提示
```

规则：

- 单击即启动，不要求双击；
- 同一时间只允许一个启动请求；
- 启动成功或确认已激活后，目标应用占满当前屏幕，启动台保留在后层；
- 目标游戏结束、关闭或主动返回后，重新显示全屏启动台，不返回盖世首页；
- 失败、超时或系统拒绝启动时不退出；
- 点击“盖世游戏”自身图标时直接退出启动台并返回原客户端窗口；
- 不通过重复拉起方式激活已运行应用。

### 7.4 修改游戏分类

- 在“全部应用”中右键或 Control + 单击应用；
- 未被识别为游戏时显示“添加到游戏”；
- 已属于游戏时显示“从游戏中移除”；
- 修改立即生效并保存到本机；
- 从“游戏”页移除后，图标从当前页消失，但仍保留在“全部应用”；
- 右键菜单不增加卸载、隐藏、排序或文件夹操作。

### 7.5 退出启动台

以下操作关闭启动台：

- 搜索为空时按 Esc；
- 使用系统退出全屏操作；
- 在“全部应用”中点击盖世游戏自身图标。

成功启动或激活目标应用不关闭启动台；启动台作为后层返回目标保留。

退出启动台后，缓存和人工分类结果保留；未完成的搜索词和滚动位置不跨会话保留。

## 8. 页面状态

| 状态 | 触发 | 展示与行为 |
|---|---|---|
| 首次扫描 | 无本地缓存 | 图标骨架屏＋“正在查找应用” |
| 正常 | 已有可用列表 | 分类、搜索框、应用网格 |
| 后台更新 | 增量扫描中 | 保持现有列表，不阻塞浏览和启动 |
| 游戏为空 | 无可启动游戏 | 提示前往“全部应用”添加游戏 |
| 搜索无结果 | 当前分类无匹配 | 展示当前分类无结果和切换分类提示 |
| 启动中 | 已发起启动请求 | 当前图标显示进度，其余应用暂不可启动 |
| 启动失败 | 系统返回失败或超时 | 留在原位置并提供原因和重试提示 |
| 应用已删除 | 路径失效或收到删除通知 | 从列表移除，不保留失效图标 |
| 部分扫描失败 | 某目录不可访问 | 使用其他目录及旧缓存，不阻塞启动台 |

## 9. 本地模块设计

### 9.1 模块边界

| 模块 | 职责 | 依赖 |
|---|---|---|
| LauncherEntry | 入口显示、全屏进入和退出 | 现有 Mac 客户端导航、窗口系统 |
| AppScanner | 扫描应用、读取 Bundle 和图标、过滤辅助进程 | 文件系统、系统应用元数据 |
| GameClassifier | 判断本地应用是否属于游戏 | 盖世本地游戏库、应用分类、本地规则 |
| LauncherStore | 缓存应用列表和人工分类 | 本地持久化 |
| AppLaunchCoordinator | 启动或激活应用、处理启动结果 | NSWorkspace/系统进程能力 |
| LauncherView | 分类、搜索、网格、滚动和状态展示 | 以上模块输出的只读视图状态 |

### 9.2 数据流

```text
标准应用目录
   ↓
AppScanner ──→ 去重后的本地应用
                    ↓
盖世本地游戏库 ─→ GameClassifier ←─ 用户人工分类
                    ↓
               LauncherStore
                    ↓
               LauncherView
                    ↓ 点击应用
           AppLaunchCoordinator
                    ↓
       成功置前且启动台留后层 / 失败留页
```

## 10. 扫描、去重与游戏识别

### 10.1 扫描目录

第一期扫描以下标准目录：

- `/Applications`；
- `~/Applications`；
- `/System/Applications`；
- `/System/Applications/Utilities`。

不递归展示 `.app` 包内部的 Helper、Updater、Renderer、Crash Handler 等辅助进程。外接磁盘和非标准自定义目录不纳入第一期主动扫描。

### 10.2 去重

1. 优先使用 Bundle ID 作为应用主键；
2. Bundle ID 相同时优先标准应用目录中的可启动版本；
3. Bundle ID 缺失时使用规范化绝对路径作为临时主键；
4. 同名但 Bundle ID 不同的应用保留为两个条目；
5. 应用路径改变但 Bundle ID 不变时继承人工分类。

### 10.3 游戏识别优先级

```text
用户人工分类
> 盖世本地游戏库的 Bundle ID/路径匹配
> 应用自身游戏分类或已知游戏 Bundle ID
> 无法确定：仅进入“全部应用”
```

不通过名称关键词进行高置信度猜测，不把未安装游戏、网页链接、云游戏入口或推荐条目放入“游戏”。

## 11. 本地数据

每个应用至少保存：

- 名称；
- Bundle ID；
- 规范化路径；
- 图标缓存标识；
- 是否可启动；
- 自动分类结果和分类来源；
- 用户人工分类结果；
- 最近扫描时间和最后发现时间。

数据全部保存在本机。重新扫描不得覆盖人工分类；应用卸载后清理对应缓存和人工分类记录。

## 12. 异常与降级

- 目录不可访问：跳过该目录，继续展示其他目录和旧缓存；
- 图标读取失败：使用系统通用应用图标；
- Bundle ID 缺失：使用规范化路径，不阻塞展示；
- 应用损坏或受 Gatekeeper 限制：展示系统返回原因，不退出启动台；
- 扫描中断：继续使用上一次缓存；
- 游戏识别失败：进入“全部应用”，允许用户手动添加；
- 应用启动超时：恢复可点击状态并提示重试；
- 系统全屏进入失败：留在盖世客户端并提示暂时无法打开启动台；
- 应用安装或删除通知缺失：下一次进入时通过增量扫描校正。

## 13. 性能与体验指标

| 指标 | 目标 |
|---|---|
| 有缓存进入可操作时间 | ≤ 500ms |
| 首次扫描首批结果 | ≤ 1.5s |
| 搜索输入到结果刷新 | ≤ 100ms |
| 200 个应用下滚动 | 无明显掉帧或输入阻塞 |
| 标准目录应用识别率 | ≥ 95% |
| 重复启动请求 | 0 |

产品验证指标：

- 启动台入口点击用户占 macOS 26 盖世活跃用户的比例；
- 首次使用后 7 日内重复使用率；
- 进入启动台到成功启动游戏的中位耗时；
- 启动台游戏启动成功率；
- 搜索使用率和无结果率；
- 人工纠正游戏分类的应用占比；
- 使用启动台用户与未使用用户的留存、游戏启动次数差异。

入口仅位于盖世客户端内，若渗透率和重复使用率长期偏低，不扩展系统级入口、整理能力或更多内容类型。

### 14.3 Demo 不代表的真实能力

HTML Demo 不模拟真实应用扫描、NSWorkspace 启动、系统进程激活或真实全屏 Space。上述能力必须通过原生技术验证证明。

## 15. 原生技术验证

在正式开发前完成一个独立技术 Spike，验证：

1. 标准应用目录扫描及系统应用读取权限；
2. Bundle ID 缺失、重复和多副本处理；
3. 图标读取和缓存性能；
4. 盖世本地游戏库与本地应用匹配；
5. NSWorkspace 冷启动应用；
6. 激活已运行应用而非重复拉起；
7. 启动失败和超时结果获取；
8. 原生全屏空间进入、目标应用置前和退出时序；
9. 多显示器下使用盖世窗口所在显示器；
10. 应用安装、删除和路径变化后的列表更新。

技术验证不通过时，Demo 交互不视为真实能力已经完成。

## 16. 验收标准

1. macOS 26 盖世客户端左侧显示“启动台”，低版本不显示；
2. 点击入口后在当前显示器进入原生全屏空间；
3. 默认打开“游戏”，且只展示本地可直接启动游戏；
4. “全部应用”展示标准目录中可启动应用，不展示 Helper 等辅助进程；
5. 游戏使用四列 `16:9` 横版封面，全部应用使用六列大图标，并均支持纵向滚动；
6. 搜索只过滤当前分类，清空后恢复原列表和同会话滚动位置；
7. 方向键、Enter、Tab、Command + F 和 Esc 行为符合本设计；
8. 右键可以修改游戏分类，重新扫描后人工结果仍保留；
