# macOS 26 盖世全屏启动台竞品对齐设计

> 日期：2026-07-22  
> 状态：方案已确认，待书面评审  
> 适用端：盖世游戏 Mac 客户端，macOS 26 及以上  
> 需求性质：2026-07-20 旧方案的 V2.0 替代方案

## 1. 变更背景

用户实际体验竞品后确认，竞品不是游戏内容页，而是 macOS 旧版 Launchpad 的替代启动台：读取本机全部应用，以全屏大图标、横向分页和文件夹方式组织，并支持搜索、应用启动、全局快捷键、F4、触控板和触发角唤起。

旧方案中的“游戏优先、游戏/全部应用双页签、横版游戏封面和纵向滚动”偏离这一定位。本次改为：

> 在盖世 Mac 客户端进程内提供一套面向本机全部应用的全屏启动台；保留盖世侧边栏入口，并在客户端运行或后台驻留时提供系统级唤起方式。

## 2. 已确认决策

| 决策项 | V2.0 结论 |
|---|---|
| 产品形态 | 盖世 Mac 客户端内部模块，不拆分独立 Launcher App |
| 入口 | 侧边栏仅增加九宫格图标，无固定文字；悬停提示“启动台” |
| 应用范围 | 展示扫描目录内全部可启动 Mac 应用，不设置游戏页签 |
| 页面结构 | 全屏壁纸模糊背景、顶部搜索、大图标网格、横向分页、底部分页圆点 |
| 整理能力 | 拖拽排序、跨页拖动、拖拽合并文件夹、文件夹重命名和移出 |
| 唤起方式 | 侧边栏、全局快捷键、F4、触控板手势、屏幕触发角 |
| 生效条件 | 盖世客户端在前台、后台或最小化时有效；彻底退出后不可用 |
| 扫描范围 | 标准应用目录并支持用户添加自定义应用目录 |
| 服务端 | 应用列表、排序、文件夹和设置均保存在本机，无运营后台 |

## 3. 目标与非目标

### 3.1 目标

1. 让 macOS 26 用户用熟悉的大图标启动台浏览和打开本机应用。
2. 让用户可以按自己的习惯分页、排序和建立文件夹。
3. 在盖世运行或后台驻留时，通过多种入口快速唤起启动台。
4. 保证应用扫描、图标读取、搜索和启动均不依赖网络。
5. 在权限不足、入口冲突或部分目录失败时仍保留可用入口。

### 3.2 非目标

- 不做独立安装、独立更新或独立 Dock 图标的 Launcher App；
- 不展示未安装游戏、云游戏、租号、商城、广告或运营推荐；
- 不在启动台内卸载应用；
- 不修改应用名称、Bundle 内容或系统 Launch Services 数据；
- 不承诺在盖世客户端被彻底退出后响应全局入口；
- 不静默修改系统快捷键、触发角或隐私权限。

## 4. 核心体验路径

### 4.1 从盖世客户端进入

```text
点击侧边栏九宫格图标
→ 当前显示器进入全屏启动台
→ 读取缓存并后台增量扫描
→ 浏览、搜索或整理应用
→ 点击应用
→ 启动未运行应用或激活已有窗口
→ 启动台保留会话状态并退到后层
```

### 4.2 从全局入口进入

```text
盖世客户端正在运行或后台驻留
→ 用户按快捷键/F4、执行触控板手势或触发屏幕角
→ 在鼠标指针所在显示器打开启动台
→ 再次触发或按 Esc
→ 关闭启动台并恢复唤起前的应用
```

### 4.3 整理应用

```text
长按或拖动应用
→ 进入整理态
├─ 拖到空位：调整排序
├─ 拖到屏幕边缘停留：切换页面后放置
├─ 拖到另一应用：创建文件夹
└─ 拖出文件夹：移回当前页面
```

## 6. 页面与交互设计

### 6.1 侧边栏入口

- 与现有一级入口使用相同图标容器和选中规则；
- 只展示九宫格图标，不显示固定名称；
- 悬停显示“启动台”；
- 点击后进入全屏，不在普通内容区加载页面；
- 退出后恢复进入前的页面和侧边栏选中状态；
- macOS 26 以下版本隐藏入口。

### 6.2 全屏启动台

- 背景使用当前桌面壁纸的模糊与降亮处理，保证应用名称可读；
- 顶部居中显示搜索框，右侧显示更多菜单；
- 中部根据显示器可用空间动态计算行列数，每一显示器分辨率下保持稳定页容量；
- 应用使用系统原始图标和本地化名称；名称过长时单行渐隐，悬停显示完整名称；
- 底部圆点反映总页数和当前页；页数变化时实时更新；
- 支持鼠标滚轮、触控板横滑、方向键和页码圆点切页；
- 退出再进入时恢复上次页码；搜索词不跨会话保留。

### 6.3 初始排序与增量更新

1. 首次扫描按本地化应用名称升序排列；系统实用工具目录默认合并为“实用工具”文件夹。
2. 用户完成第一次拖拽后，排序进入人工模式，重新扫描不得重排已有应用。
3. 新安装应用追加到最后一页末尾。
4. 应用卸载后移除对应图标，后续图标按阅读顺序前移补位。
5. Bundle ID 相同的多副本只保留优先级最高的可启动版本；同名但 Bundle ID 不同的应用均保留。

### 6.4 搜索

- 搜索范围包含所有页面和文件夹内应用；
- 按应用本地化名称做包含匹配，不区分大小写和首尾空格；
- 搜索结果展开为应用图标，不展示文件夹容器；
- 搜索结果按本地化名称升序排列；
- 输入实时刷新；无结果显示“未找到应用”；
- 有搜索词时按 Esc 先清空，搜索为空时再次按 Esc 退出启动台。

### 6.5 拖拽排序与跨页

- 按住应用开始移动后进入整理态，其他图标让位；
- 拖到页内目标位置即调整排序；
- 拖到左右边缘并停留后切换相邻页面；首尾页不继续切换；
- 拖拽过程中按 Esc 取消并恢复原位置；
- 放置成功立即写入本地排序，不等待退出启动台；
- 应用在拖拽期间被卸载时取消拖拽并刷新列表。

### 6.6 文件夹

- 将一个应用拖到另一个应用上创建文件夹；
- 新文件夹默认命名为“文件夹”，系统实用工具文件夹命名为“实用工具”；
- 打开文件夹后在全屏启动台上方展示文件夹层，背景继续模糊；
- 文件夹名称支持编辑，去除首尾空格后长度为 1—20 个字符；
- 文件夹内支持拖拽排序和将应用拖出；
- 文件夹只剩一个应用时自动解散，剩余应用占据原文件夹位置；
- 文件夹为空时删除；
- 文件夹名称只保存在本机，不涉及敏感词审核和云端同步。

### 6.7 应用启动与返回

- 单击应用即启动，不要求双击；
- 已运行应用激活已有窗口，不重复拉起；
- 发起启动后图标进入启动中状态，防止重复点击；
- 启动成功后目标应用置前，启动台保留本次页码和文件夹状态；
- 目标应用关闭后不强制抢焦点；用户回到启动台或再次唤起时恢复本次会话；
- 点击“盖世游戏”图标时退出启动台并激活盖世客户端窗口；
- 启动失败时留在启动台，恢复图标并展示系统返回原因和重试入口。

### 6.8 更多菜单与设置

更多菜单包含“设置”和“退出启动台”，不提供“添加到 Dock”。

设置内容：

| 分组 | 配置 |
|---|---|
| 快速唤起 | 自定义全局快捷键、F4、触控板手势、触发角 |
| 权限 | 辅助功能、输入监控等权限状态和“去授权”入口 |
| 触发角 | 关闭、左上、右上、左下、右下；同一时间只选一个 |
| 应用来源 | 标准目录只读展示；用户可添加、移除自定义目录 |

规则：

- 快捷键录入时检测系统保留键和盖世现有快捷键冲突；冲突时不保存；
- F4、触控板和触发角分别启用，关闭后立即停止监听；
- F4 或手势在当前系统/设备不可捕获时显示“不支持”，不得伪装成权限问题；
- 点击“去授权”打开对应系统设置，回到客户端后刷新授权状态；
- 设置变更立即保存并生效；
- 盖世客户端彻底退出后所有全局入口失效。

### 6.9 交互优先级与拖拽判定

Esc 按“弹层 → 重命名 → 拖拽 → 文件夹 → 搜索 → 启动台”的顺序逐层返回；应用启动中不响应 Esc，等待结果后再处理。

- 应用中心区域停留500毫秒进入创建文件夹预览，其他区域按排序处理；
- 左右边缘停留600毫秒切换一页，切页后800毫秒内不连续翻页；
- 拖拽期间冻结网格，扫描、卸载、分辨率和显示器变化排队到拖拽结束后处理；
- 页数减少时当前页自动收敛到最后一个有效页。

## 7. 应用扫描与权限

### 7.1 默认目录

- `/Applications`；
- `/System/Applications`；
- `/System/Applications/Utilities`；
- `~/Applications`。

### 7.2 自定义目录

- 用户通过系统目录选择器添加目录；
- 保存可持续访问授权，不直接要求完全磁盘访问；
- 用户可以移除自定义目录，移除后仅删除来源关系，不卸载应用；
- 同一路径不得重复添加；父子目录同时添加时按应用主键去重；
- 目录无权限或被删除时显示异常状态和“重新授权/移除”。

### 7.3 展示过滤

只展示可启动 `.app`，过滤应用包内部及已知辅助进程，包括 Helper、Updater、Renderer、Crash Handler、Uninstaller 和后台服务。图标读取失败时使用系统通用应用图标。

## 8. 多显示器与窗口规则

- 从侧边栏进入：在盖世客户端窗口所在显示器打开；
- 从全局入口进入：在鼠标指针所在显示器打开；
- 同一时间只允许一个启动台窗口；在另一显示器触发时将现有窗口迁移过去，不新建第二个；
- 屏幕断开时迁移到主显示器并重新计算页容量，不丢失人工排序；
- 分辨率或缩放变化只改变每页容量，不改变应用的全局排序顺序。

## 9. 本地模块

| 模块 | 职责 |
|---|---|
| LauncherEntry | 侧边栏图标、全局入口和退出恢复 |
| LauncherWindow | 全屏窗口、多显示器和壁纸背景 |
| AppScanner | 扫描、过滤、去重、图标读取和变更监听 |
| LauncherStore | 应用缓存、全局顺序、文件夹、来源和设置持久化 |
| AppGrid | 响应式分页、搜索、键盘操作和拖拽 |
| FolderCoordinator | 文件夹创建、命名、排序、移入、移出和解散 |
| GlobalTriggerCoordinator | 快捷键、F4、触控板、触发角、权限和冲突处理 |
| AppLaunchCoordinator | 冷启动、激活已有窗口、失败和超时处理 |

### 9.1 应用身份与布局继承

应用身份优先使用“签名团队标识＋Bundle ID”，其次为 Bundle ID、文件系统稳定标识和规范化路径哈希。同一身份的副本按“系统目录 → 普通应用目录 → 用户目录 → 自定义目录”保留优先版本。

- 身份不变的升级和路径移动继承排序与文件夹；
- 无稳定身份且路径变化时按新应用处理；
- 卸载后的失效布局记录保留7天，短期重装可恢复，超期清理；
- 自定义目录使用系统持续访问凭证，凭证失效只隔离该来源。

### 9.2 单实例状态

启动台状态为“已关闭、打开中、已打开、跨屏迁移中、关闭中”。500毫秒内重复触发只响应第一次；打开、关闭和迁移过程中不反向切换。退出时优先恢复唤起前应用，原应用或显示器已不可用时恢复盖世窗口。

## 10. 状态与异常

| 场景 | 处理 |
|---|---|
| 首次扫描 | 展示应用骨架屏和“正在查找应用” |
| 有缓存增量扫描 | 保留现有网格，后台追加、移除或更新应用 |
| 无任何应用 | 展示“未找到可启动应用”，提供“检查应用来源” |
| 搜索无结果 | 展示“未找到应用”，允许清空搜索 |
| 自定义目录无权限 | 保留其他来源，标记异常目录并提供重新授权 |
| 图标读取失败 | 使用系统通用应用图标 |
| 启动失败 | 留在原页，展示原因和重试 |
| 快捷键冲突 | 不保存新快捷键，展示冲突提示 |
| F4/触控板无权限 | 本入口不可用，侧边栏和已可用入口继续工作 |
| 设备不支持手势 | 展示“不支持”，不可启用 |
| 盖世彻底退出 | 全局入口无响应；重新打开盖世后恢复配置 |

## 11. 数据与指标

本地只保存应用主键、显示名、路径、图标缓存、排序、文件夹、来源、权限和唤起设置。应用绝对路径不上传；Bundle ID 上传前使用稳定哈希。

重点指标：

- 启动台入口渗透率；
- D7、D30 重复使用率；
- 周人均唤起次数；
- 各入口使用占比和成功率；
- 进入启动台到成功打开应用的 P50/P95；
- 搜索使用率和无结果率；
- 整理功能使用率；
- 应用扫描覆盖率和重复率；
- 使用启动台用户与未使用用户的盖世留存差异。

### 11.1 远程开关与灰度

不新增运营后台，复用盖世现有远程配置，支持按国内/海外包、版本、渠道、芯片和入口类型独立开关及灰度。总开关关闭时同时隐藏侧边栏并停止全部全局监听；单入口关闭不影响其他入口；本地布局数据保留。

放量顺序为内部测试、5%、20%、50%和全量。继续放量前要求无P0/P1缺陷，启动台打开成功率和应用启动成功率均不低于98%，扫描覆盖率不低于95%，且无持续全屏窗口残留。

## 12. 地区与登录差异

- 本功能为本地工具，未登录也可使用；
- 国内包使用中文固定界面；
- 海外包跟随盖世现有语言设置，产品英文名使用 `GameHub`；
- 应用名称以应用自身本地化名称为准；
- 国内与海外扫描、整理、启动和权限逻辑一致。

## 14. 技术预研门槛

正式排期前必须验证：

1. macOS 26 下 F4、触控板手势和触发角的可实现范围、权限与冲突；
2. 盖世前台、后台、最小化和彻底退出四种状态下的全局入口行为；
3. 自定义目录的持续授权与路径失效处理；
4. 多显示器、分辨率切换和全屏迁移；
5. 200 个以上应用的图标读取、分页和拖拽性能；
6. 应用安装、卸载、移动、重复和 Bundle ID 缺失时的增量更新；
7. 已运行应用激活、冷启动、Gatekeeper 拒绝和超时结果获取。

## 15. 方案自检

- 无 `TBD` 或未定义占位；
- 产品形态与已确认的“方案 B”一致，不包含独立 Launcher App 或独立 Dock 图标；
- 入口仅图标，不出现固定“启动台”文字；
- 已删除游戏优先、游戏页签、横版封面、游戏分类纠错和纵向连续滚动；
