# macOS 26 盖世全屏启动台竞品对齐设计

> 日期：2026-07-22  
> 状态：V2.4 方案已确认，Demo已实现并验收
> 适用端：盖世游戏 Mac 客户端，macOS 26 及以上  
> 需求性质：2026-07-20 旧方案的 V2.4 替代方案

## 1. 变更背景

用户实际体验竞品后确认，竞品不是游戏内容页，而是 macOS 旧版 Launchpad 的替代启动台：读取本机全部应用，以全屏大图标、横向分页和文件夹方式组织，并支持搜索、应用启动、全局快捷键、F4、触控板和触发角唤起。

旧方案中的“游戏优先、游戏/全部应用双页签、横版游戏封面和纵向滚动”偏离这一定位。V2.2 删除用户可见的扫描过程和 Demo 内部目标应用页，并补齐长按编辑、系统式删除、文件夹直接改名及独立程序坞入口；V2.3统一文件夹与程序坞引导交互；V2.4进一步修正文件夹槽位、缩略图比例，并收敛程序坞配置入口。本次改为：

> 在盖世 Mac 客户端中提供一套面向本机全部应用的全屏启动台；用户进入后直接看到全部应用，可用系统启动台式的搜索、分页、文件夹、编辑和删除能力，并能从侧边栏、全局入口或程序坞图标直接唤起。

## 2. 已确认决策

| 决策项 | V2.4 结论 |
|---|---|
| 产品形态 | 启动台仍由盖世 Mac 客户端承载；另提供轻量启动器作为独立程序坞入口，不承载普通客户端内容 |
| 入口 | 侧边栏仅增加九宫格图标，无固定文字；悬停提示“启动台” |
| 程序坞入口 | 使用与盖世 App 相同的图标；点击后直接唤起全屏启动台，盖世未运行时先拉起盖世 |
| 应用范围 | 进入后直接展示本机全部可启动 Mac 应用，不设置游戏页签，不展示扫描页 |
| 页面结构 | 全屏壁纸模糊背景、顶部搜索、大图标网格、横向分页、底部分页圆点 |
| 编辑与删除 | 长按进入图标抖动状态；可删除应用显示“−”，删除前二次确认，运行中应用禁止移到废纸篓 |
| 文件夹能力 | 拖放创建时文件夹替换目标应用槽位，后续应用前移一格；封面使用等比方形缩略图；支持移入、直接改名、移出和右上角关闭，不支持自由排序和跨页搬动 |
| 程序坞配置 | 首次引导只含标题、确认和关闭；更多菜单不重复提供入口；设置内用开关添加或移除，后台驻留默认关闭 |
| 唤起方式 | 侧边栏、全局快捷键、F4、触控板手势、屏幕触发角 |
| 生效条件 | 全局快捷键、F4、触控板和触发角仅在盖世前台、后台或最小化时有效；程序坞入口可冷启动盖世 |
| 应用发现 | 标准应用目录并支持用户添加自定义应用目录；发现和更新在后台完成，不向用户展示扫描过程 |
| 服务端 | 应用列表、文件夹和设置均保存在本机，无运营后台 |

## 3. 目标与非目标

### 3.1 目标

1. 让 macOS 26 用户用熟悉的大图标启动台浏览和打开本机应用。
2. 让用户可以通过初始名称排序、稳定槽位、分页和文件夹快速找到应用。
3. 在盖世运行或后台驻留时，通过多种入口快速唤起启动台。
4. 保证应用扫描、图标读取、搜索和启动均不依赖网络。
5. 在权限不足、入口冲突或部分目录失败时仍保留可用入口。
6. 复刻 macOS 启动台的长按编辑、文件夹改名和安全删除反馈。
7. 允许用户把独立启动台入口添加到程序坞，一次点击即可回到全屏启动台。

### 3.2 非目标

- 程序坞启动器不展示客户端首页、内容流或独立设置，不形成第二套客户端；
- 不展示未安装游戏、云游戏、租号、商城、广告或运营推荐；
- 删除仅把 `.app` 应用包移到废纸篓，不清理应用配置、存档、缓存或账号数据；
- 不修改应用名称、Bundle 内容或系统 Launch Services 数据；
- 不承诺在盖世客户端被彻底退出后响应全局入口；
- 不静默修改系统快捷键、触发角或隐私权限。

## 4. 核心体验路径

### 4.1 从盖世客户端进入

```text
点击侧边栏九宫格图标
→ 当前显示器进入全屏启动台
→ 立即展示本机全部应用，不显示扫描进度
→ 浏览、搜索或文件夹归类
→ 点击应用
→ 通过 macOS 原生能力启动未运行应用或激活已有窗口
→ 启动台保留会话状态并退到后层
```

### 4.2 从全局入口进入

```text
盖世客户端正在运行或后台驻留
→ 用户按快捷键/F4、执行触控板手势或触发屏幕角
→ 在鼠标指针所在显示器打开启动台
→ 再次触发或按 Esc
→ 关闭启动台并恢复唤起前的应用
```

### 4.3 文件夹归类

```text
拖动顶层应用
├─ 拖到另一应用：创建文件夹
├─ 拖到已有文件夹：移入文件夹
└─ 拖到空白、间隙或页面边缘：取消，不改变顺序

打开文件夹
→ 点击文件夹名称可直接改名
→ 点击应用“移出”返回顶层
→ 点击右上角关闭、外部模糊区域或按 Esc 关闭

创建或移入成功
→ 停留当前启动台页
→ 用户再次点击文件夹后才打开
```

### 4.4 编辑与删除

```text
长按任一应用约 650 毫秒
→ 应用图标轻微抖动，进入编辑模式
→ 可删除应用左上角出现“−”
├─ 应用未运行：二次确认后移到废纸篓
├─ 应用已运行：提示无法移到废纸篓，只有“好”按钮
└─ 系统应用或无删除权限：不显示“−”
```

### 4.5 添加到程序坞

```text
首次进入启动台
→ 搜索栏同行、更多按钮右侧展示一次性引导
→ 点击“确认”
→ 独立启动台图标加入程序坞
→ 后续点击该图标直接唤起全屏启动台
```

## 6. 页面与交互设计

### 6.1 侧边栏入口

- 与现有一级入口使用相同图标容器和选中规则；
- 只展示九宫格图标，不显示固定名称；
- 悬停显示“启动台”；
- 点击后进入全屏，不在普通内容区加载页面；
- 退出后恢复进入前的页面和侧边栏选中状态；
- macOS 26 以下版本隐藏入口。

### 6.2 全屏启动台

- 背景使用当前桌面壁纸的模糊与降亮处理，保证应用名称可读；
- 顶部居中显示搜索框，右侧显示更多菜单；
- 首次进入时，在搜索栏同行、更多按钮右侧展示“添加到程序坞”引导；引导采用macOS半透明深色圆角卡片，只包含标题“添加启动台到程序坞？”、蓝色“确认”和圆形关闭按钮，不显示左侧图标或副标题；
- 引导出现后保持展示，不因翻页、搜索或打开文件夹自动消失；点击“添加”成功或关闭按钮后记录本地状态，不再自动展示；
- 中部根据显示器可用空间动态计算行列数，每一显示器分辨率下保持稳定页容量；
- 应用使用系统原始图标和本地化名称；名称过长时单行渐隐，悬停显示完整名称；
- 底部圆点反映总页数和当前页；页数变化时实时更新；
- 支持鼠标滚轮、触控板横滑、方向键和页码圆点切页；
- 退出再进入时恢复上次页码；搜索词不跨会话保留。

### 6.3 初始排序与增量更新

1. 首次进入直接展示已发现应用，不展示骨架屏、扫描进度或“正在查找应用”；底层发现完成后按本地化应用名称升序排列，系统实用工具目录默认合并为“实用工具”文件夹。
2. 系统“实用工具”文件夹固定在首位；其余顶层应用与用户文件夹、文件夹内应用按当前客户端语言的本地化名称升序排列，同名项按稳定应用主键升序。
3. 新安装应用按本地化名称插入对应位置。
4. 应用卸载后移除对应图标并重新生成分页。
5. Bundle ID 相同的多副本只保留优先级最高的可启动版本；同名但 Bundle ID 不同的应用均保留。

### 6.4 搜索

- 搜索范围包含所有页面和文件夹内应用；
- 按应用本地化名称做包含匹配，不区分大小写和首尾空格；
- 搜索结果展开为应用图标，不展示文件夹容器；
- 搜索结果按本地化名称升序排列；
- 输入实时刷新；无结果显示“未找到应用”；
- 无结果页不提供“清空搜索”按钮，用户可直接修改搜索词或按 Esc 清空；
- 有搜索词时按 Esc 先清空，搜索为空时再次按 Esc 退出启动台。

### 6.5 文件夹拖放

- 将顶层应用拖到另一顶层应用上创建文件夹；
- 将顶层应用拖到已有文件夹上移入文件夹；
- 创建或移入成功后停留当前页，不自动打开文件夹；用户再次点击文件夹后才打开；
- 拖到空白处、应用间隙或页面边缘时取消，不调整顶层顺序，也不触发跨页；
- 拖放过程中按 Esc 取消，不改变文件夹归属；
- 应用在拖放期间被卸载时取消操作，刷新后保留既有槽位；仅未归类的新应用按名称插入。
- 按下后移动超过 6 点则取消 650 毫秒长按计时并进入拖放；静止达到 650 毫秒则进入编辑模式，避免拖放和长按误触。

### 6.6 文件夹

- 将一个应用拖到另一个应用上创建文件夹；新文件夹直接替换目标应用槽位，拖动源从原槽位移除，后续应用依次前移一格；
- 新文件夹默认命名为“文件夹”，系统实用工具文件夹命名为“实用工具”；
- 打开文件夹后在全屏启动台上方展示文件夹层，背景继续模糊；
- 不展示“重命名”按钮；点击文件夹名称直接切换为输入框，Enter 或失焦保存，Esc 取消；去除首尾空格后长度为 1—20 个字符；
- 文件夹封面使用 3×3、14×14 等比方形应用缩略图，缩放时保持完整，不拉成长条、不裁切成色块；
- 文件夹内应用按本地化名称升序排列；每个应用显示“移出”，点击后插入文件夹后一位；
- 文件夹层右上角显示关闭按钮；点击该按钮、外部模糊区域或按 Esc 均可关闭；
- 文件夹只剩一个应用时自动解散，剩余应用占据原文件夹位置；
- 文件夹为空时删除；
- 文件夹名称只保存在本机，不涉及敏感词审核和云端同步。

### 6.7 应用启动与返回

- 单击应用即调用 `NSWorkspace.openApplication(at:configuration:completionHandler:)`，不要求双击；
- 已运行应用通过 `NSRunningApplication` 激活已有窗口，不重复拉起；
- 发起启动后图标进入启动中状态，防止重复点击；
- completion 返回成功后目标应用置前，启动台退到后层并保留本次页码、搜索前页码和文件夹状态；
- 目标应用关闭后不强制抢焦点；用户回到启动台或再次唤起时恢复本次会话；
- 点击“盖世游戏”图标时退出启动台并激活盖世客户端窗口；
- 启动失败时留在启动台，恢复图标并展示系统返回原因和重试入口。
- Demo 仅用 Toast 表达“已调用 macOS 打开/激活 XX”，不在 Demo 内展示目标应用页面，也不提供“返回启动台”按钮。

### 6.8 更多菜单与设置

更多菜单只包含“设置”和“退出”，不重复提供程序坞添加入口。关闭首次引导后，程序坞仅在设置页管理。

设置内容：

| 分组 | 配置 |
|---|---|
| 启动 | 保持盖世后台驻留，默认关闭 |
| 快速唤起 | 自定义全局快捷键、F4、触控板手势、触发角 |
| 权限 | 辅助功能、输入监控等权限状态和“去授权”入口 |
| 触发角 | 关闭、左上、右上、左下、右下；同一时间只选一个 |
| 应用来源 | 标准目录只读展示；用户可添加、移除自定义目录 |
| 程序坞 | 使用与后台驻留一致的开关；打开执行添加，关闭执行移除；检测到用户从程序坞移除后1秒内同步为关闭 |

规则：

- 快捷键录入时检测系统保留键和盖世现有快捷键冲突；冲突时不保存；
- F4、触控板和触发角分别启用，关闭后立即停止监听；
- F4 或手势在当前系统/设备不可捕获时显示“不支持”，不得伪装成权限问题；
- 点击“去授权”打开对应系统设置，回到客户端后刷新授权状态；
- 设置变更立即保存并生效；
- 盖世客户端彻底退出后所有全局入口失效。
- 程序坞入口属于冷启动入口，不受上述限制；盖世未运行时先启动盖世，再直接进入启动台。

### 6.9 交互优先级与文件夹拖放判定

Esc 按“删除弹窗 → 设置/权限弹层 → 文件夹名称编辑 → 文件夹拖放 → 编辑模式 → 文件夹 → 搜索 → 启动台”的顺序逐层返回；应用启动中不响应 Esc，等待结果后再处理。

- 应用落到另一个应用或文件夹的命中区域时执行创建或移入；其他落点均取消；
- 文件夹拖放期间冻结网格，扫描、卸载、分辨率和显示器变化排队到拖放结束后处理；
- 拖放结束后按本地化名称重新排列，不保存人工顺序；
- 页数减少时当前页自动收敛到最后一个有效页。

### 6.10 长按编辑与删除

- 任一顶层或文件夹内应用长按 650 毫秒进入编辑模式，应用以错峰轻微旋转表现抖动；点击空白区域或按 Esc 退出；
- 可删除的第三方应用左上角显示“−”；系统应用、受系统保护应用和当前无删除权限的应用不显示；
- 正在运行的第三方应用仍显示“−”，点击后立即使用 `NSRunningApplication` 复核运行状态；
- 未运行时弹出420px收窄弹窗，主提示为“确认要删除‘XX’吗？”，下方辅助说明为“应用将被移到废纸篓。”，不显示额外配置/存档/缓存说明；按钮为“取消”“删除”；
- 用户确认后再次复核运行状态，再通过系统废纸篓能力移动 `.app`；成功后从启动台移除并重新分页；
- 若确认后应用已启动，弹出“不能将项目‘XX’移到废纸篓，因为它已打开。”，只有“好”；
- 权限、SIP、只读卷、外置卷或文件状态导致失败时不移除图标，说明失败原因并保留编辑模式；
- 删除不清理应用配置、存档和缓存，确认弹窗中的“移到废纸篓”不得替换为“彻底卸载”。

### 6.11 程序坞入口

- 独立启动器使用与盖世 App 相同的图标，名称随国内/海外包显示为“盖世启动台”或“GameHub Launchpad”；
- 点击后通过本地 IPC 请求盖世打开启动台；盖世未运行时，启动器先携带 `openLaunchpad` 参数拉起盖世，待客户端就绪后打开全屏启动台；
- 启动器不展示独立窗口，完成唤起后退出；程序坞固定项继续保留；
- 首次引导只负责一次性推荐，用户关闭后不再自动展示；设置页始终可重新执行添加；
- 添加失败时保留引导并提示系统限制；不得静默重启 Dock、修改其他程序坞项目或破坏用户排序。

## 7. 应用扫描与权限

### 7.1 默认目录

- `/Applications`；
- `/System/Applications`；
- `/System/Applications/Utilities`；
- `~/Applications`。

### 7.2 自定义目录

- 用户通过系统目录选择器添加目录；
- 保存可持续访问授权，不直接要求完全磁盘访问；
- 用户可以移除自定义目录，移除后仅删除来源关系，不卸载应用；
- 同一路径不得重复添加；父子目录同时添加时按应用主键去重；
- 目录无权限或被删除时显示异常状态和“重新授权/移除”。

### 7.3 展示过滤

只展示可启动 `.app`，过滤应用包内部及已知辅助进程，包括 Helper、Updater、Renderer、Crash Handler、Uninstaller 和后台服务。图标读取失败时使用系统通用应用图标。

## 8. 多显示器与窗口规则

- 从侧边栏进入：在盖世客户端窗口所在显示器打开；
- 从全局入口进入：在鼠标指针所在显示器打开；
- 同一时间只允许一个启动台窗口；在另一显示器触发时将现有窗口迁移过去，不新建第二个；
- 屏幕断开时迁移到主显示器并重新计算页容量，不丢失文件夹归属；
- 分辨率或缩放变化只改变每页容量，保持当前槽位顺序和文件夹归属。

### 9.1 应用身份与布局继承

应用身份优先使用“签名团队标识＋Bundle ID”，其次为 Bundle ID、文件系统稳定标识和规范化路径哈希。同一身份的副本按“系统目录 → 普通应用目录 → 用户目录 → 自定义目录”保留优先版本。

- 身份不变的升级和路径移动继承文件夹归属，并按最新本地化名称重新排序；
- 无稳定身份且路径变化时按新应用处理；
- 卸载后的失效布局记录保留7天，短期重装可恢复，超期清理；
- 自定义目录使用系统持续访问凭证，凭证失效只隔离该来源。

### 9.2 单实例状态

启动台状态为“已关闭、打开中、已打开、跨屏迁移中、关闭中”。500毫秒内重复触发只响应第一次；打开、关闭和迁移过程中不反向切换。退出时优先恢复唤起前应用，原应用或显示器已不可用时恢复盖世窗口。

## 10. 状态与异常

| 场景 | 处理 |
|---|---|
| 首次进入 | 直接展示全部应用；不显示扫描、骨架或进度页 |
| 有缓存增量扫描 | 保留现有网格，后台追加、移除或更新应用 |
| 无任何应用 | 展示“未找到可启动应用”，提供“检查应用来源” |
| 搜索无结果 | 只展示“未找到应用”和当前搜索词，不展示“清空搜索”按钮 |
| 自定义目录无权限 | 保留其他来源，标记异常目录并提供重新授权 |
| 图标读取失败 | 使用系统通用应用图标 |
| 启动失败 | 留在原页，展示原因和重试 |
| 快捷键冲突 | 不保存新快捷键，展示冲突提示 |
| F4/触控板无权限 | 本入口不可用，侧边栏和已可用入口继续工作 |
| 设备不支持手势 | 展示“不支持”，不可启用 |
| 盖世彻底退出 | 全局入口无响应；重新打开盖世后恢复配置 |
| 删除运行中应用 | 不移动应用，展示仅含“好”的系统式拦截弹窗 |
| 删除确认后应用启动 | 二次运行态检查命中后改为运行中拦截，不执行删除 |
| 删除失败 | 保留应用和编辑模式，展示权限、磁盘或系统保护原因 |
| 程序坞添加失败 | 保留设置入口和首次引导，展示失败原因，不修改其他程序坞项目 |

## 11. 数据与指标

本地只保存应用主键、显示名、路径、图标缓存、文件夹、来源、权限和唤起设置。应用绝对路径不上传；Bundle ID 上传前使用稳定哈希。

重点指标：

- 启动台入口渗透率；
- D7、D30 重复使用率；
- 周人均唤起次数；
- 各入口使用占比和成功率；
- 进入启动台到成功打开应用的 P50/P95；
- 搜索使用率和无结果率；
- 文件夹功能使用率；
- 编辑模式进入率、删除确认率、删除成功率和失败原因；
- 程序坞引导曝光、添加、关闭和程序坞入口唤起成功率；
- 应用扫描覆盖率和重复率；
- 使用启动台用户与未使用用户的盖世留存差异。

### 11.1 远程开关与灰度

不新增运营后台，复用盖世现有远程配置，支持按国内/海外包、版本、渠道、芯片和入口类型独立开关及灰度。总开关关闭时同时隐藏侧边栏并停止全部全局监听；单入口关闭不影响其他入口；本地布局数据保留。

放量顺序为内部测试、5%、20%、50%和全量。继续放量前要求无P0/P1缺陷，启动台打开成功率和应用启动成功率均不低于98%，扫描覆盖率不低于95%，且无持续全屏窗口残留。

## 12. 地区与登录差异

- 本功能为本地工具，未登录也可使用；
- 国内包使用中文固定界面；
- 海外包跟随盖世现有语言设置，产品英文名使用 `GameHub`；
- 应用名称以应用自身本地化名称为准；
- 国内与海外扫描、文件夹、启动和权限逻辑一致。
