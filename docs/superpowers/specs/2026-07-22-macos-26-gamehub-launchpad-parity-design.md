# macOS 26 盖世全屏启动台竞品对齐设计

> 日期：2026-07-22  
> 状态：V2.5 方案已确认，Demo待最终发布验收
> 适用端：盖世游戏 Mac 客户端，macOS 26 及以上  
> 需求性质：2026-07-20 旧方案的 V2.5 替代方案

## 1. 变更背景

用户实际体验竞品后确认，竞品不是游戏内容页，而是 macOS 旧版 Launchpad 的替代启动台：读取本机全部应用，以全屏大图标、横向分页和文件夹方式组织，并支持搜索、应用启动、全局快捷键、F4、触控板和触发角唤起。

旧方案中的“游戏优先、游戏/全部应用双页签、横版游戏封面和纵向滚动”偏离这一定位。V2.2 删除用户可见的扫描过程和 Demo 内部目标应用页；V2.3统一文件夹与程序坞引导；V2.4修正文件夹槽位和缩略图比例；V2.5补齐编辑态跨页整理；<mark><span style="color:#1677ff"><strong>V2.6改为Pointer同手势拖动、空白槽位排序、图标中心建夹和无提示框自动翻页，并删除全部产品内Toast。</strong></span></mark>本次改为：

> 在盖世 Mac 客户端中提供一套面向本机全部应用的全屏启动台；用户进入后直接看到全部应用，可用系统启动台式的搜索、分页、文件夹、编辑和删除能力，并能从侧边栏、全局入口或程序坞图标直接唤起。

## 2. 已确认决策

| 决策项 | V2.5 结论 |
|---|---|
| 产品形态 | 启动台仍由盖世 Mac 客户端承载；另提供轻量启动器作为独立程序坞入口，不承载普通客户端内容 |
| 入口 | 侧边栏仅增加九宫格图标，无固定文字；悬停提示“启动台” |
| 程序坞入口 | 使用与盖世 App 相同的图标；点击后直接唤起全屏启动台，盖世未运行时先拉起盖世 |
| 应用范围 | 进入后直接展示本机全部可启动 Mac 应用，不设置游戏页签，不展示扫描页 |
| 页面结构 | 全屏壁纸模糊背景、顶部搜索、大图标网格、横向分页、底部分页圆点 |
| 编辑与删除 | 长按进入图标抖动状态；可删除应用显示“−”，删除前二次确认，运行中应用禁止移到废纸篓 |
| 整理与文件夹 | 首次按名称生成布局，之后保存人工全局顺序；<mark><span style="color:#1677ff"><strong>长按后同一手势拖到空白槽位排序，仅命中图标中心时归类，边缘停留后自动翻页且无提示框</strong></span></mark>；建夹替换目标槽位；文件夹支持移入、名称直接编辑和左上角“−”移出，无右上角关闭 |
| 程序坞配置 | 首次引导只含标题、确认和关闭；更多菜单不重复提供入口；设置内用开关添加或移除；不提供后台驻留设置 |
| 唤起方式 | 侧边栏、全局快捷键、F4、触控板手势、屏幕触发角 |
| 生效条件 | 全局快捷键、F4、触控板和触发角仅在盖世前台、后台或最小化时有效；程序坞入口可冷启动盖世 |
| 应用发现 | 标准应用目录并支持用户添加自定义应用目录；发现和更新在后台完成，不向用户展示扫描过程 |
| 服务端 | 应用列表、文件夹和设置均保存在本机，无运营后台 |

## 3. 目标与非目标

### 3.1 目标

1. 让 macOS 26 用户用熟悉的大图标启动台浏览和打开本机应用。
2. 让用户可以通过初始名称排序、人工全局顺序、跨页整理、分页和文件夹快速找到应用。
3. 在盖世运行或后台驻留时，通过多种入口快速唤起启动台。
4. 保证应用扫描、图标读取、搜索和启动均不依赖网络。
5. 在权限不足、入口冲突或部分目录失败时仍保留可用入口。
6. 复刻 macOS 启动台的长按编辑、文件夹改名和安全删除反馈。
7. 允许用户把独立启动台入口添加到程序坞，一次点击即可回到全屏启动台。

### 3.2 非目标

- 程序坞启动器不展示客户端首页、内容流或独立设置，不形成第二套客户端；
- 不展示未安装游戏、云游戏、租号、商城、广告或运营推荐；
- 删除仅把 `.app` 应用包移到废纸篓，不清理应用配置、存档、缓存或账号数据；
- 不修改应用名称、Bundle 内容或系统 Launch Services 数据；
- 不承诺在盖世客户端被彻底退出后响应全局入口；
- 不静默修改系统快捷键、触发角或隐私权限。

## 4. 核心体验路径

### 4.1 从盖世客户端进入

```text
点击侧边栏九宫格图标
→ 当前显示器进入全屏启动台
→ 立即展示本机全部应用，不显示扫描进度
→ 浏览、搜索或文件夹归类
→ 点击应用
→ 通过 macOS 原生能力启动未运行应用或激活已有窗口
→ 启动台保留会话状态并退到后层
```

### 4.2 从全局入口进入

```text
盖世客户端正在运行或后台驻留
→ 用户按快捷键/F4、执行触控板手势或触发屏幕角
→ 在鼠标指针所在显示器打开启动台
→ 再次触发或按 Esc
→ 关闭启动台并恢复唤起前的应用
```

### 4.3 文件夹归类

```text
拖动顶层应用
├─ 拖到另一应用图标中心：创建文件夹
├─ 拖到已有文件夹图标中心：移入文件夹
├─ 拖到空白槽位、间隙或卡片非图标中心：按最近槽位排序
├─ 在左/右有效边缘停留500毫秒：自动翻到相邻页，继续拖动
└─ 拖出窗口、指针取消或窗口失焦：取消，不改变顺序

打开文件夹
→ 点击文件夹名称可直接改名
→ 点击应用左上角“−”返回顶层
→ 点击外部模糊区域或按 Esc 关闭

创建或移入成功
→ 停留当前启动台页
→ 用户再次点击文件夹后才打开
```

### 4.4 编辑与删除

```text
长按任一应用约 650 毫秒
→ 应用图标轻微抖动，进入编辑模式
→ 可删除应用左上角出现“−”
├─ 应用未运行：二次确认后移到废纸篓
├─ 应用已运行：提示无法移到废纸篓，只有“好”按钮
└─ 系统应用或无删除权限：不显示“−”
```

### 4.5 添加到程序坞

```text
首次进入启动台
→ 搜索栏同行、更多按钮右侧展示一次性引导
→ 点击“确认”
→ 独立启动台图标加入程序坞
→ 后续点击该图标直接唤起全屏启动台
```

## 6. 页面与交互设计

### 6.1 侧边栏入口

- 与现有一级入口使用相同图标容器和选中规则；
- 只展示九宫格图标，不显示固定名称；
- 悬停显示“启动台”；
- 点击后进入全屏，不在普通内容区加载页面；
- 退出后恢复进入前的页面和侧边栏选中状态；
- macOS 26 以下版本隐藏入口。

### 6.2 全屏启动台

- 背景使用当前桌面壁纸的模糊与降亮处理，保证应用名称可读；
- 顶部居中显示搜索框，右侧显示更多菜单；
- 首次进入时，在搜索栏同行、更多按钮右侧展示“添加到程序坞”引导；引导采用macOS半透明深色圆角卡片，只包含标题“添加启动台到程序坞？”、蓝色“确认”和圆形关闭按钮，不显示左侧图标或副标题；
- 引导出现后保持展示，不因翻页、搜索或打开文件夹自动消失；点击关闭后直接消失且不显示Toast；点击“确认”成功或关闭后记录本地状态，不再自动展示；
- 中部根据显示器可用空间动态计算行列数，每一显示器分辨率下保持稳定页容量；
- 应用使用系统原始图标和本地化名称；名称过长时单行渐隐，悬停显示完整名称；
- 底部圆点反映总页数和当前页；页数变化时实时更新；
- 支持鼠标滚轮、触控板横滑、方向键和页码圆点切页；
- 退出再进入时恢复上次页码；搜索词不跨会话保留。

### 6.3 初始排序与增量更新

1. 首次进入直接展示已发现应用，不展示骨架屏、扫描进度或“正在查找应用”；首次布局按本地化应用名称升序生成，系统实用工具目录默认合并为“实用工具”文件夹。
2. 用户完成建夹、移入、移出、改名或跨页移动后，顶层改用本机持久化的人工全局顺序；文件夹内和搜索结果仍按本地化名称升序。
3. 新安装应用追加到全局布局末尾；应用改名只更新名称，不改变人工槽位。
4. 应用卸载后移除对应图标，后续项目自然补位并按当前容量重新分页。
5. Bundle ID 相同的多副本只保留优先级最高的可启动版本；同名但 Bundle ID 不同的应用均保留。

### 6.4 搜索

- 搜索范围包含所有页面和文件夹内应用；
- 按应用本地化名称做包含匹配，不区分大小写和首尾空格；
- 搜索结果展开为应用图标，不展示文件夹容器；
- 搜索结果按本地化名称升序排列；
- 输入实时刷新；无结果只显示搜索图标和“未找到应用”，搜索词保留在输入框；
- 无结果页不显示结果数、当前搜索词说明或操作按钮，用户可直接修改搜索词或按 Esc 清空；
- 有搜索词时按 Esc 先清空，搜索为空时再次按 Esc 退出启动台。

### 6.5 Pointer拖动、排序与归类

- <mark><span style="color:#1677ff"><strong>仅编辑态顶层应用可拖动。非编辑态按住顶层应用650毫秒进入编辑后，当前指针会话继续有效；保持按压并移动超过6点即可开始拖动，不要求松手重按。</strong></span></mark>
- <mark><span style="color:#1677ff"><strong>拖动使用独立Pointer跟手浮层，不使用HTML5原生Drag and Drop。浮层挂在页面顶层，应用网格局部更新或自动翻页时仍持续跟手。</strong></span></mark>
- <mark><span style="color:#1677ff"><strong>指针进入另一顶层应用或已有文件夹的图标中心命中区时，才创建或移入文件夹；卡片名称区与外围不属于归类命中区。</strong></span></mark>
- <mark><span style="color:#1677ff"><strong>指针落在空白槽位、应用间隙或卡片非图标中心区域时，按最近网格槽位调整人工全局顺序；其他应用顺延补位。</strong></span></mark>
- 创建或移入成功后停留当前页，不自动打开文件夹；用户再次点击文件夹后才打开。
- <mark><span style="color:#1677ff"><strong>拖动进入左/右48px有效边缘并停留500毫秒后自动翻到相邻页，不显示边缘提示框，也不要求松手。翻页后锁定当前边缘，指针离开边缘后解除，避免连续误翻。</strong></span></mark>
- 首页面向左、末页面向右不触发；搜索、文件夹、设置、权限或删除弹窗打开时不接受顶层拖动。
- 按Esc、收到pointercancel、拖出窗口或窗口失焦时取消，不改变人工顺序和文件夹归属。
- 应用在拖放期间被卸载时取消操作，刷新后保留既有人工顺序；新应用追加全局末尾。
- 拖动期间只更新跟手浮层、源应用透明度、文件夹命中态和槽位指示；自动翻页时只更新应用网格与分页圆点，壁纸、搜索栏、引导、菜单及启动台根节点不重新挂载。
- 创建文件夹时局部更新应用网格与必要分页；打开设置、菜单和弹窗只更新对应浮层，不重建整屏。

### 6.6 文件夹

- 将一个应用拖到另一个应用上创建文件夹；新文件夹直接替换目标应用槽位，拖动源从原槽位移除，后续应用依次前移一格；
- 新文件夹默认命名为“文件夹”，系统实用工具文件夹命名为“实用工具”；
- 打开文件夹后在全屏启动台上方展示文件夹层，背景继续模糊；
- 不展示“重命名”按钮；点击文件夹名称直接切换为输入框，Enter 或失焦静默保存，Esc 取消；去除首尾空格后长度为1—20个字符，空名称保持输入态且不覆盖原名称；
- 文件夹封面使用 3×3、14×14 等比方形应用缩略图，缩放时保持完整，不拉成长条、不裁切成色块；
- 文件夹内应用按本地化名称升序排列；每个应用左上角显示圆形“−”，点击后插入文件夹后一位；
- 文件夹层不显示右上角关闭；点击外部模糊区域或按 Esc 关闭；
- 文件夹只剩一个应用时自动解散，剩余应用占据原文件夹位置；
- 文件夹为空时删除；
- 文件夹名称只保存在本机，不涉及敏感词审核和云端同步。

### 6.7 应用启动与返回

- 单击应用即调用 `NSWorkspace.openApplication(at:configuration:completionHandler:)`，不要求双击；
- 已运行应用通过 `NSRunningApplication` 激活已有窗口，不重复拉起；
- 发起启动后图标进入启动中状态，防止重复点击；
- completion 返回成功后目标应用置前，启动台退到后层并保留本次页码、搜索前页码和文件夹状态；
- 目标应用关闭后不强制抢焦点；用户回到启动台或再次唤起时恢复本次会话；
- 点击“盖世游戏”图标时退出启动台并激活盖世客户端窗口；
- 启动台不生成启动失败页面、失败模拟或重试条；系统无法打开应用时由 macOS 展示原生提示。
- `NSWorkspace` 返回错误或请求超时后恢复图标、解除单请求锁、保留原页和人工顺序，并上报失败或超时；用户可继续启动其他应用。
- <mark><span style="color:#1677ff"><strong>Demo不显示产品内Toast，也不在内部展示目标应用页面或“返回启动台”按钮；点击后以当前图标短暂进度态模拟调用，实际客户端交给macOS打开或激活目标应用。</strong></span></mark>

### 6.8 更多菜单与设置

更多菜单只包含“设置”和“退出”，使用固定图标列、文案列和快捷键列，两个文案左对齐；不重复提供程序坞添加入口。<mark><span style="color:#1677ff"><strong>编辑态第一次点击“•••”只退出编辑，菜单不出现；第二次点击“•••”才展开菜单，再点击“设置”打开设置。</strong></span></mark>

更多菜单、设置、文件夹、权限和删除确认均作为独立浮层局部挂载或移除；打开和关闭这些浮层不得重建壁纸、搜索栏、分页或应用网格。退出编辑仅更新应用网格的编辑控件和抖动状态。

设置内容：

| 分组 | 配置 |
|---|---|
| 程序坞 | 添加到程序坞开关；打开执行添加，关闭执行移除；用户从程序坞移除后1秒内同步为关闭 |
| 快速唤起 | 自定义全局快捷键、F4、触控板手势、触发角 |
| 权限 | 辅助功能、输入监控等权限状态和“去授权”入口 |
| 触发角 | 关闭、左上、右上、左下、右下；同一时间只选一个 |
| 应用来源 | 标准目录只读展示；用户可添加、移除自定义目录 |

规则：

- 快捷键录入时检测系统保留键和盖世现有快捷键冲突；冲突时不保存；
- F4、触控板和触发角分别启用，关闭后立即停止监听；
- F4 或手势在当前系统/设备不可捕获时显示“不支持”，不得伪装成权限问题；
- 点击“去授权”打开对应系统设置，回到客户端后刷新授权状态；
- 设置变更立即保存并生效；
- 盖世客户端彻底退出后所有全局入口失效。
- 程序坞入口属于冷启动入口，不受上述限制；盖世未运行时先启动盖世，再直接进入启动台。

### 6.9 交互优先级与文件夹拖放判定

Esc 按“删除弹窗 → 设置/权限弹层 → 文件夹名称编辑 → 应用拖放 → 编辑模式 → 文件夹 → 搜索 → 启动台”的顺序逐层返回；应用启动中不响应 Esc，等待 macOS 完成调用。<mark><span style="color:#1677ff"><strong>所有成功操作通过界面状态变化表达，不追加Toast。</strong></span></mark>

- 应用落到另一个应用或文件夹的图标中心命中区时执行创建或移入；落到空白或非中心区域时执行槽位排序；进入有效相邻页边缘停留后自动翻页；
- 拖放期间冻结网格，扫描、卸载、分辨率和显示器变化排队到拖放结束后处理；
- 拖动源或命中目标在队列中被卸载时，先取消拖放再应用队列；其他队列变化先让用户按拖动快照完成落位，再追加新应用、删除失效项；
- 拖放结束后按人工全局顺序重新分页，不按名称重排；
- 页数减少时当前页自动收敛到最后一个有效页。

### 6.10 长按编辑与删除

- 仅顶层应用静止长按650毫秒进入编辑模式，应用以错峰轻微旋转表现抖动；当前手势可继续拖动；点击空白区域或按Esc退出；文件夹内应用使用左上角“−”移出，不作为本期拖动源；
- 可删除的第三方应用左上角显示“−”；系统应用、受系统保护应用和当前无删除权限的应用不显示；
- 正在运行的第三方应用仍显示“−”，点击后立即使用 `NSRunningApplication` 复核运行状态；
- 未运行时弹出420px收窄弹窗，主提示为“确认要删除‘XX’吗？”，下方辅助说明为“应用将被移到废纸篓。”，不显示额外配置/存档/缓存说明；按钮为“取消”“删除”；
- 用户确认后再次复核运行状态，再通过系统废纸篓能力移动 `.app`；成功后从启动台移除并重新分页；
- 若确认后应用已启动，弹出“不能将项目‘XX’移到废纸篓，因为它已打开。”，只有“好”；
- 权限、SIP、只读卷、外置卷或文件状态导致失败时不移除图标，说明失败原因并保留编辑模式；
- 删除不清理应用配置、存档和缓存，确认弹窗中的“移到废纸篓”不得替换为“彻底卸载”。

### 6.11 程序坞入口

- 独立启动器使用与盖世 App 相同的图标，名称随国内/海外包显示为“盖世启动台”或“GameHub Launchpad”；
- 点击后通过本地 IPC 请求盖世打开启动台；盖世未运行时，启动器先携带 `openLaunchpad` 参数拉起盖世，待客户端就绪后打开全屏启动台；
- 启动器不展示独立窗口，完成唤起后退出；程序坞固定项继续保留；
- 首次引导只负责一次性推荐，用户关闭后不再自动展示；设置页始终可重新执行添加；
- 添加失败时保留引导并提示系统限制；不得静默重启 Dock、修改其他程序坞项目或破坏用户排序。

## 7. 应用扫描与权限

### 7.1 默认目录

- `/Applications`；
- `/System/Applications`；
- `/System/Applications/Utilities`；
- `~/Applications`。

### 7.2 自定义目录

- 用户通过系统目录选择器添加目录；
- 保存可持续访问授权，不直接要求完全磁盘访问；
- 用户可以移除自定义目录，移除后仅删除来源关系，不卸载应用；
- 同一路径不得重复添加；父子目录同时添加时按应用主键去重；
- 目录无权限或被删除时显示异常状态和“重新授权/移除”。

### 7.3 展示过滤

只展示可启动 `.app`，过滤应用包内部及已知辅助进程，包括 Helper、Updater、Renderer、Crash Handler、Uninstaller 和后台服务。图标读取失败时使用系统通用应用图标。

## 8. 多显示器与窗口规则

- 从侧边栏进入：在盖世客户端窗口所在显示器打开；
- 从全局入口进入：在鼠标指针所在显示器打开；
- 同一时间只允许一个启动台窗口；在另一显示器触发时将现有窗口迁移过去，不新建第二个；
- 屏幕断开时迁移到主显示器并重新计算页容量，不丢失文件夹归属；
- 分辨率或缩放变化只改变每页容量，保持当前槽位顺序和文件夹归属。

### 9.1 应用身份与布局继承

应用身份优先使用“签名团队标识＋Bundle ID”，其次为 Bundle ID、文件系统稳定标识和规范化路径哈希。同一身份的副本按“系统目录 → 普通应用目录 → 用户目录 → 自定义目录”保留优先版本。

- 身份不变的升级和路径移动继承人工槽位与文件夹归属，只更新名称、图标和路径；
- 无稳定身份且路径变化时按新应用处理；
- 卸载后的失效布局记录保留7天，短期重装可恢复，超期清理；
- 自定义目录使用系统持续访问凭证，凭证失效只隔离该来源。

### 9.2 单实例状态

启动台状态为“已关闭、打开中、已打开、跨屏迁移中、关闭中”。500毫秒内重复触发只响应第一次；打开、关闭和迁移过程中不反向切换。退出时优先恢复唤起前应用，原应用或显示器已不可用时恢复盖世窗口。

## 10. 状态与异常

| 场景 | 处理 |
|---|---|
| 首次进入 | 直接展示全部应用；不显示扫描、骨架或进度页 |

| 无任何应用 | 展示“未找到可启动应用”，提供“检查应用来源” |

| 自定义目录无权限 | 保留其他来源，标记异常目录并提供重新授权 |
| 图标读取失败 | 使用系统通用应用图标 |
| 系统无法打开应用 | 恢复图标、解除单请求锁并留在原页；启动台不生成失败页或重试条，由macOS展示原生提示 |
| 跨页拖动 | <mark><span style="color:#1677ff"><strong>不显示边缘提示框；进入有效边缘停留500毫秒自动翻页，跟手浮层保持；离开边缘后才能再次触发</strong></span></mark> |
| 快捷键冲突 | 不保存新快捷键，展示冲突提示 |

| 程序坞添加失败 | 保留设置入口和首次引导，展示失败原因，不修改其他程序坞项目 |

## 11. 数据与指标

本地只保存应用主键、显示名、路径、图标缓存、人工全局顺序、文件夹、来源、权限和唤起设置。应用绝对路径不上传；Bundle ID 上传前使用稳定哈希。

重点指标：

- 启动台入口渗透率；
- D7、D30 重复使用率；
- 周人均唤起次数；
- 各入口使用占比和成功率；
- 进入启动台到成功打开应用的 P50/P95；
- 搜索使用率和无结果率；
- 文件夹功能使用率；
- 人工顺序调整率和跨页移动成功率；
- 编辑模式进入率、删除确认率、删除成功率和失败原因；
- 程序坞引导曝光、添加、关闭和程序坞入口唤起成功率；
- 应用扫描覆盖率和重复率；
- 使用启动台用户与未使用用户的盖世留存差异。

### 11.1 远程开关与灰度

不新增运营后台，复用盖世现有远程配置，支持按国内/海外包、版本、渠道、芯片和入口类型独立开关及灰度。总开关关闭时同时隐藏侧边栏并停止全部全局监听；单入口关闭不影响其他入口；本地布局数据保留。

放量顺序为内部测试、5%、20%、50%和全量。继续放量前要求无P0/P1缺陷，启动台打开成功率和应用启动成功率均不低于98%，扫描覆盖率不低于95%，且无持续全屏窗口残留。

## 12. 地区与登录差异
