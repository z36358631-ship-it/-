# 盖世游戏 Mac｜macOS 26 全屏启动台功能点总结

- macOS 26 及以上版本在盖世客户端左侧一级导航显示“启动台”入口。
- macOS 25 及以下版本不显示“启动台”入口。
- 点击“启动台”后进入盖世原生全屏空间，不在普通客户端内容区打开页面。
- 启动台首次进入默认展示“游戏”分类并回到列表顶部。
- 启动台提供“游戏”和“全部应用”两个分类。
- “游戏”只展示本机已安装且可直接启动的本地游戏。
- “全部应用”展示标准应用目录中的本地可启动应用。
- 启动台不展示云游戏、租号游戏、未安装游戏、商城推荐、广告和网页链接。
- 游戏列表使用四列 `16:9` 横版封面布局。
- 全部应用使用六列大图标布局。
- 游戏和全部应用均支持纵向连续滚动。
- 同一次启动台会话分别保留游戏和全部应用的滚动位置。
- 系统扫描 `/Applications`、`~/Applications`、`/System/Applications` 和系统工具目录。
- 系统过滤 Helper、Updater、Renderer 和 Crash Handler 等辅助进程。
- 系统优先使用 Bundle ID 去重，缺失 Bundle ID 时使用规范化应用路径。
- 有本地缓存时先展示缓存并在后台增量扫描。
- 无本地缓存时展示骨架屏和“正在查找应用”。
- 游戏为空时展示空状态并提供“前往全部应用添加游戏”。
- 搜索只过滤当前分类并随输入实时刷新，不需要按回车。
- `Command + F` 聚焦搜索框。
- 方向键在应用网格中移动焦点，`Enter` 启动当前应用。
- 搜索有内容时按 `Esc` 只清空搜索。
- 搜索为空时按 `Esc` 退出启动台并返回盖世客户端。
- 用户可在全部应用中右键或 `Control + 单击`将应用添加到游戏。
- 用户可将已识别为游戏的应用从游戏中移除，但应用仍保留在全部应用。
- 人工游戏分类结果立即生效并保存在本机，重新扫描不得覆盖。
- 点击未运行应用时显示启动中并调用系统能力启动应用。
- 点击已运行应用时激活已有进程和窗口，不重复拉起应用。
- 同一时间只允许一个应用启动请求。
- 启动成功后目标游戏全屏置前，启动台保留在后层。
- 游戏结束、关闭或主动返回后重新显示启动台，不返回盖世客户端首页。
- 点击盖世游戏自身图标时退出启动台并返回原客户端窗口。
- 启动失败、系统拒绝或超时后保留启动台并恢复卡片可操作状态。
- 启动失败时展示具体原因和“重试”入口。
- 单个应用目录扫描失败时继续展示其他目录和旧缓存。
- 应用图标读取失败时使用系统通用图标。
- 应用被卸载或路径失效后从启动台移除。
- 原生全屏进入失败时留在盖世客户端并提示稍后重试。
- 应用列表、缓存和人工分类全部保存在本机，不新增运营后台。
- 启动台复用盖世现有埋点链路统计入口、就绪、搜索、分类纠正、启动结果、返回和退出行为。
- 埋点不上传应用绝对路径和原始 Bundle ID。
