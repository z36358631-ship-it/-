# 盖世游戏 Mac 端 iOS 应用库与 IPA 资源库功能对齐设计

## 1. 目标

在现有“我的 App / IPA 资源”双 Tab 基础上，对齐 PlayCover 3.1.0 的资源浏览、右键菜单、应用设置、来源导航、刷新、搜索和列表/网格能力，同时保留盖世已确认的来源校验、选择后导入、哈希确认及异常恢复流程。

本设计仅约束产品界面、交互、状态和验收结果。重签名、PlayTools 注入、PlayChain、LLDB 等底层能力由客户端技术方案另行拆解，但 Demo 中必须覆盖完整可操作状态和结果反馈。

## 2. 范围决策

### 2.1 删除

- 删除资源页“全部、游戏、工具、娱乐、社交、创意、其他”分类栏。
- 删除分类状态、分类筛选、分类空状态和批量范围中的分类条件。
- 不提供键盘映射布局导入、导出入口。

### 2.2 保留

- 保留 App 设置“杂项”中的“应用程序类型”，该字段属于应用元数据，不是主页面分类导航。
- 保留本地 IPA 二级选择确认。
- 保留来源地址校验、来源资源二级选择及最终确认。
- 保留来源哈希、兼容性、下载、校验、安装和异常恢复状态。

### 2.3 新增或补齐

- 两个 Tab 均提供刷新入口，但刷新对象和状态独立。
- 两个 Tab 均支持列表和网格。
- 两个 Tab 的列表行和网格项均支持右键。
- IPA 资源使用左侧来源树筛选，不再使用顶部来源下拉框。
- 我的 App 补齐 PlayCover 右键菜单和五页设置。

## 3. 信息架构

```text
iOS 应用
├─ 我的 App
│  ├─ 搜索、刷新、导入 IPA
│  ├─ 网格 / 列表
│  ├─ App 右键菜单
│  └─ App 设置
└─ IPA 资源
   ├─ IPA 资源库（全部已启用来源）
   ├─ 来源 A
   ├─ 来源 B
   ├─ 添加 / 管理 / 刷新来源
   ├─ 搜索、字母排序、网格 / 列表
   └─ 应用信息、下载、校验、安装
```

来源节点使用稳定 `sourceId` 作为筛选依据。显示名相同或域名相同的来源仍保持独立；旧来源无名称时依次回退为域名、JSON 文件名。

## 4. 我的 App

### 4.1 工具栏

从左到右提供：

1. 搜索：按显示名称、Bundle 名称、Bundle ID 模糊匹配。
2. 导入 IPA：列表为空、非空和搜索无结果时均常驻。
3. 刷新：重新扫描当前 App 库目录，更新名称、版本、图标、PlayTools 和异常状态。
4. 设置：未选中 App 时禁用；选中后打开该 App 设置。
5. 网格 / 列表切换：切换结果持久化。

刷新保留搜索词、视图模式和仍存在的选中项。刷新中旋转刷新图标并禁止重复触发；若选中 App 已被外部删除，则清空选择并禁用设置。

### 4.2 列表和网格

- 网格显示 App 图标、名称和运行/异常状态；名称单行截断。
- 列表显示状态、图标、名称和版本。
- 单击选中；点击内容空白处取消选择。
- 双击启动 App，启动准备期间图标叠加进度。
- 图标加载失败时显示占位和失败提示，不能把图标失败解释为 App 无效。

### 4.3 右键菜单

右键或 `Control + 单击`作用于鼠标所在 App，不依赖工具栏当前选择：

1. 设置。
2. 显示应用数据。
3. 在访达中查看。
4. 清除应用数据。
5. 清除应用偏好设置。
6. 清除 PlayChain 数据。
7. 卸载应用。

点击空白、按 `Esc`、切换 Tab 或打开菜单项后关闭菜单。

### 4.4 App 设置

设置即时保存，“好”只关闭窗口。

#### 键盘映射

- 启用键盘映射布局。
- 智能按键映射。
- 启用滚轮。
- 禁用内置鼠标处理。
- 鼠标灵敏度 0–100；主开关关闭时禁用。

#### 图像设置

- iOS 机型：多代 iPad Pro、iPhone 13/14/15/16 Pro Max。
- 分辨率：默认、自动、1080p、1440p、4K、自定义、可调整窗口。
- 长宽比：自由、自定义、4:3、16:9、16:10，按分辨率类型联动显示。
- 自动模式显示检测到的分辨率；自定义模式显示宽高输入。
- 分辨率缩放。
- 窗口化显示修复及正常/备用方式。
- 显示旋转。
- 禁用显示器息屏、隐藏标题栏、悬浮窗口。
- 高分辨率显示闪退风险提示。

#### 绕过

- 启用 PlayChain。
- PlayChain 调试，仅 PlayChain 开启后可用。
- 启用绕过越狱检测。
- 插入内省库。
- 强制插入 iOS 框架。
- 麦克风权限同步检查。
- 阻止高频睡眠请求。

涉及重签名的开关在执行期间禁用整个设置区并显示进度；失败后恢复原状态。

#### 杂项

- 应用程序类型。
- Discord 动态及自定义应用 ID、详细信息、状态、图像、清除。
- Metal 性能面板；macOS 13 以下禁用。
- LLDB 高性能调试器；“在终端中打开”依赖 LLDB 开启。
- 安装 / 移除 PlayTools。
- 在 root 目录启动。
- 限制动作更新频率。
- 忽略 Unity 键盘初始化错误。

#### 详细信息

只读显示：显示名称、Bundle 名称、Bundle ID、Bundle 版本、应用程序类型、可执行文件名、最低 OS 版本、PlayTools 状态、安装路径和别名路径。

#### 底部操作

- 重置设置。
- 重置键盘映射。
- 好。

不提供键盘映射布局导入、导出。

### 4.5 危险操作

- 清除应用数据：说明应用下载内容及数据将被删除。
- 清除偏好设置：说明 App 内设置需重新配置。
- 清除 PlayChain：说明可能需要重新登录。
- 卸载应用：展示 PlayChain、Entitlements、应用设置、键盘映射和应用数据清理选项；始终保留确认弹窗。

所有危险操作提供“取消 / 继续”，完成后刷新 App 库并反馈成功或失败。

## 5. IPA 资源

### 5.1 来源树

- 根节点“IPA 资源库”聚合所有已启用来源，并按 Bundle ID 去重。
- 子节点显示每条有效且已启用来源；点击仅显示该来源资源。
- 来源树支持展开和收起。
- 来源顺序决定聚合时同 Bundle ID 的来源优先级。
- 当前来源被停用、删除或刷新失败且不可用时，自动返回根节点。

### 5.2 工具栏

从左到右提供：

1. 搜索：仅搜索当前节点范围，按名称和 Bundle ID 模糊匹配。
2. 添加来源。
3. 刷新：根节点刷新全部已启用来源，子节点只刷新当前来源。
4. 应用信息：未选中资源时禁用。
5. 字母排序开关。
6. 网格 / 列表切换。

刷新失败时保留上次成功缓存，显示失败原因和缓存时间；刷新中禁止重复触发。

### 5.3 列表和网格

- 网格显示大图标、版本状态、名称和下载/安装进度。
- 列表显示状态图标、App 图标、名称和版本。
- 两种视图保留当前来源、搜索词、选择和任务状态，视图偏好持久化。
- 单击选中；点击空白取消。
- 双击开始下载或覆盖安装流程。

### 5.4 右键和应用信息

IPA 资源右键只有“应用信息”。顶部信息按钮和右键打开同一窗口，展示：

- Bundle 名称。
- Bundle ID。
- Bundle 版本。
- iTunes 查询地址。
- IPA 下载地址。
- 校验和。
- 来源、体积、兼容状态和风险信息。

下载地址不可访问时显示警告，但信息窗口仍可打开。

### 5.5 状态

#### 图标元数据

- 加载中：图标占位显示进度。
- 在线图标成功：显示 iTunes 图标。
- 在线图标失败但存在本地图标：显示本地图标。
- 均失败：显示三角占位；不能标记为 IPA 无效。

#### 本地版本

- 未安装：下载箭头。
- 来源版本等于本地版本：对勾。
- 来源版本低于本地版本：对勾加叉号。
- 来源版本高于本地版本：对勾加感叹号。

#### 下载与安装

```text
空闲 → 下载 → 完整性校验 → 安装准备 → 解压 → 复制 → 加入 App 库
     → 安装 PlayTools → 创建包装 → 完成 / 失败
```

- 同一时刻仅允许一个下载或安装任务。
- 下载中显示进度和取消。
- 无校验和允许继续；校验不一致必须二次确认。
- 已安装版本再次双击先做覆盖确认。
- 成功后刷新我的 App 和 IPA 状态。

### 5.6 来源管理

- 添加来源时自动补全缺失的 HTTPS 协议并校验。
- 支持有效、重复、URL 无效、JSON 无效或未找到等状态。
- 保留盖世“两级导入”：来源有效后进入资源选择，最终确认才保存来源和所选资源。
- 支持启用、停用、删除、上移、下移和刷新来源。
- 删除或停用来源不卸载已安装 App。

## 6. 共同交互规则

- 右键动作绑定被右击对象。
- 刷新不重置搜索和视图模式。
- 切换 Tab、来源或视图不中断下载、安装和启动任务。
- 列表和网格使用同一数据与状态模型。
- 加载、空状态、断网、权限不足、下载失败和安装失败均提供明确恢复入口。

## 7. 验收重点

1. 页面不存在七分类导航及对应筛选状态。
2. 两个 Tab 均有可工作的刷新，且刷新对象不同。
3. 两种视图在两个 Tab 均可切换并保持状态。
4. App 右键七项齐全，键盘映射导入/导出不存在。
5. IPA 右键仅“应用信息”。
6. 五页 App 设置可操作并覆盖联动、禁用和重置状态。
7. 来源树支持全部来源、逐来源筛选及同名来源。
8. 危险操作均有独立确认与结果反馈。
9. 图标失败与 IPA 无效状态严格区分。
10. 现有来源选择、哈希确认、批量和异常恢复能力不回归。

## 8. 评审结论

- 用户视角：通过；补齐单击选择、右键对象和刷新反馈。
- 开发视角：通过；刷新逻辑分离，来源使用 `sourceId`，分类 UI 与类型模型分离。
- 业务视角：通过；本轮交付界面与交互完整性，底层高风险能力另行技术拆解。
