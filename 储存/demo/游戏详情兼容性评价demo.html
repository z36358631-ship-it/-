1 需求描述：兼容性评价入口 ×

横屏布局

PC游戏引擎卡片内三列并排：兼容评价（宽1.3份）｜ 游玩时长 ｜ 云存档。

  • 标签「兼容评价 ›」→ 分数（26px）+ 星级 → 一行描述文字。

竖屏布局

同卡片三列并排，兼容评价列展示：标签「兼容评价 ›」→ 分数（15px）+ 星级（9px）→ 描述（单行省略）。

  • 不显示绿色圆圈，横竖屏样式统一。

数据同步

分数、星级、描述文案由 refreshEntryCard() 统一写入，横竖屏保持一致。

点击

整个兼容性列点击 → 打开兼容性评价全屏页（从右侧滑入）。

无数据时显示「暂无评价」。
2 需求描述：游戏信息 + 启动按钮区（横屏右列） ×

显示样式

横屏右列依次：状态栏（信号/WiFi/电量/时间）→ 游戏标题 + 副标题 → 评价胶囊 → 标签组 → 操作按钮组（菜单·秒玩·获取游戏）→ 开发商信息列表。

按钮交互

  • 秒玩：触发云游戏启动流程。
  • 获取游戏：跳转下载/购买页。
  • 菜单（⊞）：展开浮层「PC游戏设置 / 添加到桌面 / 卸载游戏」，点击外部收起。

滚动行为

右列跟随内容自然向下滚动，不固定在视口顶部。

3 需求描述：兼容性全屏页 — 概览卡 ×

页面入场

translateX(100%) → 0 从右侧滑入，覆盖整个 shell（z-index: 80）。顶部含返回按钮 + 标题「玩家兼容性评价」。

概览卡

  • 左：大号分数（50px 粗体）。
  • 右：星级行 + 评价条数 + 五星分布柱状图（5→1星）。

横屏限宽

横屏概览卡 max-width: 50%，不拉伸满行。

柱状图

  • 进度条宽度 = 该星级数 / 总数 × 100%，填充色 金黄色，动画过渡 0.4s。
4 需求描述：筛选工具栏 + 评价列表 ×

筛选 Chips

  • 全部：默认选中，显示全部评价。
  • 支持最多：按标签热度排序（预留）。
  • 我的：仅显示当前用户评价。

选中态蓝色 rgba(0,157,255,0.15) 填充，未选中半透明白色。

我要反馈

蓝色渐变按钮，固定工具栏右端，点击打开反馈弹窗（见标注 6）。

评价列表

  • 每条：头像 + 用户名 + 设备/系统 + 星级 + 标签组 + 文字 + 日期 + 举报。
  • 横屏：2列网格；竖屏：单列。
5 需求描述：竖屏底部操作栏 ×

显示条件

仅竖屏显示（横屏 display:none!important）。吸底固定，高度 76px。

按钮

  • 秒玩（flex:1）：深色背景，触发云游戏启动。
  • 获取游戏（flex:2）：蓝色渐变，跳转下载/购买。

按钮高度 48px，圆角 12px，点击 scale(0.96) 反馈。

6 需求描述：反馈弹窗表单 ×

触发

点击「我要反馈」，蒙层 + 弹窗从下方缩放淡入。

表单字段

  1. 评分：5颗可点击星星，hover 放大，点击确认。
  2. 设备信息：只读，显示当前设备型号。
  3. 兼容性标签（多选）：完美兼容 / 画面卡顿 / 意外闪退 / 手柄无响应 / 按键映射错误 / 声音异常。选中蓝色，未选半透明灰。
  4. 补充说明：70px 多行文本框，聚焦蓝色边框。

操作

「取消」关闭 ｜「提交」需验证至少选1个星级，否则 alert 提示。

提交后 Toast「反馈提交成功，感谢你的评价！」，2.5s 消失。
